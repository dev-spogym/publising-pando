# Category Hardening Checklist

목표: 한 카테고리씩 docs4 V1/V2 정책과 admin-pando 운영 UX를 기준으로 퍼블리싱/mock 완성도를 잠근다. 실제 API/DB 개발은 하지 않고, local state · route · DLG · side panel · toast · 목데이터로 운영 흐름을 명확히 보여준다.

## 카테고리 완료 체크

- [x] 문서 기준: docs4 V1/V2 출처와 정책이 화면에 반영됨
- [x] 목업 기준: API 호출 없이 local state / mock data로 운영 흐름이 끝까지 감
- [x] UX 기준: toast-only 액션을 줄이고 side panel / DLG / route / 상태 갱신으로 표현
- [x] 운영 기준: 차단 조건, 권한, 예외, 충돌, 취소/복구, 결과 상태가 보임
- [x] 테스트 기준: E2E + smoke/sanity/regression + visual/a11y/responsive 가능한 범위 검증
- [x] 인계 기준: 구현사가 “어떻게 구현하면 되는지” 읽히는 화면 상태와 계약 문구 유지
