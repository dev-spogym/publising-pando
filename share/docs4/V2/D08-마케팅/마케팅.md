## SCR-073 쿠폰 관리

> 연결 기능: MKT-03
### 2. UI 구성

- **요약 지표 카드 4종**: 전체 쿠폰 수 / 발급 건수 / 사용 건수 / 평균 사용률을 표시합니다. `전체 쿠폰 수`는 활성·만료·중단 쿠폰을 포함하고 삭제된 쿠폰은 제외합니다. `발급 건수`는 회원에게 발급 완료된 누적 건수, `사용 건수`는 현재 사용 완료 상태의 건수입니다. 환불로 미사용 복원된 쿠폰은 사용 건수에서 제외하고 사용 이력에는 남깁니다. `평균 사용률`은 사용 건수 ÷ 발급 건수 × 100이며 발급 0건은 0%로 표시합니다.

- **발급 버튼**: 특정 쿠폰을 선택한 뒤 쿠폰 발급 다이얼로그를 열어 회원 검색 또는 그룹/등급 선택으로 발급 대상을 구성하고 쿠폰을 지급합니다. 발급 완료 후 회원에게 별도 메시지를 자동 발송할지 여부는 고객사 확인 후 확정합니다.

## SCR-077 리퍼럴 프로그램

> 유형: 화면
> URL 경로: `/marketing/referral`
> 연결 기능: MKT-07
> 연결 다이얼로그: DLG-077-001 / `docs4/V2/D08-마케팅/마케팅.md:550` 리퍼럴등록, DLG-077-002 / `docs4/V2/D08-마케팅/마케팅.md:583` 리퍼럴삭제

### 1. 목적

기존 회원이 지인을 추천해 새 회원을 유치하면 추천인과 피추천인 모두에게 혜택을 제공하는 리퍼럴(회원 추천) 프로그램을 설정하고 실적을 관리하는 화면입니다. 구전 마케팅을 체계화해 신규 회원 유치 비용을 절감합니다.

### 2. UI 구성

- **리퍼럴 이벤트 목록**: 이벤트명 / 추천인 혜택 / 피추천인 혜택 / 기간 / 참여 수 / 상태 컬럼으로 구성됩니다.

- **이벤트 등록 버튼**: DLG-077-001 / `docs4/V2/D08-마케팅/마케팅.md:550`을 열어 새 리퍼럴 이벤트를 등록합니다.

- **프로그램 편집 모달**: 보상, 조건, 활성/비활성 상태를 수정합니다.

- **실적 현황**: 전체 추천 건수 / 성공 전환 건수 / 지급된 혜택 합계를 요약합니다.

- **추천인/피추천인 이력**: 누가 누구를 추천했는지, 전환 여부와 혜택 지급 여부를 목록으로 표시합니다.

- **이벤트 삭제**: DLG-077-002 / `docs4/V2/D08-마케팅/마케팅.md:583`를 통해 이벤트 삭제를 확인합니다.

### 3. 액션 / 흐름

1. 사이드바 "마케팅 > 리퍼럴 프로그램"으로 진입합니다.

2. "이벤트 등록"(DLG-077-001 / `docs4/V2/D08-마케팅/마케팅.md:550`)으로 추천인·피추천인 혜택과 기간을 설정합니다.

3. 이벤트가 시작되면 회원 앱에 리퍼럴 코드가 발급됩니다.

4. 추천 및 전환 실적을 이력 목록에서 모니터링합니다.

5. 혜택 지급 여부를 확인하고 미지급 건을 처리합니다.

6. 프로그램 카드의 `편집` 버튼으로 조건과 보상값을 즉시 갱신합니다.

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 데이터 미연동 | 화면은 표시되지만 추천 이력·혜택 지급 데이터는 목업 기준으로 표시됩니다. |
| 이벤트 없음 | 등록된 리퍼럴 이벤트가 없을 때 생성 안내가 표시됩니다. |
| 이벤트 진행 중 | 현재 활성화된 이벤트와 실시간 실적이 표시됩니다. |

### 5. 권한

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| Owner(지점장) / 매니저 | 전체 리퍼럴 이벤트 및 실적 | 이벤트 생성·수정·삭제, 실적 조회 |
| FC / 트레이너 / 스태프 | 접근 제한 | 리퍼럴 관리 불가 |
