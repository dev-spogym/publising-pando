## SCR-085 공지사항 관리

> 유형: 화면
> URL 경로: `/notices`
> 연결 기능: SET-EXT-04
> 연결 다이얼로그: DLG-085-001 / `docs4/V2/D09-설정관리/설정관리.md:551` 공지등록수정, DLG-085-002 / `docs4/V2/D09-설정관리/설정관리.md:638` 공지삭제확인

### 1. 목적
센터 내 공지사항을 작성·수정·삭제하고 게시 대상과 게시 기간을 설정하는 화면입니다.
전체 직원 공지, 특정 역할 대상 공지 등 수신 대상을 구분하여 필요한 정보를 원하는 시점에 전달합니다.

### 2. UI 구성
### 공지 목록
- 제목, 게시 대상, 게시 기간, 게시 상태(예정/게시 중/종료), 작성자, 작성일 표시
- 상태별 필터 탭 (전체 / 게시 중 / 예정 / 종료)
- 키워드 검색 (제목 기준)
- 공지 등록 버튼

### 공지 목록 행 액션
- 수정 버튼 → 공지 등록·수정 다이얼로그(DLG-085-001 / `docs4/V2/D09-설정관리/설정관리.md:551`) 열기
- 삭제 버튼 → 공지 삭제 확인 다이얼로그(DLG-085-002 / `docs4/V2/D09-설정관리/설정관리.md:638`)

### 공지 상세 미리보기
- 목록에서 제목 클릭 시 해당 공지 내용을 우측 패널 또는 펼침 행으로 미리보기

### 3. 액션 / 흐름
1. 화면 진입 시 공지 목록을 최신 순으로 불러온다.
2. 공지 등록 버튼 클릭 → 공지 등록·수정 다이얼로그(DLG-085-001 / `docs4/V2/D09-설정관리/설정관리.md:551`) 표시.
3. 수정 버튼 클릭 → 해당 공지의 내용을 불러온 상태로 수정 다이얼로그 열기.
4. 삭제 버튼 클릭 → 삭제 확인 다이얼로그(DLG-085-002 / `docs4/V2/D09-설정관리/설정관리.md:638`) 표시 → 확인 후 목록에서 제거.
5. 상태 필터 탭 클릭 → 해당 상태의 공지만 목록에 표시.

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 | 공지 목록 표시 (최신 순) |
| 로딩 | 목록 불러오는 중 (스켈레톤 표시) |
| 검색 결과 없음 | 검색 조건에 맞는 공지가 없을 때 안내 메시지 표시 |
| 공지 없음 | 등록된 공지가 없을 때 등록 안내 표시 |

### 5. 권한
| 계정 | 권한 |
|---|---|
| 슈퍼관리자 | 전체 지점 공지 조회·작성·수정·삭제 |
| Owner(지점장) | 소속 지점 공지 조회·작성·수정·삭제 |
| 매니저 | 소속 지점 공지 조회·작성·수정 (삭제 불가) |
| 기타 | 수신 대상에 포함된 공지 조회만 가능 |

### 6. 필수 예외처리
| 케이스 | 표시 | 후속 조치 |
|---|---|---|
| 제목 미입력 | "제목을 입력해주세요" 인라인 에러 | 저장 차단 |
| 게시 기간 역순 (종료 < 시작) | "종료일은 시작일 이후여야 합니다" 인라인 에러 | 저장 차단 |

## SCR-088 다국어 설정

### 1. 목적
시스템 전체에서 사용하는 언어와 국가·지역 관련 표시 형식을 설정하는 화면입니다.
외국인 회원이 많은 센터나 해외 지점 운영 시 관리자 화면과 키오스크 화면의 표시 언어를 다르게 구성할 수 있습니다.

### 2. UI 구성
### 시스템 언어 설정
- 관리자 화면 언어 선택 (한국어 / 영어 / 일본어 / 중국어 간체)
- 키오스크 화면 언어 선택 (위와 동일 선택지, 복수 선택 가능)
- 키오스크 기본 표시 언어 지정

### 국가·지역 설정
- 날짜 표시 형식 선택 (YYYY-MM-DD / MM/DD/YYYY / DD/MM/YYYY)
- 시간 표시 형식 선택 (24시간제 / 12시간제)
- 화폐 단위 및 표시 형식 (₩ / $ / ¥ 등)
- 시간대(Timezone) 선택

### 자동 번역 설정
- 공지사항 자동 번역 활성화 여부 (선택 시 외부 번역 서비스 연동)

### 3. 액션 / 흐름
1. 화면 진입 시 현재 저장된 언어·지역 설정을 불러와 표시한다.
2. 관리자 화면 언어 변경 후 저장 → 저장 완료 즉시 화면 전체가 선택한 언어로 전환된다.
3. 키오스크 언어 변경 후 저장 → 연결된 키오스크 기기에 즉시 반영된다.
4. 저장하지 않고 이탈 시 미저장 경고(DLG-080-001 / `docs4/V2/D09-설정관리/설정관리.md:442`)가 표시된다.

## DLG-085-001 공지 등록·수정

> 유형: 다이얼로그
> URL 경로: `/notices`
> 연결 기능: SET-EXT-04
> 연결 다이얼로그: DLG-085-001 / `docs4/V2/D09-설정관리/설정관리.md:551` 공지등록수정, DLG-085-002 / `docs4/V2/D09-설정관리/설정관리.md:638` 공지삭제확인

### 1. 목적
새 공지를 등록하거나 기존 공지를 수정하는 다이얼로그입니다.
제목, 내용, 게시 대상, 게시 기간을 설정하여 원하는 시점에 원하는 대상에게 공지를 전달합니다.

### 2. UI 구성
- "공지 등록" / "공지 수정" 제목 (모드에 따라 전환)
- 공지 제목 입력란 (필수)
- 공지 내용 입력란 (서식 없는 텍스트 또는 간단한 서식 지원, 필수)
- 게시 대상 선택
  - 전체 (모든 역할)
  - 특정 역할 선택 (primary, Owner(지점장), manager, fc, staff 등 다중 선택)
- 게시 기간 설정
  - 게시 시작일시 (날짜·시간 선택)
  - 게시 종료일시 (날짜·시간 선택, 빈칸 시 무기한)
- 상단 고정 여부 토글
- 취소 버튼
- 저장 버튼 (제목·내용 입력 시 활성)

### 3. 액션 / 흐름
1. 공지 목록에서 공지 등록 또는 수정 버튼 클릭 → 다이얼로그 표시
2. 수정 모드: 기존 공지 내용이 자동으로 채워진 상태
3. 내용 작성 후 저장 → 목록에 즉시 반영
4. 취소 → 다이얼로그 닫힘, 변경 사항 폐기

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 등록 모드 | 빈 폼 표시 |
| 수정 모드 | 기존 공지 내용 자동 로드 |
| 저장 처리 중 | 저장 버튼 비활성 |
| 저장 완료 | 다이얼로그 닫히고 목록 갱신 |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |

### 6. 필수 예외처리
| 케이스 | 표시 | 후속 조치 |
|---|---|---|
| 제목 미입력 | "제목을 입력해주세요" 인라인 에러 | 저장 차단 |
| 게시 기간 역순 (종료 < 시작) | "종료일은 시작일 이후여야 합니다" 인라인 에러 | 저장 차단 |
| 예약 발행 시각이 과거 | "예약 발행 시각을 미래로 설정해주세요" | 저장 차단 |
| 상단 고정 3개 초과 | "최대 3개까지 상단 고정 가능" 모달 | 기존 해제 선택 유도 |
| 게시 중 대상 축소 시도 | "게시 중인 공지의 대상은 축소할 수 없습니다" | 취소 유도 |
| 게시 중 즉시 삭제 시도 | "종료 처리 후 삭제 가능합니다" | 종료 처리 버튼 제공 |
| 매니저 삭제 시도 | 삭제 버튼 hidden | - |
| 첨부 파일 용량 초과 | "이미지 5MB / PDF 20MB 초과" 인라인 에러 | 파일 교체 유도 |
| 첨부 파일 개수 초과 (5개) | "공지당 첨부는 최대 5개" | 파일 교체 유도 |
| 직원 대상 공지에 푸시 설정 | "직원 공지는 앱 푸시 미지원" 안내 | 토글 자동 OFF |
| 공지 없음 | "등록된 공지가 없습니다" + [공지 등록] CTA | - |
| 검색 결과 없음 | "검색 결과가 없습니다" | 검색어 조정 안내 |
| 공지 동시 수정 충돌 | "변경 사항이 덮어씌워졌습니다" 알림 | 화면 새로고침 유도 |
| 만료 배치 실패 | 운영자에게 "자동 종료 실패" 알림 | 수동 종료 처리 안내 |
| 필독 확인율 집계 오류 | "확인율 집계 중 오류" 토스트 | 재집계 배치 재실행 |

### 7. 연동 / 자동화 참조
| 연계 대상 | 트리거 | 영향 |
|---|---|---|
| 예약 발행 배치 | 설정된 예약 발행 날짜·시각 | 공지 상태 "예정" → "게시 중" 자동 전환 |
| 만료 자동 비공개 배치 | 공지 종료일 자정 | 상태 "게시 중" → "종료" 전환, 회원앱·키오스크 숨김 |
| 푸시 발송 이벤트 | 게시 즉시 또는 예약 시각 | 수신 대상에게 앱 푸시 발송 (FCM/APNs) |
| 필독 미확인 재푸시 | 필독 공지 게시 후 24시간 | 미확인 회원에게 1회 재푸시 |
| 키오스크 공지 동기화 | 공지 상태 변경 이벤트 | 연결된 키오스크 기기에 실시간 공지 푸시 (SCR-082 / `docs4/V2/D09-설정관리/설정관리.md:69`) |
| 필독 확인율 집계 배치 | 매시간 | 필독 공지별 확인 완료 회원 수 집계 |
| 회원앱 공지 피드 | 공지 상태 "게시 중" 전환 | 회원앱 공지 탭에 실시간 노출 |
| 전 지점 공지 배포 (superAdmin) | 전체 지점 공지 저장 | 모든 테넌트의 공지 피드에 동일 공지 복제 |

### 8. 핵심 디자인 요청사항
- 공통 색상, 간격, 폰트, 토스트 규칙은 `_공통/디자인_시스템.md`, `_공통/토스트_메시지.md`를 따른다.
- 본 섹션에 별도 명시된 UI 상태와 권한 차이가 있으면 도메인 공통보다 화면 정의를 우선한다.
