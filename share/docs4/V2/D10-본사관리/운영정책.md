# D10-본사관리 본사관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.




## 1. 핵심 원칙
- 본사와 지점, 관리자 웹과 회원앱의 권한 구조를 실제 운영 기준으로 정리한다.
- 문서마다 달랐던 역할 명칭을 운영 기준으로 하나로 맞춘다.
- 관리자 웹 표준 역할 코드는 `superAdmin`, `primary`, `owner`, `manager`, `fc`, `trainer`, `staff`, `readonly` 8개를 기준으로 한다. `owner`의 화면 표시명은 `Owner(지점장)`으로 통일한다.
- `trainer`는 관리자 웹 기본 역할이다. PT트레이너, GX강사, 골프강사는 직원 직무값이며 기본 권한 템플릿은 `trainer`로 지정한다.
- 카운터/현장 지원 담당자는 별도 표준 역할로 두지 않고 `staff`에 통일한다.
- 권한은 "페이지 접근", "조회", "수정", "승인", "일괄처리"로 나눠서 본다.
- 전사 권한 템플릿
- 공통 상품 마스터
- 자동화 정책 세트
- 알림 템플릿
- 공지 템플릿
- KPI 및 감사 기준
- 회원/리드 실제 운영
- 현장 결제와 미수금 처리
- 수업 개설과 스케줄 운영
- 시설 배정
- 본사 정책 범위 안에서의 적용 설정
- 전사 환불 규모는 환불 집중 지점을 조기 포착하는 경고 지표로 본다.




## 2. 상태 / 운영 정의

- `Owner(지점장)`, `manager`, `fc`, `trainer`, `staff`, `readonly`가 로그인 직후 `/` route에서 진입하는 메인 대시보드(SCR-090 / `docs4/V2/D10-본사관리/본사관리.md:38`)로, 현재 소속 지점의 일일 운영 현황을 한눈에 파악하고 즉시 대응할 액션을 식별하는 운영 컨트롤 타워다. 핵심 지표 5종(전체/활성/HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상/오늘 출석/이번 달 매출), 회원 분포(성별·연령), 추이(월별 매출·주간 출석), 주의 대상 위젯 4종(생일·미수금·홀딩/HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상), 최근 활동 로그를 종합 표시한다. 이 지표는 D02 실제 회원 상태값 `임박(EXPIRING)`이나 자동 라벨 `만료임박`이 아니라 HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 회원 이용권 만료 step 대상이며, 기본값은 30일이지만 정책에서 N일로 변경할 수 있다. `superAdmin`/`primary`가 `/` route에 진입하면 D01 SCR-101 / `docs4/V2/D01-공통/공통.md:38` 대시보드 통합으로 전 지점 대시보드를 표시하고, 본사 권한자가 지점 전환(HQ-01 / `docs4/V2/D10-본사관리/본사관리.md:194`)으로 진입한 경우에만 SCR-090을 선택 지점 컨텍스트로 표시한다.

- **Owner(지점장) 출근 직후 1차 점검**: 로그인 → 대시보드 → 오늘 출석·미수금·HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상 회원 즉시 확인 → 액션 큐 진입(HQ-06 / `docs4/V1/D10-본사관리/본사관리.md:446`)

- **본사 슈퍼관리자 지점 전환 후 점검**: HQ-01 / `docs4/V2/D10-본사관리/본사관리.md:194` → 지점 전환 → 지점 대시보드 → 해당 지점 운영 상태 파악




## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.

- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs4/_정책미확정_확인필요.md`에 별도 관리한다.
