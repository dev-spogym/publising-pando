## SCR-090 지점 대시보드

> 유형: 화면
> URL 경로: `/`
> 연결 기능: DASH-01

### 1. 목적

`Owner(지점장)`, `manager`, `fc`, `trainer`, `staff`, `readonly` 계정이 로그인 직후 `/` route에서 진입하는 지점 운영 메인 화면입니다. 현재 계정의 소속 지점 운영 현황을 한눈에 파악합니다.
회원 현황, 매출, 출석, 주의 대상 회원(HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 회원 이용권 만료 step 대상·생일자·미수금·홀딩), 최근 활동 이력을 종합적으로 표시합니다.
`superAdmin` / `primary` 계정은 같은 `/` route에서 D01 SCR-101 / `docs4/V2/D01-공통/공통.md:38` 대시보드 통합을 표시하여 전 지점 대시보드를 봅니다. 본사 권한자가 특정 지점 컨텍스트로 전환한 경우에만 본 화면을 선택 지점 기준으로 볼 수 있습니다.

### 핵심 지표 요약 카드

- 전체 회원 수

- 활성 회원 수

- HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상 회원 수 (회원 이용권 만료 step 기준, 기본 30일/N일 변경 가능)

- 오늘 출석 수

- 이번 달 매출

### 분포 차트

- 회원 성별 분포 (원형 차트)

- 회원 연령대 분포 (막대 차트)

### 추이 차트

- 월별 매출 추이 (선 또는 막대 차트)

- 주간 출석 현황 (막대 차트)

### 주의 대상 위젯

- 오늘 생일 회원 목록

- 미수금 발생 회원 목록

- 홀딩 중인 회원 목록

- HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상 회원 목록 (각 위젯별 전체보기 링크)

### 최근 활동 로그

- 최근 시스템 활동 이력 요약 (누가 무엇을 했는지)

- 표시 범위: 현재 지점 기준 최근 10건. 슈퍼관리자는 선택 지점 컨텍스트 기준으로 표시

- 데이터 소스: 감사 로그(HQ-EXT-06 / `docs4/V2/D10-본사관리/본사관리.md:642` / SCR-097 / `docs4/V2/D10-본사관리/본사관리.md:642`) 중 운영자가 확인해야 하는 주요 이벤트

- 표시 항목: 발생 시각, 작업자, 작업 유형, 대상, 요약 문구

- 기본 노출 이벤트:

  - 회원 등록/수정/탈퇴/복구

  - 결제 등록/취소/환불

  - 이용권 등록/홀딩/연장/만료 처리

  - 수업 등록/수정/취소, 출석/노쇼 처리

  - 권한 변경, 백업/복원, 중요 설정 변경

- 제외 이벤트: 단순 조회, 페이지 진입, 자동 새로고침, 실패 없는 백그라운드 polling

- 행 클릭: 상세가 있는 이벤트는 대상 화면으로 이동, 설정/권한/감사 이벤트는 감사 로그 상세로 이동

- "전체보기" 클릭: 감사 로그(SCR-097 / `docs4/V2/D10-본사관리/본사관리.md:642`)로 이동하며 현재 지점 필터를 유지

### 상단 액션

1. 로그인 완료 → `/` route 진입 → 역할 판정.

## SCR-091 구버전 슈퍼 대시보드 리다이렉트

> 유형: 화면
> URL 경로: `/super-dashboard`
> 연결 기능: HQ-01

### 1. 목적

구버전 슈퍼 대시보드 진입 경로를 보존하기 위한 호환 화면입니다.
전 지점 통합 대시보드의 정본 화면은 D01 SCR-101 / `docs4/V2/D01-공통/공통.md:38` 대시보드 통합이며, `superAdmin` / `primary` 계정이 `/super-dashboard`에 접근하면 `/` route의 SCR-101로 이동합니다.
지점별 운영 현황과 지점 전환은 SCR-101에서 처리하고, 일반 지점 권한은 `/` route에서 SCR-090 / `docs4/V2/D10-본사관리/본사관리.md:38` 지점 대시보드로 분기합니다.

### 2. UI 구성

- 별도 KPI 카드, 지점 카드, 히스토리 로그를 구현하지 않습니다.

- 접근 시 권한과 세션을 확인한 뒤 즉시 SCR-101 / `docs4/V2/D01-공통/공통.md:38` 또는 SCR-090으로 이동합니다.

- 리다이렉트 중에는 공통 전체 화면 로딩 상태를 표시합니다.

- 권한 없음이면 공통 접근 제한 화면을 표시합니다.

### 3. 액션 / 흐름

1. `/super-dashboard` 접근 시 세션을 확인합니다.

2. `superAdmin` / `primary`이면 `/` route로 이동하고 SCR-101을 표시합니다.

3. `Owner(지점장)`, `manager`, `fc`, `trainer`, `staff`, `readonly`이면 `/` route로 이동하고 SCR-090 / `docs4/V2/D10-본사관리/본사관리.md:38` 지점 대시보드를 표시합니다.

4. 로그인 전이면 `/login`으로 이동합니다.

5. 권한이 없거나 세션이 유효하지 않으면 공통 접근 제한 안내를 표시합니다.

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 리다이렉트 중 | 권한 판정 후 `/` route로 이동하는 동안 공통 로딩 표시 |
| 세션 없음 | `/login`으로 이동 |
| 권한 없음 | 공통 접근 제한 안내 표시 |

### 5. 권한

| 계정 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| superAdmin / primary | 별도 화면 없음. `/` route의 SCR-101 / `docs4/V2/D01-공통/공통.md:38` | SCR-101로 이동 |
| Owner(지점장) / manager / fc / trainer / staff / readonly | 별도 화면 없음. `/` route의 SCR-090 / `docs4/V2/D10-본사관리/본사관리.md:38` | SCR-090으로 이동 |
| 세션 없음 | 로그인 필요 안내 | `/login`으로 이동 |

### 6. 필수 예외처리

| 케이스 | 표시 | 후속 조치 |
|---|---|---|
| 권한 판정 실패 | "권한을 확인하지 못했습니다" | 재시도 또는 로그인 이동 |
| 리다이렉트 실패 | "대시보드로 이동하지 못했습니다" | `/` 재진입 버튼 표시 |

### 7. 연동 / 자동화 참조

| 연계 대상 | 트리거 | 영향 |
|---|---|---|
| SCR-101 / `docs4/V2/D01-공통/공통.md:38` 대시보드 통합 | `/super-dashboard` 진입 + superAdmin/primary | `/` route에서 SCR-101 / `docs4/V2/D01-공통/공통.md:38` 표시 |

### 상담/퍼널 세분화

- `WI`는 방문 문의, `TI`는 전화 문의로 표시합니다.

- WI vs TI 건수는 방문 문의와 전화 문의를 분리한 막대 또는 2분할 카드로 표시합니다.

- TI 방문 전환율은 전화 문의 중 실제 대면 상담으로 이어진 비율을 표시합니다.

- TI 등록률과 WI 등록률은 동일 기간 각 문의 유형 대상자 중 등록 완료 비율로 표시합니다.

- FC 보드(SCR-095 / `docs4/V2/D10-본사관리/본사관리.md:503`)의 상담 액션 큐와 연결해 전화 문의 후 미방문, 방문 후 미등록 대상을 후속 조치 대상으로 노출합니다.

### 재등록 / 이탈 회복 KPI

- 기존의 `재등록률` 미구현 KPI는 이탈 정의가 확정되기 전까지 재등록 성공률로 대체합니다.

- 만료일별 고정 버킷 위젯은 추가하지 않고, HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 회원 이용권 만료 본사 step 또는 지점 추가 step 대상 단일 지표를 유지합니다.

### GX 세부 종목 분포

- 요가 / 필라테스 / 스피닝 / 줌바 / GX 기타 5종 분포를 차트로 표시

- 상품 마스터 `category=LESSON_PASS`, `lessonType=GX`, `subCategory`를 우선 기준으로 집계

- 외부 시트 연동 시 `[월매출]`의 `상품/종목명`과 `[주간월간보고]`의 `주차별 상품/종목 구간`을 상품 마스터 세부종목에 매핑

- 인기 종목은 증설 후보로, 비인기 종목은 시간표 축소 또는 폐지 후보로 표시

### OT / 수업 운영 KPI

- 강습 OT 신규 배정 건수와 골프 OT 신규 배정 건수를 별도 카드로 표시합니다.

- OT 등록 전환율은 `등록 / OT 진행` 산식으로 표시하며, PT 완료율과 합산하지 않습니다.

- OT 거부율은 `거절 / OT 진행`으로 표시하고, `거절`에는 `거부+보류`를 함께 집계했다고 툴팁에 명시합니다.

- OT 이월률은 이월된 OT 재고 규모를 표시하며, `이월 건수 / OT배정 인원 × 100`을 표준 산식으로 사용합니다. 기존 운영 리포트에서 다른 분모가 확인되면 보조 지표로 분리합니다.

- OT 체험 → 등록률은 `[OT관리-강습/골프]` 체험 대상 중 등록 완료 비율로 표시하며, 온보딩의 일반 체험 → 등록 전환율과 별도 카드로 분리합니다.

- 골프 OT 전체 현황은 강습 OT와 별도 섹션 또는 탭으로 표시해 골프 프로그램의 영업 파이프라인을 독립적으로 확인합니다.

### 강습 세션 유형별 운영 KPI

- PT / GX / 골프 / 기타 4종의 강습세션수를 카드 또는 누적 막대 차트로 표시합니다.

- 월간 합계와 주차별 추이를 함께 제공해 특정 종목의 운영량 쏠림과 월중 증감을 확인합니다.

- 공식 원천은 수업/출석/일정 원장에 저장된 `강습 세션 유형`입니다.

- `[주간월간보고]`의 `주차별 강습 합계` 및 `달성 강습 합계`, `[강습업무보고]`의 `주간 수업수`는 4종 분해 공식 원천이 아니라 합계 검산용으로만 사용합니다.

- 수업 유형과 동일한 PT/GX/골프/기타 4종 축으로 표시하며, GX 세부 종목 5종 분포와는 별도 축으로 분리합니다.

- 연락 완료 건수

- 리드 전환율

- 이번 달 신규 등록 회원 수

- 7일 이용률

- 30일 활동률

- PT 체험 참여율

- 일일입장 → 회원 전환율

- GX 첫 참여율

- 초기 이탈 회원 수

### 상담/가입 경로 분포

- `[주간월간보고]`의 `등록자 기준 가입경로` 9개 버킷(간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타)을 막대 차트로 표시합니다.

- 외부 신규 DB 원천은 `docs/03_템플릿/지점파일템플릿/2025년 12월 광화문점 신규통합 DB.xlsx`의 `신규통합DB` 시트 형식입니다. `경로` 필드는 문의 유형(방문(WI) / 전화문의(TI) / 체험 / 일일입장)으로 매핑하고, `상세 경로1`과 `상세 경로2`는 가입경로 9종으로 통합 매핑합니다.

- `[주간월간보고]`의 `WI 가입경로`와 `TI 가입경로`는 각각 별도 9개 버킷으로 분리해 표시합니다. 두 구간 모두 보고서 표시용 버킷이며 순서는 간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타입니다.

- 이탈위험 라벨 또는 이탈위험 세그먼트에 해당하는 회원은 별도 섹션으로 강조합니다.

### 3. 액션 / 흐름

1. 화면 진입 시 현재 지점의 이번 달 신규 회원 온보딩 현황을 자동 조회합니다.

## SCR-H1001 자동화 정책 라이브러리

> 유형: 화면
> URL 경로: `/hq/automation-policies`
> 연결 기능: HQ-09
> 연결 다이얼로그: DLG-H1001-001 / `docs4/V2/D10-본사관리/본사관리.md:1239` 정책세트편집

### 1. 목적

- 본사가 전사 자동화 정책 세트를 관리하는 화면이다.

- 만료 알림 정책은 `회원 이용권 만료`, `결제기한 만료`, `락커 만료` 3개로 분리한다.

- 세 정책 모두 본사가 기준일 step을 생성해 지점에 배포한다.

- 지점은 본사 step의 기준일을 수정하지 않고 Owner(지점장)만 ON/OFF할 수 있다.

- `회원 이용권 만료`와 `락커 만료`는 지점 Owner(지점장)이 해당 지점 전용 추가 step을 생성·수정·삭제할 수 있다.

- `결제기한 만료`는 지점 추가 step을 허용하지 않는다. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 결제링크 만료 알림은 D-3/D-1 고정 정책으로 별도 관리한다.

- 미수/연체 알림 N일, 반복 주기, 발송 시각, 기본 채널은 `결제기한 만료` 본사 정책 step에 포함한다. 본사(superAdmin/primary)가 설정해 지점에 배포하고, 지점은 ON/OFF만 할 수 있다.

- 수업 운영 자동화 정책은 본사에서만 생성·수정·배포한다. 대상은 미승인 일괄 등록, 수업 템플릿 본사 동기화, 대기열 자동 알림/배정, 신규 회원 첫 노쇼 면제 정책이다.

- 자동화의 경우 고객사의 정책 협의 후 개발한다. 그렇지 않을 경우 수동으로 진행하거나 V2로 진행한다.

### 2. UI 구성

- **상단 헤더**: 화면 제목 `자동화 정책 라이브러리`, 마지막 배포 일시, 전체 정책 세트 수, 배포 실패 지점 수, [정책 세트 생성], [PDF 내보내기] 버튼을 표시한다.

- **정책 유형 필터**: 전체 / 회원 이용권 만료 / 결제기한 만료 / 락커 만료 / 수업 운영 자동화 / 운영 자동화로 필터링한다.

- **정책 세트 카드 영역**: 정책 유형별 요약 카드를 표시한다.

| 카드 | 표시 항목 | 카드 액션 |

| 회원 이용권 만료 | 정책명, 버전, 본사 step 수, 지점 추가 step 허용 여부, ON 지점 수, OFF 지점 수 | 상세 보기, 복제, 배포 |
| 결제기한 만료 | 정책명, 버전, 본사 step 수, 미수/연체 알림 N일, 지점 추가 step 불가 표시, ON 지점 수 | 상세 보기, 복제, 배포 |

## SCR-H1002 커스텀 대시보드 빌더

> 유형: 화면
> URL 경로: `/dashboard/builder`
> 연결 기능: HQ-10

### 1. 목적
위젯을 드래그앤드롭 방식으로 배치하여 자신만의 대시보드를 구성하는 화면입니다.
자주 확인하는 지표와 정보를 원하는 레이아웃으로 배치하여 업무 효율을 높입니다.

### 2. UI 구성
### 위젯 라이브러리 패널 (좌측)
- 추가 가능한 위젯 목록 (카테고리별 분류: 매출 / 회원 / 출석 / 수업 / 공지 / 할 일 등)
- 각 위젯 썸네일 미리보기
- 위젯 검색

### 캔버스 영역 (우측)
- 현재 대시보드 레이아웃 편집 영역
- 위젯 드래그앤드롭으로 배치 및 이동
- 위젯 크기 조절 (모서리 드래그)
- 위젯별 설정 버튼 (위젯 내 표시 기간, 지점 등 세부 설정)
- 위젯별 제거 버튼

### 레이아웃 옵션
- 그리드 열 수 선택 (2 / 3 / 4열)
- 그리드 선 표시/숨김 토글

### 저장 및 관리
- 현재 레이아웃 저장 버튼
- 저장된 레이아웃 불러오기
- 기본값으로 초기화 버튼
- 레이아웃 이름 지정 및 저장 (여러 레이아웃 프리셋 관리)

### 3. 액션 / 흐름
1. 화면 진입 시 현재 저장된 대시보드 레이아웃을 불러온다.
2. 좌측 라이브러리에서 위젯을 드래그하여 캔버스에 배치.
3. 배치된 위젯을 드래그로 이동하거나 모서리를 드래그로 크기 조정.
4. 위젯 설정 버튼 클릭 → 해당 위젯의 데이터 범위·필터 설정 패널 표시.
5. 저장 버튼 클릭 → 레이아웃 저장 → 대시보드에 즉시 반영.

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 | 저장된 레이아웃 편집 모드 |
| 위젯 드래그 중 | 드래그 대상 위젯 강조, 배치 가능 위치 표시 |
| 수정 중 | 변경 사항 있을 때 저장 버튼 활성 |
| 저장 완료 | 완료 메시지 후 대시보드 반영 |

| 계정 | 볼 수 있는 것 | 할 수 있는 것 |

## SCR-H1003 벤치마크 비교

> 유형: 화면
> URL 경로: `/benchmark`
> 연결 기능: HQ-11

### 1. 목적
유사한 규모·업종의 다른 센터와 자사 지점의 KPI를 익명으로 비교 분석하는 화면입니다.
업계 평균 대비 우리 센터의 위치를 파악하여 경쟁력 강화 전략을 수립합니다.

### 2. UI 구성
### 비교 기준 설정
- 비교 대상 센터 규모 선택 (회원 수 기준 소 / 중 / 대)
- 비교 대상 업종 선택 (노출 순서 고정: 헬스 / 필라테스 / PT샵 / 골프 / 요가 / 크로스핏 / 복싱 / 수영 / 태권도 / 스피닝 / 기타)
- 비교 지표 선택 (매출 / 회원 유지율 / 출석률 / 신규 회원 등)
- 기간 선택 (월간 / 분기 / 연간)

### 벤치마크 차트
- 우리 지점 수치 vs 업계 평균 vs 상위 25% 비교 막대 차트
- 지표별 우리 지점의 백분위 표시 (예: "상위 35%")
- 개선 여지가 큰 지표 강조 표시

### 인사이트 요약
- 강점 지표 목록 (업계 평균 대비 우수한 항목)
- 개선 필요 지표 목록 (업계 평균 대비 미달 항목)
- 개선을 위한 액션 아이템 제안

### 3. 액션 / 흐름
1. 비교 기준(규모, 업종, 지표, 기간)을 선택한다.
2. 분석 실행 버튼 클릭 → 벤치마크 데이터 로드 및 차트 표시.
3. 지표별 차트를 확인하고 인사이트 요약을 검토한다.
4. 내보내기 버튼으로 결과를 PDF로 저장한다.

### 4. 핵심 상태
| 상태 | 설명 |

## SCR-H1005 NPS 설문

> 유형: 화면
> URL 경로: `/nps`
> 연결 기능: HQ-13

### 1. 목적
회원 대상 NPS(순추천지수) 설문 결과를 조회하고 분석하는 화면입니다.
회원 만족도 추이를 파악하고, 부정적 피드백을 빠르게 식별하여 서비스 개선에 활용합니다.

### 2. UI 구성
### NPS 요약 카드
- 전체 NPS 점수 (-100 ~ +100)
- 응답자 분류 비율 (추천자 / 중립자 / 비추천자)
- 전기간 대비 NPS 점수 변화

### NPS 추이 차트
- 기간별 NPS 점수 변화 선 차트
- 응답자 수 추이

### 응답 분포
- 0~10 점수별 응답 수 막대 차트
- 지점별 NPS 비교 테이블

### 응답 목록
- 설문 응답 목록 (점수, 응답일, 자유 의견, 회원 정보)
- 점수 범위 필터 (추천자 9~10 / 중립 7~8 / 비추천 0~6)
- 자유 의견 키워드 분석 (자주 등장하는 키워드 태그 클라우드)

### 설문 발송 관리
- NPS 설문 발송 주기 설정 (월 1회 / 분기 1회)
- 발송 대상 설정 (활성 회원 전체 / 특정 이용권 보유 회원)
- 발송 채널 선택 (Push / 카카오톡 / SMS)
- 최근 발송 이력 목록

### 3. 액션 / 흐름
1. 화면 진입 시 최근 분기 기준 NPS 결과를 불러온다.
2. 기간 필터 변경 → NPS 점수 및 차트 자동 갱신.
3. 비추천자 필터 선택 → 비추천 응답 목록만 표시하여 빠른 대응 가능.
4. 자유 의견 키워드 클릭 → 해당 키워드가 포함된 응답 목록 필터링.
5. `설문 발송` 탭에서 대상과 채널을 선택하고 발송한다.
6. 발송 후 최근 발송 이력에 캠페인을 추가하고 결과 탭으로 복귀한다.

### 4. 핵심 상태

| 기본 | NPS 요약 및 응답 목록 표시 |
| 로딩 | 데이터 불러오는 중 (스켈레톤 표시) |
| 응답 없음 | 선택 기간에 응답 없음 안내 |
| 설문 미발송 | 아직 설문을 발송한 이력이 없을 때 설문 설정 안내 |
