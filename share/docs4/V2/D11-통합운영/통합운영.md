## 범위 제외: 키오스크 운영 현황

> 관리자 CRM D11 구현 대상이 아닙니다.

### 정리
- 키오스크 운영 현황 화면, 단말별 장애 콘솔, 원격 재시작, 로그 다운로드, KIO-402 관리자 패널 연동은 KIOSK 기획문서에서 정의합니다.
- D11은 키오스크 운영 화면을 별도로 만들지 않고, 키오스크에서 발생한 출석 이벤트와 상태 수신값을 SCR-I001 / `docs4/V2/D11-통합운영/통합운영.md:37` 통합 출석 관리에 표시합니다.
- 키오스크가 오프라인이거나 오류 상태인 경우 D11은 출석 화면 또는 알림에서 상태 배지만 노출하며, 복구 조작은 제공하지 않습니다.

### D11 반영 방식
| 표시 위치 | 표시 내용 | 허용 액션 |
|---|---|---|
| SCR-I001 / `docs4/V2/D11-통합운영/통합운영.md:37` 통합 출석 관리 | 최근 키오스크 출석 이벤트, 실패 사유, 마지막 수신 시각 | 새로고침, 실패 로그 필터 |
| SCR-101 / `docs4/V2/D01-공통/공통.md:38` 대시보드 통합 | 실시간 출석 확인 카드 | 회원 상세 이동 |
| NFR-19 / `_공통/공통참조_코드인덱스.md:33` 알림 센터 | 키오스크/출입 게이트 오프라인 알림 | 알림 확인, 담당자 지정 |

### 구현 금지
- D11에 `/kiosk-ops` 화면, 원격 재시작 버튼, 로그 다운로드 버튼, 기기 상세 모달, 전사 키오스크 가동률 콘솔을 만들지 않습니다.
- 키오스크 운영 관제와 단말 관리자 기능은 KIOSK 문서 기준으로 개발합니다.

## DLG-I001 수동 출석 등록

> 유형: 다이얼로그
> URL 경로: `/attendance`
> 연결 기능: IOT-07
> 연결 다이얼로그: DLG-I001 / `docs4/V2/D11-통합운영/통합운영.md:214` 수동출석등록, DLG-I002 / `docs4/V2/D11-통합운영/통합운영.md:316` 옷락커배정

### 1. 목적
키오스크·QR 등 기기를 통한 출석이 불가한 상황에서 관리자(스태프)가 직접 회원 또는 직원의 출석을 수동으로 등록한다. 별도 기기 없이 시스템에서 직접 출석 처리가 가능하다.

### 2. UI 구성
### 팝업 구조

모달 팝업 형태로 통합 출석 관리 화면(SCR-I001 / `docs4/V2/D11-통합운영/통합운영.md:37`) 위에 오버레이로 표시된다.

### 입력 항목

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| 구분 | 필수 | 회원 / 직원 선택 |
| 이름 또는 번호 검색 | 필수 | 회원명, 회원번호, 사번으로 검색 |
| 출석 시각 | 필수 | 기본값: 현재 시각 (수정 가능) |
| 비고 | 선택 | 수동 등록 사유 메모 |

### 버튼

| 버튼 | 설명 |
|------|------|
| 등록 | 입력값 유효성 검사 후 출석 저장 |
| 취소 | 팝업 닫기, 입력값 초기화 |

### 3. 액션 / 흐름
1. 통합 출석 관리 화면 → '수동 출석 등록' 버튼 클릭 → 팝업 열림
2. 구분(회원/직원) 선택
3. 이름 또는 번호 입력 → 자동완성 목록에서 선택
4. 출석 시각 확인 (필요 시 수정)
5. 비고 입력 (선택)
6. 등록 버튼 클릭 → 성공 시 팝업 닫힘 + 출석 목록 즉시 갱신
7. 실패 시 인라인 오류 메시지 표시

### 4. 핵심 상태
| 상태 | 설명 |
|------|------|
| 기본(입력 대기) | 입력 항목 비어있음, 등록 버튼 비활성 |
| 검색 중 | 자동완성 드롭다운 표시 |
| 검색 결과 없음 | '일치하는 회원/직원이 없습니다.' 메시지 |
| 입력 완료 | 등록 버튼 활성화 |
| 등록 성공 | 팝업 닫힘 + 토스트 '출석이 등록되었습니다.' |
| 등록 실패 | 인라인 오류 메시지 표시 |

### 5. 권한

## DLG-I002 옷 락커 배정

> 유형: 다이얼로그
> URL 경로: `/locker`
> 연결 기능: IOT-04
> 연결 다이얼로그: DLG-I001 / `docs4/V2/D11-통합운영/통합운영.md:214` 수동출석등록, DLG-I002 / `docs4/V2/D11-통합운영/통합운영.md:316` 옷락커배정

### 1. 목적
출석 완료 후 옷 락커가 미배정 상태인 회원에게 빈 락커를 선택하여 배정한다. 통합 출석 관리 또는 옷 락커 운영 관리 화면에서 호출되는 팝업 형태이다.

### 2. UI 구성
### 팝업 구조

모달 팝업 형태. 출석 관리(SCR-I001 / `docs4/V2/D11-통합운영/통합운영.md:37`) 또는 옷 락커 운영 관리(SCR-I004 / `docs4/V2/D11-통합운영/통합운영.md:89`)에서 오버레이로 표시된다.

### 회원 정보 표시 영역 (읽기 전용)

| 항목 | 설명 |
|------|------|
| 회원명 | 배정 대상 회원 이름 |
| 출석 시각 | 해당 회원의 체크인 시각 |
| 이용권 상품명/유형 | 실제 구매한 회원권·수강권 상품명 또는 이용권 유형 |
| 고정 락커 | 보유 중인 고정 물품 락커 번호 (있는 경우) |

### 락커 선택 영역

빈 락커 목록을 번호 순으로 표시하며, 선택 가능한 락커를 클릭하여 배정한다.

| 상태 | 설명 |
|------|------|
| 선택 가능 | 클릭하여 배정 |
| 사용 중 | 선택 불가 (회색 처리) |
| 점검 중 | 선택 불가 (경고 표시) |

### 버튼

| 버튼 | 설명 |
|------|------|
| 배정 | 선택한 락커를 해당 회원에게 배정 |
| 취소 | 팝업 닫기, 배정하지 않음 |

### 3. 액션 / 흐름
1. 출석 목록(SCR-I001 / `docs4/V2/D11-통합운영/통합운영.md:37`) 또는 락커 운영(SCR-I004 / `docs4/V2/D11-통합운영/통합운영.md:89`)에서 '락커 배정' 버튼 클릭
2. 팝업 열림 → 대상 회원 정보 자동 표시
3. 빈 락커 목록에서 번호 클릭하여 선택
4. 배정 버튼 클릭 → 배정 완료
5. 성공 시 팝업 닫힘 + 출석 목록 및 락커 그리드 즉시 갱신
6. 빈 락커 없는 경우 → '배정 가능한 락커가 없습니다.' 안내

### 4. 핵심 상태
| 상태 | 설명 |
|------|------|
| 기본(락커 선택 대기) | 빈 락커 목록 표시, 배정 버튼 비활성 |
| 락커 선택됨 | 선택 락커 강조 표시, 배정 버튼 활성화 |
