# D11-통합운영 통합운영 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.




## 1. 핵심 원칙
- D11은 CRM 운영 화면에서 발생·수신된 출석, 락커, 체성분, 건강 요약 데이터를 표시하는 도메인이다.
- 키오스크 설정은 D09-설정관리에서만 수행하고, 키오스크 운영 화면과 단말 관제는 KIOSK 기획문서에서 다룬다.
- IoT는 독립 운영 화면을 만들지 않고 CRM 화면에 상태 배지, 최근 수신 시각, 이벤트 결과로만 표시한다.
- CRM에서 사용하는 고정 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종이다.
- D11은 수업 생성·예약·횟수 차감·노쇼 정책을 정의하지 않는다. 해당 기준은 D04-수업관리를 따르고, D11은 수업/출석/락커/체성분에서 발생한 운영 결과를 통합 조회·처리하는 화면만 정의한다.
- 통합 출석은 회원 / 직원 / 게스트 출석 이벤트를 한 화면에서 조회하고, 수동 출석 등록과 당일 옷 락커 배정까지 연결한다.
- 락커 운영은 출석 연동 당일용 옷 락커와 상품 계약 기반 고정 물품 락커를 구분한다.
- 체성분은 InBody API 수신값과 운영자 수기 입력값을 조회·검수·확정하는 범위까지 다루며, 장비 제어는 제공하지 않는다.




## 2. 상태 / 운영 정의

- **키오스크 운영 책임 분리**: 단말 운영 현황, 원격 재시작, 로그 다운로드, 장애 대응 콘솔은 KIOSK 기획문서에서 처리한다.




## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.




## 4. KPI / 성과 측정
| KPI | 산식/의미 | 측정 주기 |

| 예외 발생률 | 예외·차단 건수 / 전체 시도 건수 | 주 단위 |

| 운영 응답률 | 알림·요청 후 운영자가 처리한 비율 | 주 단위 |
