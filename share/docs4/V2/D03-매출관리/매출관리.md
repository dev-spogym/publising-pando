## SCR-S001 매출 현황

> 유형: 화면
> URL 경로: `/sales`
> 연결 기능: SAL-01
> 연결 다이얼로그: DLG-S001 / `docs4/V2/D03-매출관리/매출관리.md:764` 매출상세

### 1. 목적

센터에서 발생한 전체 매출 거래를 한눈에 조회하고 분석하는 매출관리 도메인의 핵심 허브 화면입니다. 담당자는 오늘·이번 주·이번 달 매출을 즉시 확인하고, 구매자나 상품명으로 원하는 거래를 빠르게 찾을 수 있습니다. 매출 내역 외에도 기간별·상품별·결제수단별·담당자별 집계, 환불 내역, 미납 내역을 탭으로 전환하며 관리합니다.

### 2. UI 구성

- **매출유형 필터 탭**: 전체 / 신규 / 재등록 / 휴면복귀 / 종목추가 / 업그레이드 중 하나를 선택해 해당 유형의 거래만 표시합니다.

- **분석 탭 (7개)**: 매출 내역 / 기간별 매출 / 상품별 내역 / 결제수단별 / 담당자별 / 환불 내역 / 미납 내역을 탭으로 전환합니다. 환불·미납 탭에는 해당 건수가 배지로 표시됩니다.

- **매출 목록 테이블**: 구매일 / 매출유형 / 유형 / 상품명 / 구매자 / 담당자 / 금액 / 결제수단 / 결제경로 / 현금·카드·포인트 / 미수금 / 링크 상태 / 상태 등을 컬럼으로 제공합니다. 상품명 클릭 시 상품 상세로, 구매자 클릭 시 회원 상세로 이동합니다.

- **신규 결제 버튼**: POS 판매 화면(SCR-S002 / `docs4/V2/D03-매출관리/매출관리.md:105`)으로 바로 이동합니다.

- **엑셀 내보내기**: 현재 필터가 적용된 전체 매출 내역을 엑셀 파일로 내려받습니다.

### 3. 액션 / 흐름

1. 사이드바에서 "매출관리 > 매출 현황"을 클릭해 진입합니다.

2. 상단 지표 카드로 이번 달 순매출·결제수단별 합계·미수금 현황을 파악합니다.

8. 매출유형 탭(신규/재등록 등)을 선택해 특정 유형만 볼 수 있습니다.

9. 분석 탭을 전환해 기간별·상품별·결제수단별·담당자별 집계를 확인합니다.

### 5. 권한

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| superAdmin / primary | 전체 지점 매출 조회·분석 가능 | 엑셀 내보내기, 신규 결제(POS) 이동, 모든 필터·탭 사용 |
| Owner(지점장) / 매니저 | 소속 지점 매출 전체 | 엑셀 내보내기, 신규 결제(POS) 이동, 모든 필터·탭 사용 |

## SCR-S002 POS 판매

> 유형: 화면
> URL 경로: `/pos`
> 연결 기능: SAL-EXT-05
> 연결 다이얼로그: DLG-S002 / `docs4/V2/D03-매출관리/매출관리.md:801` 구매자검색

### 1. 목적

센터 현장에서 직원이 직접 상품을 선택하고 구매자를 지정해 결제 등록 화면으로 넘기는 현장 판매(POS) 화면입니다. 카운터에서 회원이 방문했을 때 회원권·수강권(PT/GX/골프)·락커·운동복·일반 상품을 빠르게 장바구니에 담고, 현금가·카드가를 선택한 뒤 결제 처리 화면(SCR-S003 / `docs4/V2/D03-매출관리/매출관리.md:218`)으로 이동합니다. 이 화면은 `회원/상품 선택과 장바구니 구성`을 담당하며, 실제 결제는 인포데스크(FC)가 외부 POS 또는 현금 수납으로 완료한 뒤 SCR-S003에서 영수증 파일을 첨부해 등록합니다. 비대면 고객을 위한 결제링크 발송 경로는 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 고객사 확인 후에만 노출합니다.

### 2. UI 구성

- **카테고리 탭**: 회원권 / 수강권 / 락커 / 운동복 / 일반 탭을 전환하며 해당 상품군만 표시합니다. POS 판매와 상품관리의 상품 대분류는 이 5개로 통일합니다.

- **상단 도구 바**: 상품 타입 안내, 상품 생성, 상품 삭제, 타입별 필터, 정렬(이름순/생성순/수정순), 상품명 검색을 제공합니다. 상품 생성은 신규 상품 등록 패널을 빈 폼으로 열고, 저장 후 POS 판매 화면으로 복귀해 새 상품을 목록에 반영합니다.

- **상품 검색**: 상품명으로 실시간 검색해 원하는 상품을 빠르게 찾을 수 있습니다.

- **상품 카드 그리드**: 카테고리별 상품을 카드 형태로 나열합니다. 각 카드에는 상품 유형 배지(회원권·수강권·락커·운동복·일반), 상품명, 기간, 횟수, 현금가, 카드가가 표시됩니다. 클릭하면 장바구니에 추가됩니다.

- **구매 회원 패널 (우측 상단)**: 이름 검색으로 구매 회원을 지정합니다. 선택된 회원은 이름·전화번호로 표시되며 X 버튼으로 해제할 수 있습니다.

- **장바구니 패널 (우측 하단)**: 담긴 상품 목록, 수량 조절(-/+), 현금/카드 가격 유형 전환, 개별 상품 삭제, 전체 삭제 버튼이 있습니다. 여러 항목 선택 시 "일괄 변경"으로 가격 정책·판매 담당자·판매 메모를 묶어서 바꿀 수 있습니다. 장바구니가 비어있으면 빈 상태 안내를 표시합니다.

- **귀속 요약 바**: 장바구니 하단에 상품 정책 기준 `결제지점 / 이용지점 / 매출 귀속 지점 / 정산 지점 / 인센티브 귀속자` 기본값을 요약 표시합니다. 법인/홈&오피스/주말/통합 회원권이 포함되면 경고 배지를 함께 노출합니다.

- **빈 상태 프롬프트**: 현재 카테고리에 상품이 없으면 "상품을 추가해주세요" 문구와 상품 생성 CTA를 중앙에 표시합니다.

- **구매자 검색 모달**: 이름 또는 전화번호로 회원을 검색해 선택합니다. 결과가 없으면 "검색 결과가 없습니다" 안내를 표시합니다.

- **총 합계 및 결제하기 버튼**: 장바구니 하단에 총 결제 금액을 표시하고, 장바구니에 상품이 있을 때만 "결제하기" 버튼이 활성화됩니다. 클릭하면 장바구니 상품, 구매자, 가격 유형, 총 합계, 귀속 정보를 결제 처리 화면으로 전달하고 영수증 첨부 기반 현장 결제 등록 흐름을 시작합니다.

- **결제링크 발송 진입 버튼**: PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 고객사 확인 전까지는 노출하지 않습니다. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후에는 회원이 선택된 상태에서만 노출하고, 클릭하면 동일 장바구니 정보를 가진 채 결제 처리 화면으로 이동하며 해당 화면에서 결제링크 발송 경로를 선택합니다.

- **상품 등록 버튼**: POS 판매 중 상품이 없거나 현장에서 바로 판매할 신규 상품을 먼저 등록해야 할 때 신규 상품 등록 패널을 여는 단축 버튼입니다. 저장 후 POS 판매 화면으로 복귀하며, 신규 상품은 상품 목록에 반영됩니다.

### 3. 액션 / 흐름

1. 사이드바에서 "매출관리 > POS 판매"를 클릭하거나, 매출 현황에서 "신규 결제(POS)" 버튼을 클릭해 진입합니다.

2. 카테고리 탭을 선택하거나 상품명·타입별 필터를 조합해 원하는 상품을 찾습니다.

3. 상품 카드를 클릭하면 장바구니에 추가됩니다. 동일 상품을 다시 클릭하면 수량이 1 증가합니다.

4. 우측 "구매 회원" 패널에서 회원을 먼저 지정하거나, 비회원 상태로 장바구니를 구성합니다.

5. 장바구니에서 수량 조절(-/+), 현금/카드 가격 전환, 개별 삭제 또는 일괄 변경으로 구성을 조정합니다.

6. 총 합계와 귀속 요약 바를 확인하고 "결제하기" 버튼을 누르면 장바구니 상품, 구매자, 가격 유형, 총 합계, 귀속 정보를 결제 처리 화면으로 전달합니다. 이후 인포데스크(FC)는 외부 POS/현금/계좌이체 수납 결과를 혼합결제 형태로 등록합니다. 결제링크 발송은 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후에만 선택할 수 있습니다.

7. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 고객사 확인 후 회원이 현장에 없거나 링크 결제를 원하면 "결제링크 발송" 경로로 결제 처리 화면에 진입합니다. 확정 전에는 이 경로를 노출하지 않습니다.

## SCR-S003 결제 처리

> 유형: 화면
> URL 경로: `/pos/payment`
> 연결 기능: PAY-06
> 연결 다이얼로그: DLG-S003 / `docs4/V2/D03-매출관리/매출관리.md:843` 결제확인, DLG-S004 / `docs4/V2/D03-매출관리/매출관리.md:887` 중복결제경고, DLG-S016 / `docs4/V2/D03-매출관리/매출관리.md:1321` 결제링크발송

### 1. 목적

POS 판매 화면(SCR-S002 / `docs4/V2/D03-매출관리/매출관리.md:105`)에서 장바구니에 담은 상품을 현장 판매로 확정하는 화면입니다. 실제 카드/현금/계좌이체 결제는 인포데스크(FC)가 외부 POS 또는 현장 수납 절차에서 먼저 완료하고, CRM에서는 혼합결제 행, 포인트 사용액, 결제일시, 영수증 파일, VAN/POS 승인 결과, 지점 귀속 필드를 등록해 정상 결제와 회원권/수강권/락커/운동복/일반 상품 구매를 확정합니다. CRM은 현장 카드 PG 승인을 직접 수행하지 않으며, 비대면 고객을 위한 결제링크 발송 경로(PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218`)는 고객사 확인 후에만 노출합니다.

### 2. UI 구성

- **결제 상품 목록 (좌측)**: POS에서 담아온 상품명·수량·소계를 목록으로 표시하고, 하단에 총 결제 금액을 크게 표시합니다.

- **회원 검색 (좌측)**: 포인트를 사용하거나 회원권/수강권/락커/운동복/일반 상품을 구매 회원에게 연결하는 결제에서는 회원 지정이 필수입니다. 이름 또는 전화번호로 검색해 회원을 선택하면 보유 포인트가 표시됩니다. 선택된 회원은 X 버튼으로 해제할 수 있습니다.

- **수동 할인 입력**: v1에서는 PRD-04 / `docs4/V2/D05-상품관리/상품관리.md:218` 할인 정책과 MBR-EXT-03 / `docs4/V2/D02-회원관리/회원관리.md:270` 등급 혜택을 결제 금액에 자동 적용하거나 후보로 조회하지 않습니다. 할인 정책은 D05에서 등록·관리만 가능하며 결제 개발 기준으로 삼지 않습니다. 본 화면에는 운영자가 수동으로 확정한 할인 항목과 할인 금액만 입력·표시하고, 할인 입력 후 최종 결제 금액과 잔액이 즉시 재계산됩니다.

- **현장 결제 등록 정보**: 결제수단별 수납 행을 1개 이상 입력합니다. 각 행에는 `결제수단(카드/현금/계좌이체) / 금액 / 승인번호 또는 이체확인번호 / 승인시각 / 영수증 파일 / 메모`를 입력합니다. 영수증 파일은 필수이며 이미지/PDF 첨부를 기준으로 합니다.

- **혼합결제 입력 그리드**: 카드+현금, 카드+계좌이체, 현금+계좌이체, 카드+현금+계좌이체 조합을 모두 허용합니다. 포인트 사용액은 혼합결제 행과 별도 필드로 관리합니다.

- **VAN/POS 결과 박스**: 승인번호, 단말 ID, 승인 결과, 응답메시지, 외부 거래번호, 영수증 매칭 상태를 표시합니다. 수기 입력 또는 연동 조회 결과를 모두 저장할 수 있습니다.

- **지점 귀속 설정**: 결제지점, 이용지점, 매출 귀속 지점, 정산 지점, 인센티브 귀속자를 표시합니다. 상품 기본값을 자동 채우되 운영자가 조정할 수 있습니다.

- **수납 방식 선택**: 현장 전액 등록 / 계약금 등록 / 잔액 등록 중 하나를 선택합니다. `결제링크 발송`은 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 고객사 확인 후에만 네 번째 옵션으로 노출합니다.

  - **계약금 등록**: 당일 결제가격과 포인트 사용액을 입력하고, 남은 금액은 미수금 또는 할부 계획으로 넘깁니다. 이때 이용권 즉시 개시 여부를 선택할 수 있습니다.

  - **결제링크 발송**: PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후에만 노출합니다. 금액과 운영자가 수동 확정한 할인 금액을 확인한 뒤 고객이 직접 전액 결제할 링크를 발송합니다. 결제링크는 계약금/잔액/할부 결제에 사용하지 않습니다.

- **잔액 요약 박스**: 정가 / 할인 / 당일 수납액 / 잔액 / 결제 경로를 한눈에 표시합니다.

- **잔액 처리 방식 선택**: 계약금 등록에서 잔액이 발생하면 수기 분할 미수금(PAY-03 / `docs4/V2/D03-매출관리/매출관리.md:519`) 또는 정기 할부(SAL-EXT-01 / `docs4/V2/D03-매출관리/매출관리.md:600`) 중 하나를 선택합니다. 수기 분할은 별도 회차 없이 단일 미수 건으로 자동 등록됩니다.

- **이용권 개시 옵션**: 계약금 수납일 때만 노출되며 `즉시 개시 / 완납 후 개시` 중 하나를 선택합니다. 이 옵션은 결제 방식이 아니라 이용권 활성화 시점입니다. `완납 후 개시`는 잔액이 PAY-03 / `docs4/V2/D03-매출관리/매출관리.md:519` 미수금 또는 SAL-EXT-01 / `docs4/V2/D03-매출관리/매출관리.md:600` 할부로 남아 있을 때 이용권을 개시 대기 상태로 두고, 미수금 완료 또는 마지막 할부 회차 납입으로 잔액이 0원이 되는 시점에 자동 활성화하는 의미입니다. 할부 완납 후 새 결제를 다시 한다는 뜻이 아닙니다.

- **금액 일치 검증**: `혼합결제 행 합계 + 포인트 사용액`이 장바구니 최종 합계와 정확히 일치해야 결제 등록 버튼이 활성화됩니다. 입력 합계가 부족하거나 초과하면 차액을 붉은색으로 표시합니다.

- **포인트 사용 검증**: 포인트 사용 시 회원 선택이 필수이며, 보유 포인트가 부족하면 결제 등록을 차단합니다.

- **전자계약 후속 선택**: 현장 결제 등록 완료 직후 "계약서 미작성 / 계약서 작성" 선택 단계를 노출합니다. 계약서 작성 선택 시 강습 / 개인락커 / 양도 / 회원권 / 일일이용 유형 중 맞는 계약서를 고릅니다.

- **결제링크 발송 버튼**: PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후 수납 방식이 `결제링크 발송`일 때 노출됩니다. 회원 연락처, 최종 결제금액, 만료일시를 확인한 후 DLG-S016 / `docs4/V2/D03-매출관리/매출관리.md:1321` 결제링크 발송 다이얼로그를 통해 발송합니다. DLG-S016은 별도 화면이 아니라 발송 전 확인/실행 팝업입니다.

- **결제 등록 버튼**: 현장 결제 등록 필수값과 금액 일치 조건이 충족되면 활성화됩니다. 클릭 시 결제 확인 모달이 열립니다.

- **결제 확인 모달**: 상품 내역·장바구니 합계·결제가격·포인트 사용액·영수증 첨부 여부·선택 회원을 최종 확인하고 "결제 완료 등록" 버튼으로 처리합니다. 처리 중에는 버튼이 "등록 중..."으로 변경되고 비활성화됩니다.

- **중복 결제 경고**: 동일 회원이 최근 같은 금액을 결제한 이력이 있으면 경고 다이얼로그가 표시됩니다. "계속 진행" 또는 "취소"를 선택할 수 있습니다.

- **이전으로 돌아가기**: POS 판매 화면(SCR-S002 / `docs4/V2/D03-매출관리/매출관리.md:105`)으로 돌아갑니다.

### 3. 액션 / 흐름

1. POS 판매 화면에서 "결제하기"를 눌러 진입합니다. 장바구니 상품이 자동으로 불러와집니다.

3. 운영자가 할인 금액을 수동 입력·확정하고, 수납 방식(현장 전액 등록/계약금 등록/잔액 등록)을 선택합니다. PRD-04 / `docs4/V2/D05-상품관리/상품관리.md:218` / MBR-EXT-03 / `docs4/V2/D02-회원관리/회원관리.md:270` 할인 후보 조회·자동 적용은 v1에서 제공하지 않습니다. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후에만 결제링크 발송 옵션을 추가로 선택할 수 있습니다.

4. 현장 등록인 경우 외부 POS 또는 현금/계좌이체 수납 완료 후 혼합결제 행, 포인트 사용액, 결제일시, 영수증 파일, VAN/POS 결과, 귀속 필드를 입력합니다.

6. `혼합결제 행 합계 + 포인트 사용액 = 장바구니 최종 합계`가 맞고 영수증 파일이 첨부되면 "결제 등록" 버튼이 활성화됩니다.

8. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후 결제링크 발송을 선택한 경우 회원 연락처, 최종 결제금액, 링크 만료일시를 확인한 뒤 DLG-S016 / `docs4/V2/D03-매출관리/매출관리.md:1321` 발송 다이얼로그를 엽니다. 이 경로는 전액 결제 전용이며 계약금/잔액 결제에는 사용하지 않습니다.

9. 동일 회원의 중복 결제가 감지되면 경고 다이얼로그에서 진행 여부를 선택합니다.

10. 현장 결제 등록 완료 시 결제완료와 회원권/수강권/락커/운동복/일반 상품 구매 완료를 즉시 반영하고, 계약서 미작성/작성 선택 단계를 노출합니다.

11. 링크 발송 완료 시 회원상세 결제내역에는 `미결제`만 표시합니다. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 결제링크 상태값 `발송됨`은 화면 문구로 노출하지 않고 재발송·무효화 버튼 노출 기준으로만 사용합니다. 이 상태는 결제완료가 아니며 환불 버튼을 노출하지 않습니다.

### 1. 목적

회원이 실제 계약금·잔액·할부 등 계약상 납입 의무를 완료하지 못한 미수금을 추적하고 관리하는 화면입니다. 결제링크 발송만 된 건은 계약금이나 계약상 미수금이 아니므로 본 화면의 미수금 원장에 생성하지 않습니다. 결제링크 결제 전 상태는 회원상세 결제내역과 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 결제링크 상태로만 표시합니다.

### 2. UI 구성

- **요약 지표 카드 (4개)**: 미수금 총액 / 미수금 건수 / 본사 정책 기준 연체 알림 대상 건수 / 이번 달 회수액을 상단에 표시합니다. HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 결제기한 만료 본사 정책 step에서 설정한 N일 기준에 해당하는 건이 있을 때 해당 카드에 경고 강조 테두리가 표시됩니다.

- **상태 탭 (5개)**: 전체 / 미결제 / 일부결제 / 연체 / 완료 탭을 제공하며 각 탭에 해당 건수 배지가 표시됩니다. 결제링크 발송만 된 건은 이 탭에 포함하지 않습니다.

  - 미수금액: 붉은색으로 강조 표시합니다.

  - 결제 기한: 연체된 건은 날짜를 붉은색 굵은 글씨로 표시합니다.

### 2. UI 구성

- **혼합결제 환불 분해표**: 카드/현금/계좌이체/마일리지(포인트)별 원수납액, 기환불액, 이번 환불 배분액, 잔여 환불 가능액을 행 단위로 보여줍니다. 수단별 자동 분해 정책 확정 전에는 환불 수단과 배분 메모를 수기로 기록합니다.

  - **전체 취소**: 이번 환불 가능액 전액을 환불 처리합니다. 사유 입력이 필요합니다.

  - **부분 환불**: 환불할 금액을 직접 입력합니다. 이번 환불 가능액을 초과할 수 없습니다. 사유 입력이 필요합니다.

- **환불 수단 확인**: 원래 결제수단과 결제경로(현장 영수증 등록 / 결제링크)에 따라 환불 방법을 안내합니다. 현장 등록 건은 외부 POS/현금 환불 완료 증빙을 확인하고, 결제링크 건은 PG 환불 상태를 확인합니다. 외부 처리 상태와 영수증/증빙 메모는 수기로 입력할 수 있습니다.

- **쿠폰/마일리지 복원 요약**: 환불 시 결제에 사용된 마일리지(포인트)를 우선 복원하고, 사용 쿠폰은 미사용 상태로 복원합니다. 복원 대상·금액·상태를 처리 전 확인합니다.

## DLG-S016 결제링크 발송

> 유형: 다이얼로그

> 연결 다이얼로그: DLG-S003 / `docs4/V2/D03-매출관리/매출관리.md:843` 결제확인, DLG-S004 / `docs4/V2/D03-매출관리/매출관리.md:887` 중복결제경고, DLG-S016 / `docs4/V2/D03-매출관리/매출관리.md:1321` 결제링크발송

### 1. 목적

PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 고객사 확인 후 현장에서 확정한 상품·금액·운영자가 수동 확정한 할인 금액으로 고객이 직접 전액 결제할 수 있는 링크를 발송합니다. DLG-S016은 결제 처리 화면에서 [결제링크 발송]을 눌렀을 때 뜨는 발송 전 확인/실행 팝업이며, 별도 목록 화면이 아닙니다. 발송 후 고객은 결제수단만 선택할 수 있고 금액·상품 구성은 수정할 수 없습니다.
본 섹션의 만료일·알림·활성 링크 제한·PG 완료 동기화 기준은 2026-05-23 판도 내부 기준안입니다. 고객사 최종 확인 전까지 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 자동 만료·자동 알림·활성 링크 제한·PG 결제 완료 webhook은 개발 대기 항목으로 두며, 확정 전 구현 기준으로 삼지 않습니다.

> 2026-05-23 기준 본 섹션은 판도 내부 기준안이며 고객사 최종 확인 대기 상태입니다. 고객사 답변에 따라 권한·문구·알림 정책은 변경될 수 있습니다.

### 2. UI 구성

| 구성요소 | 내용 |
|---|---|
| 회원 정보 | 수신 대상 회원 이름, 연락처, 앱 연결 여부 |
| 결제 요약 | 상품명, 수량, 정가, 할인, 최종 결제 금액. 결제링크는 전액 결제 전용이며 계약금/잔액 링크는 제공하지 않음 |

| 취소 버튼 | 발송 없이 이전 화면으로 복귀 |

### 3. 액션 / 흐름

1. 결제 처리 화면에서 상품, 할인, 최종 결제 금액을 고정합니다. 결제링크는 전액 결제 전용이므로 계약금/잔액 결제로는 진입할 수 없습니다.

2. 결제링크 발송 전 확인 팝업(DLG-S016 / `docs4/V2/D03-매출관리/매출관리.md:1321`)이 열립니다.

3. 수신 연락처와 발송 채널, 만료일시를 확인합니다.

4. 동일 결제 건에 `발송됨` 상태의 활성 링크가 있으면 신규 발송을 차단하고 기존 링크 재발송 또는 결제링크 무효화를 안내합니다.

5. 발송 버튼을 누르면 새 링크가 생성되어 고객에게 발송됩니다.

6. 발송 완료 후 원화면으로 돌아가며 회원상세 결제내역에는 `미결제`만 표시됩니다. `발송됨`은 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 내부 결제링크 상태값이며 화면 문구로 노출하지 않고 재발송·무효화 버튼 노출 기준으로만 사용합니다.

7. 결제링크가 발송된 회원의 회원상세 결제내역 행에는 [재발송], [결제링크 무효화] 버튼을 표시합니다.

8. 결제링크 무효화 시 사유 입력은 필수이며, 무효화 처리자와 처리시각을 이력에 남깁니다.

9. 무효화 또는 만료 상태에서는 같은 행에 [결제링크 발송] 버튼을 다시 표시해 새 DLG-S016을 열 수 있습니다.

10. 회원이 링크 결제를 완료하면 즉시 이용권을 활성화하고 회원·운영자 모두에게 결제 완료 알림을 발송합니다.
