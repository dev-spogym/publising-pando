## SCR-M001 회원 목록

> 연결 기능: MBR-01
### 1. 목적

센터에 등록된 전체 회원을 한눈에 조회하고, 다양한 조건으로 원하는 회원을 찾아 상세 페이지로 이동하거나 일괄 작업을 수행하는 회원관리 도메인의 핵심 허브 화면입니다. 본사 관계자가 `전체 지점(통합)`을 선택한 경우 모든 지점 회원이 조회되며, 각 회원의 소속 지점이 목록과 엑셀 내보내기에 표시됩니다. FC는 HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 회원 이용권 만료 step 대상 회원을 검색해 리텐션 상담을 진행하고, 매니저는 장기 미방문 회원에게 메시지를 발송하며, Owner(지점장)은 전체 회원 데이터를 엑셀로 내려받아 분석에 활용합니다.

### 2. UI 구성

- **상단 액션 바**: 선택된 회원 수(`선택된 목록 n/N`)를 표시하고, 메시지 전송 / 상태 변경 / 엑셀 다운로드 액션을 상단에서 즉시 실행합니다. 지점 운영 설정에 따라 "출석시 상세 팝업" 토글을 노출합니다.

- **엑셀 내보내기**: 현재 필터를 적용한 전체 데이터를 엑셀 파일로 내려받습니다.

### 3. 액션 / 흐름

1. 사이드바에서 "회원 관리 > 회원 목록"을 클릭하거나, 대시보드 지표 카드를 클릭해 진입합니다.

개별 회원의 상세 정보를 탭 영역으로 분류해 조회하고 관리하는 회원관리 도메인의 중심 화면입니다. 화면 상단의 회원 프로필 요약과 주요 액션 버튼이 스크롤 중에도 고정되어 어느 탭에서든 즉시 접근 가능합니다. 회원 상태(활성 / 만료 / 예정 / 임박 / 홀딩 / 미등록 / 탈퇴)에 따라 표시되는 배너, 활성화되는 버튼, 안내 메시지가 완전히 달라지며, 정지 플래그·삭제·복구까지 회원의 전체 라이프사이클을 단일 페이지에서 관리합니다.

### 2. UI 구성

- **만료 알림 배너**: HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 회원 이용권 만료 본사 step 또는 해당 지점의 지점 추가 step 결과가 존재하고 해당 step이 ON일 때 상단에 고정 표시. 색상과 강조 수준은 step 긴급도 기준을 따른다.

- **회원 프로필 요약 (고정 영역)**: 프로필 사진 / 관심회원 별표 / 회원명 / 등급 배지 / 상태 배지 / 만료일까지 남은 일수 / 연락처 / 이메일. 하단에 미수금·마일리지·이용권 계약 수·락커 번호 요약 표시.

- **지점·귀속 요약 카드**: 프로필 요약 하단에 `소속지점 / 기본 이용지점 / 담당 상담자(FC) / 담당 트레이너 / 기본 매출 귀속 지점 / 기본 인센티브 귀속자`를 별도 카드로 표시합니다.

- **운영 코크핏**: 프로필 바로 아래에 `지금 바로 처리할 운영 이슈` 영역을 배치합니다. HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상, 자동 라벨 `만료임박`, 미수금 존재, 장기 미방문, 홀딩 상태, 이용 정지 같은 리스크를 우선순위 순으로 카드형 경보로 노출합니다. D02 실제 회원 상태값 `임박(EXPIRING)` 배너, HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상 카드, 자동 라벨 `만료임박` 카드는 별도 조건으로 판정합니다.

- **추천 액션 큐**: 운영 코크핏 우측에 재등록 안내 발송, 미수 결제 처리, 추가 상품 제안, 홀딩 처리/해제, 상담이력 확인 같은 다음 액션 버튼을 문맥에 따라 다르게 배치합니다.

  - 이용권 탭에서는 상품별 `운영 정책 구분(단일지점/법인/홈&오피스/주말/통합)`과 `이용 가능 지점` 요약을 함께 표시합니다.

- **계약 액션 버튼 묶음**: 계약 정보 탭 상단에 업그레이드 / 양도 / 환불 / 기간정지 / 기간정지 종료 / 기간정지 취소 버튼을 배치합니다. 회원 상태와 상품 상태에 따라 버튼 활성 조건이 달라집니다. 환불 버튼은 D02에서 환불을 직접 완료하지 않고 D03 매출관리의 SCR-S012 / `docs4/V2/D03-매출관리/매출관리.md:694` / DLG-S013 / `docs4/V2/D03-매출관리/매출관리.md:1181` 환불 플로우로 진입합니다.

### 3. 액션 / 흐름

1. 회원 목록(SCR-M001 / `docs4/V2/D02-회원관리/회원관리.md:38`)에서 회원명을 클릭하거나, 대시보드 위젯에서 특정 회원을 선택해 진입합니다.

3. 운영 코크핏에서 HQ-09 / `docs4/V2/D10-본사관리/본사관리.md:710` 만료 알림 대상, 자동 라벨 `만료임박`, 미수금, 장기 미방문 등 우선 대응 이슈를 먼저 확인합니다.

5. 탭을 선택해 이용권 현황, 출석 이력, 결제 내역 등 원하는 정보를 확인합니다.

6. 계약 정보 탭에서는 카드 보드에서 활성 상품을 빠르게 확인하고, 필요 시 "유효한 상품만" 토글로 만료 상품을 숨깁니다.

7. 상품 카드 상단 액션 또는 탭 상단 버튼으로 업그레이드·양도·환불·기간정지를 수행합니다. 환불은 선택한 결제/계약 정보를 기준으로 DLG-M013을 거쳐 D03 환불 플로우(SCR-S012 / `docs4/V2/D03-매출관리/매출관리.md:694` 또는 DLG-S013 / `docs4/V2/D03-매출관리/매출관리.md:1181`)에 연결합니다.

8. 결제이력 또는 결제내역 탭에서 결제경로, 미수잔액, 결제링크 상태를 확인합니다.

9. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후 미결제 결제링크 행에는 [재발송]과 [결제링크 무효화] 버튼을 표시합니다. 확정 전에는 두 버튼을 노출하지 않습니다. 무효화 시 사유 입력은 필수이며, 무효화 처리자와 처리 시각을 히스토리에 기록합니다.

10. PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 확정 후 링크 미결제 건이 만료 또는 무효화된 상태라면 권한자에게 [결제링크 발송] 버튼이 표시되어 새 링크를 만들 수 있습니다. 단, 발송됨 상태의 활성 링크가 있으면 새 링크 생성은 차단하고 기존 링크 재발송 또는 무효화만 허용합니다.

11. 필요한 액션 버튼(수동출석·메시지·상품구매 등)을 클릭해 작업을 수행합니다.

## DLG-M013 환불 처리

> 유형: 진입 다이얼로그
> 연결 기능: MBR-02
> 연결 다이얼로그: DLG-S013 / `docs4/V2/D03-매출관리/매출관리.md:1181` 환불처리, DLG-S014 / `docs4/V2/D03-매출관리/매출관리.md:1231` 환불상세결과, DLG-S006 / `docs4/V2/D03-매출관리/매출관리.md:970` 환불상세

### 1. 목적

회원상세의 결제이력/결제내역/계약 카드에서 선택한 결제 건을 D03 매출관리 환불 플로우로 넘기기 위한 진입 다이얼로그입니다.

D02에서는 환불 금액을 확정하거나 환불을 완료하지 않습니다. 실제 환불 처리 기준은 D03 SCR-S012 / `docs4/V2/D03-매출관리/매출관리.md:694` 결제 취소 / 부분 환불과 DLG-S013 / `docs4/V2/D03-매출관리/매출관리.md:1181` 환불 처리를 따릅니다. 따라서 회원상세에서 환불을 시작해도 환불 자동 산식 미확정, 기사용 차감금/위약금/기환불 누계/이번 환불 가능액/최종 환불액 수기 입력, Owner(지점장) 즉시 처리, manager/fc 승인 요청, 쿠폰/마일리지 복원 정책은 D03과 동일하게 적용됩니다.

### 2. UI 구성

- D03 환불 플로우 안내: "환불 금액·차감금·승인·쿠폰/마일리지 복원은 매출관리 환불 기준으로 처리됩니다."

- 쿠폰/마일리지 사용 여부 요약: 사용 마일리지(포인트), 사용 쿠폰, 복원 대상 여부

- 환불 처리로 이동 버튼: D03 SCR-S012 / `docs4/V2/D03-매출관리/매출관리.md:694` 또는 DLG-S013 / `docs4/V2/D03-매출관리/매출관리.md:1181`을 선택 결제 건 컨텍스트로 엽니다.

- 취소 버튼 (다이얼로그 닫기)

- 미결제 결제링크 행에는 환불 버튼을 표시하지 않습니다. URL 직접 접근 등 예외 상황에서만 방어 메시지를 표시하고 PAY-06 / `docs4/V2/D03-매출관리/매출관리.md:218` 결제링크 무효화 안내로 보냅니다.

### 3. 액션 / 흐름

1. 관리자가 회원상세 결제이력/결제내역/계약 카드에서 [환불] 버튼을 클릭합니다.

2. DLG-M013이 선택 결제 건의 원결제 정보, 기존 환불 이력, 쿠폰/마일리지 사용 여부를 읽기 전용으로 표시합니다.

4. 시스템은 선택 결제 건 컨텍스트를 유지한 상태로 D03 SCR-S012 / `docs4/V2/D03-매출관리/매출관리.md:694` 결제 취소 / 부분 환불 또는 DLG-S013 / `docs4/V2/D03-매출관리/매출관리.md:1181` 환불 처리를 엽니다.

5. D03에서 기사용 차감금, 위약금, 기환불 누계, 이번 환불 가능액, 최종 환불액, 환불 사유, 외부 처리 상태를 수기 입력합니다.

6. Owner(지점장)은 D03에서 즉시 환불 완료 처리할 수 있습니다. manager/fc는 승인 요청만 저장하며, Owner(지점장) 승인 전에는 환불 완료 상태가 되지 않습니다.

7. 환불 완료 후 D02 회원상세 결제이력/결제내역/계약 카드에 환불 상태, 승인자, 처리자, 쿠폰/마일리지 복원 결과를 갱신합니다.

8. 취소를 누르면 DLG-M013을 닫고 환불 요청을 생성하지 않습니다.

### 1. 목적

관리자가 회원의 마일리지를 수동으로 적립하거나 차감하는 다이얼로그입니다. 시스템 오류 보정, 이벤트 지급 등의 용도로 사용합니다.

### 2. UI 구성

- 제목: "마일리지 조정"

- 현재 마일리지 잔액 표시 (읽기 전용)

- 조정 유형 선택 (적립 / 차감)

- 조정 포인트 입력 (숫자 입력)

- 조정 후 예상 잔액 자동 계산 표시

- 조정 사유 입력 (필수, 텍스트 입력 또는 사유 목록 선택)

- 확인 버튼

- 취소 버튼

### 3. 액션 / 흐름

1. 관리자가 마일리지 조정 버튼 클릭 → 다이얼로그 표시, 현재 잔액 표시

2. 조정 유형(적립/차감) 선택 및 포인트 입력 → 조정 후 예상 잔액 자동 계산

3. 사유 입력 후 확인 클릭 → 마일리지 반영, 조정 이력 기록

4. 취소 클릭 → 다이얼로그 닫힘, 변경 없음
