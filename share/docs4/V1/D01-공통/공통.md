# D01-공통 공통

> 개발사용 정리 문서입니다. 이 파일만으로 해당 도메인의 화면·다이얼로그 구현 기준을 확인할 수 있도록 재구성했습니다.




## 도메인 개요
- 화면/다이얼로그 구현 시 이 파일을 먼저 확인합니다.
- 본 문서가 해당 도메인 화면·다이얼로그 구현 기준입니다. 구현 판단을 위해 다른 폴더를 확인하지 않습니다.
- API/DB 상세는 백엔드 설계 범위이므로 제외하고, 화면/기능/UX/운영정책 기준만 남겼습니다.




## SCR-100 로그인

> 유형: 화면
> URL 경로: `/login`
> 연결 기능: AUTH-01
> 연결 다이얼로그: DLG-000 / `docs4/V1/D01-공통/공통.md:164` 세션 만료

### 1. 목적
본사 또는 지점에서 등록한 `직원 로그인 계정`으로 CRM 시스템에 접속하는 유일한 진입 화면입니다.
본 시스템은 `직원 1명 = 로그인 계정 1개`를 기본 원칙으로 하며, 공개 회원가입이나 공용 운영 계정 없이 직원에게 발급된 로그인 ID와 비밀번호로만 접근합니다. 로그인 후에는 계정 역할과 접근 지점 범위에 따라 각자의 전용 작업공간으로 자동 이동합니다.

### 2. UI 구성
- 지점 선택 드롭다운
- 브랜드 로고 및 서비스명
- 로그인 ID 입력란
- 계정 이메일 안내 영역 (비밀번호 재설정 / 보안 알림용)
- 비밀번호 입력란 (표시/숨김 전환 버튼 포함)
- 로그인 상태 유지 체크박스
- 로그인 버튼
- 비밀번호 재설정 링크
- 테스트 계정 프리셋 패널
- 이미 로그인된 사용자 자동 우회 처리
- 첫 로그인 임시 비밀번호 변경 강제 안내
- 계정 잠금 / 비활성 / 휴직 상태 안내 배너
- 2단계 인증(2FA) 추가 인증 단계
- 자동 로그아웃 / 동시 세션 정책 안내

### 3. 액션 / 흐름
1. 직원 등록 시 발급된 로그인 ID와 비밀번호 입력 후 로그인 버튼 클릭
2. 지점 선택값, 계정 활성 상태, 잠금 상태, 권한 템플릿, 접근 지점 범위를 함께 확인
3. 첫 로그인 또는 임시 비밀번호 상태라면 비밀번호 강제 변경 단계로 우선 이동
4. 2단계 인증(2FA) 활성 계정은 추가 인증 코드 입력 단계 진행
5. 인증 성공 → 계정 역할에 맞는 기본 작업공간으로 자동 이동
6. 이미 로그인된 상태라면 로그인 화면을 건너뛰고 해당 작업공간으로 바로 이동
7. 퇴사/휴직/비활성 계정은 로그인 차단 후 관리자 문의 안내를 표시

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 | 로그인 ID·비밀번호 입력 대기 |
| 로그인 중 | 버튼 클릭 후 처리 중 (버튼 비활성) |
| 오류 | 잘못된 정보 입력 시 안내 메시지 표시 |
| 계정 잠금 | 연속 실패 횟수 초과 시 30분 잠금, 남은 시간 카운트다운 표시, 30분 후 자동 해제 또는 superAdmin / primary / Owner(지점장) 수동 해제 |
| 계정 비활성 | 휴직·퇴사·보안 비활성 상태로 로그인 차단 |
| 2단계 인증 | 추가 인증 코드 입력 단계 |
| 첫 로그인 | 임시 비밀번호 변경 강제 |
| 서비스 점검 | 시스템 점검 중 안내 화면 |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 지점 전체 데이터를 조회하고 업무 액션을 수행할 수 있습니다. |
| trainer / fc / staff | 본인 담당 범위 또는 배정된 업무 데이터 중심으로 조회합니다. |
| readonly | 조회만 가능하며 생성·수정·삭제·확정 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 메뉴와 직접 URL 접근을 차단하고 권한 안내 화면으로 전환합니다. |

계정 잠금 수동 해제 기준: 잠금 계정의 수동 해제는 D07 직원관리 > 직원 상세(STF-02 / `docs4/V1/D07-직원관리/직원관리.md:22`) > 계정 / 보안 카드의 [잠금 해제] 버튼에서만 실행합니다. superAdmin / primary는 전체 지점 직원 계정을 해제할 수 있고, Owner(지점장)는 본인 소속 지점 직원 계정만 해제할 수 있습니다. manager / fc / trainer / staff / readonly는 수동 잠금 해제를 실행할 수 없습니다. 해제 시 실패 카운트를 0으로 초기화하고 SCR-097 / `docs4/V1/D10-본사관리/본사관리.md:386` 감사 로그에 처리자, 대상 계정, 처리 시각, 사유를 남깁니다.

### 6. 필수 예외처리
| 케이스 | 처리 |
|---|---|
| 이메일 형식 오류 | 제출 전 인라인 유효성 안내 |
| 존재하지 않는 이메일 | "이메일 또는 비밀번호가 올바르지 않습니다" (계정 존재 여부 노출 차단) |
| 비밀번호 불일치 | 동일 메시지로 통일 처리 |
| 5회 연속 실패 | 30분 계정 잠금, 남은 시간 표시, "30분 후 다시 시도하거나 관리자에게 문의하세요" 안내 |
| 2FA 코드 만료 | "인증 코드가 만료되었습니다. 재발송해주세요" 안내 |
| 2FA 코드 불일치 | "인증 코드가 올바르지 않습니다" 안내 |
| 서비스 점검 중 | 점검 안내 화면으로 대체 표시 |
| 네트워크 단절 | 오프라인 안내 + 재시도 버튼 |

### 8. 핵심 디자인 요청사항
- 공통 색상, 간격, 폰트, 토스트 규칙은 `_공통/디자인_시스템.md`, `_공통/토스트_메시지.md`를 따른다.
- 본 섹션에 별도 명시된 UI 상태와 권한 차이가 있으면 도메인 공통보다 화면 정의를 우선한다.

## SCR-104 알림 센터

> 유형: 화면
> URL 경로: `(전체 화면 공통 - 상단 알림 아이콘)`
> 연결 기능: 없음 (V1 범위 직접 연결 기능 없음; 공통/보조 화면 또는 V2 기능은 V2 문서에서 관리)

### 1. 목적
HQ-09 / `docs4/V1/D10-본사관리/본사관리.md:1393` 만료 알림 대상, 수업 취소, 결제 완료 등 주요 운영 이벤트를 실시간으로 알려주는 알림 모음 패널입니다.

### 2. UI 구성
- 상단 알림 아이콘 (미읽음 수 배지 표시)
- 패널 상단 문맥 바로가기 영역
  - 현재 화면이 회원/매출/설정 중 어디인지에 따라 다른 운영 바로가기 4개 표시
  - 예: 회원 화면에서는 `회원 목록`, `메시지 발송`, `전자계약`, `출석 관리`
  - 예: 매출 화면에서는 `신규 결제`, `매출 현황`, `자동 알림`, `회원 목록`
- 알림 목록 (시간순 정렬)
  - 알림 유형 아이콘
  - 알림 내용 요약
  - 발생 시각
  - 읽음/미읽음 상태
- 전체 읽음 처리 버튼
- 전체 삭제 버튼 (권한 있는 계정만)
- 알림 설정 바로가기

### 3. 액션 / 흐름
1. 상단 알림 아이콘 클릭 → 알림 패널 열림
2. 패널 상단의 문맥 바로가기 버튼으로 현재 업무에 필요한 화면으로 즉시 이동
3. 알림 항목 클릭 → 관련 화면으로 이동 + 읽음 처리
4. "전체 읽음" 버튼으로 한 번에 처리
5. ESC 또는 바깥 클릭으로 패널 닫기

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 미읽음 있음 | 아이콘에 숫자 배지 표시 |
| 전부 읽음 | 배지 없음 |
| 알림 없음 | "새 알림이 없습니다" 안내 |

### 5. 권한
| 계정 유형 | 받는 알림 종류 | 관리 가능 액션 |
|---|---|---|
| 슈퍼관리자 | 전체 지점 시스템 알림 | 전체 읽음 / 전체 삭제 / 설정 |
| 본사 관리자 / Owner(지점장) / 매니저 | 소속 지점 운영 알림 | 전체 읽음 / 전체 삭제 / 설정 |
| FC / 코치 | 담당 수업·회원 관련 알림 | 개별 읽음 / 개별 삭제만 |
| 스태프 | 출석·회원 관련 알림 | 개별 읽음 / 개별 삭제만 |

### 6. 필수 예외처리
| 케이스 | 처리 |
|---|---|
| 알림 없음 | "새 알림이 없습니다" 빈 상태 표시 |
| 알림 로드 실패 | 패널 내 오류 안내 + 재시도 버튼 |
| 전체 삭제 실패 | "삭제 중 오류가 발생했습니다" 토스트 안내 |
| 실시간 연결 끊김 | 재연결 시도, 재연결 실패 시 수동 새로고침 유도 |
| 권한 없는 삭제 시도 | 전체 삭제 버튼 미노출 처리로 사전 차단 |

### 8. 핵심 디자인 요청사항

## DLG-000 세션 만료

> 유형: 다이얼로그
> URL 경로: `(전체 화면 공통 - 자동 감지)`
> 연결 기능: MFN-SCR-100
> 연결 다이얼로그: 없음

### 1. 목적
사용자가 일정 시간 동안 아무 활동을 하지 않거나 세션이 만료되었을 때 자동으로 표시되는 알림 다이얼로그입니다.
현재 작업 중이던 위치를 기억해두었다가 재로그인 후 바로 돌아올 수 있습니다.

### 2. UI 구성
- "세션이 만료되었습니다" 안내 메시지
- 재로그인 버튼
- 현재 페이지 위치 안내 (재로그인 후 복귀 예정)

### 3. 액션 / 흐름
1. 세션 만료 감지 → 현재 작업 화면 위에 다이얼로그 자동 표시
2. 재로그인 버튼 클릭 → 로그인 화면으로 이동 (현재 URL 기억)
3. 로그인 완료 → 만료 전 화면으로 자동 복귀

### 4. 핵심 상태

| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |
