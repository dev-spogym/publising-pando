# D01-공통 공통 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.




## 1. 핵심 원칙

- 이 문서는 관리자 웹과 회원앱을 포함한 전체 서비스의 운영 기준을 한 번에 파악하기 위한 상위 문서다.

- 목표는 "기능이 많다"가 아니라 "센터가 실제로 굴러간다"를 기준으로 문서를 정렬하는 데 있다.

- 포함

- 리드, 회원, 상품, 결제, 이용권, 수업, 출석, 시설, 자동화, 회원앱

- 본사와 지점의 권한 차이

- 화면 노출 우선순위와 상태 기준

- 제외

- 외부 결제사, 알림사, IoT 연동 상세 구현

- 상태값은 운영 기준으로 한 벌만 쓴다.

- 화면 라벨과 내부 enum이 다르더라도, 기획 문서에서는 먼저 운영 라벨을 고정한다.

- 같은 개념이 문서마다 다른 이름으로 등장하면 이 문서 세트 기준으로 통합한다.

- 본사는 정책, 템플릿, 표준, 기본값을 관리한다.

- 지점은 본사가 허용한 범위 안에서 적용하고 운영한다.

- 지점이 전사 기준을 새로 만드는 구조는 피한다.

- 1순위: 리드, 회원, 결제, 이용권, 출석, 예약, 만료 알림

- 2순위: 수업 통계, 시설 운영, 직원 운영, 본사 KPI

- 화면은 실제 센터 운영 동선을 따라 설계한다.




## 2. 상태 / 운영 정의

- **직원 1명 = 로그인 계정 1개**를 기본 원칙으로 한다. 직원 상태와 로그인 계정 상태는 직원관리와 공통 인증 정책에서 함께 관리한다.

- 로그인 계정은 직원 단위로 생성·잠금·비활성화·재활성화된다. 퇴사·휴직·보안 사고가 발생하면 직원 상태와 계정 상태를 함께 변경한다.

- 5회 연속 로그인 실패 시 계정은 30분 동안 자동 잠금 처리된다. 잠금 중에는 남은 시간을 표시하고, 30분 경과 후 자동 해제하거나 superAdmin / primary / Owner(지점장)가 수동 해제한다.

- 장시간 자리 비움은 30분 무활동 기준으로 자동 로그아웃 처리한다. 토큰 만료 후에는 로그인 화면으로 복귀하고, 작성 중 데이터는 각 화면의 임시저장 정책을 따른다.

- 퇴사자 계정 비활성화 시 모든 활성 세션을 즉시 강제 만료하고 재로그인을 차단한다.

| 항목 | V1 운영 기준 | 연결 기능 |
|---|---|---|
| 로그인 실패 잠금 | 5회 연속 실패 시 30분 자동 잠금 | AUTH-01 / `docs4/V1/D01-공통/공통.md:21` |
| 로그인 상태 유지 | 체크박스 ON 시 refresh token 만료 연장 | MFN-SCR-100 / `docs4/V1/D01-공통/공통.md:38` |

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

| 운영 응답률 | 알림·요청 후 운영자가 처리한 비율 | 주 단위 |

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs4/_정책미확정_확인필요.md`에 별도 관리한다.
