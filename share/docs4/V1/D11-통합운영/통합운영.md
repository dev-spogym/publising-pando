# D11-통합운영 통합운영

> 개발사용 정리 문서입니다. 이 파일만으로 해당 도메인의 화면·다이얼로그 구현 기준을 확인할 수 있도록 재구성했습니다.




## 도메인 개요

- 화면/다이얼로그 구현 시 이 파일을 먼저 확인합니다.

- 본 문서가 해당 도메인 화면·다이얼로그 구현 기준입니다. 구현 판단을 위해 다른 폴더를 확인하지 않습니다.

- API/DB 상세는 백엔드 설계 범위이므로 제외하고, 화면/기능/UX/운영정책 기준만 남겼습니다.

- D11은 통합 출석, 락커 운영, 체성분/건강 요약처럼 CRM 안에서 운영자가 보는 화면만 담당합니다.

- 키오스크 설정 화면은 D09-설정관리의 SCR-082 / `docs4/V1/D09-설정관리/설정관리.md:432` 키오스크 설정과 `/settings/iot` 페이지의 SCR-082A / `docs4/V1/D09-설정관리/설정관리.md:526` 키오스크 IoT 설정 탭, SCR-083 / `docs4/V1/D09-설정관리/설정관리.md:620` IoT 출입 관리 탭에서만 관리합니다.

- IoT는 별도 운영 화면을 만들지 않고, CRM 화면에서 상태 배지·출입 이벤트·측정 결과 수신값을 조회하는 수준으로만 반영합니다.

- CRM에서 고정으로 인식하는 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종입니다. 카드 리더, 얼굴인식 카메라 등은 별도 기기 종류가 아니라 출입 게이트 또는 키오스크의 인증 수단/부속 모듈로 봅니다.




## SCR-I001 통합 출석 관리

> 유형: 화면
> URL 경로: `/attendance`
> 연결 기능: 없음 (V1 범위 직접 연결 기능 없음; 공통/보조 화면 또는 V2 기능은 V2 문서에서 관리)
> 연결 다이얼로그: DLG-I001 / `docs4/V1/D11-통합운영/통합운영.md:781` 수동출석등록, DLG-I002 / `docs4/V1/D11-통합운영/통합운영.md:883` 옷락커배정

### 1. 목적

회원과 직원의 출석 현황을 하나의 화면에서 실시간으로 통합 조회·관리한다. 키오스크 QR, 얼굴인식, 앱 QR, 수동 입력 등 다양한 출석 채널의 데이터를 통합 집계하여 운영 효율을 높이고, 출석 후 옷 락커 배정 상태까지 함께 확인할 수 있다.

### 상단 통계 카드

| 항목 | 설명 |
|------|------|
| 오늘 총 출석 | 회원 + 직원 합산 입장 수 |
| 현재 센터 내 | 입장 후 퇴실 미처리 인원 수 |
| 미배정 락커 | 출석은 완료됐으나 옷 락커 미배정 인원 수 |
| 실패 건수 | 금일 `결과=실패` 건수. 만료·미등록·미수금 차단·직원 출석 허용 OFF·얼굴인식 시간 초과·허용 시간 외를 포함하며, `결과=중복`은 별도 중복 건수로 집계 |

### 필터 영역

| 필터 | 선택 옵션 |
|------|-----------|
| 기간 | 오늘(기본값), 이번 주, 이번 달, 직접 선택 |
| 구분 | 전체, 회원, 직원, 게스트 |
| 결과 | 전체, 성공, 실패, 중복, 퇴실 |
| 채널 | 앱 QR, 키오스크 QR, 얼굴인식, 수동 |
| 락커 상태 | 전체, 미배정, 수동배정, 자동배정, 대상 아님 |
| 검색 | 이름, 연락처, 회원번호, 사번 |

### 출석 테이블

| 컬럼명 | 설명 |
|--------|------|
| 시간 | 입장 시각 |
| 구분 | 회원 / 직원 / 게스트 |
| 이름 | 이름 + 회원번호 또는 사번 |
| 채널 | 앱 QR / 키오스크 QR / 얼굴인식 / 수동 |
| 결과 | 성공 / 실패 / 중복 / 퇴실 |
| 옷 락커 | 미배정 / 수동배정(번호) / 자동배정(번호) / 대상 아님 |
| 고정 락커 | 보유 시 락커 번호, 없으면 '-' |
| 비고 | 실패 사유 또는 운영 메모 |

### 실시간 출석 확인 카드

| 항목 | 표시 규칙 |
|------|-----------|
| 카드 수 | 데스크톱 기준 최대 10개 유지 |
| 초과 처리 | 10개 초과 시 가장 먼저 출석한 카드부터 제거 |
| 중복 처리 | 동일 회원 10분 이내 재시도는 `결과=중복` 이벤트로 기록합니다. 동일 회원의 2시간 이내 재입장은 방문/세그먼트 집계와 실시간 카드 노출에서 1회로 묶습니다. |
| 포함 채널 | 키오스크 QR, 앱 QR, RFID/밴드, 얼굴인식, 수동 출석 |
| 표시 정보 | 사진/기본 아바타, 이름, 회원번호, 성별/나이, 이용권, 만료일, 출석 시각, 출석 채널 |
| 주의 표시 | 만료, 만료 임박, 미수금, 중복, 이용권 없음, 사진 미등록 |
| 만료 임박 기준 | HQ-09 / `docs4/V1/D10-본사관리/본사관리.md:1393` 회원 이용권 만료 본사 step 또는 지점 추가 step 중 해당 지점에서 ON인 step 기준. 키오스크 출입 규칙은 출입 허용/차단 판정에만 사용 |
| 신규 이벤트 | 카드 4~5초 강조 + 알림음 + 토스트 |
| 사진 없음 | 기본 아바타 표시 + "사진 필요" 태그 + 현장 촬영 안내 토스트 |
| 클릭 | 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`) 기본정보 탭으로 이동 |

### 헤더 버튼

| 버튼 | 설명 |
|------|------|
| 수동 출석 등록 | DLG-I001 / `docs4/V1/D11-통합운영/통합운영.md:781` 팝업 호출 |
| 실패 로그 | 실패 건만 필터링하여 표시 |
| 최근 이벤트 | 키오스크·출입 게이트·앱·수동 출석 이벤트 사이드 패널 열기 |

### 행 단위 액션

| 액션 | 설명 |
|------|------|
| 퇴실 처리 | 입장 상태의 회원·직원을 퇴실로 전환 |
| 락커 배정 | 미배정 상태에서 옷 락커 배정 화면 또는 인라인 연동 |
| 회원 상세 | 해당 회원의 상세 화면으로 이동 |

### 3. 액션 / 흐름

1. 화면 진입 시 기본값: 오늘 날짜, 전체 구분, 전체 결과

2. 키오스크·앱에서 출석 이벤트 발생 → 테이블 상단에 즉시 반영

3. 실시간 출석 확인 카드에 회원 사진·기본 정보·주의 태그 표시

4. FC가 동일인 여부를 확인하고 필요 시 카드 클릭 → 회원 상세 기본정보로 이동

5. 미배정 락커 카드 클릭 → 미배정 인원만 필터링

6. 출석 행 클릭 → 해당 회원 상세 화면으로 이동

7. 행 내 락커 배정 버튼 클릭 → DLG-I002 / `docs4/V1/D11-통합운영/통합운영.md:883`(옷 락커 배정) 팝업 호출

8. 수동 출석 등록 버튼 → DLG-I001 / `docs4/V1/D11-통합운영/통합운영.md:781` 팝업 호출 → 등록 완료 시 목록 자동 갱신

## 범위 제한: IoT 상태/이벤트 수신 기준

> 기존 IoT 연동 관리는 D11의 독립 운영 화면으로 구현하지 않습니다.

### 1. 목적

D11은 IoT 장비를 등록·삭제·제어하는 화면을 제공하지 않습니다. D11에서 필요한 것은 통합 출석, 락커 운영, 체성분 화면에 표시할 상태값과 이벤트 수신값입니다.

### 2. 고정 기기 종류

| 기기 종류 | CRM에서 사용하는 값 | D11 표시 위치 |
|---|---|---|

전체 회원의 체성분 측정 현황을 통합 관리한다. 관리자 CRM은 InBody 측정기를 제어하지 않고, InBody API로 수신된 측정 사용자와 체성분 정보값을 조회·검수·확정 처리한다. 수기 등록은 현장 입력 보조 수단이며, 장비 파일 업로드와 장비 설정은 본 화면 범위에서 제외한다.

동일한 `/body-composition` route를 D02 SCR-M006 / `docs4/V1/D02-회원관리/회원관리.md:545` 체성분 관리와 공유합니다. 지점 권한자는 소속 지점 회원 범위만 조회·검수하고, 본사 권한자는 전 지점 회원의 체성분 수신 현황과 검수 큐를 통합 조회합니다. 회원 개인별 추이·상담 활용 화면은 D02 SCR-M006 / `docs4/V1/D02-회원관리/회원관리.md:545` 기준이며, 본 화면은 전 지점 수신/검수/확정 운영 기준을 정의합니다.

#### 검수 처리 기준

검수 처리는 InBody API 수신값 또는 수기 입력값이 바로 회원 이력으로 확정되기 어려운 경우 관리자가 확인하는 workflow입니다.

| 검수 유형 | 발생 조건 | 기본 처리 |
|---|---|---|
| 회원 매칭 실패 | InBody 수신값의 회원 식별 정보가 기존 회원과 일치하지 않음 | 회원 직접 매칭 또는 무시 |
| 중복 의심 | 동일 회원, 동일 측정일시, 주요 수치가 동일하거나 5분 이내 반복 수신 | 기존 데이터 유지, 신규 데이터 무시 또는 별도 확정 |
| 값 이상치 | 체중 20~300kg, 골격근량 5~80kg, 체지방률 3~60% 범위를 벗어남 | 관리자 확인 후 확정 또는 무시 |
| 미확정 수기 입력 | 스태프 또는 트레이너가 입력했으나 관리자 확인이 필요한 정책 | 관리자 확정 |

처리 액션은 다음 세 가지로 제한합니다.

| 액션 | 결과 |
|---|---|
| 회원 직접 매칭 | 선택한 회원의 체성분 이력으로 연결하고 `확정 대기` 또는 `확정` 상태로 전환 |
| 무시 | 원본 데이터는 보관하되 회원 이력에는 반영하지 않음 |
| 확정 처리 | 회원 건강 요약 및 회원 상세 체성분 이력에 노출 |

검수 처리 이력은 처리자, 처리 시각, 처리 전 상태, 처리 후 상태, 사유를 남깁니다.

## SCR-I007 회원 건강 연동 요약

> 유형: 화면
> URL 경로: `/members/health`
> 연결 기능: IOT-05
> 연결 다이얼로그: DLG-I003 / `docs4/V1/D11-통합운영/통합운영.md:711` 체성분수기등록

### 1. 목적

회원별 체성분, 운동 이력, 건강 목표, 외부 건강 앱(Health Connect) 연동 상태를 하나의 화면에서 통합 요약한다. 관리자가 회원의 건강 상태를 빠르게 파악하고 상담 근거로 활용할 수 있다. v1 기준 Health Connect는 걸음수 / 이동 거리 / 활동 kcal / 운동 세션만 요약 반영한다. Health Connect는 키오스크/IoT 장비 연동이 아니라 회원앱 MA-159에서 회원이 직접 동의·연결하는 Android 모바일 OS 선택 연동이며, admin은 연결/해제/권한 변경을 수행하지 않고 조회만 한다.

### 요약 카드 영역

| 카드 | 설명 |
|------|------|
| 최근 출석 | 해당 회원의 마지막 체크인 시각 |
| 오늘 옷 락커 | 당일 배정된 락커 번호 또는 미배정 표시 |
| 고정 락커 | 보유 중인 고정 물품 락커 번호 또는 없음 |
| 최근 InBody | 체중·골격근량·체지방률 최신 수치 (확정 데이터 기준) |
| 최근 7일 활동량 | 걸음수 합계, 활동 kcal, 운동 일수 (Health Connect 연동 시) |

### 체성분 이력 섹션

| 항목 | 설명 |
|------|------|
| 측정일 | 체성분 측정 날짜 |

| 골격근량 / 체지방률 | 핵심 지표 그래프 |

### 운동 이력 섹션

| 항목 | 설명 |
|------|------|
| 최근 수업 이력 | 출석한 수업명, 날짜, 강사 |
| 목표 달성률 | 월간 운동 목표 대비 달성 비율 |

### Health Connect 연동 상태 위젯

| 항목 | 설명 |
|------|------|
| 연동 상태 | 연결됨 / 부분 권한 / 동기화 지연 / 미연결 / 미지원 배지 |
| 최근 동기화 시각 | 마지막으로 데이터가 수신된 시각 |
| 수집 범위 | 걸음수 / 거리 / 활동 kcal / 운동 세션 |
| 소스 앱 | Samsung Health / Google Fit 등 최근 수집 소스 |
| 활동량 비교 | 센터 출석 운동 vs 외부 활동량 비교 |
| 안내 문구 | 회원앱 MA-159의 Android 하이브리드 앱 경유 연동이며 admin에서는 조회 전용임을 명시 |

### 3. 액션 / 흐름

4. Health Connect 미연동 또는 부분 권한 상태 → 회원 앱(MA-159) 연동 / 권한 수정 안내

6. 상담 메모 바로가기 → 해당 회원 상담 탭으로 이동

## DLG-I003 체성분 수기 등록

> 유형: 다이얼로그
> URL 경로: `/body-composition`
> 연결 기능: IOT-05
> 연결 다이얼로그: DLG-I003 / `docs4/V1/D11-통합운영/통합운영.md:711` 체성분수기등록

### 1. 목적
InBody 등 측정 장비 없이 관리자가 직접 체성분 수치를 수동으로 입력하여 등록한다. 측정지 원본을 보고 수기로 전산화하거나, 장비 연동이 불가한 환경에서 임시 등록 시 사용한다.

### 2. UI 구성
### 팝업 구조

모달 팝업 형태. 체성분 통합 관리(SCR-I006 / `docs4/V1/D11-통합운영/통합운영.md:470`) 화면에서 오버레이로 표시된다.

### 입력 항목

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| 회원 검색 | 필수 | 이름 또는 회원번호로 회원 선택 |
| 측정일시 | 필수 | 기본값: 현재 시각 (수정 가능) |
| 체중 (kg) | 필수 | 소수점 1자리 입력 |

## DLG-I001 수동 출석 등록

> 유형: 다이얼로그
> URL 경로: `/attendance`
> 연결 기능: IOT-07
> 연결 다이얼로그: DLG-I001 / `docs4/V1/D11-통합운영/통합운영.md:781` 수동출석등록, DLG-I002 / `docs4/V1/D11-통합운영/통합운영.md:883` 옷락커배정

### 1. 목적
키오스크·QR 등 기기를 통한 출석이 불가한 상황에서 관리자(스태프)가 직접 회원 또는 직원의 출석을 수동으로 등록한다. 별도 기기 없이 시스템에서 직접 출석 처리가 가능하다.

### 2. UI 구성
### 팝업 구조

모달 팝업 형태로 통합 출석 관리 화면(SCR-I001 / `docs4/V1/D11-통합운영/통합운영.md:32`) 위에 오버레이로 표시된다.

| 출석 시각 | 필수 | 기본값: 현재 시각 (수정 가능) |

### 버튼

|------|------|
| 등록 | 입력값 유효성 검사 후 출석 저장 |
| 취소 | 팝업 닫기, 입력값 초기화 |

### 3. 액션 / 흐름
1. 통합 출석 관리 화면 → '수동 출석 등록' 버튼 클릭 → 팝업 열림
2. 구분(회원/직원) 선택
3. 이름 또는 번호 입력 → 자동완성 목록에서 선택
4. 출석 시각 확인 (필요 시 수정)
5. 비고 입력 (선택)
6. 등록 버튼 클릭 → 성공 시 팝업 닫힘 + 출석 목록 즉시 갱신
7. 실패 시 인라인 오류 메시지 표시

### 4. 핵심 상태
| 상태 | 설명 |

빈 락커 목록을 번호 순으로 표시하며, 선택 가능한 락커를 클릭하여 배정한다.
