# D11-통합운영 통합운영 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.




## 1. 핵심 원칙
- D11은 CRM 운영 화면에서 발생·수신된 출석, 락커, 체성분, 건강 요약 데이터를 표시하는 도메인이다.
- 키오스크 설정은 D09-설정관리에서만 수행하고, 키오스크 운영 화면과 단말 관제는 KIOSK 기획문서에서 다룬다.
- IoT는 독립 운영 화면을 만들지 않고 CRM 화면에 상태 배지, 최근 수신 시각, 이벤트 결과로만 표시한다.
- CRM에서 사용하는 고정 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종이다.
- D11은 수업 생성·예약·횟수 차감·노쇼 정책을 정의하지 않는다. 해당 기준은 D04-수업관리를 따르고, D11은 수업/출석/락커/체성분에서 발생한 운영 결과를 통합 조회·처리하는 화면만 정의한다.
- 통합 출석은 회원 / 직원 / 게스트 출석 이벤트를 한 화면에서 조회하고, 수동 출석 등록과 당일 옷 락커 배정까지 연결한다.
- 락커 운영은 출석 연동 당일용 옷 락커와 상품 계약 기반 고정 물품 락커를 구분한다.
- 체성분은 InBody API 수신값과 운영자 수기 입력값을 조회·검수·확정하는 범위까지 다루며, 장비 제어는 제공하지 않는다.




## 2. 상태 / 운영 정의

- **만료 회원 출석 재결제 유도**: 만료된 회원이 키오스크에서 실패 처리되면 스태프가 즉시 회원 상세 이동 → 재결제 안내

| 항목 | 표시 기준 | 클릭/필터 동작 |
|---|---|---|
| 실패 건수 | 금일 `결과=실패` 건수. 만료, 미등록, 미수금 차단, 직원 출석 허용 OFF, 얼굴인식 시간 초과, 허용 시간 외를 포함한다. `결과=중복`은 실패 건수에 합산하지 않고 중복 건수로 별도 집계한다. | 결과=실패 필터 적용 |
| 락커 상태 | 전체 / 미배정 / 수동배정 / 자동배정 / 대상 아님 | 선택한 락커 상태 필터 적용 |
| 구분 | 회원 / 직원 / 게스트 배지 | 구분 필터 적용 |
| 회원 정보 | 사진 또는 기본 아바타, 이름, 회원번호, 성별/나이, 이용권, 만료일, 출석 시각, 출석 채널 | 회원 상세 또는 출석 상세로 이동 |

- **얼굴인식 도입 단계적 활성화**: 모듈 활성 + ka.face.stillSec 1초→2초 단계 조정으로 인식률 ↑

- **얼굴 데이터 미등록 회원 안내**: 얼굴 데이터 미등록 전용 TTS 문구로 얼굴 등록을 권유

- **설정 책임 분리**: 키오스크 화면 구성, TTS, 체크인 수단, 출입 규칙, 기기 등록은 D09-설정관리에서 처리한다.

- **IoT 표시 기준**: D11은 출입 게이트/키오스크/락커 컨트롤러/InBody 측정기의 상태를 OK / 불안정 / 오류 / 점검 중 배지와 마지막 수신 시각으로 표시한다. 상태 판정은 D09의 기기 헬스체크 정책을 따르며, 기본 주기는 5분이고 본사(primary/superAdmin)만 전역 기준을 변경할 수 있다.

- **InBody 수신 기준**: CRM은 InBody API로 측정 사용자, 측정 시각, 체성분 정보값만 수신한다. 관리자 CRM에서 InBody 측정 시작, 장비 설정, 파일 업로드, 장비 제어는 제공하지 않는다.

- **고정 기기 종류 4종**: 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기. 카드 리더, 얼굴인식 카메라는 별도 기기 종류가 아니라 인증 수단 또는 부속 모듈이다.

- 출석과 연동되는 당일용 옷 락커(SCR-I004 / `docs4/V1/D11-통합운영/통합운영.md:240`)와 상품 계약 기반 고정 물품 락커(SCR-I005 / `docs4/V1/D11-통합운영/통합운영.md:351`)를 통합 운영한다. 락커 그리드에서 사용 현황을 한눈에 파악하고, 미배정 회원 배정·회수·고장 처리·계약 만료 연장까지 처리한다.

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

## 6. 정책 결정 근거

- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.

- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.

- Health Connect는 키오스크/IoT 장비 연동이 아니라 회원앱 MA-159에서 회원이 직접 동의·연결하는 Android 선택 연동이다. SCR-I007의 Health Connect 영역은 admin 조회 전용이며, 키오스크/IoT 구현 범위로 해석하지 않는다.

- 키오스크 설정과 운영 관제는 D11에서 제외한다. 설정은 D09, 단말 운영 화면은 KIOSK 기획문서가 책임진다.

- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs4/_정책미확정_확인필요.md`에 별도 관리한다.
