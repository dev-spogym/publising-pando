# D03-매출관리 매출관리

> 개발사용 정리 문서입니다. 이 파일만으로 해당 도메인의 화면·다이얼로그 구현 기준을 확인할 수 있도록 재구성했습니다.




## 도메인 개요
- 화면/다이얼로그 구현 시 이 파일을 먼저 확인합니다.
- 본 문서가 해당 도메인 화면·다이얼로그 구현 기준입니다. 구현 판단을 위해 다른 폴더를 확인하지 않습니다.
- API/DB 상세는 백엔드 설계 범위이므로 제외하고, 화면/기능/UX/운영정책 기준만 남겼습니다.




## SCR-S001 매출 현황

> 연결 기능: SAL-01
### 1. 목적

완료된 결제 건에 대해 전체 취소 또는 일부 금액 환불을 요청·승인·처리합니다. 환불 자동 계산 산식은 미확정이므로 기사용 차감금, 위약금, 기환불 누계, 이번 환불 가능액, 최종 환불액은 모두 운영자가 수기로 입력하고 사유를 남깁니다. 실제 환불 실행은 Owner(지점장) 승인 후 가능하며, Owner(지점장)이 직접 처리하는 경우 승인대기 없이 바로 완료할 수 있습니다. 단, 수기 입력된 값은 환불 내역, 매출 차감, 정산 영향 요약, 인센티브 귀속 영향 계산에 사용합니다. 즉 V1은 환불 금액을 자동 산출하지 않지만, 운영자가 입력한 값을 기준으로 합계·차감·정산 표시를 계산합니다.

### 2. UI 구성

- **요약 지표 카드 (4개)**: 순 매출 / 카드 결제 / 현금 결제 / 미수금을 상단에 크게 표시합니다.

- **운영 코크핏**: 미수 추적 / 환불 검토 / 재등록 성과 / 고할인 거래를 우선순위 카드로 묶어 운영자가 지금 처리해야 할 매출 이슈를 first fold에서 확인합니다.

- **처리 큐**: 미수 규모, 환불 금액, 재등록 매출 영향도를 기준으로 상위 거래 3~4건을 따로 모아 보여주고 `거래 보기`, `회원 보기`를 즉시 실행할 수 있습니다.

- **기간 프리셋 버튼**: 오늘 / 이번 주 / 이번 달을 클릭하면 날짜 범위가 자동 설정됩니다.

- **검색 및 필터 바**: 구매자 이름 또는 상품명 통합 검색, 상품 대분류(회원권·수강권·락커·운동복·일반) 필터, 상태(완료·환불·미납) 필터, 결제경로(POS·링크·수기) 필터, 날짜 범위 직접 지정.

  - 담당자별 탭은 담당자별 총매출뿐 아니라 신규 매출, 재등록 매출, 기타 매출, 판매 건수, 평균 단가를 함께 표시합니다.

- **하단 요약 바**: 현재 필터 기준의 총매출 / 순매출 / 현금 / 카드 / 환불 / 미납금 / 할인 합계를 한 줄로 표시합니다.

- **매출 상세 모달**: 테이블 행을 클릭하면 결제일·상태·구매 정보·금액 정보·결제 내역·결제경로·링크 상태·카드 승인 정보·메모를 담은 모달이 열립니다. 모달 하단의 "회원 상세 보기" 버튼으로 해당 회원 페이지로 이동할 수 있습니다.

#### 하단 요약 바 7종 산출 기준

| 항목 | 산출 기준 |
|---|---|
| 총매출 | 현재 필터 결과의 할인·환불 차감 전 판매금액 합계 |
| 순매출 | 총매출 - 환불 합계 - 할인 합계 |
| 현금 | 현금으로 수납 완료된 금액 합계 |
| 카드 | 카드로 수납 완료된 금액 합계 |
| 환불 | 환불 완료 처리된 금액 합계. 환불 요청·승인대기 건은 제외 |
| 미납금 | 계약/결제 등록 후 아직 수납되지 않은 잔액 합계 |
| 할인 합계 | 운영자가 수동 확정한 할인 금액 합계. v1에서는 자동 할인 후보 금액을 포함하지 않음 |

### 3. 액션 / 흐름

3. 운영 코크핏 카드 중 `미수 추적`, `환불 검토`, `재등록 성과`를 눌러 필요한 탭과 필터 조합으로 즉시 이동합니다.

4. 처리 큐에서 영향도가 큰 거래를 먼저 보고 `거래 보기` 또는 `회원 보기`를 바로 실행합니다.

5. 기간 프리셋(오늘/이번 주/이번 달) 또는 날짜 직접 입력으로 조회 기간을 설정합니다.

6. 검색창에 구매자 이름이나 상품명을 입력하거나, 유형·상태 필터를 적용해 원하는 거래를 찾습니다.

7. 필요 시 결제경로 필터로 현장 결제, 링크 결제, 수기 반영 건을 구분합니다.

10. 담당자별 탭에서는 신규/재등록 매출 분리를 확인해 신규 획득과 기존 회원 유지 성과를 구분합니다.

11. 테이블 행을 클릭하면 해당 거래의 상세 정보를 모달로 확인합니다.

12. 필요 시 "신규 결제(POS)" 버튼으로 현장 판매 화면으로 이동합니다.

13. 엑셀 버튼으로 현재 조회 결과를 파일로 저장합니다.

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 로딩 중 | 데이터를 불러오는 동안 테이블 영역에 스켈레톤(뼈대 화면)을 표시합니다. |
| 정상 목록 | 조건에 맞는 거래가 1건 이상일 때 테이블과 하단 요약 바를 표시합니다. |
| 검색 결과 없음 | 필터·검색 적용 후 결과가 0건일 때 "검색 결과가 없습니다" 안내를 표시합니다. |
| 전체 데이터 없음 | 해당 기간에 매출 데이터가 없을 때 빈 화면 안내를 표시합니다. |
| 오류 | 데이터 조회에 실패하면 오류 메시지 알림(토스트)을 표시합니다. |
| 매출 상세 모달 열림 | 테이블 행 클릭 시 상세 정보 모달이 화면 위에 나타납니다. |

## SCR-S003 결제 처리

> 연결 기능: PAY-01
### 2. UI 구성

- **초기화 버튼**: 상단에 위치하며, 클릭 시 결제 상태를 초기 상태로 리셋합니다.

### 3. 액션 / 흐름

2. 좌측에서 상품 목록과 총 결제 금액을 확인합니다.

## SCR-S004 매출 통계

> 연결 기능: SAL-03
### 1. 목적

다양한 관점(상품별·상품 유형별·결제수단별·종목별·개월별 판매 비율·GX 세부 종목·법인권·직군별·담당자별)으로 매출 데이터를 시각적으로 분석하는 통계 화면입니다. Owner(지점장)과 매니저는 이 화면에서 어떤 상품이 잘 팔리는지, 어떤 결제수단이 주로 사용되는지, 단기/장기 회원권 판매 믹스가 어떤지, GX 하위 종목별 수요가 어떤지, 법인권(B2B) 매출이 어느 정도인지, 담당자별 신규/재등록 매출과 골프프로별 매출이 어떻게 나뉘는지를 차트와 표로 파악하고 운영 전략 수립에 활용합니다.

### 2. UI 구성

- **최상위 탭 (3개)**: 전체 통계 / 직군별 / 담당자별을 전환합니다. 담당자별 탭은 총매출, 신규 매출, 재등록 매출, 기타 매출, 판매 건수, 평균 단가를 함께 표시합니다.

- **날짜 필터 (전체 통계 탭)**: 이번 달·지난 달·최근 3개월·최근 6개월·올해 프리셋 버튼과 시작일~종료일 직접 입력, 조회 버튼을 제공합니다.

- **하위 탭 (전체 통계 탭)**: 상품별 / 상품타입별 / 결제수단별 / 종목별 / 개월별 / GX종목별 / 법인권을 선택합니다. 각 탭 선택에 따라 차트 유형이 달라집니다.

  - 상품별·종목별: 가로 막대 차트 (상위 10개)

  - 상품타입별·결제수단별: 도넛 차트

  - 개월별: 1개월·3개월·6개월·12개월·기타 판매 비율 막대 차트

  - GX종목별: 상품 마스터 `category=LESSON_PASS`, `lessonType=GX`, `subCategory` 기준의 요가·필라테스·스피닝·줌바·GX 기타 5종 분포 차트

  - 법인권: 레거시 `[월매출]` 자료의 `구분` 값이 `법인권`인 거래를 집계하는 가로 막대 차트 또는 요약 테이블

- **GX 세부종목 5종 분류**: 요가 / 필라테스 / 스피닝 / 줌바 / GX 기타. 이 5종은 상품 마스터의 `category=LESSON_PASS`, `lessonType=GX`, `subCategory` 기준으로 고정 집계하며, 강습 세션 유형(PT/GX/골프/기타)과 섞지 않습니다.

- **전월 대비 토글**: 켜면 현재 기간 통계 옆에 전월 동기 요약과 비교(증감액·증감률)가 추가 표시됩니다. 증가는 초록, 감소는 빨강으로 강조됩니다.

- **집계 테이블**: 분석 항목명 / 판매 건수 / 매출액 / 비율(%) 컬럼으로 구성됩니다. 담당자별 탭에서는 신규 매출, 재등록 매출, 기타 매출, 평균 단가 컬럼이 추가됩니다.

- **골프프로별 매출**: `[매출종합]` `골프매출` 탭의 `등록강사` 기준으로 프로별 골프 매출 합계와 비중을 표시합니다.

- **총 매출 표시**: 현재 필터 기준의 총 매출액을 차트 영역 상단에 표시합니다.

## SCR-S005 통계 관리

> 유형: 화면
> URL 경로: `/sales/statistics-management`
> 연결 기능: SAL-05

### 2. UI 구성

- **기간 입력**: 시작일~종료일을 직접 입력해 조회 범위를 설정합니다. 기본값은 이번 달 전체입니다.

- **필터 드롭다운 (4종)**: 수업 유형 / 담당자 / 구분 / 회원 상태 필터를 선택합니다. 각 드롭다운의 선택지는 기간·지점·계정 권한 범위에서 조회 가능한 데이터의 고유값으로 동적으로 생성됩니다.

  - 수업 유형: `전체` + PT / GX / 골프 / 기타 중 해당 데이터에 존재하는 값만 표시합니다.

  - 담당자: `전체` + 해당 데이터에 존재하는 담당자명을 표시하며, 담당자가 없는 행은 "미지정"으로 표시합니다.

  - 구분: `전체` + 통계 테이블의 `구분` 컬럼에 존재하는 값을 표시합니다. 매출현황 탭은 신규 / 재등록 / 휴면복귀 / 종목추가 / 업그레이드 / 기타 기준을 사용합니다.

  - 회원 상태: `전체` + 활성 / 만료 / 예정 / 임박 / 홀딩 / 미등록 / 탈퇴 중 해당 데이터에 존재하는 값만 표시합니다. 정지는 상태값이 아니라 제한 플래그이므로 포함하지 않습니다.

  - 동적 생성은 필터 옵션 등록/수정 기능이 아니라, 조회 가능한 데이터에서 드롭다운 선택지를 자동 구성하는 표시 방식입니다.

- **현황 탭 (4개)**: 수업현황 / 매출현황 / 예약현황 / 이용현황을 탭으로 전환합니다. 탭에 따라 동일한 데이터에서 해당 유형의 행만 걸러 보여줍니다.

- **통계 테이블**: 일자 / 담당자 / 회원명 / 성별 / 휴대폰 / 구분 / 판매금액 / 할인금액 / 받을금액(VAT 제외 순매출) / 카드 / 현금 / 계좌이체 / 마일리지(포인트) 사용액 / 미수발생 / 미수입금 / 받은금액 / 비고, 총 17개 컬럼을 제공합니다.

  - 상품권은 v1 계약 범위의 결제수단으로 정의되어 있지 않으므로 통계 컬럼에서 제외합니다.

  - 마일리지(포인트)는 카드/현금/계좌이체 같은 수납 행이 아니라, PAY-01 / `docs4/V1/D03-매출관리/매출관리.md:161` 결제 처리에서 별도 입력되는 사용액을 집계용으로 표시합니다.

- **VAT 제외 매출 매핑**: `판매금액`은 VAT 포함 총 판매금액, `받을금액`은 VAT 제외 순매출로 구분해 표시합니다. 레거시 `[월매출]` 자료의 `순매출 산출값`이 있으면 `받을금액`에 우선 매핑하고, 값이 없으면 VAT 포함 결제액에서 공급가액을 환산합니다. 이 항목은 `판도 스프레드시트 산식 참조`가 필요한 항목이며, 판도에서 `[월매출]`의 컬럼 정의와 산식 기준을 제공해야 자동 집계 기준으로 확정됩니다.

  - 일반 과세 상품: `VAT 포함 금액 ÷ 1.1`을 원 단위 반올림합니다.

  - 면세 상품: VAT가 0원이므로 판매금액과 받을금액을 동일하게 봅니다.

  - 이 항목은 VAT 포함/제외 금액을 구분하기 위한 표시·집계 기준이며, 별도 버튼이나 신규 업무 기능이 아닙니다.

- **합계 행**: 테이블 하단에 전체 건수와 각 금액 컬럼의 합계를 노란색 배경으로 표시합니다.

- **조회 버튼**: 설정한 기간과 필터로 데이터를 불러옵니다.

- **신규 버튼**: 신규 데이터를 등록하지 않고 모든 필터와 조회 결과를 초기화합니다. 기간은 이번 달, 필터는 `전체`, 현황 탭은 기본 탭(수업현황), 테이블은 빈 상태로 되돌립니다.

- **종료 버튼**: 이전 화면으로 돌아갑니다. 직전 화면 이력이 없으면 매출관리 > 매출현황(SCR-S001 / `docs4/V1/D03-매출관리/매출관리.md:21`)으로 이동합니다.

- 위 액션 버튼은 통계 관리 화면의 조회/초기화/CSV/복귀 조작입니다. 급여 입력, 매출 입력, 결제 처리 같은 업무 입력 액션과 연결하지 않습니다.

| 액션 버튼 | 결과 | 제한 |
|---|---|---|
| 조회 | 현재 기간·필터·탭 조건으로 테이블, 합계 행, 드롭다운 옵션 갱신 | 시작일이 종료일보다 늦으면 오류 안내 |
| 신규 | 기간/필터/탭/결과 초기화. 신규 데이터 생성 없음 | 조회 중 비활성 |
| 종료 | 직전 화면 복귀, 이력 없으면 SCR-S001 / `docs4/V1/D03-매출관리/매출관리.md:21` 매출현황 이동 | 복귀 실패 시 토스트 표시 |

### 3. 액션 / 흐름

1. 사이드바에서 "매출관리 > 통계 관리"를 클릭해 진입합니다.

2. 시작일~종료일을 입력하거나 기본값(이번 달)을 그대로 사용합니다.

3. 수업 유형·담당자·구분·회원 상태 드롭다운으로 원하는 조건을 설정합니다.

4. "조회" 버튼을 눌러 데이터를 불러옵니다.

5. 수업현황·매출현황·예약현황·이용현황 탭을 전환하며 원하는 뷰를 확인합니다.

6. 하단 합계 행에서 건수와 VAT 제외 순매출을 포함한 금액 총계를 확인합니다.

7. "엑셀" 버튼으로 현재 결과를 CSV 파일로 내려받습니다.

8. 조건을 초기화하려면 "신규" 버튼을 누릅니다.

## SCR-S009 할부결제 관리

> 유형: 화면
> URL 경로: `/sales/installment`
> 연결 기능: SAL-EXT-01
> 연결 다이얼로그: DLG-S007 / `docs4/V1/D03-매출관리/매출관리.md:1708` 할부상세, DLG-S008 / `docs4/V1/D03-매출관리/매출관리.md:1814` 납입처리, DLG-S009 / `docs4/V1/D03-매출관리/매출관리.md:1890` 할부등록

### 1. 목적

회원이 분할 납부 방식으로 결제한 할부 계약의 전체 현황을 한눈에 파악하고, 매월 납입 현황을 추적하며 미납된 회차를 관리하는 화면입니다. 운영자는 이 화면을 통해 어느 회원이 몇 회차를 납부했고 남은 금액이 얼마인지 즉시 확인하고, 미납 회차에 대한 후속 조치를 취할 수 있습니다. 이 화면은 `정기 분납 계획` 전용이며 계약금만 받고 잔액을 단기 미수금으로 남긴 건은 SCR-S008에서 우선 관리합니다.

### 2. UI 구성

- **요약 지표 카드 (4개)**: 진행 중인 할부 계약 수 / 이번 달 납입 예정 건수 / 이번 달 납입 완료 건수 / 미납 총액을 상단에 표시합니다. 각 카드는 클릭 시 같은 화면의 계약 목록 필터를 자동 적용합니다.

- **상태 탭 4개**: 전체 / 진행중 / 완납 / 미납. 진행중·완납·미납은 서로 중복되지 않으며, 미납 회차가 1건 이상 있으면 미납 탭에 우선 분류합니다.

- **검색 및 필터**: 회원명 검색, 기간 범위 필터, 계약 출처 필터를 제공합니다.

- **할부 계약 목록 테이블**: 회원명 / 상품명 / 계약 출처 / 선납금 / 총 할부 금액 / 납입 완료 금액 / 잔여 금액 / 총 회차 / 납입 회차 / 다음 납입일 / 상태, 총 11개 컬럼을 제공합니다.

- **회차별 납입 현황**: 계약 행을 클릭하거나 펼치면 회차별(1회·2회·3회 등) 납입 예정일·납입 완료일·금액·상태를 상세히 확인할 수 있습니다.

- **미납 관리**: 납입 기한이 지난 미납 회차에 경고 표시를 하고, 납입 처리 버튼으로 상태를 업데이트할 수 있습니다.

- **할부 등록 버튼**: Owner(지점장) / manager 이상에게만 [+ 할부 등록] 버튼을 노출합니다. 클릭 시 `할부 등록` 다이얼로그가 열리고, 회원·상품·총 할부 금액·총 회차·첫 납입일을 입력해 직접 등록 할부 계약을 생성합니다.

#### 요약 카드 클릭 필터

| 카드 | 클릭 시 적용 필터 | 목록 표시 결과 |
|---|---|---|
| 진행 중인 할부 계약 수 | 상태 탭 `진행중` | 계약 상태가 진행중인 할부 계약만 표시 |
| 이번 달 납입 예정 건수 | `이번 달 납입 예정` 필터 칩 | 이번 달에 예정 상태 회차가 1개 이상 있는 계약만 표시 |
| 이번 달 납입 완료 건수 | `이번 달 납입 완료` 필터 칩 | 이번 달에 완료 처리된 회차가 1개 이상 있는 계약만 표시 |
| 미납 총액 | 상태 탭 `미납` | 납입 기한이 지난 미납 회차가 1개 이상 있는 계약만 표시 |

카드 클릭은 화면 이동이 아니라 목록 필터를 세팅하는 동작입니다. 회원명 검색·계약 출처 필터는 유지하고, 상태 탭·월 기준 필터는 클릭한 카드 기준으로 갱신합니다. `전체` 탭 또는 [필터 초기화]를 누르면 카드 클릭 필터가 해제됩니다.

#### 상태 탭 기준

| 탭 | 포함 기준 | 표시 기준 |
|---|---|---|
| 전체 | 권한 범위 내 정기 분납 계약 전체 | 환불 완료로 해지된 계약, 삭제/무효 처리된 계약은 제외 |
| 진행중 | 잔여 회차가 있고 현재 미납 회차가 없는 계약 | 예정 회차가 남아 있는 정상 납입 계약 |
| 완납 | 모든 회차가 완료이고 잔여 금액이 0원인 계약 | 중도 상환으로 잔여 회차가 일괄 완납된 계약 포함 |
| 미납 | 납입 예정일 + 1일이 지나 미납 상태인 회차가 1건 이상 있는 계약 | 미납 회차가 있으면 진행중이 아니라 미납 탭에 표시 |

탭 옆 배지는 위 상태 탭 조건에 해당하는 계약 수를 표시합니다. 환불 진행 중인 계약은 조회는 가능하지만 납입 처리는 차단하고, 환불 완료로 해지된 계약은 기본 운영 대상에서 제외합니다.

### 3. 액션 / 흐름

1. 사이드바에서 "매출관리 > 할부결제 관리"를 클릭해 진입합니다.

2. 상단 지표 카드로 이번 달 납입 예정·완료·미납 현황을 파악합니다.

3. 상태 탭을 선택해 진행중·완납·미납 계약을 구분해 봅니다.

4. 회원명 검색으로 특정 회원의 할부 계약을 찾습니다.

5. 계약 출처와 선납금을 함께 확인해 현장 계약금 연계 건인지 구분합니다.

6. 계약 행을 선택해 회차별 납입 현황 상세를 확인합니다.

7. 미납 회차에 납입 처리 버튼으로 상태를 업데이트합니다.

8. 신규 정기 분납 계약이 필요하면 [+ 할부 등록] 버튼으로 할부 등록 다이얼로그를 열어 등록합니다.

9. 엑셀 버튼으로 현재 목록을 파일로 저장합니다.

## SCR-S010 세금계산서 발행

> 유형: 화면
> URL 경로: `/sales/invoice`
> 연결 기능: SAL-EXT-02
> 연결 다이얼로그: DLG-S010 / `docs4/V1/D03-매출관리/매출관리.md:2003` 세금계산서상세, DLG-S011 / `docs4/V1/D03-매출관리/매출관리.md:2105` 세금계산서발행

### 1. 목적

법인 회원 또는 사업자 회원에게 세금계산서를 발행하고, 발행 이력을 조회·관리하는 화면입니다. 세금계산서 발행이 필요한 거래를 선택해 발행 처리하고, 발행된 세금계산서의 상태(발행 완료·전송 완료·오류)를 추적합니다. 회계 마감 및 세무 신고에 필요한 발행 이력을 한 곳에서 관리할 수 있습니다.

### 2. UI 구성

- **요약 지표 카드 (3개)**: 이번 달 발행 건수 / 이번 달 발행 총액 / 미발행 대기 건수를 상단에 표시합니다.

- **발행 탭 / 이력 탭**: 발행 탭에서 세금계산서 발행을 처리하고, 이력 탭에서 기발행된 세금계산서 목록을 조회합니다.

- **발행 대상 조회**: 세금계산서 발행이 필요한 결제 건을 기간·상태 필터로 조회합니다.

- **세금계산서 발행 폼**: 공급받는 자의 사업자번호·상호·대표자·업태·종목·이메일을 입력하고, 발행 대상 결제 건에서 자동 채워진 공급 품목(상품명·수량·VAT 제외 단가·공급가액)을 확인한 뒤 발행합니다.

- **발행 이력 테이블**: 발행일 / 공급받는 자 / 공급가액 / 부가세 / 합계 / 상태 / 액션, 총 7개 컬럼을 제공합니다. 상태 배지로 발행 완료·전송 완료·오류·취소 발행을 구분합니다.

- **이메일 전송**: 발행 완료된 세금계산서를 공급받는 자 이메일로 전송합니다.

- **취소 / 재발행 모달**: 발행완료 건은 취소, 취소 건은 재발행으로 전환할 수 있습니다.

#### 공급 품목 자동 채움 기준

| 항목 | 자동 채움 기준 | 표시 기준 |
|---|---|---|
| 상품명 | 선택한 발행 대상 결제 건의 상품명 | 복수 상품 결제는 상품별 행으로 표시 |
| 수량 | 결제 상품 수량 | 회원권·수강권·락커 등 수량 개념이 없는 상품은 1 |
| 단가 | 품목별 최종 판매금액을 수량으로 나눈 VAT 제외 단가 | 할인 반영 후 금액 기준 |
| 금액 | 단가 × 수량 | 품목별 공급가액 |
| 과세 구분 | 상품 과세/면세 설정 또는 원결제 세액 기준 | 면세 품목은 부가세 0원 |

카드·현금·계좌이체·포인트 같은 결제수단은 공급 품목으로 만들지 않습니다. 마이너스 세금계산서는 원 세금계산서의 품목 행을 기준으로 음수 금액을 자동 채움합니다.

#### 발행 이력 7개 컬럼

| 컬럼 | 표시 기준 | 클릭/동작 |
|---|---|---|
| 발행일 | 세금계산서 발행 기준일 | 정렬 가능 |
| 공급받는 자 | 거래처 상호 + 사업자번호 마스킹 | 텍스트 표시 |
| 공급가액 | 부가세 제외 공급가액 합계 | 천 단위 콤마 |
| 부가세 | 과세 품목 공급가액의 10%, 면세 품목은 0원 | 천 단위 콤마 |
| 합계 | 공급가액 + 부가세 | 천 단위 콤마 |
| 상태 | 발행 완료 / 전송 완료 / 오류 / 취소 발행 | 상태 배지 표시 |
| 액션 | [상세] / [이메일 전송] / [재발행] | [상세]는 세금계산서 상세 다이얼로그를 열고, [이메일 전송]은 등록 이메일로 재전송, [재발행]은 오류·취소 대상만 가능 |

[상세] 클릭은 선택한 세금계산서의 상세 내용을 확인하는 기능입니다. 세금계산서 상세 다이얼로그에서 세금계산서 번호, 발행일시, 공급자/공급받는자 정보, 공급 품목, 공급가액, 부가세, 합계, 이메일 전송 이력을 확인합니다.

### 3. 액션 / 흐름

1. 사이드바에서 "매출관리 > 세금계산서 발행"을 클릭해 진입합니다.

2. 상단 지표 카드로 이번 달 발행 현황과 미발행 대기 건수를 파악합니다.

3. 발행 탭에서 발행 대상 결제 건을 선택합니다.

4. 발행 폼에서 공급받는 자 정보와 품목 내역을 확인·입력하고 발행 버튼을 누릅니다.

5. 발행 완료 후 이메일 전송 버튼으로 상대방에게 세금계산서를 전달합니다.

6. 이력 탭에서 기발행 목록을 조회하고 상태를 확인합니다.

7. 엑셀 버튼으로 이력을 파일로 저장합니다.

## SCR-S011 매출 예측

> 연결 기능: 없음 (V1 범위 직접 연결 기능 없음; 공통/보조 화면 또는 V2 기능은 V2 문서에서 관리)

- **예측 기간 선택**: 다음 달·다음 분기·연간 예측 중 원하는 기간을 선택합니다.

- **매출 목표 설정**: 기간별 매출 목표 금액을 직접 입력하면 달성률이 실시간으로 업데이트됩니다.

- **매출 예측 차트**: 과거 실적(선 그래프)과 예측값(점선 그래프)을 하나의 차트에 겹쳐 표시합니다. 목표 라인도 함께 표시됩니다.

- **달성률 게이지**: 현재 누적 매출이 목표의 몇 퍼센트인지 프로그레스 바와 숫자로 표시합니다.

- **상품별 예측 기여도**: 어떤 상품이 향후 매출에 가장 큰 기여를 할 것으로 예측되는지 순위 목록으로 제공합니다.

- **월별 예측 테이블**: 월별 예측 매출·목표·달성률·전년 동기 대비 항목을 표 형태로 제공합니다.

### 3. 액션 / 흐름

2. 상단 지표 카드로 이번 달 예측 매출과 목표 달성률을 확인합니다.

3. 예측 기간(다음 달·분기·연간)을 선택합니다.

4. 매출 목표 금액을 입력해 달성률 게이지가 변하는 것을 확인합니다.

5. 예측 차트에서 과거 추세와 예측 궤적을 비교합니다.

6. 상품별 기여도 목록으로 어느 상품에 집중해야 하는지 파악합니다.

## SCR-S012 결제 취소 / 부분 환불

> 연결 기능: SAL-EXT-04
### 2. UI 구성

- **결제 건 검색**: 회원명·상품명·결제일로 환불 처리할 결제 건을 조회합니다.

- **결제 내역 확인**: 선택한 결제 건의 결제일·상품명·결제금액·결제수단·결제자 정보를 확인합니다.

- **환불 수기 입력 박스**: 원결제금액 / 기사용 차감금 / 위약금 / 기환불 누계 / 이번 환불 가능액 / 최종 환불액을 표시합니다. 원결제금액을 제외한 계산 항목은 모두 담당자가 수기로 입력합니다.

- **정책 미확정 항목 수기 입력**: 사용분 차감 기준, 위약금, 기환불 누계, 이번 환불 가능액, 최종 환불액, 조정 사유를 입력합니다. 자동 계산식이 확정되기 전까지 운영 판단 값을 기록하는 정책 입력 영역입니다.

- **처리 유형 선택**: "전체 취소"와 "부분 환불" 중 하나를 선택합니다.

- **환불 사유 입력**: 단순 변심·결제 오류·회원 요청·중복 결제·서비스 불만 등 프리셋 사유를 선택하거나 직접 입력합니다.

- **귀속 영향 요약**: 결제지점, 이용지점, 매출 귀속 지점, 정산 지점, 인센티브 귀속자에 대한 환불 차감 영향을 요약 표시합니다. 다중 귀속 정책 확정 전에는 각 항목을 운영자가 수기로 입력합니다. 시스템은 입력된 귀속값과 환불 수기 입력값을 기준으로 매출 차감액, 정산 차감액, 인센티브 차감 대상을 계산해 표시합니다.

- **승인 상태 입력**: 처리 상태를 요청 / 승인대기 / 완료로 기록합니다. manager/fc가 작성한 건은 승인대기 상태가 되며, Owner(지점장)이 승인자·승인 메모를 남깁니다.

- **처리 이력**: 해당 결제 건에 대해 이전에 처리된 환불 이력이 있으면 하단에 목록으로 표시합니다.

2. 회원명·상품명·결제일로 환불 처리할 결제 건을 검색합니다.

## DLG-S009 할부 등록

| 선납금 | 이미 수납된 금액. 직접 등록은 0원 기본값으로 입력 가능, 결제 연계는 계약금 자동 반영 |

## DLG-S011 세금계산서 발행

> 유형: 다이얼로그
> 연결 기능: SAL-EXT-02
> 연결 다이얼로그: DLG-S010 / `docs4/V1/D03-매출관리/매출관리.md:2003` 세금계산서상세, DLG-S011 / `docs4/V1/D03-매출관리/매출관리.md:2105` 세금계산서발행

### 1. 목적

특정 매출 건에 대한 세금계산서를 발행합니다. 공급받는자의 사업자 정보와 이메일을 입력하고, 선택한 결제 건의 상품 라인을 공급 품목으로 자동 채워 세금계산서를 생성하고 전송합니다.

### 2. UI 구성

| 구성요소 | 내용 |
|---|---|
| 원 매출 정보 | 발행 대상 매출 번호, 결제일, 금액 (읽기 전용) |
| 공급받는자 사업자번호 | 거래처 사업자등록번호 입력 |
| 공급받는자 상호 | 거래처 법인명 또는 상호 입력 |
| 대표자명 | 거래처 대표자 이름 입력 |
| 업태·업종 | 거래처 업태 및 업종 입력 |
| 이메일 주소 | 세금계산서를 수신할 이메일 주소 입력 |
| 발행 일자 | 세금계산서 발행 기준일 (기본값: 오늘) |
| 공급 품목 | 선택한 결제 건의 상품명, 수량, VAT 제외 단가, 공급가액을 품목 행으로 자동 표시 |
| 공급가액 | 부가세 제외 금액 (자동 계산) |
| 부가세액 | 공급가액의 10% (자동 계산) |
| 발행 버튼 | 입력 내용으로 세금계산서 발행 및 이메일 전송 |
| 취소 버튼 | 발행을 취소하고 이전 화면으로 복귀 |

### 3. 액션 / 흐름

1. 매출 상세 또는 세금계산서 발행 화면에서 발행 버튼을 선택합니다.

2. 세금계산서 발행 다이얼로그가 열리며 원 매출 금액과 공급 품목 행이 자동 입력됩니다.

3. 공급받는자 정보(사업자번호, 상호, 이메일 등)를 입력합니다.

4. 공급 품목의 상품명·수량·VAT 제외 단가·공급가액을 확인합니다.

5. 공급가액과 부가세액이 자동 계산되어 표시됩니다.

6. 발행 버튼을 눌러 세금계산서를 발행하고 이메일로 전송합니다.

7. 발행 완료 후 DLG-S010 / `docs4/V1/D03-매출관리/매출관리.md:2003`(세금계산서 상세)에서 결과를 확인할 수 있습니다.

### 3-1. 공급 품목 자동 채움 기준

| 항목 | 자동 채움 기준 | 표시 기준 |
|---|---|---|
| 상품명 | 발행 대상 결제 건의 상품명 | 복수 상품 결제는 상품별 행으로 표시 |

## DLG-S013 환불 처리

> 연결 기능: PAY-02, SAL-EXT-04
### 1. 목적

## DLG-S015 환불 요청

> 연결 기능: PAY-02, SAL-EXT-04
### 1. 목적

회원 또는 직원이 환불을 요청하는 첫 번째 단계 팝업입니다. 환불 대상 매출 건과 요청 사유를 입력하여 환불 요청을 접수합니다. 실제 환불 처리는 담당자 확인 후 환불 처리 전 최종 확인 팝업에서 별도로 진행됩니다.

### 2. UI 구성

| 구성요소 | 내용 |

| 요청자 구분 | 요청자가 회원인지 직원인지 구분 표시 |
