# D03-매출관리 매출관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.




## 1. 핵심 원칙
- 상품 판매부터 계약, 결제, 이용권 생성, 환불까지를 하나의 운영 흐름으로 정리한다.
- 매출 화면과 회원 상세 화면이 서로 다른 상태를 말하지 않도록 기준을 통일한다.
- 상품 관리
- 할인 정책
- POS 판매
- 결제 처리
- 매출 현황
- 환불 관리
- 미수금 관리
- 선수익금 조회
- 회원권
- 수강권
- 락커
- 운동복
- 일반 상품
- 수강권은 PT / GX / 골프 / 기타 수강권을 세부 유형으로 가진다.
- GX 수강권은 2단계 세부종목을 필수로 가진다.
- 패키지는 상품 대분류가 아니라 상품 구성/판매 옵션으로 표시한다.




## 2. 상태 / 운영 정의

- **카운터 매니저 일반 회원권 등록**: POS에서 1년 회원권 장바구니 구성 → 외부 POS 카드 결제 완료 → 영수증 첨부 → 회원권 즉시 활성

- 영수증 파일 첨부는 현장 판매 결제 등록의 필수 조건이다. 첨부가 없으면 결제 완료 등록 버튼은 활성화되지 않는다.

- `혼합결제 행 합계 + 포인트 사용액 = 장바구니 최종 합계`가 정확히 일치해야 결제 완료 등록이 가능하다. 불일치하면 차액을 붉은색으로 표시하고 버튼을 비활성화한다.

| 환불/취소 상태 배지 | 표시 색상 | 의미 |
|---|---|---|
| 요청 | 회색 | 환불/취소 요청이 등록된 상태 |
| 승인대기 | 노랑 | Owner(지점장) 승인 또는 확인을 기다리는 상태 |
| 완료 | 초록 | 환불/취소 처리가 완료된 상태 |
| 반려·거절 | 빨강 | 승인자가 반려했거나 처리할 수 없는 상태 |

### 화면 상태

### 상태 탭 (5개)

- 전체 / 미결제 / 일부결제 / 연체 / 완료. 각 탭에 건수 배지.

- **만료/알림**: 링크 유효기간은 발송일시 + 7일 고정. 만료 D-3/D-1에 회원과 운영자에게 자동 알림을 발송하며 ON/OFF는 제공하지 않는다.




## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.




## 4. KPI / 성과 측정
| KPI | 산식/의미 | 측정 주기 |
|---|---|---|
| 처리 완료율 | 완료 건수 / 대상 건수 | 일/주 단위 |
| 예외 발생률 | 예외·차단 건수 / 전체 시도 건수 | 주 단위 |
| 정책 준수율 | 권한·상태·필수값 위반 없이 처리된 비율 | 월 단위 |
| 운영 응답률 | 알림·요청 후 운영자가 처리한 비율 | 주 단위 |




## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs4/_정책미확정_확인필요.md`에 별도 관리한다.
