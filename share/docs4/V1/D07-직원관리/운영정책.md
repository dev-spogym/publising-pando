# 운영정책




# D07-직원관리 직원관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.




## 1. 핵심 원칙

- 본사와 지점, 관리자 웹과 회원앱의 권한 구조를 실제 운영 기준으로 정리한다.

- 문서마다 달랐던 역할 명칭을 운영 기준으로 하나로 맞춘다.

- 관리자 웹 표준 역할 코드는 `superAdmin`, `primary`, `owner`, `manager`, `fc`, `trainer`, `staff`, `readonly` 8개를 기준으로 한다. `owner`의 화면 표시명은 `Owner(지점장)`으로 통일한다.

- `trainer`는 관리자 웹 기본 역할이다. PT트레이너, GX강사, 골프강사는 직원 직무값이며 기본 권한 템플릿은 `trainer`로 지정한다.

- 카운터/현장 지원 담당자는 별도 표준 역할로 두지 않고 `staff`에 통일한다.

- 권한은 "페이지 접근", "조회", "수정", "승인", "일괄처리"로 나눠서 본다.

- 전사 권한 템플릿

- 공통 상품 마스터

- 자동화 정책 세트

- 알림 템플릿

- 공지 템플릿

- KPI 및 감사 기준

- 회원/리드 실제 운영

- 현장 결제와 미수금 처리

- 수업 개설과 스케줄 운영

- 시설 배정

- 본사 정책 범위 안에서의 적용 설정

- 전사 환불 규모는 환불 집중 지점을 조기 포착하는 경고 지표로 본다.

- 직원 지각 기준은 D09 SCR-086 / `docs4/V1/D09-설정관리/설정관리.md:865` 출석 관리 설정의 직원 근태 기준에서 설정한 지점별 직원 지각 허용 시간을 따른다. 기본값은 10분이며, 정규 출근 시각 + 10분까지는 정상, 10분 초과 출근부터 지각으로 분류한다. 소속 지점 Owner(지점장)이 설정값을 변경할 수 있다.

- 직원 근태 지각 기준은 수업/회원 출석의 지각·노쇼 기준과 별도 정책값이다. 수업 지각 기준을 직원 출퇴근 자동 분류에 재사용하지 않는다.

- 직원 등록/수정, 재직 상태 변경, 퇴사 처리, 근태 수동 보정, 급여 수당·공제 편집, 급여 확정/취소, 급여 명세서 무효화는 SCR-097 / `docs4/V1/D10-본사관리/본사관리.md:386` 히스토리 로그 기록 대상이다. 변경 이력에는 변경자, 대상 직원, 변경 전 값, 변경 후 값, 변경 사유, 변경 일시를 남기며, D07에서는 변경을 수행하고 D10 SCR-097 / `docs4/V1/D10-본사관리/본사관리.md:386` 히스토리 로그에서 조회한다.




## 2. 상태 / 운영 정의

- 직원 근태 상태 배지는 정상, 지각, 결근, 조퇴, 외근, 휴가를 사용한다.

- 근태 기록 출처 배지는 키오스크, IoT, 수동 보정, 누락 추가를 사용한다.

- 근태 목록 테이블 8개 컬럼: 날짜 / 직원명 / 출근 시각 / 퇴근 시각 / 근무 시간 / 상태 / 출처 / 보정 이력.

- 매니저는 정당 사유가 있는 근태 기록을 외근 또는 휴가로 보정할 수 있으며, 보정 사유와 보정 이력을 반드시 남긴다.

- 보정 이력은 `보정 N회` 배지로 표시하고, 매니저+가 hover하면 변경 전/후 시각, 변경 상태, 보정 사유, 보정자, 보정 일시를 확인할 수 있다.

- **자동 수집 데이터 + 수동 보정 충돌**: 수동 보정 우선 적용 + "수동 보정됨" 배지

- **전월 급여 조회 (read-only)**: 시스템 날짜 기준 현재 월보다 이전 지급월은 마감 월로 잠그고 조회만 허용한다.

- 급여를 처리할 대상 월 선택. 기본값은 시스템 날짜 기준 당월이며, 미래 월은 선택할 수 없다. 전월 이전 월은 선택 가능하지만 잠금 아이콘과 `마감` 배지를 표시하고 read-only로 제공한다.

- 직원 목록의 역할/직무 표시는 `역할` 컬럼과 `직무 배지`로 나누어 표시한다. 역할은 권한 템플릿이며, 직무 배지는 PT트레이너·GX강사·골프강사 등 실제 업무 구분이다.

- 급여 확정 상태 배지는 `미확정`과 `확정`만 사용한다.

- 이미 확정된 급여도 당월 내에는 확정 취소 후 재편집 가능하다. 전월 이전 급여는 편집, 확정, 확정 취소, 자동 재계산, 급여 정책 재적용을 차단한다.

- 수당/공제 편집은 당월 미확정 급여에서만 가능하다. 자동 생성된 기본급, 자동 수당, 자동 공제, 수수료 원본 항목은 직접 덮어쓰지 않고, 보정이 필요하면 수동 수당 또는 수동 공제 항목으로 추가한다.

- 수동 수당 항목은 식대, 교통비, 특별 수당, 성과 수당, 기타 수당을 사용한다. 수동 공제 항목은 지각·결근 공제, 4대 보험 안내 항목, 선지급·가불 차감, 기타 공제를 사용한다.

- 수동 수당/공제 저장 시 항목명, 금액, 변경 사유가 필수이며, 저장 후에도 급여 상태는 미확정으로 유지된다. 확정은 별도의 급여 확정 액션으로 처리한다.

- 급여 확정 취소 시 기존 급여 명세서는 CRM 내부 `무효` 상태로 전환한다. 무효 명세서는 발송, 재발송, PDF 저장, 인쇄를 차단하고, 기존 CRM/앱 링크 접근 시 무효 안내만 표시한다. 이미 발송된 외부 이메일·앱 푸시 자체는 회수하거나 삭제하지 않는다.

- 재확정 시 기존 무효 명세서는 보존하고 새 명세서를 별도 생성한다. 새 명세서 발송은 자동 실행하지 않고 SCR-065에서 운영자가 [발송] 또는 [일괄 발송]으로 실행한다.

- PAY-STF-03 / `docs4/V1/D07-직원관리/직원관리.md:687` 01은 조회 기간 선택 기능이며 재발송 안내를 포함하지 않는다. 재발송 안내는 이미 발송된 명세서에서 [발송]을 다시 누를 때 표시하는 확인 문구이고, 기존 발송 이력 유지와 새 발송 이력 추가를 안내한다.

- 급여 명세서 목록 9개 컬럼: 직원명 / 역할 / 지급 월 / 기본급 / 실지급액 / 발송 상태 / 발송 일시 / 수신 확인 / 액션.

- 발송 상태 배지는 `미발송`, `발송완료`, `무효` 3종으로 고정한다. `미발송`은 발송 가능, `발송완료`는 재발송 가능, `무효`는 발송·재발송·PDF 저장·인쇄를 차단한다.

- 수신 확인은 발송 전 `-`, 발송 후 `미열람`, 직원 열람 시 `열람완료`, 무효 명세서·만료 링크·추적 불가 채널은 `확인불가`로 표시한다. `열람완료`는 열람 일시를 함께 표시한다.

- 발송 일시는 마지막 성공 발송 일시를 표시한다. 재발송 시 마지막 발송 일시는 갱신하되 발송 일시·수신자·채널·상태(성공/실패) 이력은 누적 보존하며 매니저 이상이 조회할 수 있다.

- PAY-STF-03-06 / `docs4/V1/D07-직원관리/운영정책.md:121` 일괄 발송 진행 상태는 발송 대기, 발송 중, 완료, 일부 실패, 실패로 표시한다.

- 센터에 소속된 모든 직원의 현황을 한눈에 파악하고, 직무·재직 상태별로 빠르게 찾을 수 있는 직원관리 도메인의 핵심 허브 화면(SCR-060 / `docs4/V1/D07-직원관리/직원관리.md:22`)이다. 본사 관계자가 `전체 지점(통합)`을 선택한 경우에는 전 지점 직원이 합산 조회되며 각 직원의 소속 지점이 함께 표시된다. Owner(지점장)은 소속 지점 인력 현황을 조회하고 신규 직원을 등록하며, 매니저는 특정 직원에게 메시지를 전송하거나 권한 범위 내 상태 변경을 진행한다. 퇴사 처리 실행·확정은 `Owner(지점장)+`만 가능하며 manager 이하에게는 [퇴사 처리] 버튼과 퇴사 처리 화면 직접 접근을 제공하지 않는다. 이 화면에서 다루는 값은 직원의 `직무`이며, 실제 관리자웹 메뉴 접근 권한은 SCR-081 / `docs4/V1/D09-설정관리/설정관리.md:268` 권한 설정에서 별도로 매핑한다.

- **퇴사 일괄 처리**: `Owner(지점장)+`가 계약 만료 5명 동시 퇴사 → 다중 선택 → [퇴사 처리] → 퇴사 처리 확인 다이얼로그에서 "퇴사처리" 입력 → 퇴사일·사유·직원별 인수인계·계정 비활성화 입력 단계 진행

- 요약 카드는 클릭으로 재직 상태 필터를 자동 적용해 단계별 드릴다운을 지원한다. 본사 통합 모드에서는 직원 목록의 `소속 지점` 컬럼에 직원별 실제 지점명을 표시하고, 지점 필터와 지점별 합산 통계를 함께 제공해 다지점 운영 가시성을 확보한다.

| 요약 카드 | 산정 기준 | 클릭 동작 |
|---|---|---|
| 재직 중 인원 | `status=active` 직원 수 | 재직 필터 자동 적용 |
| 휴직 중 인원 | `status=onLeave` 직원 수 | 휴직 필터 자동 적용 |




## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.
