# D07-직원관리 직원관리

> 개발사용 정리 문서입니다. 이 파일만으로 해당 도메인의 화면·다이얼로그 구현 기준을 확인할 수 있도록 재구성했습니다.




## 도메인 개요
- 화면/다이얼로그 구현 시 이 파일을 먼저 확인합니다.
- 본 문서가 해당 도메인 화면·다이얼로그 구현 기준입니다. 구현 판단을 위해 다른 폴더를 확인하지 않습니다.
- API/DB 상세는 백엔드 설계 범위이므로 제외하고, 화면/기능/UX/운영정책 기준만 남겼습니다.
- 직원 등록/수정, 재직 상태 변경, 퇴사 처리, 근태 수동 보정, 급여 수당·공제 편집, 급여 확정/취소, 급여 명세서 무효화는 SCR-097 / `docs4/V1/D10-본사관리/본사관리.md:386` 히스토리 로그 기록 대상입니다. D07은 직원·근태·급여 변경을 수행하고, 변경 이력 조회는 D10 SCR-097 / `docs4/V1/D10-본사관리/본사관리.md:386` 히스토리 로그에서 수행합니다.




## SCR-060 직원 목록

> 유형: 화면
> URL 경로: `/staff`
> 연결 기능: STF-01, STF-02
> 연결 다이얼로그: DLG-060-001 / `docs4/V1/D07-직원관리/직원관리.md:856`, DLG-060-002 / `docs4/V1/D07-직원관리/직원관리.md:953`

### 1. 목적

센터에 소속된 모든 직원의 현황을 한눈에 파악하고, 직무·재직 상태별로 빠르게 찾을 수 있는 직원관리 도메인의 핵심 허브 화면입니다. 본사 관계자가 `전체 지점(통합)`을 선택한 경우에는 전 지점 직원이 합산 조회되며 각 직원의 소속 지점이 함께 표시됩니다. Owner(지점장)은 소속 지점 인력 현황을 조회하고 신규 직원을 등록하며, 매니저는 특정 직원에게 메시지를 전송하거나 권한 범위 내 상태 변경을 진행합니다. 퇴사 처리 실행·확정은 `Owner(지점장)+`만 가능하며 manager 이하에게는 [퇴사 처리] 버튼과 퇴사 처리 화면 직접 접근을 제공하지 않습니다. 이 화면에서 다루는 값은 직원의 `직무`이며, 실제 관리자웹 메뉴 접근 권한은 SCR-081 / `docs4/V1/D09-설정관리/설정관리.md:268` 권한 설정에서 별도로 관리합니다.

### 2. UI 구성

- **요약 지표 카드 (4개)**: 전체 직원 수 / 재직 중 인원 / 휴직 중 인원 / 퇴사 인원을 한눈에 표시합니다. 카드 클릭 시 해당 상태로 자동 필터링됩니다.

- **검색 바**: 직원 이름 또는 연락처로 통합 검색합니다.

- **직무 필터**: Owner(지점장) / 매니저 / FC / PT트레이너 / GX강사 / 골프강사 / 스태프 중 선택해 해당 직무 직원만 표시합니다.

- **소속 지점 필터**: 본사 통합 모드에서만 표시되며, 전 지점 직원 목록 중 특정 지점 직원만 좁혀 봅니다.

- **재직 상태 필터**: 재직 / 휴직 / 퇴사 중 선택해 필터링합니다.

- **직원 목록 테이블**: 번호 / 직원명 / 소속 지점 / 직무 배지 / 연락처 / 입사일 / 재직 상태 배지 / 메모 / 더보기 9개 컬럼으로 구성됩니다.

- **페이지네이션**: 한 페이지에 10건씩 표시하며 페이지 번호로 이동합니다.

- **직원 등록 버튼**: 신규 직원 등록용 빈 폼을 열어 직원명, 연락처, 직무, 소속 지점, 로그인 계정, 권한 템플릿, 급여 기본 정보를 입력할 수 있게 합니다.

#### 요약 지표 카드 4종

| 카드명 | 집계 기준 | 클릭 동작 |
|---|---|---|
| 전체 직원 수 | 권한 범위 내 재직·휴직·퇴사 직원을 모두 합산 | 재직 상태 필터를 전체로 전환하고 퇴사 직원까지 포함해 표시 |
| 재직 중 인원 | 재직 상태가 `재직`인 직원 수 | 재직 상태 필터를 재직으로 적용 |

#### 직원명 클릭 동작

직원명을 클릭하면 선택 직원의 상세 패널/화면을 열어 기본 정보, 담당 회원, 담당 스케줄, 실적 요약, 근태 요약, 인사 메모, 계정 상태를 확인합니다. PC에서는 목록 우측 패널 또는 상세 화면으로 표시하고, 모바일에서는 전체 화면 상세로 표시합니다. 상세를 닫으면 직원 목록의 검색어, 필터, 페이지 상태를 유지합니다.

#### 모바일 표시

모바일에서는 위 직원 목록 테이블을 직원 카드 목록으로 전환합니다. 카드에는 직원명, 직무 배지, 재직 상태, 소속 지점, 연락처를 우선 표시하고, 메모와 더보기 메뉴는 카드 하단에 배치합니다. 카드를 누르면 직원 상세가 전체 화면으로 열리며, 닫기/뒤로 버튼으로 목록에 복귀합니다.

#### 본사 통합 모드의 소속 지점 표시

본사 통합 모드는 본사 권한자가 여러 지점 직원을 한 목록에서 볼 때 사용하는 조회 방식입니다. 이 모드에서 `소속 지점` 컬럼은 각 직원이 실제로 소속된 지점명을 행마다 표시합니다. 예를 들어 강남점 직원은 `강남점`, 분당점 직원은 `분당점`으로 표시됩니다.

단일 지점 모드에서는 모든 직원이 현재 지점 소속이므로 `소속 지점` 컬럼에는 현재 지점명이 표시됩니다. 본사 통합 모드에서는 지점 필터가 함께 노출되며, 필터에서 특정 지점을 선택하면 해당 지점 소속 직원만 목록에 남고 요약 카드도 선택 지점 기준으로 다시 집계됩니다.

### 3. 액션 / 흐름

1. 사이드바에서 "직원 관리"를 클릭해 직원 목록 화면으로 진입합니다.

2. 요약 카드로 전체 인원 현황을 파악하고, 검색 또는 필터를 조합해 원하는 직원을 찾습니다.

3. 직원명을 클릭하면 해당 직원의 상세 패널/화면을 열어 기본 정보, 담당 회원, 담당 스케줄, 실적, 근태, 인사 메모를 확인합니다.

4. 체크박스로 직원을 선택하면 일괄 작업 버튼이 활성화되어 메시지 전송 또는 상태 변경을 진행합니다. `Owner(지점장)+`는 추가로 퇴사 처리를 진행할 수 있습니다.

5. 우측 상단 "직원 등록" 버튼으로 신규 직원 등록용 빈 폼을 열고 직원명, 연락처, 직무, 소속 지점, 로그인 계정, 권한 템플릿, 급여 기본 정보를 입력합니다.

6. `Owner(지점장)+`가 퇴사 처리 버튼을 클릭하면 퇴사 처리 확인 다이얼로그가 표시됩니다.

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 로딩 중 | 데이터를 불러오는 동안 테이블 자리에 로딩 표시를 보여줍니다. |
| 정상 목록 | 직원이 1명 이상 존재할 때 테이블과 페이지네이션을 표시합니다. |
| 검색 결과 없음 | 검색·필터 적용 후 결과가 0건이면 "직원 데이터가 없습니다." 안내를 표시합니다. |
| 전체 직원 없음 | 필터 없이 0건이면 직원 등록 안내 메시지를 함께 표시합니다. |
| 오류 | 데이터 로드 실패 시 오류 메시지와 다시 시도 안내를 표시합니다. |

### 5. 권한

| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |

| 직원 본인 | 본인 프로필 및 급여 명세서 | 본인 정보 조회 |

| 케이스 | 표시 | 후속 조치 |

| 전체 직원 0건 | 빈 상태 일러스트 + [직원 등록] CTA | 신규 등록 유도 |
| 데이터 로드 실패 | 오류 박스 + [다시 시도] | 재요청 트리거 |

## SCR-061 직원 등록/수정

> 유형: 화면
> URL 경로: `/staff/new 또는 /staff/edit`
> 연결 기능: STF-EXT-01

### 1. 목적
신규 직원을 시스템에 등록하거나 기존 직원의 정보를 수정하는 화면입니다. 이름·연락처·직무 등 기본 정보와 함께 소속 센터, 로그인 계정, 권한 템플릿 연결 상태, 급여 정보까지 한 화면에서 설정합니다. 등록된 직원은 직원 목록(SCR-060 / `docs4/V1/D07-직원관리/직원관리.md:22`)에 즉시 반영되며, 직원 1명당 로그인 계정 1개를 기본 원칙으로 합니다. 여기서 입력하는 `직무`는 인사/운영 분류값이며, 실제 관리자웹 메뉴 접근 권한은 SCR-081 / `docs4/V1/D09-설정관리/설정관리.md:268` 권한 설정에서 별도로 매핑합니다.

### 2. UI 구성
- **기본 정보 섹션**: 직원명(필수) / 직군(관리직·강사·FC·스태프 등) / 직무(PT트레이너·GX강사·골프강사 등) / 직급 / 팀 / 입사일 / 연락처 / 성별을 입력합니다.
- **담당 서비스 섹션**: 해당 직원이 담당할 서비스 항목(예: PT, 필라테스, GX, 골프 등)을 선택합니다.
- **급여 정보 섹션**: 급여 형태(월급제·시급제·건별·혼합)와 기본급을 입력합니다. 필요 시 급여 정책 템플릿(PAY-STF-02 / `docs4/V1/D07-직원관리/직원관리.md:496`)과 연결합니다.
- **로그인 계정 섹션**: 로그인 ID, 이메일, 임시 비밀번호(신규 시), 계정 상태(활성/잠금), 첫 로그인 후 비밀번호 변경 강제 여부를 설정합니다. 저장 시 직원 계정과 관리자웹 로그인 계정이 함께 생성/수정됩니다.
- **권한/보안 요약 섹션**: 역할별 권한 템플릿 요약, 로그인 차단 여부, 지점 접근 범위를 요약 표시합니다.
- **캘린더 색상**: 수업 캘린더에서 직원을 구분하기 위한 색상입니다. v1에서는 시스템 자동 배정을 기본값으로 하며, 수동 색상 선택·미리보기·색상 충돌 경고는 선택/후순위 기능으로 분리할 수 있습니다.
- **저장 / 취소 버튼**: 저장 시 유효성 검사 후 목록으로 돌아갑니다. 취소 클릭 시 DLG-061-001 / `docs4/V1/D07-직원관리/직원관리.md:1056`이 표시됩니다.

### 3. 액션 / 흐름
1. 직원 목록에서 "직원 등록" 버튼을 클릭해 신규 직원 등록용 빈 폼을 엽니다.
2. 또는 목록 테이블의 직원명 클릭 후 수정 버튼을 눌러 기존 데이터가 채워진 폼으로 진입합니다. (수정)
3. 필수 항목(이름, 로그인 ID, 이메일, 직군/직무, 소속 센터)을 입력하고 임시 비밀번호 및 계정 상태를 설정합니다.
4. "저장" 버튼을 클릭하면 유효성 검사 후 직원 정보와 로그인 계정이 함께 저장됩니다.
5. 저장 성공 시 직원 목록(SCR-060 / `docs4/V1/D07-직원관리/직원관리.md:22`)으로 돌아갑니다.
6. 입력 중 "취소"를 클릭하면 이탈 확인 다이얼로그(DLG-061-001 / `docs4/V1/D07-직원관리/직원관리.md:1056`)가 표시됩니다.

### 캘린더 색상 처리

1. 신규 직원 저장 시 시스템이 지점 내 사용 빈도가 낮은 색상을 자동 배정합니다.
2. 색상은 수업 캘린더에서 강사/직원 식별용으로만 사용합니다.
3. 같은 지점 내 색상 중복은 허용합니다. 중복 방지는 v1 필수 정책이 아닙니다.
4. 수동 색상 선택 UI를 제공하는 경우 6종 팔레트에서 선택하며, 이미 많이 사용 중인 색상은 안내 문구만 표시합니다.

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 신규 등록 | 모든 필드가 빈 상태로 표시됩니다. |

| 계정 잠금 | 로그인 가능 상태가 차단되고, 직원은 재직 상태를 유지한 채 관리자 승인을 기다립니다. |

### 8. 핵심 디자인 요청사항
- 공통 색상, 간격, 폰트, 토스트 규칙은 `_공통/디자인_시스템.md`, `_공통/토스트_메시지.md`를 따른다.
- 본 섹션에 별도 명시된 UI 상태와 권한 차이가 있으면 도메인 공통보다 화면 정의를 우선한다.

## SCR-062 직원 퇴사 처리

> 유형: 화면
> URL 경로: `/staff/resignation`
> 연결 기능: STF-04
> 연결 다이얼로그: DLG-060-002 / `docs4/V1/D07-직원관리/직원관리.md:953`

### 1. 목적
퇴사 예정인 직원의 퇴사일을 설정하고, 해당 직원이 담당하던 미완료 업무·회원·수업을 인수인계한 뒤 시스템 계정을 비활성화하는 화면입니다. 본 시스템은 `직원 1명 = 로그인 계정 1개` 정책을 따르므로 퇴사 처리가 완료되면 해당 직원의 로그인 계정도 함께 잠금 또는 비활성화되며, 이미 로그인 중인 상태도 종료 처리되고 재로그인이 차단됩니다.

### 2. UI 구성
- **퇴사 대상 직원 정보**: 퇴사 처리할 직원의 이름·역할·입사일을 요약해 표시합니다.
- **1단계 퇴사일 설정**: 실제 퇴사 확정일과 예정일을 입력하고 즉시 퇴사 여부를 결정합니다.
- **퇴사 사유 입력**: 자발 / 권고 / 계약 만료 / 기타 중 선택하고 내부 메모를 남깁니다.
- **2단계 미완료 업무 인수인계**: 해당 직원이 담당하던 회원, 예약된 수업, 진행 중인 계약 등을 인계받을 담당자를 지정합니다. 시스템은 인계자를 자동 선택하지 않습니다.
- **3단계 스케줄 정리 미리보기**: 2단계에서 지정한 인계자를 기준으로 퇴사일 이후 수업 담당자 변경 예정, 개인 일정 삭제/종료 예정, 충돌 항목을 확인합니다.
- **4단계 계정 비활성화 확인**: 퇴사 처리 완료 시 시스템 로그인 계정 잠금/비활성화, 기존 로그인 상태 종료, 재로그인 차단 정책을 안내합니다.
- **처리 완료 / 취소 버튼**: 최종 확인 후 퇴사 처리를 완료하거나 취소합니다.

### 3. 액션 / 흐름
1. `Owner(지점장)+`가 직원 목록에서 대상 직원을 체크박스로 선택한 뒤 "퇴사 처리" 버튼을 클릭합니다.
2. 1단계에서 퇴사 확정일 / 예정일 / 퇴사 사유를 입력합니다.
3. 2단계에서 회원·수업·계약 인수인계가 필요한 경우 인계자를 지정합니다.
4. 3단계에서 선택한 인계자 기준 스케줄 반영 미리보기를 확인합니다. 수업 시간 충돌, 휴직/퇴사 인계자, 권한 불일치가 있으면 처리 완료를 진행할 수 없습니다.
5. 4단계에서 계정 비활성화, 기존 로그인 상태 종료, 재로그인 차단 안내를 확인하고 "처리 완료" 버튼을 클릭합니다.
6. 최종 확인 다이얼로그(DLG-060-002 / `docs4/V1/D07-직원관리/직원관리.md:953`)가 표시되며, `퇴사처리` 문구가 일치할 때만 최종 실행됩니다.
7. 당일 또는 과거 날짜 퇴사 처리라면 계정이 즉시 비활성화되고 기존 로그인 상태가 종료되며 재로그인이 차단됩니다.
8. 미래 퇴사일이라면 예정 상태로 등록되며 확정일 도래 시 자동 비활성화됩니다.
9. 처리 완료 후 직원 목록으로 돌아가며 해당 직원의 재직 상태 배지는 "퇴사"로 표시됩니다.

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 1단계 정상 | 퇴사 대상 직원 정보와 퇴사 정보 입력 폼이 표시됩니다. |
| 2단계 인수인계 필요 | 인계 대상 목록과 인계자 선택 영역이 표시됩니다. |
| 2단계 인수인계 없음 | 인수인계 섹션이 "담당 업무 없음"으로 표시되고 다음 단계로 이동합니다. |
| 3단계 스케줄 정리 | 선택 인계자 기준 수업 담당자 변경 예정, 개인 일정 삭제/종료 예정, 충돌 항목이 표시됩니다. |
| 4단계 보안 확인 | 계정 비활성화 / 기존 로그인 상태 종료 / 재로그인 차단 안내가 표시됩니다. |
| 처리 중 | 처리 완료 버튼이 로딩 상태로 변경됩니다. |
| 즉시 퇴사 | 계정 즉시 차단 안내 배너가 표시됩니다. |
| 예약 퇴사 | 확정일 도래 시 자동 처리 안내가 표시됩니다. |

## SCR-063 직원 근태 관리

> 유형: 화면
> URL 경로: `/staff/attendance`
> 연결 기능: PAY-STF-01

### 1. 목적

직원별 출퇴근 기록을 조회하고 지각·결근·초과근무 현황을 파악하는 화면입니다. 월별·직원별로 근무 시간을 집계해 급여 산정의 기초 데이터로 활용하며, 관리자가 출퇴근 기록을 수동으로 보정할 수 있습니다.

### 2. UI 구성

- **조회 기간 선택**: 월 단위 또는 특정 날짜 범위를 선택해 해당 기간의 근태 데이터를 조회합니다.

- **직원 선택 필터**: 특정 직원 또는 전체 직원을 선택해 조회합니다.

- **요약 지표 카드 (5개)**: 해당 기간의 정상 출근 일수 / 지각 횟수 / 결근 횟수 / 총 근무 시간 / 평균 출근율을 요약합니다.

- **근태 목록 테이블**: 날짜 / 직원명 / 출근 시각 / 퇴근 시각 / 근무 시간 / 상태 / 출처 / 보정 이력 8개 컬럼으로 구성됩니다.

- **수동 보정 기능**: 관리자가 출퇴근 시각을 직접 수정하거나 누락된 기록을 추가할 수 있습니다.

- **월별 집계 요약**: 직원별 월 총 근무 시간과 출근율을 한눈에 비교하는 요약 뷰를 제공합니다.

- **직원 지각 기준 표시**: 현재 지점의 직원 지각 허용 시간을 상단 정책 배지로 표시합니다. 기본값은 10분이며, 정규 출근 시각 + 10분까지는 정상, 10분 초과 출근부터 지각으로 분류합니다. 이 값은 D09 SCR-086 / `docs4/V1/D09-설정관리/설정관리.md:865` 출석 관리 설정 > 직원 근태 기준 설정에서 Owner(지점장)이 변경할 수 있습니다.

#### 요약 지표 카드 5종

| 카드 | 표시 기준 | 클릭 동작 |
|---|---|---|
| 정상 출근 일수 | 조회 기간 내 정상 출근으로 분류된 일수. 지각·결근·조퇴는 제외합니다. | 상태 필터를 정상으로 적용 |
| 지각 횟수 | 조회 기간 내 지각으로 분류된 횟수. 지점별 직원 지각 허용 시간 기준을 따릅니다. | 상태 필터를 지각으로 적용 |
| 결근 횟수 | 조회 기간 내 영업일이면서 출근 기록이 없고 휴가·외근도 등록되지 않은 횟수입니다. | 상태 필터를 결근으로 적용 |
| 총 근무 시간 | 조회 기간 내 퇴근 시각 - 출근 시각 합계입니다. 수동 보정이 있으면 보정 후 시간을 기준으로 표시합니다. | 근무 시간 내림차순 정렬 |
| 평균 출근율 | `(정상 출근 + 지각) / 총 영업일 × 100`으로 표시합니다. 휴가·외근과 입사일 이전 영업일은 분모에서 제외합니다. | 월별 집계 요약으로 이동 또는 해당 값 기준 정렬 |

#### 근태 목록 테이블 8개 컬럼

| 컬럼 | 표시 내용 | 동작/규칙 |
|---|---|---|
| 날짜 | `YYYY-MM-DD (요일)` | 조회 기간 기준 정렬 |
| 직원명 | 직원 이름 | 클릭 시 직원 상세 확인 |
| 출근 시각 | `HH:MM` | 누락 시 `-` 표시 |
| 퇴근 시각 | `HH:MM` | 누락 시 `-` 표시 |
| 근무 시간 | 출근~퇴근 기준 합계 | 수동 보정이 있으면 보정 후 시간 기준 |
| 상태 | 근태 상태 배지. 선택지는 정상, 지각, 결근, 조퇴, 외근, 휴가 | 상태 필터와 연동 |
| 출처 | 기록 생성 경로 배지. 선택지는 키오스크, IoT, 수동 보정, 누락 추가 | 기록 생성 경로 표시 |
| 보정 이력 | `보정 N회` 또는 `-` | hover 시 변경 전/후, 사유, 보정자, 보정 일시 표시 |

근태 상태 배지 6종: 정상 / 지각 / 결근 / 조퇴 / 외근 / 휴가.

근태 출처 배지 4종: 키오스크 / IoT / 수동 보정 / 누락 추가.

보정 이력 hover는 보정 이력이 있는 행에서만 동작합니다. 매니저+가 `보정 N회` 배지에 마우스를 올리면 최근 보정 이력의 변경 전/후 시각, 변경 상태, 보정 사유, 보정자, 보정 일시를 툴팁 또는 작은 팝오버로 표시합니다. 직원 본인에게는 보정 이력 상세를 노출하지 않습니다.

### 3. 액션 / 흐름

1. 사이드바 "직원 관리 > 근태 관리"로 진입합니다.

2. 조회 기간(월 또는 날짜 범위)과 대상 직원을 선택합니다.

3. 근태 목록 테이블에서 날짜별 출퇴근 기록을 확인합니다.

4. 요약 카드에서 지각·결근 현황을 파악합니다.

5. 오류 기록이 있으면 해당 행의 수정 버튼을 클릭해 시각을 보정합니다.

6. 급여 계산 시 월별 집계 요약 데이터를 참고합니다.

## SCR-064 급여 관리

> 유형: 화면
> URL 경로: `/payroll`
> 연결 기능: PAY-STF-02
> 연결 다이얼로그: DLG-064-001 / `docs4/V1/D07-직원관리/직원관리.md:1145`, DLG-064-002 / `docs4/V1/D07-직원관리/직원관리.md:1283`

### 1. 목적

직원별 당월 급여를 계산하고 수당·공제 내역을 검토한 뒤 최종 급여를 확정하는 화면입니다. 근태 데이터와 연동되어 기본급에 각종 수당이 자동 반영되며, 관리자가 항목을 수동으로 편집한 후 일괄 확정할 수 있습니다. 또한 우측 정책 영역에서 직급별 급여 정책 템플릿을 관리해 정산 계산식의 기준값을 유지합니다.

### 2. UI 구성

- **지급 월 선택**: 급여를 처리할 대상 월을 선택합니다. 기본값은 시스템 날짜 기준 당월이며, 미래 월은 선택할 수 없습니다. 전월 이전 월은 선택 가능하지만 read-only 마감 월로 표시합니다.

- **요약 지표 카드**: 전체 지급 예정 금액 / 확정 완료 인원 / 미확정 인원 / 이전 월 대비 증감을 표시합니다.

- **직원별 급여 테이블**: 직원명 / 역할 / 급여 형태 / 기본급 / 수당 합계 / 공제 합계 / 수수료 / 실지급액 / 확정 상태 / 액션 10개 컬럼으로 구성됩니다.

- **급여 정책 템플릿 패널**: 화면 우측 또는 하단 보조 패널에서 직급별 정책 목록을 조회합니다. `매출 타입` / `수업 타입` 탭으로 정책 분류를 전환합니다.

- **직군별 산식 탭**: `FC / PT / GX / 공통` 탭을 제공하며, 각 직군의 정률제 / 고정급제 / 시급제 / 혼합제 산식을 따로 조회합니다.

- **정책 액션 버튼**: 정책 추가 / 정책 삭제 / 정책 수정 버튼을 제공하며, 정책 편집은 DLG-064-003 / `docs4/V1/D07-직원관리/직원관리.md:1390`에서 수행합니다.

- **정책 요약 필드**: 선택된 정책의 직급, 직군, 지급 방식, 기본급, 수업단가, 수업료(%), 매출커미션(%), 재등록 커미션(%), 환불 차감 규칙, 적용 센터/팀 범위를 요약 표시합니다.

- **수당·공제 편집**: 각 직원 행의 편집 버튼 클릭 시 DLG-064-001 / `docs4/V1/D07-직원관리/직원관리.md:1145`이 열려 항목별 금액을 수동으로 조정합니다.

- **급여 확정 버튼**: 개별 또는 전체 직원의 급여를 확정합니다. 확정 전 DLG-064-002 / `docs4/V1/D07-직원관리/직원관리.md:1283` 확인 다이얼로그가 표시됩니다.

- **확정 취소**: 이미 확정된 급여도 당월 내에는 확정 취소 후 재편집할 수 있습니다.

### 3. 액션 / 흐름

1. 사이드바 "직원 관리 > 급여 관리"로 진입합니다.

2. 지급 월을 선택하면 해당 월 직원별 급여 데이터가 로드됩니다.

3. 근태 데이터 기반으로 기본급과 수당이 자동 계산되어 표시됩니다.

4. 정책 기준이 바뀐 경우 우측 정책 패널에서 정책을 추가/수정해 계산 기준을 먼저 정비합니다.

5. FC/PT/GX별 산식 탭에서 기준 룰을 먼저 확인한 뒤, 수정이 필요한 직원의 편집 버튼을 클릭해 수당·공제 항목을 조정합니다(DLG-064-001 / `docs4/V1/D07-직원관리/직원관리.md:1145`).

6. 검토 완료 후 "급여 확정" 버튼을 클릭하면 확정 확인 다이얼로그(DLG-064-002 / `docs4/V1/D07-직원관리/직원관리.md:1283`)가 표시됩니다.

7. 확정 후 해당 직원은 급여 명세서(SCR-065 / `docs4/V1/D07-직원관리/직원관리.md:687`)에서 내역을 조회할 수 있습니다.

## SCR-065 급여 명세서

> 유형: 화면
> URL 경로: `/payroll/statements`
> 연결 기능: PAY-STF-03

### 1. 목적

확정된 급여 명세서를 조회하고 직원에게 발송하는 화면입니다. 직원 본인은 본인의 월별 급여 명세서를 확인할 수 있으며, 관리자는 전체 직원의 명세서를 조회하고 개별 또는 일괄 발송할 수 있습니다.

### 2. UI 구성

- **조회 기간 선택**: 월 단위로 급여 명세서를 조회합니다. 기본값은 가장 최근 확정된 월입니다. 이 기능은 조회 월 선택 기능이며 재발송 또는 재발송 안내 기능을 포함하지 않습니다.

- **직원 선택 필터**: 특정 직원 또는 전체 직원의 명세서를 조회합니다.

- **명세서 목록 테이블**: 직원명 / 역할 / 지급 월 / 기본급 / 실지급액 / 발송 상태 / 발송 일시 / 수신 확인 / 액션 9개 컬럼으로 구성됩니다.

- **명세서 상세 조회**: 특정 직원의 행을 클릭하면 기본급 / 각종 수당 상세 / 공제 항목 상세 / 실지급액이 표시되는 상세 패널 또는 모달이 열립니다.

- **개별 발송 버튼**: 해당 직원에게 급여 명세서를 이메일 또는 앱 알림으로 발송합니다.

- **일괄 발송 버튼**: 체크박스로 선택한 직원 전체에게 명세서를 일괄 발송합니다.

- **인쇄 / PDF 저장**: 명세서를 인쇄하거나 PDF 파일로 저장합니다.

#### 명세서 목록 테이블 9개 컬럼

| 컬럼 | 표시 기준 |
|---|---|
| 직원명 | 명세서 대상 직원 이름. 클릭 시 급여 명세서 상세를 엽니다. |
| 역할 | `owner`, `manager`, `fc`, `trainer`, `staff` 중 해당 직원의 기본 역할 배지입니다. |
| 지급 월 | `YYYY-MM` 형식의 급여 대상 월입니다. |
| 기본급 | 해당 월 기본급 금액입니다. |
| 실지급액 | 기본급 + 수당 합계 - 공제 합계 + 수수료로 산출한 최종 지급액입니다. |
| 발송 상태 | `미발송`, `발송완료`, `무효` 3종 배지 중 하나입니다. |
| 발송 일시 | 마지막 발송 성공 일시입니다. 미발송은 `-`로 표시하고, 재발송 시 마지막 성공 일시로 갱신합니다. |
| 수신 확인 | 발송 전 `-`, 발송 후 `미열람`, 열람 시 `열람완료`와 열람 일시, 추적 불가 또는 무효 명세서는 `확인불가`로 표시합니다. |
| 액션 | [발송] / [재발송] / [PDF 저장] / [인쇄]입니다. 발송 상태와 권한에 따라 비활성화합니다. |

#### 발송 상태 배지와 수신 확인

| 구분 | 상태 | 화면 동작 |
|---|---|---|
| 발송 상태 | 미발송 | 생성된 명세서가 아직 성공 발송되지 않은 상태입니다. [발송]을 사용할 수 있습니다. |
| 발송 상태 | 발송완료 | 이메일 또는 앱 알림 중 하나 이상 성공 발송된 상태입니다. [재발송]을 사용할 수 있습니다. |
| 발송 상태 | 무효 | 확정 취소로 기존 명세서가 무효 처리된 상태입니다. 발송·재발송·PDF 저장·인쇄를 차단합니다. |
| 수신 확인 | - | 발송 전이라 수신 확인 대상이 없습니다. |
| 수신 확인 | 미열람 | 발송 성공 후 직원의 이메일·앱 링크 열람 이력이 없습니다. |
| 수신 확인 | 열람완료 | 직원이 명세서를 열람했으며 열람 일시를 함께 표시합니다. |
| 수신 확인 | 확인불가 | 무효 명세서, 만료 링크, 추적 불가 채널로 열람 여부를 확정할 수 없습니다. |

#### 무효 명세서 처리

PAY-STF-02에서 급여 확정 취소가 발생하면 같은 직원·지급월의 기존 명세서는 CRM 내부 `무효` 상태로 전환됩니다. 이는 CRM 데이터 상태 변경이며, 이미 발송된 외부 이메일·앱 푸시 자체를 회수하거나 삭제하는 기능은 아닙니다.

| 항목 | 기준 |
|---|---|
| 목록 표시 | 발송 상태 컬럼에 `무효` 배지 표시 |
| 차단 액션 | [발송], [재발송], [PDF 저장], [인쇄] 비활성 |
| 상세 조회 | 운영자는 무효 사유와 이력 확인 가능. 직원 본인은 상세 금액 조회 차단 |
| 기존 링크 접근 | "무효 처리된 명세서입니다. 재확정 후 새 명세서를 확인해주세요" 안내 표시 |
| 재확정 후 | 기존 무효 명세서는 보존하고 새 명세서가 별도 생성됨. 발송은 운영자가 [발송] 또는 [일괄 발송]으로 실행 |

#### 재발송 안내 기준

재발송 안내는 별도 화면이나 자동 발송 기능이 아니라 명세서 상태에 따라 표시하는 확인 문구입니다.

| 상황 | 안내 문구 | 후속 동작 |
|---|---|---|
| 발송완료 명세서에서 [발송] 재클릭 | "이미 발송된 명세서입니다. 다시 발송하면 기존 발송 이력은 유지되고 새 발송 이력이 추가됩니다." | 확인 시 재발송 실행 |
| 무효 명세서에서 발송 시도 | "무효 명세서는 발송할 수 없습니다. 급여 재확정 후 새 명세서를 발송해주세요." | 발송 차단 |
| 미확정 월 조회 | "급여 미확정 상태입니다. 급여 확정 후 명세서를 발송할 수 있습니다." | 발송 버튼 비활성 |

### 3. 액션 / 흐름

1. 사이드바 "직원 관리 > 급여 명세서"로 진입합니다.

2. 조회 월과 대상 직원을 선택합니다.

3. 명세서 목록에서 원하는 직원의 행을 클릭해 상세 내역을 확인합니다.

4. "발송" 버튼을 클릭하면 해당 직원에게 급여 명세서가 전송됩니다.

5. 여러 직원을 선택한 뒤 "일괄 발송"으로 한 번에 전송합니다.

6. 직원 본인은 앱 또는 이메일로 받은 명세서 링크에서 직접 조회합니다.

## DLG-064-001 급여 상세 편집

> 유형: 다이얼로그
> 연결 기능: PAY-STF-02
> 연결 다이얼로그: DLG-064-001 / `docs4/V1/D07-직원관리/직원관리.md:1145`, DLG-064-002 / `docs4/V1/D07-직원관리/직원관리.md:1283`

### 1. 목적

자동 계산된 급여 항목 중 수당, 공제 등 일부 항목을 관리자가 직접 조정해야 할 때 사용하는 편집 다이얼로그입니다. 기본급은 직원 정보에서 가져오고, 이 다이얼로그에서는 부가 항목만 수정합니다.

### 2. UI 구성

- **직원 정보 요약**: 직원명 / 역할 / 지급월 / 급여 형태 / 확정 상태를 read-only로 표시합니다.

- **자동 계산 요약**: 기본급 / 자동 수당 / 자동 공제 / 수수료 / 현재 실지급액을 read-only로 표시합니다.

- **수당 항목 목록**: 식대 / 교통비 / 특별 수당 / 성과 수당 / 기타 수당을 수동 항목으로 추가·수정·삭제합니다. 금액은 0 이상만 허용합니다.

- **공제 항목 목록**: 지각·결근 공제 / 4대 보험 안내 항목 / 선지급·가불 차감 / 기타 공제를 수동 항목으로 추가·수정·삭제합니다. 금액은 0 이상만 허용합니다.

- **변경 사유 입력**: 수당/공제 저장 시 필수 입력합니다.

- **실지급액 미리보기**: 기본급 + 수당 합계 - 공제 합계 + 수수료를 입력값 변경 즉시 다시 계산해 표시합니다.

- **저장 버튼**: 변경 내용을 저장하고 다이얼로그를 닫습니다.

- **취소 버튼**: 변경 내용을 저장하지 않고 다이얼로그를 닫습니다.

## DLG-064-003 급여 정책 추가

> 유형: 다이얼로그
> 연결 기능: PAY-STF-02
> 연결 다이얼로그: DLG-064-001 / `docs4/V1/D07-직원관리/직원관리.md:1145`, DLG-064-002 / `docs4/V1/D07-직원관리/직원관리.md:1283`

### 1. 목적

직급·직군별 급여 정산 기준이 되는 정책 템플릿을 등록하거나 수정하는 다이얼로그입니다. 월별 급여 확정 전에 계산 기준을 정비하는 용도로 사용합니다.

### 2. UI 구성

- **정책 분류 탭**: 매출 타입 / 수업 타입 탭으로 정책 성격을 전환합니다.

- **직군 선택 탭**: FC / PT / GX / 공통 중 적용 직군을 선택합니다.

- **지급 방식 선택**: 정률제 / 고정급제 / 시급제 / 혼합제 중 하나를 선택합니다.

- **직급 입력**: 정책이 적용될 직급명을 입력합니다. (예: 트레이너, 수석 트레이너, FC 팀장)

- **기본급 입력**: 월 기본급 금액을 입력합니다.

- **수업단가 입력**: 수업 1건당 지급 단가를 입력합니다.

- **수업료(%) 입력**: 수업 매출 기준 정률 정산 비율을 입력합니다.

- **매출커미션(%) 입력**: PT/상품/재등록 매출에 연동되는 커미션 비율을 입력합니다.

- **재등록 커미션(%) 입력**: FC 재등록 매출에 별도 비율을 입력합니다.

- **환불 차감 규칙**: 환불 확정 건 발생 시 이전 지급분 차감 여부와 차감 비율을 설정합니다.

- **타지점 수업 귀속 규칙**: 다른 지점에서 수행한 강습의 귀속 기준(수행 지점 / 판매 지점 / 정산 지점)을 선택합니다.

- **적용 범위**: 소속 센터 / 팀 / 직군 제한값을 선택합니다.

- **미리보기**: 예시 정산 시나리오를 기준으로 예상 지급액을 계산해 보여줍니다.

- **저장 / 취소 버튼**: 정책을 저장하거나 닫습니다.

### 3. 액션 / 흐름

1. 관리자가 SCR-064에서 정책 분류 탭을 고릅니다.

2. [정책 추가] 또는 기존 정책의 [수정]을 클릭합니다.

3. 직군, 지급 방식, 직급, 기본급, 수업단가, 수업료(%), 매출커미션(%), 재등록 커미션(%), 환불 차감 규칙, 적용 범위를 입력합니다.

4. 미리보기 계산값을 확인한 뒤 저장합니다.

5. 저장된 정책은 급여 관리 화면의 정책 패널과 해당 월 자동 계산식에 반영됩니다.

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

### 5. 권한

| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |
