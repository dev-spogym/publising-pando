### 2. UI 구성

- **요약 지표 카드 (4개)**: 전체 회원 수 / 활성 회원 수 / 정책상 조치 대상 회원 수 / 이번 달 만료 수를 한눈에 표시. 카드 클릭 시 카드별 정의된 목록 필터가 적용됩니다. `정책상 조치 대상 회원 수`는 HQ-09 / `docs4/V1/D10-본사관리/본사관리.md:1393` 회원 이용권 만료 step 대상, `이번 달 만료 수`는 당월 실제 만료일 도래 회원입니다.

- **운영 포커스 바**: 요약 카드 아래에 `지금 바로 봐야 할 회원군` 블록을 배치합니다. 재등록 집중 보기(HQ-09 / `docs4/V1/D10-본사관리/본사관리.md:1393` 회원 이용권 만료 step 대상), 이탈 위험 보기(최근 30일 이상 미방문), 관심회원 보기 같은 바로가기 카드를 제공해 운영자가 즉시 목록 문맥을 전환할 수 있게 합니다.

- **액션 큐**: 운영 포커스 바 우측에 선택 회원 기준의 메시지 발송, 상태 변경, 수동 출석 실행 버튼을 모읍니다. 선택 회원이 없을 때는 사용 가이드를, 있을 때는 선택된 회원 이름 요약을 노출합니다.

- **운영 보기 탭**: 회원 목록 / 회원권 목록 / 수강권 목록 / 락커 목록 / 운동복 목록으로 보기 방식을 전환합니다. 센터 운영 방식에 따라 일부 탭은 숨길 수 있습니다.

- **저장 뷰 탭**: 상담내역 / 상담예약 / 재등록대상 / 고객관리 같은 운영 저장 뷰를 상단 서브 탭으로 제공합니다. CRM-01 리드관리와 분리된 회원 원장 관점의 조회 뷰입니다.

- **상태 필터 탭**: 전체 · 활성 · 만료 · 예정 · 임박 · 홀딩 · 미등록 · 탈퇴 총 8개 탭을 건수 배지와 함께 표시합니다. `전체`는 집계 탭이고, 나머지 7개는 실제 회원 상태값입니다. 클릭 시 해당 상태의 회원만 표시되며, `임박`은 자동 라벨의 만료임박과 이름은 비슷하지만 회원 상태 필터 값입니다. `정지`는 실제 회원 상태값이 아니라 현재 상태 위에 적용되는 이용 제한 플래그입니다.

- **검색 및 필터 바**: 이름·연락처·운동 ID 통합 검색, 계약상품·성별·최종만료일·최근방문일 필터를 제공합니다. 추가 필터로 관심회원만 보기, 미방문 기간, 회원구분, 문의 유형(방문(WI) / 전화문의(TI) / 체험 / 일일입장), 가입경로(간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타), 만료 상품 숨기기, 출석 시 상세 팝업 기본값을 제어합니다.

- **일괄 작업 바**: 회원을 1명 이상 선택하면 화면 하단에 자동으로 노출됩니다. 상태 변경 / 메시지 전송 / 출석 처리 / 관심회원 등록 / 일괄 탈퇴 / 지점이관 버튼을 제공합니다. 지점이관은 회원 1명 선택 시에만 진행합니다.

- **일괄 탈퇴**: Owner(지점장) 이상 권한에서만 노출합니다. 선택 회원 중 미수금, 환불 필요 이용권, 진행 예약, 가족 대표 회원이 있으면 DLG-M005에서 차단 대상과 처리 가능 대상을 분리해 보여줍니다.

- **회원 목록 테이블**: 선택 체크박스 / No / 상태 배지 / 회원명 / 성별 / 생년월일 / 나이 / 연락처 / 보유 이용권 / 소속 지점 / 마지막 방문일 / 등록일 컬럼으로 구성합니다. 행 클릭 또는 OPEN 액션으로 상세 진입이 가능합니다.

- **우측 상세 팝업 패널**: 상단 토글이 켜져 있으면 회원 행 선택 시 우측 패널이 슬라이드 인 됩니다. 패널에는 프로필, 방문기록 유무, 출석 체크 / 상품 구매 / 메시지 / 더보기 액션, 기본 인적사항, 상담 담당자, 문의 유형(방문(WI) / 전화문의(TI) / 체험 / 일일입장), 가입경로(간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타), 운동목적, 구독 플랜, 외부 연동 상태를 요약 표시합니다.

- **페이지 이동**: 한 페이지에 20 / 50 / 100건 선택 가능, 페이지 번호 이동.

- **회원 추가 버튼**: 신규 회원 등록 화면(SCR-M002 / `docs4/V1/D02-회원관리/회원관리.md:134`)으로 이동.

- **삭제·탈퇴 회원 보기 토글 (헤더)**: 활성화하면 삭제 또는 탈퇴 처리된 회원이 별도 섹션 또는 별도 탭으로 표시됩니다. 토글 OFF 상태에서는 삭제·탈퇴 회원이 목록에서 숨겨집니다.

- **회원 복구 액션**: 삭제·탈퇴 섹션에서 회원 행의 [복구] 버튼 클릭 시 복구 확인 다이얼로그(DLG-M005 / `docs4/V1/D02-회원관리/회원관리.md:1370` 역연계)를 통해 회원을 다시 활성 상태로 복원합니다. 권한은 Owner(지점장)·매니저·슈퍼관리자에 한정합니다.

### 3. 액션 / 흐름

2. 운영 포커스 바에서 재등록 대상, 장기 미방문, 관심회원 같은 우선 회원군으로 빠르게 전환합니다.

3. 운영 보기 탭 또는 저장 뷰 탭으로 현재 업무 문맥(회원 원장, 재등록 대상, 상담 예약 등)을 고릅니다.

4. 요약 지표 카드로 전체 현황을 파악하고, 상태 탭 또는 검색·필터를 조합해 원하는 회원을 찾습니다.

5. "출석시 상세 팝업" 토글이 켜져 있으면 회원을 선택하는 즉시 우측 상세 팝업이 열리고, 꺼져 있으면 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`)로 이동합니다.

6. 회원 행의 "이관" 버튼 또는 단일 선택 후 하단 "지점이관" 버튼으로 회원 이관 화면(SCR-M005 / `docs4/V1/D02-회원관리/회원관리.md:434`)에 진입합니다.

7. 여러 회원을 체크박스로 선택하면 상단 액션 바, 액션 큐, 하단 일괄 작업 바가 동시에 활성화되어 상태 변경·메시지·출석 처리·일괄 탈퇴 등을 처리합니다.

8. 일괄 탈퇴 선택 시 DLG-M005를 열고 선택 인원, 차단 인원, 탈퇴 사유, 감사 로그 기록 여부를 최종 확인합니다.

9. 권한이 있는 역할은 우측 상단 "회원 추가" 버튼으로 신규 등록(SCR-M002 / `docs4/V1/D02-회원관리/회원관리.md:134`)으로 이동합니다.

### 1. 목적

새로운 회원을 시스템에 등록하고 첫 이용권 결제까지 완료하는 화면입니다. 필수 정보(Step 1)와 추가 정보(Step 2)를 단계별로 입력받아 빠른 등록과 상세 정보 입력을 분리하되, 결제 없이 회원 등록을 완료할 수 없습니다. Step 2의 [결제 진행] 후 결제 화면(SCR-S003 / `docs4/V1/D03-매출관리/매출관리.md:161`)으로 이어지며, 첫 정상 결제가 완료된 시점을 회원 등록 완료일과 신규 세그먼트 기준일로 봅니다.

### 2. UI 구성

- **단계 표시기 (Step 1 / Step 2)**: 현재 진행 단계를 상단에 표시. Step 1이 완료되어야 Step 2로 이동 가능.

- **프로필 사진 업로더**: 원형 이미지 영역을 클릭해 사진 등록. 미등록 시 이니셜 아바타 자동 생성.

- **[Step 1] 기본 인적 사항**

  - 이름 (필수), 성별 (필수, 남성/여성 토글)

  - 연락처 (필수) + 중복 확인 버튼 — 동일 연락처 회원이 있으면 경고 표시

  - 회원구분 (일반 / 기명법인 / 무기명법인, 필수)

  - 법인 회원 선택 시 법인 회사명 입력란 추가 노출

  - 생년월일, 키(cm) (선택)

- **[Step 1] 관리 정보**: 소속지점 / 기본 이용지점 / 담당 상담자(FC) / 담당 트레이너 / 문의 유형(방문(WI) / 전화문의(TI) / 체험 / 일일입장) / 가입경로(간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타) / 운동 목적 / 기본 매출 귀속 지점 / 기본 인센티브 귀속자 (모두 선택). 담당 상담자는 상담 성과 기준이며, 실제 결제 완료 시 판매 담당자/인센티브 귀속자는 결제 화면에서 다시 확인한다.

- **귀속 정책 안내 박스**: 회원 등록 시 저장하는 값은 회원 기본값입니다. 실제 결제 시점에는 결제지점, 이용지점, 매출 귀속 지점, 정산 지점, 인센티브 귀속자를 다시 확인·조정할 수 있다는 안내를 표시합니다.

- **[Step 2] 추가 연락 정보**: 별칭/닉네임 / 이메일 / 주소(주소 검색 버튼) / 상세 주소 / 회사명 / 출석번호

- **[Step 2] 기타 설정**: 광고성 정보 수신 동의 체크박스, 메모 입력란(최대 500자, 글자 수 표시)

- **하단 고정 버튼**: Step 1에서는 [취소] [다음], Step 2에서는 [이전] [결제 진행]

- **결제 진행 안내 영역**: Step 2 하단에 "결제 완료 후 회원 등록이 확정됩니다" 문구를 표시하고, 결제 취소 시 작성 내용은 임시 저장 상태로 돌아간다는 안내를 제공합니다.

- **초기화 버튼**: 입력한 내용이 있을 때만 활성화. 확인 후 전체 초기화.

### 3. 액션 / 흐름

1. 회원 목록(SCR-M001 / `docs4/V1/D02-회원관리/회원관리.md:21`)의 "회원 추가" 버튼을 눌러 진입합니다.

2. Step 1에서 이름·성별·연락처·회원구분을 입력하고, 연락처 중복 확인을 완료합니다.

3. "다음" 버튼을 누르면 Step 2로 이동합니다. (필수 항목 미입력 또는 중복 확인 미완료 시 이동 불가)

4. Step 2에서 추가 정보를 선택적으로 입력하고 "결제 진행"을 누릅니다.

5. 입력값 검증과 연락처 중복 확인이 통과되면 결제 화면(SCR-S003 / `docs4/V1/D03-매출관리/매출관리.md:161`)으로 이동합니다.

6. 첫 정상 결제가 완료되면 회원 등록이 확정되고 해당 회원의 상세 페이지(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`)로 이동합니다.

7. 결제 취소 또는 실패 시 회원 등록은 완료되지 않으며, 작성 중인 등록 정보로 돌아갑니다.

8. 취소 또는 이탈 시, 입력한 내용이 있으면 확인 다이얼로그가 나타납니다.

## SCR-M003 회원 수정

> 유형: 화면
> URL 경로: `/members/edit`
> 연결 기능: MBR-03
> 연결 다이얼로그: DLG-M006 / `docs4/V1/D02-회원관리/회원관리.md:1465` 전화번호중복, DLG-M007 / `docs4/V1/D02-회원관리/회원관리.md:1534` 취소확인, DLG-M008 / `docs4/V1/D02-회원관리/회원관리.md:1603` 초기화확인, DLG-M027 / `docs4/V1/D02-회원관리/회원관리.md:2877` 주소검색

### 1. 목적

이미 등록된 회원의 기본 정보를 수정하는 화면입니다. 회원 등록(SCR-M002 / `docs4/V1/D02-회원관리/회원관리.md:134`)과 동일한 입력 구조를 사용하되, 기존 데이터가 자동으로 채워진 상태에서 시작합니다. 수정 완료 후 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`)로 돌아가 변경 결과를 바로 확인할 수 있습니다.

### 2. UI 구성

- **단계 표시기 (Step 1 / Step 2)**: 등록 화면과 동일한 2단계 구조. 기존 값이 각 단계에 미리 채워져 있습니다.

- **프로필 사진**: 기존 사진이 표시되며, 클릭으로 새 사진으로 교체 가능.

- **[Step 1] 기본 인적 사항**: 이름 / 성별 / 연락처 + 중복 확인 / 회원구분 / 법인 회사명(법인인 경우) / 생년월일 / 키. 기존 연락처는 이미 확인된 것으로 처리되어 변경 시에만 중복 확인 필요.

- **[Step 1] 관리 정보**: 소속지점 / 기본 이용지점 / 담당 상담자(FC) / 담당 트레이너 / 문의 유형(방문(WI) / 전화문의(TI) / 체험 / 일일입장) / 가입경로(간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타) / 운동 목적 / 기본 매출 귀속 지점 / 기본 인센티브 귀속자

- **[Step 2] 추가 연락 정보**: 별칭/닉네임 / 이메일 / 주소 / 상세 주소 / 회사명 / 출석번호

- **[Step 2] 기타 설정**: 광고성 정보 수신 동의 / 메모(최대 500자)

- **하단 고정 버튼**: Step 1에서는 [취소] [다음], Step 2에서는 [이전] [저장]

## SCR-M005 회원 이관

> 유형: 화면
> URL 경로: `/members/transfer`
> 연결 기능: MBR-ADV-02
> 연결 다이얼로그: DLG-M023 / `docs4/V1/D02-회원관리/회원관리.md:2641` 이관확인

### 1. 목적

특정 회원을 현재 소속 지점에서 다른 지점으로 이관하는 화면입니다. 회원이 거주지 이동이나 개인 사정으로 이용 지점을 변경할 때, 기존 이용권과 이력 데이터를 그대로 유지한 채 새로운 지점으로 소속을 변경합니다. 이관 처리 후 회원은 새 지점의 관리 대상이 됩니다.

### 2. UI 구성

- **이관 대상 회원 정보**: 이관할 회원의 이름·연락처·현재 이용권·소속 지점·기본 이용지점·기본 매출 귀속 지점을 요약 표시. 회원 상세에서 넘어온 경우 자동으로 채워집니다.

- **이관 대상 지점 선택**: 이관할 지점을 드롭다운으로 선택. 현재 소속 지점은 선택 불가.

- **소속 지점 기준 처리**: 본사 통합 모드에서 진입해도 이관 원 지점은 화면의 현재 선택 지점이 아니라 회원 row의 실제 `branchId`를 기준으로 확정합니다.

- **이관 전 확인 정보**:

  - 이관 후에도 유지되는 항목: 이용권 잔여일·잔여 횟수·결제 이력·출석 이력·상담 메모

  - 이관 후 변경되는 항목: 소속 지점·기본 이용지점·담당 FC·담당 트레이너·기본 매출 귀속 지점·기본 인센티브 귀속자(새 지점 기준으로 재배정 필요)

- **귀속 필드 재배정 표**: `소속지점 / 기본 이용지점 / 기본 매출 귀속 지점 / 정산 지점 기본값 / 담당 상담자 / 담당 트레이너 / 인센티브 귀속자`를 한 번에 확인하고 이관 후 값을 지정하는 표를 제공합니다.

- **예외 회원권 안내**: 법인 회원권, 홈&오피스, 주말 이용권, 통합 회원권은 단순 소속지점 이동만으로 끝나지 않으므로 이용 가능 지점과 정산 지점 정책을 함께 점검해야 한다는 안내를 표시합니다.

- **이관 사유 입력**: 이관 이유를 간략하게 입력 (선택 항목, 내부 기록용)

- **확인 버튼 및 안내 문구**: "이관 후 현재 지점에서 해당 회원을 조회할 수 없습니다" 등 주의 사항 명시.

- **이관 확인 다이얼로그**: 이관 버튼 클릭 시 최종 확인 다이얼로그(DLG-M023 / `docs4/V1/D02-회원관리/회원관리.md:2641`) 표시 후 처리.

### 3. 액션 / 흐름

1. 회원 목록(SCR-M001 / `docs4/V1/D02-회원관리/회원관리.md:21`)의 행별 "이관" 버튼, 단일 선택 후 벌크 "지점이관" 버튼, 또는 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`) 프로필 영역의 "지점이관" 버튼을 눌러 진입합니다.

2. 이관할 회원 정보가 자동으로 표시됩니다.

3. 이관 대상 지점을 드롭다운에서 선택합니다.

## SCR-M006 체성분 관리

> 유형: 화면
> URL 경로: `/body-composition`
> 연결 기능: MBR-ADV-03
> 연결 다이얼로그: DLG-M015 / `docs4/V1/D02-회원관리/회원관리.md:2093` 체성분등록, DLG-M016 / `docs4/V1/D02-회원관리/회원관리.md:2174` 체성분덮어쓰기, DLG-M017 / `docs4/V1/D02-회원관리/회원관리.md:2243` 목표설정

### 1. 목적

특정 회원의 체성분(체중·골격근량·체지방률·BMI·기초대사량) 변화 추이를 시계열 그래프와 측정 기록 표로 시각화하는 화면입니다. 트레이너와 FC가 회원의 운동·영양 개선 효과를 판정하고, 재상담 또는 재계약의 근거 자료로 활용합니다. InBody 장비와 연동한 자동 수집도 지원합니다.

동일한 `/body-composition` route를 D11 SCR-I006 / `docs4/V1/D11-통합운영/통합운영.md:470` 체성분 통합 관리와 공유합니다. 지점 권한자는 소속 지점 회원 또는 선택 회원의 체성분 기록만 조회·입력·수정할 수 있고, 본사 권한자는 전 지점 회원의 측정 현황과 수신/검수 상태를 통합 조회합니다. 회원 개인별 추이·상담 활용 기준은 본 화면(SCR-M006 / `docs4/V1/D02-회원관리/회원관리.md:545`)을 따르고, 전 지점 수신 로그·검수 운영 기준은 D11 SCR-I006을 따릅니다.

### 2. UI 구성

- **회원 선택**: 상단 드롭다운에서 조회할 회원을 선택. 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`) 체성분 탭에서 진입하면 해당 회원이 자동 선택됩니다.

- **회원 요약 카드**: 선택한 회원의 이름·현재 이용권·상태를 한 줄로 표시. 클릭 시 회원 상세로 이동.

- **탭**: 측정 기록 / 추이 분석 두 가지 보기 전환. "측정 추가" 버튼은 탭 우측에 고정.

- **현재값 요약 카드 (4개)**: 현재 체중 / 골격근량 / 체지방률 / BMI를 최신 측정값으로 표시. 이전 측정값 대비 변화량(증감 방향·색상)을 함께 표시.

  - 체중·체지방률 감소는 긍정(초록), 증가는 부정(빨강)

  - 골격근량 증가는 긍정(초록), 감소는 부정(빨강)

  - 측정 기록이 1건뿐이면 변화량 표시 없음

- **추이 그래프**: 측정일을 x축, 수치를 y축으로 하는 꺾은선 그래프. 체중(산호색)·골격근량(민트색)·체지방률(앰버색) 세 가지 지표를 각각 토글로 on/off 가능. 최소 1개는 항상 켜져 있어야 합니다. 기록이 2건 이상일 때만 그래프 표시.

- **측정 기록 표**: 날짜 / 체중 / 골격근량 / 체지방률 / BMI / 기초대사량 / 체수분 / 수정·삭제 버튼 순으로 구성. 최신 측정일 기준 내림차순 정렬.

- **측정 추가 다이얼로그**: "측정 추가" 버튼 클릭 시 열림. 날짜·체중·골격근량·체지방률·체수분을 입력하며, 나머지 값(BMI·기초대사량·체지방량)은 자동 계산.

- **동일 날짜 덮어쓰기 확인**: 같은 날짜에 이미 측정값이 있으면 덮어쓸지 물어보는 확인 창이 나타납니다.

### 3. 액션 / 흐름

1. 사이드바 "회원관리 > 체성분 관리"를 클릭하거나, 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`) 체성분 탭에서 진입합니다.

2. 상단 드롭다운에서 조회할 회원을 선택합니다. (회원 상세에서 넘어오면 자동 선택)

3. 현재값 요약 카드로 최신 체성분 수치와 변화 방향을 확인합니다.

4. 그래프에서 측정 지표를 토글하며 원하는 추이를 시각적으로 확인합니다.

5. "측정 추가" 버튼으로 새 측정값을 입력합니다.

6. 기존 기록을 수정하거나 삭제할 수 있습니다 (권한 및 시간 제한에 따라 다름).

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 로딩 중 | 데이터 조회 중 스켈레톤 화면 표시 |
| 측정 기록 0건 | "측정 기록이 없습니다" 안내 + 측정 추가 유도 버튼 |

## SCR-M007 회원 병합

> 유형: 화면
> URL 경로: `/members/merge`
> 연결 기능: MBR-EXT-01
> 연결 다이얼로그: DLG-M028 / `docs4/V1/D02-회원관리/회원관리.md:2958` 회원병합확인

### 1. 목적
동일인으로 판단되는 중복 회원 계정 두 개를 하나로 합치는 화면입니다. 연락처 변경이나 재등록 등의 이유로 같은 회원이 중복 등록된 경우, 양쪽 데이터를 하나의 주 계정으로 통합합니다. 출석 이력·결제 이력·상담 메모 등 중요한 이력 데이터는 병합 후에도 유지됩니다.

### 2. UI 구성
- **중복 회원 검색**: 이름·연락처·생년월일로 중복이 의심되는 회원을 검색합니다.
- **주 계정 지정**: 병합 결과로 남길 계정(주 계정)과 삭제될 계정(부 계정)을 지정합니다.
- **병합 전 정보 비교 화면**: 주 계정과 부 계정의 주요 정보(이름·연락처·이용권·등록일·최근 방문일)를 나란히 표시해 최종 확인을 돕습니다.
- **병합 항목 선택**: 이름, 연락처, 프로필 사진 등 일부 항목에서 어느 쪽 계정 값을 채택할지 선택할 수 있습니다.
- **병합 후 처리 안내**:
  - 부 계정의 출석 이력·결제 이력·상담 메모·체성분 기록은 주 계정으로 이전
  - 부 계정은 병합 처리 완료 후 비활성 상태로 전환 (즉시 삭제 아님)
- **최종 확인 다이얼로그**: 병합은 되돌리기 어려운 작업임을 명시하고 "병합" 입력 확인 후 처리.

### 3. 액션 / 흐름
1. 회원 목록 또는 회원 상세에서 "회원 병합" 메뉴로 진입합니다.
2. 중복으로 의심되는 회원을 검색하여 주 계정과 부 계정을 각각 지정합니다.
3. 나란히 비교된 두 계정의 정보를 확인하고, 병합 후 남길 정보를 선택합니다.
4. "병합" 버튼을 클릭하면 최종 확인 다이얼로그가 나타납니다. 확인 문구를 입력 후 처리합니다.
5. 병합 완료 후 주 계정의 회원 상세(SCR-M004 / `docs4/V1/D02-회원관리/회원관리.md:373`)로 이동합니다.

## SCR-M008 가족 회원

> 유형: 화면
> URL 경로: `/members/family`
> 연결 기능: MBR-EXT-02
> 연결 다이얼로그: DLG-M029 / `docs4/V1/D02-회원관리/회원관리.md:3039` 가족연결

### 1. 목적
동일 가족 구성원인 회원들을 서로 연결하여 가족 단위로 관리하는 화면입니다. 가족 회원 연결을 통해 패밀리 이용권 혜택 적용, 가족 단위 결제 현황 파악, 특정 가족 묶음 대상 마케팅 등을 지원합니다. 가족 관계를 직관적으로 파악하고 구성원 추가·제거를 간단하게 처리합니다.

### 2. UI 구성
- **가족 그룹 목록**: 등록된 가족 그룹을 카드 형태로 표시. 각 카드에는 그룹 이름(예: 김씨 가족)과 구성원 수, 대표 회원 이름이 표시됩니다.
- **가족 그룹 상세**: 그룹을 클릭하면 구성원 목록(이름·관계·이용권 상태·최근 방문일)이 펼쳐집니다.
- **가족 구성원 추가**: 기존 회원 검색 후 그룹에 추가. 관계(부모·자녀·배우자·형제·기타)를 선택합니다.
- **가족 그룹 생성**: 새 가족 그룹 생성 버튼. 그룹 이름 입력 후 최초 구성원을 추가하는 방식.
- **구성원 제거**: 특정 구성원을 그룹에서 분리합니다. 분리 후 해당 회원 데이터는 그대로 유지.
- **가족 단위 요약 정보**: 가족 전체 이용권 현황, 가족 중 활성 회원 수, 총 마일리지 합산 등.

### 3. 액션 / 흐름
1. 사이드바 "회원관리 > 가족 회원" 또는 회원 상세 내 관련 메뉴에서 진입합니다.
2. 기존 가족 그룹 목록을 확인하거나 "새 그룹 만들기" 버튼으로 그룹을 생성합니다.
3. 그룹에 구성원을 추가할 때는 회원 이름·연락처로 검색 후 관계를 선택합니다.
4. 특정 구성원을 제거하거나 그룹 자체를 삭제할 수 있습니다.
5. 가족 그룹 카드를 클릭하면 구성원 전체의 이용 현황을 한눈에 확인합니다.

### 4. 핵심 상태

|---|---|
| 그룹 없음 | 등록된 가족 그룹이 없을 때 "가족 그룹이 없습니다" 안내 + 그룹 생성 버튼 |
| 그룹 목록 | 가족 그룹 카드 목록 표시 |
| 구성원 상세 | 그룹 선택 후 구성원 목록 및 이용 현황 표시 |
| 구성원 추가 중 | 회원 검색 및 관계 선택 입력 단계 |
| 처리 중 | 그룹 생성·수정·삭제 중 로딩 표시 |

## SCR-M009 등급 관리

> 유형: 화면
> URL 경로: `/members/grade`
> 연결 기능: MBR-EXT-03
> 연결 다이얼로그: DLG-M030 / `docs4/V1/D02-회원관리/회원관리.md:3116` 등급변경

### 1. 목적

회원 등급 체계(예: 브론즈·실버·골드·플래티넘·다이아몬드)의 기준을 설정하고, 각 등급별 혜택을 관리하는 화면입니다. 누적 결제 금액, 이용 기간, 방문 횟수 등의 조건으로 등급 산정 기준을 정의하고, 조건을 충족한 회원에게 자동으로 등급을 부여합니다. 등급별 차별화된 혜택 제공으로 회원 충성도와 재방문율을 높이는 데 활용합니다.

### 2. UI 구성

- **등급 목록 패널**: 현재 설정된 등급(브론즈·실버·골드·플래티넘·다이아몬드)을 순서대로 표시. 각 등급에 해당하는 현재 회원 수 표시.

- **등급 기준 설정**:

  - 기준 항목 선택: 누적 결제 금액 / 총 이용 기간(개월) / 누적 방문 횟수 중 하나 또는 조합

  - 각 등급의 최솟값을 입력하면 초과 조건으로 자동 계산

  - 예: 골드 = 누적 결제 300만원 이상 + 이용 기간 12개월 이상

- **등급별 혜택 설정**:

  - 마일리지 적립률 (예: 실버는 기본 1%, 골드는 2%)

  - 이용권 할인율

  - 락커·주차 등 부가 서비스 우선 배정

  - 생일 특별 혜택 여부

- **등급 자동 갱신 주기 설정**: 월간 / 분기 / 연간 중 선택. 갱신 시 조건을 재계산해 등급을 자동으로 올리거나 내립니다.

- **등급별 회원 현황**: 각 등급에 속한 회원 목록을 조회하는 링크 제공.

- **저장 및 즉시 적용**: 설정 변경 후 저장하면 다음 갱신 주기 또는 즉시 적용 여부를 선택할 수 있습니다.

- **혜택 편집 모달**: 등급별 혜택을 추가/삭제/저장하는 모달 제공.

- **회원 보기 모달**: 해당 등급 회원의 방문 횟수와 이용권 정보를 조회하는 모달 제공.

### 3. 액션 / 흐름

1. 사이드바 "회원관리 > 등급 관리"로 진입합니다.

2. 현재 설정된 등급 기준과 혜택을 확인합니다.

3. 등급 조건(결제 금액·이용 기간·방문 횟수 기준치)을 수정합니다.

4. 각 등급별 혜택(마일리지 적립률·할인율 등)을 설정합니다.

5. 갱신 주기를 선택하고 저장합니다.

6. 변경 즉시 적용 여부를 선택해 반영합니다.

7. `해당 회원 보기` 버튼으로 등급별 실제 회원 목록을 확인합니다.

## SCR-M010 세그먼트 관리

> 유형: 화면
> URL 경로: `/members/segment`
> 연결 기능: MBR-EXT-04

### 1. 목적
공통된 특성이나 행동 패턴을 가진 회원들을 그룹(세그먼트)으로 묶어 타겟 마케팅과 맞춤 관리에 활용하는 화면입니다. "30일 이상 방문/출석이 없는 활성 회원", "본사/지점 만료 알림 step 대상 회원", "첫 정상 결제 완료일로부터 30일 이내 회원" 등 다양한 조건 조합으로 세그먼트를 생성하고, 해당 그룹에 메시지 발송이나 혜택 제공 등의 액션을 연결합니다.

### 2. UI 구성
- **세그먼트 목록**: 생성된 세그먼트를 카드 형태로 표시. 각 카드에는 세그먼트 이름·설명·현재 해당 회원 수·생성일·마지막 업데이트 시각이 표시됩니다.
- **자동 7종 세그먼트 카드**: 신규 / 이탈위험 / 만료임박 / 활발 / 관심필요 / 만료후미등록 / 충성의 현재 대상 수와 마지막 계산 시각을 표시합니다. 자동 7종은 시스템 정의이므로 조건·라벨명·삭제 버튼을 제공하지 않습니다.
- **세그먼트 생성 버튼**: 새 세그먼트 만들기. 이름, 설명, 조건을 입력합니다.
- **조건 빌더**: AND / OR 조합으로 여러 조건을 추가합니다.
  - 회원 상태 (활성 / 만료 / 예정 / 임박 / 홀딩 / 미등록 / 탈퇴) 및 정지 플래그 여부
  - 최근 방문일 기준: 키오스크 입장, 수업 출석, PT/GX 참여, 수동 출석을 모두 포함하며 동일 회원의 2시간 이내 중복 입장은 1회로 집계
  - 이용권 만료일 기준: HQ-09 / `docs4/V1/D10-본사관리/본사관리.md:1393` 회원 이용권 만료 정책 step 및 지점 추가 step 중 ON 상태인 step 대상
  - 첫 결제일 기준: 첫 정상 결제 완료일로부터 N일 이내 / 이후
  - 성별 / 연령대 / 회원구분 / 문의 유형(방문(WI) / 전화문의(TI) / 체험 / 일일입장) / 가입경로(간판 / 블로그 / 인스타 / 당근 / 지역카페 / 현수막 / 전단지 / 회원소개 / 기타)
  - 누적 결제 금액 범위
- **미리보기**: 조건 설정 중 "미리보기" 버튼을 누르면 현재 조건에 해당하는 회원 수와 샘플 명단이 표시됩니다.
- **자동 갱신 설정**: 세그먼트 조건을 매일·주간·월간 자동으로 재계산할지 설정합니다.
- **세그먼트 액션 연결**: 세그먼트 카드에서 "메시지 발송" 또는 "이용권 혜택 적용" 등의 액션을 바로 실행할 수 있는 버튼을 제공합니다.
- **회원 보기 모달**: 세그먼트에 포함된 샘플 회원 목록과 운영 메모를 즉시 확인합니다.
- **메시지 발송 모달**: 선택한 세그먼트 기준 공지 메시지를 바로 발송합니다.
- **세그먼트 수정·삭제**: 기존 세그먼트의 조건이나 이름을 변경하거나 삭제합니다.

### 3. 자동 7종 세그먼트 상세 정의
자동 7종은 시스템이 매일 04:00에 재계산하고, 회원 상태·결제·출석 이벤트가 발생하면 관련 회원을 실시간 재판정합니다. 커스텀 세그먼트는 자동 7종과 별도로 운영자가 추가 생성할 수 있습니다.

| 라벨 | 판정 기준 | 개발·운영 해석 |
|---|---|---|
| 신규 | 첫 정상 결제 완료일로부터 30일 이내 | 결제 없이 회원 등록은 완료될 수 없으므로 등록일이 아니라 첫 결제 완료일을 기준으로 합니다. 환불·결제취소 건은 첫 결제 기준에서 제외합니다. |
| 만료후미등록 | 마지막 회원 이용권 만료일 +60일 경과, 만료 이후 재등록 결제 없음 | 재등록 결제가 발생하면 즉시 후보에서 제외합니다. |
| 이탈위험 | 활성 회원이 최근 30일 이상 방문/출석 없음 | 방문/출석은 키오스크 입장, 수업 출석, PT/GX 참여, 수동 출석을 모두 포함합니다. 동일 회원의 2시간 이내 중복 입장은 1회로 봅니다. |
| 만료임박 | HQ-09 / `docs4/V1/D10-본사관리/본사관리.md:1393` 회원 이용권 만료 본사 step 또는 지점 추가 step 중 ON 상태인 step의 대상 | 본사 step은 지점에 배포되고, Owner(지점장)은 ON/OFF 및 지점 추가 step을 관리합니다. 결제기한 만료·락커 만료 정책과 섞지 않습니다. |
| 관심필요 | 최근 90일 이내 종합평가 기록이 없거나 상담 기록이 없는 회원 | 지속적인 관심이 필요한 회원을 빠뜨리지 않기 위한 라벨입니다. 둘 중 하나라도 없으면 후보에 포함합니다. |
| 충성 | 누적 결제 기간 12개월 이상 + 골드 이상 등급 | 누적 결제 기간은 결제 완료된 이용권 기간 합산 기준입니다. 홀딩 기간은 제외하고, 지점 이관 시 이전 지점 이용기간을 포함합니다. 다른 자동 라벨과 중복 표시할 수 있습니다. |
| 활발 | 최근 30일 이내 방문/출석 8회 이상 | 키오스크 입장, 수업 출석, PT/GX 참여, 수동 출석을 모두 포함합니다. 동일 회원의 2시간 이내 중복 입장은 1회로 봅니다. |

- 기본 자동 라벨은 `이탈위험 > 만료임박 > 신규 > 활발 > 관심필요 > 만료후미등록` 우선순위로 회원당 1개만 표시합니다.
- `충성`은 보조 라벨로 별도 판정하며, 기본 자동 라벨 조건을 동시에 만족하면 기본 라벨 옆에 함께 표시합니다.
- 기본 자동 라벨 조건을 만족하지 않고 `충성` 조건만 만족하면 회원 목록과 상세에는 `충성` 라벨만 표시합니다.
- 방문/출석 집계 기준은 세그먼트 관리, 회원 목록 배지, 회원 상세 헤더, 본사 통합 KPI에서 동일하게 적용합니다.

### 4. 액션 / 흐름
1. 사이드바 "회원관리 > 세그먼트 관리"로 진입합니다.
2. 기존 세그먼트 목록을 확인하거나 "새 세그먼트 만들기"를 클릭합니다.
3. 세그먼트 이름과 설명을 입력하고, 조건 빌더로 필터 조건을 설정합니다.
4. "미리보기"로 현재 조건에 해당하는 회원 수를 확인합니다.
5. 자동 갱신 주기를 설정하고 저장합니다.
6. 저장된 세그먼트 카드에서 "메시지 발송" 등의 액션을 바로 실행합니다.
7. `보기` 버튼으로 세그먼트 샘플 회원 목록을 열어 운영 메모를 확인합니다.

## DLG-M001 회원 상태 변경 확인

> 유형: 다이얼로그
> 연결 기능: MBR-01, MBR-02
> 연결 다이얼로그: DLG-M001 / `docs4/V1/D02-회원관리/회원관리.md:1073` 상태변경, DLG-M005 / `docs4/V1/D02-회원관리/회원관리.md:1370` 탈퇴/복구, DLG-M022 / `docs4/V1/D02-회원관리/회원관리.md:2553` 수동출석, DLG-M023 / `docs4/V1/D02-회원관리/회원관리.md:2641` 이관확인

### 1. 목적

관리자가 회원의 상태(활성, 홀딩, 탈퇴 등)를 변경하기 전, 의도하지 않은 변경을 방지하기 위해 최종 확인을 받는 다이얼로그입니다.

### 2. UI 구성

- 제목: "상태를 변경하시겠습니까?"

- 안내 메시지: 변경 대상 회원명, 현재 상태 → 변경될 상태 표시

- 확인 버튼 (변경 실행)

- 취소 버튼 (다이얼로그 닫기)

### 3. 액션 / 흐름

1. 관리자가 회원 상태 변경 버튼 클릭 → 다이얼로그 표시

2. 확인 → 상태 변경 처리 후 다이얼로그 닫힘, 화면 내 상태 값 즉시 갱신

3. 취소 → 다이얼로그 닫힘, 상태 변경 없음

### 4. 핵심 상태

| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

## DLG-M005 탈퇴 처리

### 1. 목적

회원을 탈퇴 처리하기 전, 개인정보 마스킹 처리 및 서비스 이용 중단 사실을 안내하고 관리자의 최종 확인을 받는 다이얼로그입니다.

### 2. UI 구성

- 제목: "탈퇴 처리하시겠습니까?"

- 경고 메시지: "탈퇴 처리 시 개인정보가 마스킹되며 서비스 이용이 중단됩니다."

- 탈퇴 사유 선택 (드롭다운: 본인 요청, 장기 미이용, 기타)

- 대상 회원명 표시. 일괄 탈퇴에서는 선택 회원 수, 처리 가능 회원 수, 차단 회원 수를 함께 표시합니다.

- 차단 대상 목록: 미수금, 환불 필요 이용권, 진행 예약, 가족 대표 회원, 이미 탈퇴·삭제된 회원을 사유별로 묶어 표시합니다.

- 일괄 탈퇴 확인 입력: 2명 이상 처리 시 `탈퇴` 문구를 직접 입력해야 확인 버튼이 활성화됩니다.

- 확인 버튼 (탈퇴 실행)

- 취소 버튼 (다이얼로그 닫기)

### 3. 액션 / 흐름

1. 관리자가 탈퇴 처리 버튼 클릭 → 다이얼로그 표시

2. 단일 처리면 회원명과 상태를, 일괄 처리면 선택 회원 요약과 차단 대상 목록을 확인

3. 탈퇴 사유 선택. 일괄 탈퇴는 확인 문구 `탈퇴` 입력

### 2. UI 구성

- 제목: "지점 이관 확인"

- 이관 대상 회원 정보 표시 (회원명, 현재 지점)

- 이관 대상 지점 표시 (이관될 지점명)

- 이관 시 함께 이전되는 정보 안내 (이용권, 출석 이력, 체성분 등)

- 이관 후 변경 사항 안내 메시지

- 취소 버튼

2. 이관 정보 및 안내 사항 확인

3. 확인 클릭 → 회원 소속 지점 변경, 해당 지점 Owner(지점장)·매니저에게 이관 알림 발송

| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |
