## SCR-092 지점 관리

> 유형: 화면
> URL 경로: `/branches`
> 연결 기능: CTR-01
> 연결 다이얼로그: DLG-092-001 / `docs4/V1/D10-본사관리/본사관리.md:712` 신규지점등록, DLG-092-002 / `docs4/V1/D10-본사관리/본사관리.md:830` 비활성화확인

### 1. 목적

전체 지점의 현황을 조회하고, 신규 지점을 등록하거나 기존 지점의 상태를 변경하는 화면입니다.
지점 간 회원 이동 신청 및 지점별 통합 성과 비교도 이 화면에서 처리합니다.

### 지점 목록

- 지점 목록 테이블 9개 컬럼: 지점명 / 지점 코드 / 주소 / 연락처 / 회원 수 / 직원 수 / 운영 상태 / 등록일 / 액션.

- 행 클릭 → 지점 상세 정보 패널 또는 상세 페이지 이동

| 컬럼 | 표시 기준 |
|---|---|
| 지점명 | 지점 운영 표시명. 행 클릭 시 지점 상세 정보 패널 또는 상세 화면을 엽니다. |
| 지점 코드 | 본사가 지점을 식별하는 고유 코드입니다. |
| 주소 | 지점 대표 주소입니다. 긴 주소는 말줄임 처리하고 상세에서 전체 주소를 확인합니다. |
| 연락처 | 지점 대표 연락처입니다. |
| 회원 수 | 해당 지점의 활성 회원 수입니다. |
| 직원 수 | 해당 지점 소속 재직 직원 수입니다. 휴직자는 포함하고 퇴사자는 제외합니다. |
| 운영 상태 | `운영 중`, `오픈 예정`, `임시휴업`, `폐점` 4종 배지 중 하나입니다. |
| 등록일 | 시스템에 지점이 등록된 날짜입니다. `YYYY-MM-DD` 형식으로 표시합니다. |
| 액션 | [수정] / [비활성화] 또는 상태별 가능한 액션입니다. 권한이 없으면 숨기거나 비활성화합니다. |

### 검색 및 필터

- 지점명 또는 주소 키워드 검색

- 운영 상태 필터 (전체 / 운영 중 / 오픈 예정 / 임시휴업 / 폐점)

### 상단 액션

- 신규 지점 등록 버튼 → 신규 지점 등록 다이얼로그 표시. 지점명, 지점 코드, 주소, 대표 연락처, 운영 시작일, 초기 Owner(지점장), 본사 표준 정책 적용 기준을 입력합니다.

### 지점별 행 액션

- 수정 버튼 → 지점 정보 수정

- 비활성화 버튼 → 비활성화/운영 상태 변경 확인 다이얼로그 표시. 대상 지점명, 현재 상태, 변경 후 상태, 적용 예정일, 변경 사유, 활성 회원 수, 진행 예약 수, 회원·직원 영향 범위를 확인한 뒤 상태 변경을 확정합니다.

### 통합 현황 비교 탭

- 지점별 회원 수, 매출, 출석률을 기간(월간 / 주간 / 연간) 기준으로 비교합니다.

- 차트는 총매출 비교 막대 차트, 활성 회원 수 비교 막대 차트, 출석률 비교 막대 차트 3개를 표시합니다.

- 통합 비교 테이블 9개 컬럼: 순위 / 지점명 / 운영 상태 / 총매출 / 활성 회원 수 / 출석 수 / 출석률 / 매출 증감률 / 상세.

- 차트 막대를 클릭하면 같은 지점의 테이블 행을 강조합니다.

- 테이블 행 또는 [상세 보기]를 클릭하면 선택 지점과 기간을 유지한 상태로 지점 성과 리포트 화면에 진입합니다.

### 3. 액션 / 흐름

1. 화면 진입 시 전체 지점 목록을 불러온다.

2. 신규 지점 등록 버튼 클릭 → 신규 지점 등록 다이얼로그 표시 → 지점 기본 정보와 초기 Owner(지점장) 지정 → 저장 후 목록에 추가.

3. 비활성화 버튼 클릭 → 비활성화/운영 상태 변경 확인 다이얼로그 표시 → 영향 범위와 사유 확인 후 운영 상태 변경.

4. 검색어 입력 또는 필터 선택 → 조건에 맞는 지점만 목록에 표시.

## SCR-093 지점 성과 리포트

> 유형: 화면
> URL 경로: `/branch-report`
> 연결 기능: HQ-04

### 1. 목적

지점별 매출, 회원, 수업 성과를 비교 분석하는 리포트 화면입니다.
지점 간 성과 차이를 시각적으로 파악하고, 기간별 추이를 확인하여 운영 전략 수립에 활용합니다.

### 필터 및 기간 선택

- 조회 기간 선택 (주간 / 월간 / 분기 / 연간, 직접 날짜 입력)

- 지점 선택 (전체 또는 특정 지점 복수 선택)

- 비교 기준 선택 (전기간 대비 / 지점 간 비교)

### 매출 성과 섹션

- 지점별 총 매출 비교 차트 (막대 차트)

- 지점별 매출 순위 테이블

- 목표 달성률 표시

### 회원 성과 섹션

- 지점별 신규 회원 수, 활성 회원 수, 이탈 회원 수 비교

- 회원 유지율 차트

### 수업 성과 섹션

- 지점별 수업 운영 횟수, 예약률, 출석률 비교

- 인기 수업 유형 분석

#### 인기 수업 산정 기준

인기 수업은 "많이 예약되고 실제 출석도 안정적으로 이루어진 수업 유형"을 의미합니다.

| 항목 | 기준 |
|---|---|
| 대상 | 조회 기간 내 운영 완료된 그룹 수업 및 PT/GX/골프/기타 수업 유형 |
| 제외 | 취소된 수업, 정원 미설정 수업, 운영 횟수 3회 미만 수업 유형 |
| 예약률 | 예약 인원 합계 / 정원 합계 × 100 |
| 출석률 | 실제 출석 인원 합계 / 예약 인원 합계 × 100 |
| 인기 점수 | 예약률 50% + 출석률 50% |
| 동점 처리 | 출석 인원 합계가 높은 수업 → 운영 횟수가 많은 수업 → 가나다순 |
| 표시 방식 | 상위 5개 수업 유형, 점수·예약률·출석률·운영 횟수 함께 표시 |

예약 인원 합계가 10명 미만이면 표본 부족으로 표시하고 인기 순위 산정에서 제외합니다.

### 3. 액션 / 흐름

1. 화면 진입 시 현재 월 기준 전체 지점 성과 데이터를 불러온다.

2. 기간 또는 지점 필터 변경 → 차트와 테이블 자동 갱신.

과거 데이터 패턴을 학습하여 미래 운영 계획 수립과 선제적 대응에 활용합니다.

- 대상 지점 선택

### 예측 인사이트

- 예측 인사이트 항목 3개: 근거 요인 설명 / 이탈 위험 회원 목록 / 권장 대응 액션.
- 근거 요인은 영향도가 큰 항목을 최대 세 개까지 표시합니다. 각 요인은 요인명, 영향 방향, 근거 수치, 기준 기간을 함께 보여줍니다.

- 컬럼은 시나리오, 예측 기준값, 시작 대비 증감, 증감률, 신뢰 구간, 주요 해석으로 구성합니다.

### 상담/퍼널 세분화

- `WI`는 방문 문의, `TI`는 전화 문의로 표시합니다.

- WI vs TI 건수는 방문 문의와 전화 문의를 분리한 막대 또는 2분할 카드로 표시합니다.

- TI 방문 전환율은 전화 문의 중 실제 대면 상담으로 이어진 비율을 표시합니다.

- TI 등록률과 WI 등록률은 동일 기간 각 문의 유형 대상자 중 등록 완료 비율로 표시합니다.

- FC 보드(SCR-095 / `docs4/V1/D10-본사관리/본사관리.md:267`)의 상담 액션 큐와 연결해 전화 문의 후 미방문, 방문 후 미등록 대상을 후속 조치 대상으로 노출합니다.

## SCR-H1001 자동화 정책 라이브러리

> 유형: 화면
> URL 경로: `/hq/automation-policies`
> 연결 기능: HQ-09
> 연결 다이얼로그: DLG-H1001-001 / `docs4/V1/D10-본사관리/본사관리.md:1110` 정책세트편집

### 1. 목적

- 본사가 전사 자동화 정책 세트를 관리하는 화면이다.

- 만료 알림 정책 유형 3개: 회원 이용권 만료 / 결제기한 만료 / 락커 만료.

- 세 정책 모두 본사가 기준일 step을 생성해 지점에 배포한다.

- 지점은 본사 step의 기준일을 수정하지 않고 Owner(지점장)만 ON/OFF할 수 있다.

- `회원 이용권 만료`와 `락커 만료`는 지점 Owner(지점장)이 해당 지점 전용 추가 step을 생성·수정·삭제할 수 있다.

- `결제기한 만료`는 지점 추가 step을 허용하지 않는다. 결제링크 만료 알림은 `docs4/_V1_정책확인필요.md`의 결제링크 정책이 고객사/판도 확인으로 확정된 뒤 D-3/D-1 고정 정책으로 별도 관리한다.

- 미수/연체 알림 N일, 반복 주기, 발송 시각, 기본 채널은 `결제기한 만료` 본사 정책 step에 포함한다. 본사(superAdmin/primary)가 설정해 지점에 배포하고, 지점은 ON/OFF만 할 수 있다.

- 수업 운영 자동화 정책은 본사에서만 생성·수정·배포한다. 대상은 미승인 일괄 등록, 수업 템플릿 본사 동기화, 대기열 자동 알림/배정, 신규 회원 첫 노쇼 면제 정책이다.

- 자동화의 경우 고객사의 정책 협의 후 개발한다. 그렇지 않을 경우 수동으로 진행하거나 V2로 진행한다.

### 2. UI 구성

- **상단 헤더**: 화면 제목, 마지막 배포 일시, 전체 정책 세트 수, 배포 실패 지점 수, `정책 세트 생성` 버튼, `PDF 내보내기` 버튼을 표시한다.

- **정책 유형 필터**: 전체 / 회원 이용권 만료 / 결제기한 만료 / 락커 만료 / 수업 운영 자동화 / 운영 자동화로 필터링한다.

- **정책 세트 카드 영역**: 정책 유형별 요약 카드를 표시한다.

| 카드 | 표시 항목 | 카드 액션 |
|---|---|---|
| 회원 이용권 만료 | 정책명, 버전, 본사 step 수, 지점 추가 step 허용 여부, ON 지점 수, OFF 지점 수 | 상세 보기, 복제, 배포 |
| 결제기한 만료 | 정책명, 버전, 본사 step 수, 미수/연체 알림 N일, 지점 추가 step 불가 표시, ON 지점 수 | 상세 보기, 복제, 배포 |
| 락커 만료 | 정책명, 버전, 본사 step 수, 지점 추가 step 허용 여부, 회수 정책 분리 안내 | 상세 보기, 복제, 배포 |
| 수업 운영 자동화 | 정책명, 버전, 적용 지점 수, 자동화 항목 수, 개발 대기 항목 여부 | 상세 보기, 복제, 배포 |
| 운영 자동화 | 정책명, 버전, 적용 범위, 마지막 배포 일시, 실패 지점 수 | 상세 보기, 복제, 배포 |

- **정책 세트 테이블**: 정책 세트를 목록으로 조회하고 행 단위 액션을 제공한다.

| 컬럼 | 표시 기준 |
|---|---|
| 정책명 | 운영자가 식별하는 정책 세트명 |
| 정책 유형 | 회원 이용권 만료 / 결제기한 만료 / 락커 만료 / 수업 운영 자동화 / 운영 자동화 |
| 버전 | 현재 배포 버전 |
| 적용 범위 | 전체 지점 / 선택 지점 / 미적용 |
| 본사 step 수 | 본사가 정의한 step 개수 |
| 지점 추가 step | 허용 / 불가 |
| 지점 ON/OFF 현황 | ON 지점 수 / OFF 지점 수 |
| 마지막 배포 | 배포 일시와 배포자 |
| 상태 | 초안 / 배포됨 / 중지됨 / 배포 실패 |
| 액션 | 상세 / 수정 / 복제 / 중지 / 배포 / 시뮬레이션 |

- **Step 미리보기 패널**: 정책 행 또는 카드를 선택하면 기준일(D-숫자/D-Day/D+숫자), 발송 시각, 반복 주기, 기본 채널(회원앱 Push 기본, KakaoTalk fallback 선택), 메시지 템플릿, 기본 활성값, 지점 제어 가능 여부를 표시한다. 결제기한 만료 step에서는 미수/연체 알림 N일과 운영자 알림 발송 시각을 함께 표시한다.

- **지점 적용 현황 패널**: 선택한 정책을 적용 중인 지점 목록, 지점별 ON/OFF 상태, 마지막 적용 일시, 실패 사유, 수동 재적용 버튼을 표시한다.

| 버튼 | 동작 |
|---|---|
| 정책 세트 생성 | DLG-H1001-001 / `docs4/V1/D10-본사관리/본사관리.md:1110` 정책 세트 편집 다이얼로그를 빈 폼으로 연다 |
| 상세 | 선택 정책의 step 미리보기와 지점 적용 현황을 우측 패널에 표시한다 |
| 수정 | 정책 세트 편집 다이얼로그를 기존 값으로 열고 저장 전 영향 범위를 표시한다 |
| 복제 | 기존 정책명을 복사한 새 초안 정책을 만들고 이름 입력을 요구한다 |
| 중지 | 신규 적용을 차단하고 기존 적용 지점 영향 범위를 고지한 뒤 중지 상태로 전환한다 |
| 배포 | 대상 지점, 변경된 step, 채널, 발송 시각을 확인한 뒤 배포한다 |
| 시뮬레이션 | 최근 회원/미수/락커 데이터를 기준으로 예상 발송 대상 수와 제외 건수를 계산해 보여준다 |
| PDF 내보내기 | 정책 세트, step 목록, 적용 지점, 버전 정보를 PDF로 생성한다 |

### 3. 액션 / 흐름

1. `정책 세트 생성` 버튼 클릭 시 DLG-H1001-001 / `docs4/V1/D10-본사관리/본사관리.md:1110` 정책 세트 편집 생성 다이얼로그를 연다.

2. 정책 카드 선택 시 우측 상세 패널에 적용 범위, 정책 유형, 본사 step, 지점 추가 step 허용 여부, 신규 적용 상태를 즉시 반영한다.

3. `허용 범위 설정` 버튼 클릭 시 적용 범위, 신규 적용 상태, 지점 추가 step 허용 여부를 조정하는 다이얼로그를 연다.

4. 사용 중지된 정책은 신규 적용만 차단하고 기존 사용 지점은 영향 범위를 고지한다.
