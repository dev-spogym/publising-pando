# 비기능 요구사항(NFR) 통합 정의서

## 1. 문서 목적

이 문서는 `docs4` 전체에서 반복 참조되는 비기능 요구사항의 단일 기준입니다.

NFR은 특정 화면의 고유 기능이 아니라 여러 화면에서 공통으로 필요한 운영 품질 기준입니다. 예를 들어 만료 알림 배치, 알림 센터, 보안 잠금, 감사 로그, 대용량 다운로드, 외부 연동 실패 재시도 같은 항목은 한 화면의 버튼 기능이 아니지만 여러 기능이 동일 기준으로 동작해야 합니다.

화면 문서에는 NFR-19 / `docs4/_공통/공통참조_코드인덱스.md:33`처럼 코드만 적지 않고, 반드시 "무엇을 누구에게 어떤 채널로 알리는지"를 함께 적습니다.

## 2. 적용 원칙

- 이 문서는 NFR 코드의 의미를 정의하는 기준 문서입니다.
- 화면별 UI, 버튼, 필드, 권한은 각 도메인 문서를 우선합니다.
- 공통 에러 표현은 `_공통/에러_예외_표준.md`를 따릅니다.
- 공통 권한 역할은 `_공통/권한매트릭스.md`를 따릅니다.
- 자동화 실행 목록은 `_공통/자동화_크론.md`를 따릅니다.
- 외부 연동, fallback, 재시도는 `_공통/외부연동_현황.md`와 본 문서 기준을 함께 봅니다.
- 문서 간 충돌 시 `docs4` 내부에서 값 하나로 보정해야 하며, 개발자가 `docs/admin`으로 역추적하지 않도록 합니다.

## 3. 정식 NFR 코드

현재 `docs4`에서 정식 코드로 반복 사용되는 NFR은 NFR-05 / `docs4/_공통/공통참조_코드인덱스.md:26`, NFR-19 / `docs4/_공통/공통참조_코드인덱스.md:33`입니다.

| 코드 | 명칭 | 의미 | 구현 판단 기준 |
|---|---|---|---|
| NFR-05 / `docs4/_공통/공통참조_코드인덱스.md:26` | 만료/기한 알림 배치 및 결과 동기화 | 회원 이용권 만료, 결제기한 만료, 락커 만료처럼 날짜 기준으로 대상자를 산출하고 화면 배지·배너·알림 큐에 반영하는 공통 배치 기준 | 대상 정책 유형, 산출 시각, 지점 step 허용 여부, 결과 소비 화면이 명시되어야 함 |

## 4. NFR-05 / `docs4/_공통/공통참조_코드인덱스.md:26` 만료/기한 알림 배치

## 5. NFR-19 / `docs4/_공통/공통참조_코드인덱스.md:33` 알림 센터 / 발송 큐

### 5.1 공통 정의

NFR-19 / `docs4/_공통/공통참조_코드인덱스.md:33`는 운영 이벤트를 사용자에게 전달하고, 미열람 수, 발송 결과, 재시도, 이동 링크를 관리하는 공통 알림 기준입니다.

NFR-19 / `docs4/_공통/공통참조_코드인덱스.md:33`는 마케팅 메시지 발송 기능 자체가 아닙니다. 마케팅 캠페인, 쿠폰 발급 메시지, 일괄 메시지 발송처럼 화면 고유 기능은 해당 도메인 문서가 주관하고, 그 결과 알림·실패 알림·완료 알림만 NFR-19 / `docs4/_공통/공통참조_코드인덱스.md:33` 기준으로 처리합니다.

### 5.2 채널 기준

| 채널 | 사용 기준 |
|---|---|
| 관리자 인앱 알림 | 관리자 웹 알림 센터에 표시하는 기본 운영 알림 |
| 회원앱 Push | 회원 대상 자동 알림의 기본 채널 |
| 카카오톡 fallback | 지점 설정이 켜져 있고 회원앱 Push 실패 또는 미설치인 경우 선택 사용 |
| SMS | 정책 step 또는 화면에서 명시된 경우에만 사용. 자동 fallback으로 임의 전환하지 않음 |

### 5.3 알림 이벤트에 반드시 포함할 정보

| 항목 | 설명 |
|---|---|
| 이벤트명 | 예: 환불 승인 요청, 수업 시간 변경, 엑셀 다운로드 완료 |
| 발생 조건 | 어떤 상태 변경, 배치 결과, 실패 조건에서 생성되는지 |
| 수신자 | 회원, 담당 FC, Owner(지점장), manager, superAdmin 등 |
| 채널 | 관리자 인앱, 회원앱 Push, 카카오톡 fallback, SMS 중 무엇인지 |
| 메시지 목적 | 확인 요청, 완료 안내, 실패 알림, 조치 요청 등 |
| 이동 대상 | 클릭 시 이동할 화면 또는 열릴 다이얼로그 |
| 실패 처리 | 재시도 여부, 제외 처리, 운영자 알림 여부 |
| 이력 | 발송자/발송시각/상태/읽음 여부/실패 사유 |

### 5.4 재시도 기준

- 회원앱 Push와 카카오톡 fallback은 화면 또는 정책에서 별도 기준이 없으면 30분 간격 최대 3회 재시도합니다.
- 최종 실패 시 운영자에게 실패 건 목록을 알립니다.
- 일괄 메시지 발송에서 카카오톡 비친구 또는 실패 대상은 실패/제외 건수로 표시합니다.
- SMS 자동 전환은 기본 제공하지 않습니다. SMS는 해당 기능에서 명시한 경우에만 발송합니다.

## 6. 코드 없이 적용되는 공통 NFR 기준

아래 항목은 현재 별도 NFR 코드가 없지만 비기능 요구사항입니다. 화면 문서에서 필요할 때는 기능 설명으로 직접 적고, 관련 공통 문서를 함께 참조합니다.

- 인증/세션/보안: 로그인 잠금, 2FA, IP 제한, 세션 만료, 토큰 갱신 기준은 D01-공통 운영정책과 D01-공통 공통 문서를 따른다.
- 권한/RBAC: 8개 기본 역할, 화면 접근, 조회/수정/승인/일괄처리 권한은 `_공통/권한매트릭스.md`와 D09 설정관리 기준을 따른다.
- 감사 로그/히스토리: 변경 액션, 권한 변경, 환불, 결제링크 무효화, 보안 이벤트 이력은 D10 히스토리 로그와 각 도메인 연동 표를 따른다.
- 대용량/비동기 처리: 엑셀/CSV 대용량 다운로드, 일괄 작업, 백그라운드 처리는 `_공통/자동화_크론.md`와 `_공통/에러_예외_표준.md`를 따른다.
- 에러/예외: 401/403/404/5xx/timeout, 빈 목록, 권한 부족, 재시도 안내는 `_공통/에러_예외_표준.md`와 `_공통/토스트_메시지.md`를 따른다.
- 개인정보/마스킹: 탈퇴/삭제 90일 경과 PII 마스킹, 권한자 사유 입력 후 일시 해제는 D02 회원관리와 `_공통/자동화_크론.md`의 X11 / `docs4/_공통/공통참조_코드인덱스.md:75` 기준을 따른다.
- 외부 연동 장애 대응: POS/PG/IoT/Push/KakaoTalk 장애, 재시도, 수동 대체 흐름은 `_공통/외부연동_현황.md`와 관련 도메인 문서를 따른다.
- 기기 헬스체크: 출입 게이트, 키오스크, 락커 컨트롤러, InBody 측정기 상태 확인은 D09 설정관리와 D11 통합운영 기준을 따른다.
