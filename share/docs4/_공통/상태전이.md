# 공통 상태 전이

## 기본 상태 축
| 축 | 주요 상태 |
|---|---|
| 데이터 로드 | 로딩 / 정상 / 빈 상태 / 오류 |
| 권한 | 허용 / 읽기 전용 / 차단 |
| 작업 | 대기 / 처리 중 / 성공 / 부분 실패 / 실패 |
| 계정 | ACTIVE / LOCKED / INACTIVE |
| 지점 | 운영 중 / 오픈 예정 / 임시휴업 / 폐점 |
| 기기 | OK / 불안정 / 오류 / 점검 중 |
| 회원 실제 상태 | ACTIVE(활성) / EXPIRED(만료) / SCHEDULED(예정) / EXPIRING(임박) / HOLDING(홀딩) / UNREGISTERED(미등록) / WITHDRAWN(탈퇴) |
| 회원 제한 플래그 | 정지. 실제 회원 상태값이 아니라 현재 회원 상태 위에 추가되는 이용 제한 플래그 |
| 리드 상담 단계 | 신규 / 연락완료 / 상담예정 / 방문완료 / 등록완료 / 미전환 / 보류 |
| 결제링크 | 발송됨 / 결제완료 / 만료 / 무효화 / 발송실패 |
| 출석 결과 | 성공 / 실패 / 중복 |

## 원칙
- 상태값은 화면 표시 배지와 저장 상태를 혼동하지 않습니다.
- 표시 상태가 저장 상태를 가공한 값이면 문서에 산식을 명시합니다.
- 리드 `등록완료`는 첫 정상 결제 완료 이벤트로만 확정되는 시스템 단계이며, 칸반 이동은 회원 등록/결제 흐름 시작 트리거입니다.
- 결제링크 발송 후 결제 완료 전 결제내역 화면에는 `미결제`만 표시합니다. 결제링크 저장 상태값 `발송됨`은 화면 문구로 노출하지 않고 재발송·무효화 버튼 노출 기준으로만 사용합니다.
- 출석 `중복`은 실패 건수에 합산하지 않고 중복 건수로 별도 집계합니다.
