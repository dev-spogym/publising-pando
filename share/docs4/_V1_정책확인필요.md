# V1 정책 확인 필요

이 문서는 `docs4/V1` 안에 화면 또는 정책 후보로 남아 있지만, 고객사 또는 판도 측 최종 판단이 있어야 개발 기준으로 확정할 수 있는 항목만 모은 목록입니다. 아래 항목은 이번 V1 개발 과정에서 정책 확인을 받은 뒤 개발 기준으로 반영합니다. 확인 전에는 최종 구현 완료 기준으로 보지 않습니다.

| 구분 | 현재 문서 위치 | 확인 필요 내용 | V1 개발 기준 |
|---|---|---|---|
| 결제링크 정책 | D03 결제/매출, D02 회원상세 결제내역 | 전액 결제 전용, 링크 7일 고정, D-3/D-1 만료 전 알림, 활성 링크 1개, 무효화 권한과 사유 입력, 결제 완료 즉시 이용권 활성화 기준의 고객사 확정 | 고객사 확정 전에는 자동 만료, 자동 알림, 활성 링크 제한, 결제 완료 자동 활성화, 미결제 링크 환불 처리를 구현하지 않는다. 미결제 링크는 환불 대상이 아니며 회원상세 결제내역에 `미결제` 상태로만 표시한다. |
| 자동 알림 13종 | D08 자동 알림, D09 자동화 설정, D10 자동화 정책 라이브러리 | 계약서에 예시형으로 적힌 자동 알림 13종의 정확한 명칭, 수신 주체, 채널, 발송 시점 | 확정 전에는 문서에 명시된 회원 이용권 만료, 결제기한 만료, 락커 만료, 생일 축하, 장기 미방문, 등록 감사, 첫 방문 환영, 이용권 만료 당일, 홀딩 종료, 결제 완료, 환불 처리 범위만 후보로 관리한다. 목록 밖 알림은 자동 포함하지 않는다. |
| 출석 N회 달성 알림 | D08 자동 알림, D04/D11 출석 이벤트 | N회 기준값, 발송 대상, 발송 채널, 반복 발송 여부 | 트리거 후보로만 표시하고, 조건 입력과 자동 발송은 확정 후 개발한다. |
| 폐점 데이터 보관 기간 | D10 지점/본사 운영정책 | 지점 폐점 후 데이터 보관 N개월, 보관 만료 후 아카이브 또는 삭제 기준 | 폐점 상태 전환과 이관/환불 안내까지만 V1 기준으로 삼고, 보관 기간 만료 후 자동 아카이브는 확정 후 개발한다. |
| 쿠폰 발급 안내 메시지 | D08 쿠폰 관리 | 쿠폰 발급 시 회원에게 자동 메시지를 보낼지, 항상 발송인지 직원 입력 시에만 발송인지, Push/카카오톡/SMS 채널과 실패 재시도 기준 | V1에서는 쿠폰 발급 자체만 개발 기준으로 삼고, 안내 메시지 자동 발송·재시도·실패 처리는 확정 후 개발한다. |
| 외부 Google Spreadsheet 산식 | D03 매출, D04 수업, D10 본사 KPI/보고 | `[매출종합]`, `[월매출]`, `[OT관리-강습/골프]`, `[OT관리-강습]`, `[강습업무보고]`, `[주간월간보고]` 등 운영 시트의 컬럼 정의와 산식 | `판도 스프레드시트 산식 참조`가 필요한 항목이다. 해당 시트명은 현재 운영 원천을 뜻하며, 판도에서 각 시트의 컬럼 정의, 산식, 집계 기준을 개발사에 제공해야 자동 집계 개발 기준으로 확정된다. 제공 전에는 화면 표시 항목과 수기/임시 입력 기준만 개발한다. |
| 전자계약 | D08 SCR-075 전자 계약 | 회원 계약/직원 근로계약의 V1 포함 여부, 템플릿 종류, 자동 로드 기준, 원격 서명 링크, 재발송 채널, 보관/조회 정책 | 화면과 기능 후보는 유지하되, 템플릿과 서명/발송 정책은 스포짐 또는 판도 측 의견 수령 후 확정한다. 확정 전에는 구현 완료 기준으로 보지 않는다. |

## V1 제외 / V2 이관

| 구분 | V2 문서 위치 | 처리 기준 |
|---|---|---|
| 캠페인 A/B 테스트 | D08 SCR-079 A/B 테스트, DLG-079-001 A/B 테스트 등록, DLG-079-002 A/B 테스트 삭제 확인 | V1 제외 대상이다. V1에서는 목록, 등록, 삭제, 자동 분배, 승리안 채택, 우선순위 계산을 개발하지 않는다. 해당 기능은 V2 문서에서 후속 범위로 관리한다. |

## 전달 기준

- 확인 필요 항목은 삭제하지 않고 `docs4/V1` 본문에 후보 또는 보류 상태로 남길 수 있다.
- 단, 본문에는 `고객사/판도 확인 후 개발` 또는 이 문서 참조를 명시해 개발사가 확정 구현 범위로 오해하지 않게 한다.
- 확정 답변이 오면 이 문서의 항목을 제거하고, 해당 도메인 문서와 운영정책 문서에 확정값을 직접 반영한다.
- V1 제외 / V2 이관 항목은 `docs4/V1` 본문에서 제거하거나 제외 안내만 남기고, 실제 화면/다이얼로그 정의는 `docs4/V2` 기준으로 관리한다.
