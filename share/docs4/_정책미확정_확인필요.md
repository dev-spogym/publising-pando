# 정책 미확정 확인 필요

이 문서는 `docs4` 기준으로 개발 착수 전 고객사 또는 판도 운영 판단이 필요한 정책값만 모은 목록입니다. API/DB 설계 방식이 아니라 화면·기능·UX·운영정책 기준에서 확정이 필요한 항목입니다.

V1 화면 안에 남아 있는 보류성 항목은 `docs4/_V1_정책확인필요.md`에서 별도로 관리합니다. 개발사는 해당 문서의 항목을 확인 전 확정 구현 범위로 해석하지 않습니다.

## 미확정에서 제외된 항목

- **지점 운영 임계값 중 미수/연체 알림 N일**: 고객사별 고정값을 별도로 확정받는 항목이 아니라 `HQ-09 결제기한 만료` 본사 정책 세트에 포함한다. 본사(superAdmin/primary)가 기준일(D+N), 반복 주기, 발송 시각, 기본 채널을 설정해 지점에 배포하고, 지점은 해당 step을 ON/OFF만 할 수 있다. 지점 추가 step은 허용하지 않는다.
- **회원 세그먼트 운영 임계값**: 이탈위험은 최근 30일 이상 미방문, 만료후미등록은 마지막 회원 이용권 만료일 +60일 경과로 확정 고정값이다.

## 고객사 추가 확인 질문

- **쿠폰 발급 안내 메시지**: 쿠폰 발급 완료 후 회원에게 자동 메시지를 보낼지 확인이 필요하다. 발송한다면 항상 기본 템플릿으로 발송할지, 직원이 발급 메시지를 입력한 경우에만 발송할지, 발송 채널을 Push / 카카오톡 / SMS 중 무엇으로 할지 확정해야 한다. 고객사 확인 전까지 MKT-03은 쿠폰 발급만 개발 기준으로 삼고, 메시지 자동 발송·입력 필드·재시도 정책·발송 실패 처리는 개발 대기한다.
