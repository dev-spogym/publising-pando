# docs3 정합성 관리

`docs3`는 개발 납품용 기획서가 아니라, `docs/admin`과 `docs2`의 정합성을 추적하기 위한 관리 폴더입니다.

## 산출물

- `00_정합성관리_마스터.xlsx`: ID, route, 배지/알림, 정책값, 권한, 연동, 이슈를 엑셀 시트로 관리합니다.
- `reports/정합성_이슈_리포트.md`: 자동 검출된 우선 검토 이슈입니다.
- `reports/docs2_목록정합성_리포트.md`: 같은 개념의 목록/상태/옵션 세트가 docs2 내부에서 다르게 쓰이는 후보입니다.
- `reports/고객사_판단필요_목록.md`: 고객사 또는 판도 대표 판단이 필요한 후보입니다.
- `reports/우선수정_묶음.md`: 개발사 질의 가능성이 높은 이슈를 기능 묶음 단위로 정리한 우선 처리 목록입니다.
- `reports/개발사_질의_방지_체크리스트.md`: 문서 수정 시 빠뜨리면 개발사 질의가 재발하는 항목입니다.

## 사용 원칙

1. 개발사 질문이 들어오면 먼저 엑셀의 `정합성이슈`, `배지알림정의`, `ID참조검증` 시트를 확인합니다.
2. 수정이 필요한 항목은 `docs/admin` 정본과 `docs2` 개발용 문서에 모두 반영합니다.
3. 고객사 정책 판단이 필요한 항목은 문서를 임의 확정하지 않고 `고객사판단필요`에 남깁니다.
4. 배지/알림/상태/대시보드/권한/정책값 문장은 표시 위치, 기준, 클릭 이동, 설정 주체를 반드시 명시합니다.

## 재생성

```bash
node --import tsx scripts/build-docs3-consistency.ts
```
