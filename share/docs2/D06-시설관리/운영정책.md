# D06-시설관리 시설관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.

## 1. 핵심 원칙
- 시설 운영과 부가서비스를 회원 운영과 끊기지 않게 정리한다.
- 락커, 카드, 운동룸, 타석, 상품 재고 같은 실물 자산을 일관된 기준으로 관리한다.
- 락커
- 사물함 배정
- RFID/밴드 카드
- 운동룸
- 골프 타석
- 상품 재고
- 시설자산은 `자산 마스터`, `현재 상태`, `배정 대상`, `이력`으로 본다.
- 회원 이용권과 연결되는 자산은 만료일과 연동해 운영한다.
- 단발성 점검 상태와 장기 비활성 상태를 같은 개념으로 쓰지 않는다.
- 배정형 자산은 언제 누구에게 연결되었는지 이력이 남아야 한다.
- 락커는 회원 부가 이용권 또는 옵션 상품과 연결한다.
- 상태는 `사용가능 / 사용중 / 만료임박 / 만료 / 고장 / 비활성` 기준으로 본다.
- 만료임박 판정은 자동화 정책과 같은 기준을 따른다.
- 락커 배정과 해제는 회원 상세에서도 진입 가능해야 한다.
- 카드 자체 상태와 회원 매핑 상태를 분리한다.
- 카드 분실, 재발급, 비활성 처리는 이력 중심으로 남긴다.

## 2. 상태 / 운영 정의
- 센터 내 모든 락커의 배정 현황과 상태를 한눈에 파악하고 관리하는 화면(SCR-050 / `D06-시설관리/시설관리.md:23`)이다. 구역별(A/B/C) 그리드 배치도로 각 락커의 사용 여부·만료 임박·고장 상태를 시각적으로 확인하고, 회원 배정·이동·회수·고장 처리를 즉시 수행한다. 만료 임박 락커를 한 번에 일괄 해제하거나 여러 빈 락커를 한 명의 회원에게 일괄 배정하는 기능으로 운영 부담을 줄인다.
- **락커 만료 임박 일괄 회수**: 매월 말 락커 만료 정책 step 대상 락커를 일괄 해제하여 운영 회의에서 처리 (DLG-050-007 / `D06-시설관리/시설관리.md:1339`)
- **장기 결석 회원 락커 정리**: 회원 상태 점검 후 락커 회수 + 회원에게 알림 발송
- **고장 락커 격리**: 잠금 불량·키 분실 시 즉시 고장 토글로 배정 차단 + 수리 접수 (FAC-EXT-02 / `D06-시설관리/시설관리.md:549` 연계)
- **이용권 종료 자동 회수**: D09 설정관리의 `이용권 만료 이벤트 자동 회수` 정책이 ON일 때만 이용권 만료 일자 도달 시 락커 자동 해제
- **마감 시간 일괄 해제**: 일일 사물함 22시 overtime 상태 갱신은 고정 ON이며, 실제 일괄 회수는 D09 설정관리의 `일일 사물함 22:00 실제 일괄 회수` 정책이 ON일 때만 자동 실행. OFF일 때 잔여 미반환은 운영자가 D06에서 [만료 일괄 해제]로 수동 회수
- D06은 락커/사물함 배정, 수동 회수, 일괄 회수, 회수 이력 확인을 담당한다. 자동 회수의 기본값·ON/OFF·실행 조건은 D09 SCR-080A / `D09-설정관리/설정관리.md:132` 지점 자동화 적용에서 관리하고, D06 화면에는 현재 적용 중인 정책 상태만 표시한다.
- **시간 초과 회원 알림**: 만료 시각 경과 시 깜빡이 표시 + 회원에게 알림 발송, 운영자 즉시 인지
- **상태 비정상 사물함 격리**: 잠금 불량 시 클릭 차단 + 점검 등록(FAC-EXT-02 / `D06-시설관리/시설관리.md:549`)
- 상태 비정상 | error |
- 일일 사물함 / 개인 사물함 / 골프 사물함. 각 탭에 카운트 배지.
- 활성(사용중) | status=active |
- 미사용·해제·분실 합계 | 비활성 카드 |
- **상태 필터**: 활성 / 분실 / 해제
- 상태 | 활성/분실/해제 배지 |
- 행 [분실] → 상태 "분실" 변경 + 회원 알림.
- 센터 내 운동룸(GX룸·PT룸·스피닝룸·필라테스룸 등)의 현황과 예약 슬롯을 관리하는 화면(SCR-053 / `D06-시설관리/시설관리.md:288`)이다. 각 룸의 운영 상태를 확인하고, 오늘의 시간대별 예약 혼잡도를 시각적으로 파악할 수 있다. 새 운동룸을 등록하거나 기존 룸 정보를 수정·삭제하고, 운영 상태를 빠르게 전환할 수 있다.
- 상태 전환·등록·수정·삭제 모두 SCR-097 / `D10-본사관리/본사관리.md:694` 감사 로그에 기록되며, 상태 변경은 즉시 SCR-C001 / `D04-수업관리/수업관리.md:24` 캘린더 예약 시스템에 반영된다.
- 운영중 | status=operating |
- 유형 배지 | GX / PT / 스피닝 / 필라테스 / 기타 |
- 운영 상태 배지 | 운영중/점검중/고장 |

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs2/_정책미확정_확인필요.md`에 별도 관리한다.
