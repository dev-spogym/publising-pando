## 화면
## SCR-100 로그인

> 유형: 화면
> URL 경로: `/login`
> 연결 기능: MFN-SCR-100 / `D01-공통/공통.md:20`, AUTH-01 / `D01-공통/공통.md:20`
> 연결 다이얼로그: DLG-000 / `D01-공통/공통.md:626` 세션 만료

### 1. 목적
본사 또는 지점에서 등록한 `직원 로그인 계정`으로 CRM 시스템에 접속하는 유일한 진입 화면입니다.
본 시스템은 `직원 1명 = 로그인 계정 1개`를 기본 원칙으로 하며, 공개 회원가입이나 공용 운영 계정 없이 직원에게 발급된 로그인 ID와 비밀번호로만 접근합니다. 로그인 후에는 계정 역할과 접근 지점 범위에 따라 각자의 전용 작업공간으로 자동 이동합니다.

### 2. UI 구성
- 지점 선택 드롭다운
- 브랜드 로고 및 서비스명
- 로그인 ID 입력란
- 계정 이메일 안내 영역 (비밀번호 재설정 / 보안 알림용)
- 비밀번호 입력란 (표시/숨김 전환 버튼 포함)
- 로그인 상태 유지 체크박스
- 로그인 버튼
- 비밀번호 재설정 링크
- 테스트 계정 프리셋 패널
- 이미 로그인된 사용자 자동 우회 처리
- 첫 로그인 임시 비밀번호 변경 강제 안내
- 계정 잠금 / 비활성 / 휴직 상태 안내 배너
- 2단계 인증(2FA) 추가 인증 단계
- 자동 로그아웃 / 동시 세션 정책 안내

### 3. 액션 / 흐름
1. 직원 등록 시 발급된 로그인 ID와 비밀번호 입력 후 로그인 버튼 클릭
2. 지점 선택값, 계정 활성 상태, 잠금 상태, 권한 템플릿, 접근 지점 범위를 함께 확인
3. 첫 로그인 또는 임시 비밀번호 상태라면 비밀번호 강제 변경 단계로 우선 이동
4. 2단계 인증(2FA) 활성 계정은 추가 인증 코드 입력 단계 진행
5. 인증 성공 → 계정 역할에 맞는 기본 작업공간으로 자동 이동
6. 이미 로그인된 상태라면 로그인 화면을 건너뛰고 해당 작업공간으로 바로 이동
7. 퇴사/휴직/비활성 계정은 로그인 차단 후 관리자 문의 안내를 표시

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 | 로그인 ID·비밀번호 입력 대기 |
| 로그인 중 | 버튼 클릭 후 처리 중 (버튼 비활성) |
| 오류 | 잘못된 정보 입력 시 안내 메시지 표시 |
| 계정 잠금 | 연속 실패 횟수 초과 시 30분 잠금, 남은 시간 카운트다운 표시, 30분 후 자동 해제 또는 superAdmin / primary / Owner(지점장) 수동 해제 |
| 계정 비활성 | 휴직·퇴사·보안 비활성 상태로 로그인 차단 |
| 2단계 인증 | 추가 인증 코드 입력 단계 |
| 첫 로그인 | 임시 비밀번호 변경 강제 |
| 서비스 점검 | 시스템 점검 중 안내 화면 |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 지점 전체 데이터를 조회하고 업무 액션을 수행할 수 있습니다. |
| trainer / fc / staff | 본인 담당 범위 또는 배정된 업무 데이터 중심으로 조회합니다. |
| readonly | 조회만 가능하며 생성·수정·삭제·확정 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 메뉴와 직접 URL 접근을 차단하고 권한 안내 화면으로 전환합니다. |

계정 잠금 수동 해제 기준: 잠금 계정의 수동 해제는 D07 직원관리 > 직원 상세(STF-02 / `D07-직원관리/직원관리.md:21`) > 계정 / 보안 카드의 [잠금 해제] 버튼에서만 실행합니다. superAdmin / primary는 전체 지점 직원 계정을 해제할 수 있고, Owner(지점장)는 본인 소속 지점 직원 계정만 해제할 수 있습니다. manager / fc / trainer / staff / readonly는 수동 잠금 해제를 실행할 수 없습니다. 해제 시 실패 카운트를 0으로 초기화하고 SCR-097 / `D10-본사관리/본사관리.md:694` 감사 로그에 처리자, 대상 계정, 처리 시각, 사유를 남깁니다.

### 6. 필수 예외처리
| 케이스 | 처리 |
|---|---|
| 이메일 형식 오류 | 제출 전 인라인 유효성 안내 |
| 존재하지 않는 이메일 | "이메일 또는 비밀번호가 올바르지 않습니다" (계정 존재 여부 노출 차단) |
| 비밀번호 불일치 | 동일 메시지로 통일 처리 |
| 5회 연속 실패 | 30분 계정 잠금, 남은 시간 표시, "30분 후 다시 시도하거나 관리자에게 문의하세요" 안내 |
| 2FA 코드 만료 | "인증 코드가 만료되었습니다. 재발송해주세요" 안내 |
| 2FA 코드 불일치 | "인증 코드가 올바르지 않습니다" 안내 |
| 서비스 점검 중 | 점검 안내 화면으로 대체 표시 |
| 네트워크 단절 | 오프라인 안내 + 재시도 버튼 |

### 8. 핵심 디자인 요청사항
- 공통 색상, 간격, 폰트, 토스트 규칙은 `_공통/디자인_시스템.md`, `_공통/토스트_메시지.md`를 따른다.
- 본 섹션에 별도 명시된 UI 상태와 권한 차이가 있으면 도메인 공통보다 화면 정의를 우선한다.

## SCR-101 대시보드 통합

> 유형: 화면
> URL 경로: `/`
> 연결 기능: MFN-SCR-101 / `D01-공통/공통.md:96`

### 1. 목적
로그인 직후 `/` route에서 역할을 판정한 뒤, `superAdmin` 또는 `primary` 계정에게 표시되는 전 지점 통합 대시보드입니다.
전체 지점 KPI, 지점별 운영 이슈, 전사 매출·출석·회원 현황을 한 화면에서 확인합니다.
`Owner(지점장)`, `manager`, `fc`, `trainer`, `staff`, `readonly` 계정은 같은 `/` route에 진입해도 본 화면이 아니라 D10 SCR-090 / `D10-본사관리/본사관리.md:20` 지점 대시보드를 표시하며, 해당 계정의 소속 지점 데이터만 봅니다.

### 2. UI 구성
- 오늘 집중 업무 코크핏 (HQ-09 / `D10-본사관리/본사관리.md:894` 만료 알림 대상 / 미수금 / 홀딩 관리 우선순위 카드)
- 역할별 빠른 액션 패널 (회원, 매출, 자동 알림, Today Tasks 등 계정 역할에 맞는 바로가기)
- 오늘의 핵심 수치 요약 (오늘 출석, 당일 매출, HQ-09 / `D10-본사관리/본사관리.md:894` 만료 알림 대상 회원 수 등)
- 실시간 출석 확인 카드 (키오스크·QR·RFID·얼굴인식·수동 출석 회원 동일인 확인)
- 수업 현황 위젯 (오늘 수업 일정 및 예약 현황)
- 회원 현황 위젯 (신규 등록, HQ-09 / `D10-본사관리/본사관리.md:894` 만료 알림 대상 등)
- 매출 현황 위젯 (당월 누적 등)
- 빠른 액션 버튼 (회원 등록, 결제 처리 등)
- 엑셀 내보내기 버튼 (권한 있는 계정만)

### 3. 액션 / 흐름
1. 로그인 완료 → `/` route 진입 → 역할 판정.
   - `superAdmin` / `primary`: SCR-101 / `D01-공통/공통.md:96` 전 지점 통합 대시보드 표시.
   - 그 외 역할: SCR-090 / `D10-본사관리/본사관리.md:20` 지점 대시보드 표시.
2. 상단 오늘 집중 업무 카드에서 HQ-09 / `D10-본사관리/본사관리.md:894` 만료 알림 대상·미수·홀딩 이슈를 먼저 확인한다.
3. 역할별 빠른 액션 패널에서 현재 계정에 맞는 운영 화면으로 즉시 이동한다.
4. 실시간 출석 확인 카드에서 신규 입장 회원의 사진·기본 정보·주의 태그를 확인한다.
5. 동일인 확인 또는 보완이 필요하면 출석 카드 클릭 → 회원 상세 기본정보로 이동한다.
6. 각 위젯을 클릭하면 해당 상세 화면으로 이동
7. 빠른 액션 버튼으로 주요 업무 즉시 시작
8. 슈퍼관리자는 상단 지점 필터로 다른 지점 현황 조회 가능

## SCR-102 사이드바 네비게이션

> 유형: 화면
> URL 경로: `(전체 화면 공통)`
> 연결 기능: MFN-SCR-102 / `D01-공통/공통.md:168`

### 1. 목적
모든 화면에서 공통으로 사용되는 좌측 메뉴 영역으로, 시스템의 주요 기능으로 이동할 수 있는 네비게이션입니다.
계정 역할에 따라 접근 가능한 메뉴 항목이 다르게 표시됩니다.

### 2. UI 구성
- 브랜드 로고 (클릭 시 계정 역할의 기본 작업공간으로 이동)
- 지점명 표시
- 메뉴 항목 목록 (아이콘 + 텍스트)
- 현재 위치 강조 표시
- 접기/펼치기 버튼 (아이콘만 보기로 전환)
- 슈퍼관리자 전용 본사 관리 섹션
- 권한 없는 메뉴 자동 숨김 로직

### 3. 액션 / 흐름
1. 원하는 메뉴 클릭 → 해당 화면으로 이동
2. 현재 위치한 메뉴가 강조 표시됨
3. 현재 경로에 맞는 메뉴 그룹이 자동으로 펼쳐짐
4. 브랜드 로고 클릭 시 현재 역할의 기본 작업공간으로 복귀
5. 좁은 화면 또는 접기 버튼 클릭 시 아이콘만 표시
6. 모바일에서는 햄버거 버튼으로 사이드바 열기/닫기

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 펼침 | 메뉴명 + 아이콘 모두 표시 |
| 접힘 | 아이콘만 표시 (마우스 올리면 툴팁) |

## DLG-002 이탈 경고

> 유형: 다이얼로그
> URL 경로: `(전체 화면 공통 - 미저장 상태에서 이탈 시)`
> 연결 기능: DLG-002 / `D01-공통/공통.md:724`
> 연결 다이얼로그: DLG-002 / `D01-공통/공통.md:724` 이탈경고

### 1. 목적
사용자가 등록/수정 화면에서 저장하지 않은 내용이 있는 상태로 다른 화면으로 이동하려 할 때, 데이터 손실을 방지하기 위해 표시하는 경고 다이얼로그입니다.

### 2. UI 구성
- "저장하지 않은 내용이 있습니다" 경고 메시지
- 나가기 버튼 (변경사항 버리고 이탈)
- 계속 편집 버튼 (다이얼로그 닫고 편집 계속)

### 3. 액션 / 흐름
1. 미저장 상태에서 화면 이탈 시도
2. 이 다이얼로그 자동 표시
3. "나가기" → 변경사항 취소 후 이동
4. "계속 편집" → 다이얼로그 닫고 현재 화면 유지

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |

### 8. 핵심 디자인 요청사항
- 공통 색상, 간격, 폰트, 토스트 규칙은 `_공통/디자인_시스템.md`, `_공통/토스트_메시지.md`를 따른다.
- 본 섹션에 별도 명시된 UI 상태와 권한 차이가 있으면 도메인 공통보다 화면 정의를 우선한다.
