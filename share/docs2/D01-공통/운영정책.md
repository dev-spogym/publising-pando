# D01-공통 공통 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.

## 1. 핵심 원칙
- 이 문서는 관리자 웹과 회원앱을 포함한 전체 서비스의 운영 기준을 한 번에 파악하기 위한 상위 문서다.
- 목표는 "기능이 많다"가 아니라 "센터가 실제로 굴러간다"를 기준으로 문서를 정렬하는 데 있다.
- 포함
- 리드, 회원, 상품, 결제, 이용권, 수업, 출석, 시설, 자동화, 회원앱
- 본사와 지점의 권한 차이
- 화면 노출 우선순위와 상태 기준
- 제외
- 외부 결제사, 알림사, IoT 연동 상세 구현
- 상태값은 운영 기준으로 한 벌만 쓴다.
- 화면 라벨과 내부 enum이 다르더라도, 기획 문서에서는 먼저 운영 라벨을 고정한다.
- 같은 개념이 문서마다 다른 이름으로 등장하면 이 문서 세트 기준으로 통합한다.
- 본사는 정책, 템플릿, 표준, 기본값을 관리한다.
- 지점은 본사가 허용한 범위 안에서 적용하고 운영한다.
- 지점이 전사 기준을 새로 만드는 구조는 피한다.
- 1순위: 리드, 회원, 결제, 이용권, 출석, 예약, 만료 알림
- 2순위: 수업 통계, 시설 운영, 직원 운영, 본사 KPI
- 3순위: 고도화 탭, 실험적 대시보드, 확장형 분석 기능
- 화면은 실제 센터 운영 동선을 따라 설계한다.

## 2. 상태 / 운영 정의
- **Owner(지점장) 출근 직후 로그인**: 자동 로그인 토글 ON 상태 + 단말 기억 → 즉시 지점 대시보드(SCR-090 / `D10-본사관리/본사관리.md:20` / DASH-01 / `D10-본사관리/본사관리.md:20`) 진입
- **5회 연속 실패 → 30분 잠금**: 부정 시도 차단 + 알림 센터로 본인·관리자에게 알림
- **장시간 자리 비움 후 자동 로그아웃**: 30분 무활동 감지 → 토큰 만료 → 로그인 화면 복귀, 작성 중 데이터 자동 임시저장
- **퇴사자 계정 비활성화**: 직원 관리(STF)에서 비활성화 → 즉시 모든 활성 세션 강제 만료 + 재로그인 차단
- 5회 연속 실패 | 30분 자동 잠금 |
- 직원 관리, 로그인, 권한 설정, 퇴사 처리, 프로필/보안 설정에 공통으로 적용되는 `직원 계정 정책`의 기준 문서다. 본 시스템은 `직원 1명 = 로그인 계정 1개`를 기본 원칙으로 하며, 직원 정보와 로그인 자격은 운영 UX에서는 하나의 흐름으로 다루고 보안 정책은 계정 상태를 통해 제어한다.
- 3. 로그인 계정은 직원 단위로 생성·잠금·비활성화·재활성화된다.
- 5. 퇴사·휴직·보안 사고 시 계정 상태가 직원 상태와 함께 움직인다.
- ## 생명주기 상태
- 직원 상태 | 계정 상태 | 로그인 가능 | 설명 |
- 등록/수정형 화면에서 저장하지 않은 변경사항이 있는 상태로 이탈하려 할 때, 데이터 유실을 막기 위해 표시하는 공통 경고 다이얼로그다. 사용자는 현재 화면에 머물지, 변경사항을 버리고 이동할지 명확히 선택해야 한다.
- dirty 상태가 없으면 표시하지 않는다.
- 4. 상태 변경, 배치, 알림, 감사 로그는 `자동화 / 배치 / 이벤트 연계` 또는 본문 내 연계 표를 같이 적용합니다.
- 4. 상태 변경, 배치, 알림, 감사 로그는 `자동화 / 배치 / 이벤트 연계` 또는 본문 내 연계 표를 같이 적용합니다.
- 로그인 상태 유지 | checkbox (remember me) — 체크 시 토큰 만료 연장 |
- 2FA 활성 계정 전용으로, 기본 인증 성공 후 추가로 표시되는 입력 영역이다.
- MFN-SCR-100-02 / `D01-공통/운영정책.md:42` | 비밀번호 빈값 차단 | 빈 상태 제출 시 인라인 오류 메시지 |
- MFN-SCR-100-03 / `D01-공통/운영정책.md:43` | 로그인 상태 유지 | 체크박스 ON → refresh token 만료 연장 |
- MFN-SCR-100-04 / `D01-공통/운영정책.md:44` | 로그인 버튼 클릭 | 처리 중 버튼 비활성화 + 스피너 표시 |
- 로그인 직후 역할 판정 이후 표시되는 공통 통합 현황 화면이다. 오늘의 출석 수, 당일 매출, HQ-09 / `D10-본사관리/본사관리.md:894` 만료 알림 대상 회원 수 등 핵심 운영 수치를 카드 형태로 한눈에 확인하고, 수업·회원·매출 위젯을 통해 즉시 조치가 필요한 사안을 식별한다. 빠른 액션 버튼으로 회원 등록·결제 처리 같은 주요 업무를 즉시 시작할 수 있어, 별도의 탐색 없이 하루 운영을 시작하는 허브 역할을 한다.

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs2/_정책미확정_확인필요.md`에 별도 관리한다.
