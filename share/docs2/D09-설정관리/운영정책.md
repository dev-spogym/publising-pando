# D09-설정관리 설정관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.

## 1. 핵심 원칙
- 본사와 지점, 관리자 웹과 회원앱의 권한 구조를 실제 운영 기준으로 정리한다.
- 문서마다 달랐던 역할 명칭을 운영 기준으로 하나로 맞춘다.
- 관리자 웹 표준 역할 코드는 `superAdmin`, `primary`, `owner`, `manager`, `fc`, `trainer`, `staff`, `readonly` 8개를 기준으로 한다. `owner`의 화면 표시명은 `Owner(지점장)`으로 통일한다.
- `trainer`는 관리자 웹 기본 역할이다. PT트레이너, GX강사, 골프강사는 직원 직무값이며 기본 권한 템플릿은 `trainer`로 지정한다.
- 카운터/현장 지원 담당자는 별도 표준 역할로 두지 않고 `staff`에 통일한다.
- 권한은 "페이지 접근", "조회", "수정", "승인", "일괄처리"로 나눠서 본다.
- 전사 권한 템플릿
- 공통 상품 마스터
- 자동화 정책 세트
- 알림 템플릿
- 공지 템플릿
- KPI 및 감사 기준
- 회원/리드 실제 운영
- 현장 결제와 미수금 처리
- 수업 개설과 스케줄 운영
- 시설 배정
- 본사 정책 범위 안에서의 적용 설정
- 전사 환불 규모는 환불 집중 지점을 조기 포착하는 경고 지표로 본다.
- 키오스크와 IoT의 관리자 설정 화면은 D09가 책임진다.
- 키오스크 단말 운영 화면, 장애 대응 콘솔, 원격 재시작, 로그 다운로드는 KIOSK 기획문서 범위다.
- CRM에서 설정 가능한 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종으로 고정한다.
- 카드 리더와 얼굴인식 카메라는 별도 기기 종류가 아니라 출입 게이트 또는 키오스크의 인증 수단/부속 모듈로 관리한다.
- 수업/회원 출석 지각 기준과 직원 근태 지각 기준은 별도 정책값이다. 회원·수업 출석 기준값은 미설정을 허용하며, 값이 비어 있으면 해당 자동 분류·자동 알림·자동 노쇼 처리를 실행하지 않고 수동 처리 대상으로 남긴다. 직원 지각 기준은 소속 지점 Owner(지점장)이 SCR-086 / `D09-설정관리/설정관리.md:825` 출석 관리 설정에서 설정한 직원 지각 허용 시간을 따른다. 기본값은 10분이며, 정규 출근 시각 + 10분까지는 정상, 10분 초과 출근부터 지각으로 분류한다.
- 락커 만료 알림 step과 실물 자산 자동 회수 실행 정책은 분리한다. 알림 step은 본사 정책 + 지점 추가 step 구조이고, 락커/사물함/운동복 자동 회수 실행은 SCR-080A / `D09-설정관리/설정관리.md:132` 지점 자동화 적용의 지점 운영 정책값으로 Owner(지점장)이 제어한다.
- 권한, 역할, 정책, 자동화, 기기, 백업 설정 변경은 모두 SCR-097 / `D10-본사관리/본사관리.md:694` 히스토리 로그 기록 대상이다. 변경 이력에는 변경자, 대상 지점/역할/정책, 변경 전 값, 변경 후 값, 변경 사유, 변경 일시를 남기며, D09에서는 변경을 수행하고 D10 SCR-097 / `D10-본사관리/본사관리.md:694` 히스토리 로그에서 조회한다.

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

| KPI | 산식/의미 | 측정 주기 |
|---|---|---|
| 처리 완료율 | 완료 건수 / 대상 건수 | 일/주 단위 |
| 예외 발생률 | 예외·차단 건수 / 전체 시도 건수 | 주 단위 |
| 정책 준수율 | 권한·상태·필수값 위반 없이 처리된 비율 | 월 단위 |
| 운영 응답률 | 알림·요청 후 운영자가 처리한 비율 | 주 단위 |

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
- 키오스크 운영 화면은 D09에 만들지 않는다. D09는 설정값 관리만 담당하며, 운영 관제는 KIOSK 기획문서 범위다.
- IoT 기기 종류는 4종으로 고정하고, 문서 내 다른 표현이 있으면 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기로 정규화한다.
- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs2/_정책미확정_확인필요.md`에 별도 관리한다.
