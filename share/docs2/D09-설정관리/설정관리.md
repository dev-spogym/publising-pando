## 도메인 개요
- 화면/다이얼로그 구현 시 이 파일을 먼저 확인합니다.
- 본 문서가 해당 도메인 화면·다이얼로그 구현 기준입니다. 구현 판단을 위해 다른 폴더를 확인하지 않습니다.
- API/DB 상세는 백엔드 설계 범위이므로 제외하고, 화면/기능/UX/운영정책 기준만 남겼습니다.
- 키오스크와 IoT의 관리자 설정 화면은 D09가 책임집니다.
- 키오스크 단말 운영 화면, 단말 장애 대응 콘솔, 원격 재시작, 로그 다운로드는 KIOSK 기획문서 범위이며 D09에서 제공하지 않습니다.
- CRM에서 고정으로 관리하는 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종입니다. 카드 리더와 얼굴인식 카메라는 별도 기기 종류가 아니라 인증 수단 또는 부속 모듈로 관리합니다.
- 권한, 역할, 정책, 자동화, 기기, 백업 설정을 저장·삭제·복원하면 SCR-097 / `D10-본사관리/본사관리.md:694` 히스토리 로그에 변경자, 대상 지점/역할/정책, 변경 전 값, 변경 후 값, 변경 사유, 변경 일시를 남깁니다. D09 각 화면은 설정 변경을 수행하고, 변경 이력 조회는 D10 SCR-097 / `D10-본사관리/본사관리.md:694` 히스토리 로그에서 수행합니다.

## SCR-080 센터 설정

### 1. 목적
센터 운영에 필요한 기본 정보를 한 곳에서 관리하는 설정 허브 화면입니다.
센터명·주소·연락처 등 기본정보, 알림 수신 방식, 화면 테마, 물품 재고를 탭별로 구분하여 관리합니다.

### 2. UI 구성
### 공통 영향도 프리뷰
- 현재 탭에서 저장 시 영향을 받는 운영 항목 요약
- 예: 운영시간 변경 시 출석/예약 정책 검토 필요, KakaoTalk fallback 변경 시 발송량·비용 영향 표시
- 변경 없음 상태와 저장 전 검토 필요 상태를 배지로 표시

### 기본정보 탭
- 센터 로고 업로드 영역
- 센터명 (필수)
- 사업자등록번호
- 센터 소개
- 대표 연락처
- 주소 (주소 검색 버튼 포함)
- 업종 선택 (정확한 노출 순서: 헬스 / 필라테스 / PT샵 / 골프 / 요가 / 크로스핏 / 복싱 / 수영 / 태권도 / 스피닝 / 기타)
- 평일 운영시간 (오픈~마감)
- 주말 운영시간 (오픈~마감)
- 저장 버튼 (변경 사항이 있을 때만 활성화)

### 알림설정 탭
- 앱 푸시 알림 항목별 ON/OFF 스위치 (입장 알림, 만료 임박, 결제 완료, 수업 예약·취소)
- 이메일 알림 항목별 ON/OFF 스위치 (주간 리포트, 만료 예정 안내)
- KakaoTalk fallback 항목별 ON/OFF 스위치 (결제 완료, 만료 임박 등 회원앱 Push 실패·미설치 시 보조 발송)
- 메시지 플랫폼 연동 요약 카드 (연동 플랫폼, 발신 프로필, 승인 템플릿 수, 최근 동기화 시각)
- 예상 비용은 참고용 조회값으로만 노출되며 관리자에서 단가를 직접 수정하지 않음

### 테마설정 탭
- 화면 모드 선택 (라이트 / 다크 / 시스템 설정)
- 메인 컬러 색상 선택
- 보조 컬러 색상 선택
- 글자 크기 선택 (작게 / 기본 / 크게)

### 물품 관리 탭
- 물품별 재고 수량 입력 (수건 대·소, 운동복 상·하의)
- 1일 발급 한도 설정
- 현재 잔여 수량 입력

### 3. 액션 / 흐름
1. 화면 진입 시 저장된 센터 설정 정보를 불러와 각 탭에 표시한다.
2. 상단 영향도 프리뷰에서 저장 시 함께 바뀌는 운영 정책을 먼저 확인한다.
3. 정보 수정 후 저장 버튼 클릭 → 변경 사항 저장 완료 메시지 표시 (3초 후 자동 사라짐).
4. 알림설정 탭에서는 메시지 플랫폼 연동 상태와 최근 동기화 시각을 확인한 뒤 저장한다.
5. 저장하지 않고 다른 탭으로 이동 시 미저장 경고 다이얼로그(DLG-080-001 / `D09-설정관리/설정관리.md:1202`) 표시.
6. 주소 검색 버튼 클릭 시 주소 검색 팝업 실행 후 선택한 주소가 자동 입력된다.

## SCR-080A 지점 자동화 적용

> 유형: 화면
> URL 경로: `/settings/automation`
> 연결 기능: SET-EXT-01 / `D09-설정관리/설정관리.md:132`
> 연결 다이얼로그: DLG-080A-001 / `D09-설정관리/설정관리.md:1280` 정책적용확인

### 1. 목적
- 지점이 본사 정책 세트를 적용받고 지점별 활성 상태를 관리하는 화면이다.
- 만료 정책은 회원 이용권 만료, 결제기한 만료, 락커 만료로 분리해 표시한다.
- 본사 step의 기준일은 읽기 전용이며 Owner(지점장)만 정책/step ON/OFF를 변경할 수 있다.
- 회원 이용권 만료와 락커 만료는 Owner(지점장)이 지점 추가 step을 생성·수정·삭제할 수 있다.
- 결제기한 만료는 지점 추가 step을 만들 수 없고 본사 step ON/OFF만 가능하다.
- 락커 만료 알림 step과 실물 자산 자동 회수 실행 정책은 분리한다. 락커 만료 알림은 본사 정책 + 지점 추가 step 구조를 따르고, 실제 락커/사물함/운동복 자동 회수 실행은 지점 운영 정책값으로 Owner(지점장)이 ON/OFF한다.

### 2. UI 구성

- **상단 헤더**: 화면 제목 `지점 자동화 적용`, 현재 지점명, 마지막 저장 일시, 마지막 변경자, 본사 정책 동기화 상태, [변경 이력] 버튼을 표시한다.
- **정책 적용 요약 카드**: 적용 중인 본사 정책 세트 수, 활성화된 본사 step 수, 지점 추가 step 수, 자동 회수 ON 정책 수, 최근 실패 알림 수를 카드로 표시한다. 각 카드는 클릭 시 아래 목록을 해당 조건으로 필터링한다.
- **적용 범위 탭**: 전체 / 본사 정책 step / 지점 추가 step / 실물 자산 자동 회수로 구분한다. 본사 정책 step은 기준일·발송 시각·템플릿을 읽기 전용으로 표시하고, 지점 추가 step은 Owner(지점장)이 생성·수정·삭제할 수 있는 항목만 표시한다.
- **정책 유형 탭**: 회원 이용권 만료 / 결제기한 만료 / 락커 만료 / 락커·사물함 자동 회수 / 운동복 자동 회수.
- **Step 목록 테이블**: 알림 step을 행 단위로 표시한다.

| 컬럼 | 표시 기준 |
|---|---|
| 정책 유형 | 회원 이용권 만료 / 결제기한 만료 / 락커 만료 |
| step 구분 | 본사 step / 지점 추가 step 배지 |
| 기준일 | D-숫자 / D-Day / D+숫자. 본사 step은 읽기 전용 |
| 발송 시각 | 본사 정책 또는 지점 추가 step의 발송 시각 |
| 반복 주기 | 1회 / 매일 / N일 간격 등 정책값 |
| 채널 | 회원앱 Push 기본, KakaoTalk fallback 선택, SMS 명시 선택 |
| 템플릿 | 발송 메시지 템플릿명 또는 미리보기 |
| 지점 ON/OFF | Owner(지점장)만 변경 가능. 본사 강제 ON 항목은 잠금 표시 |
| 액션 | 지점 추가 step은 수정 / 삭제, 본사 step은 상세 보기만 표시 |
| 마지막 변경 | 마지막 변경자와 변경 일시 |

- **지점 step 추가 버튼**: 회원 이용권 만료 / 락커 만료 탭에서만 노출한다. 결제기한 만료 탭에서는 버튼을 숨기고 "결제기한 만료는 본사 정책 step만 사용할 수 있습니다" 안내를 표시한다.
- **실물 자산 자동 회수 정책 섹션**: 자동 회수 정책명, 기본값, 현재 ON/OFF, 마지막 변경자, 적용 화면(D06)을 표로 표시한다.
- **하단 액션 영역**: [저장], [취소], [변경 요약 보기], [24시간 내 롤백] 버튼을 표시한다. 변경 사항이 없으면 [저장]과 [변경 요약 보기]는 비활성화한다.

### 락커·사물함 자동 회수 정책

### 3. 액션 / 흐름

## SCR-081 권한 설정

> 유형: 화면
> URL 경로: `/settings/permissions`
> 연결 기능: SET-02 / `D09-설정관리/설정관리.md:235`
> 연결 다이얼로그: DLG-081-001 / `D09-설정관리/설정관리.md:1357` 권한초기화, DLG-081-002 / `D09-설정관리/설정관리.md:1425` 충돌경고, DLG-081-003 / `D09-설정관리/설정관리.md:1496` 역할생성, DLG-081-004 / `D09-설정관리/설정관리.md:1570` 역할삭제, DLG-081-005 / `D09-설정관리/설정관리.md:1642` 역할복사, DLG-081-006 / `D09-설정관리/설정관리.md:1715` 역할영향분석

### 1. 목적
관리자웹의 권한 템플릿별로 메뉴 접근 및 기능 사용 권한을 설정하는 화면입니다.
직원관리에서 등록한 직원은 로그인 계정과 1:1로 연결되며, 이 화면에서는 그 로그인 계정에 연결할 권한 템플릿을 관리합니다. 직무(PT트레이너·GX강사·골프강사 등)와는 별도로, 기본 제공 권한 템플릿과 센터 맞춤형 역할을 추가하고 각 템플릿/역할이 어떤 메뉴에서 무엇을 할 수 있는지를 매트릭스 형태로 한눈에 확인하고 조정합니다.

### 2. UI 구성
### 권한 템플릿/역할 목록 패널 (좌측)
- 기본 제공 권한 템플릿 목록: 슈퍼관리자(`superAdmin`), 최고관리자(`primary`), Owner(지점장)(`owner`), 매니저(`manager`), FC(`fc`), 트레이너(`trainer`), 스태프(`staff`), 조회전용(`readonly`)
- `PT트레이너`, `GX강사`, `골프강사`는 직원 직무값이며 기본 권한 템플릿은 `trainer`로 지정합니다.
- 카운터/현장 지원 담당자는 별도 역할을 만들지 않고 `staff` 권한 템플릿으로 통일합니다.
- 각 역할 카드에 해당 역할이 배정된 직원 수 표시
- 직원 등록 시 발급된 로그인 계정이 어떤 권한 템플릿에 연결되는지 요약 표시
- 새 역할 추가 버튼 (+)
- 역할 카드별 복사·삭제 액션

### RBAC 역할 체계 기준
- 관리자 웹 표준 역할 코드는 `superAdmin`, `primary`, `owner`, `manager`, `fc`, `trainer`, `staff`, `readonly` 8개를 기준으로 합니다.
- 커스텀 역할은 기본 역할 외에 추가로 생성하는 역할이며, 기본 역할명과 이름이 중복될 수 없습니다.
- 직원 직무값은 PT트레이너, GX강사, 골프강사 등 인사/운영 분류값입니다.
- 권한 역할값은 로그인 후 메뉴 접근과 기능 권한을 결정하는 값입니다.
- 각 활성 직원 로그인 계정은 하나의 권한 템플릿만 연결합니다.

### 권한 매트릭스 패널 (우측)
- 좌측에서 선택한 역할의 메뉴별 권한 표시
- 메뉴 그룹: 회원, 수업, 매출, 상품, 직원, 급여, 시설, 메시지, 설정 (총 9그룹)
- 권한 항목: 접근 / 조회 / 생성 / 수정 / 삭제 (5종)
- 전체 허용 / 전체 거부 버튼
- 권한 충돌 감지 경고 영역

### 상단 액션 버튼
- 초기화 버튼 (마지막 저장 상태로 되돌리기)
- 변경 사항 저장 버튼 (변경 사항이 있을 때만 활성)

### 권한 변경 영향도 프리뷰
- 변경된 메뉴 수, 새로 열리는 메뉴 수, 차단되는 메뉴 수, 영향 직원 수를 상단 카드로 표시
- 새 접근 허용 / 쓰기 권한 확대 / 접근 차단 예정 메뉴를 각각 요약 표시
- 저장 전에 운영 중단 가능성이 있는 메뉴 차단을 빠르게 검토할 수 있음
- 영향도 프리뷰와 DLG-081-006 / `D09-설정관리/설정관리.md:1715`은 저장 전 확인 안내이며, 별도 예측 리포트나 시뮬레이션 기능으로 확장하지 않음
- 영향 분석 다이얼로그에는 배정 직원 수, 담당 회원 합계, 영향 직원 목록, 추가/제거 권한, 민감 기능 변경 여부를 표시합니다.
- 역할 삭제 시 배정 직원이 1명 이상이면 재배정 역할 선택 전에는 삭제할 수 없습니다.

### 민감 기능 식별 표기
- 매트릭스 셀 중 다음 6종은 민감 기능으로 시각 표기(아이콘 또는 강조 색)되며, `superAdmin` 또는 `primary`(본사 최고관리자) 외 역할에서는 체크할 수 없고, 기존 체크가 포함된 변경 저장도 차단합니다.
  - 회원 영구 삭제
  - 환불 처리
  - 급여 확정·지급
  - 슈퍼관리자 권한 부여/회수
  - 데이터 복원
  - 감사로그 내보내기

### 3. 액션 / 흐름
1. 화면 진입 시 권한 템플릿/역할 목록이 좌측에 표시되고, 첫 번째 권한 템플릿(`superAdmin`)의 권한이 자동 선택되어 우측 매트릭스에 표시된다.
2. 좌측에서 역할 선택 → 해당 역할의 권한 매트릭스가 우측에 표시된다.
3. 상단 영향도 프리뷰에서 열리는 메뉴, 막히는 메뉴, 영향 직원 수를 먼저 확인한다.
4. `superAdmin` 권한 템플릿은 모든 권한이 고정(수정 불가)으로 표시된다.
5. 권한 체크박스 변경 → 충돌 여부 실시간 감지 → 충돌 발생 시 경고 영역에 내용 표시.
6. 저장 버튼 클릭 → 변경된 역할에 배정된 직원이 1명 이상 있을 경우 **역할 변경 영향 분석 다이얼로그(DLG-081-006 / `D09-설정관리/설정관리.md:1715`)** 가 먼저 표시되어 배정 직원 수, 담당 회원 합계, before/after 권한 차이를 확인한 뒤 진행한다. 충돌이 있으면 충돌 경고 다이얼로그(DLG-081-002 / `D09-설정관리/설정관리.md:1425`)도 추가로 표시된다.

## SCR-082 키오스크 설정

> 유형: 화면
> URL 경로: `/settings/kiosk`
> 연결 기능: IoT-01 / `D09-설정관리/설정관리.md:352`
> 연결 다이얼로그: DLG-080-001 / `D09-설정관리/설정관리.md:1202` 미저장경고

### 1. 목적
센터 입구에 설치된 키오스크 화면의 구성과 동작 방식을 설정하는 화면입니다.
회원이 키오스크를 통해 출석 체크를 하는 방식, 화면에 표시할 메시지와 배경, 사용 가능한 기능 항목을 관리합니다.
본 화면은 설정 화면이며, 키오스크 단말 운영 현황·장애 대응·원격 재시작·로그 다운로드는 KIOSK 기획문서 범위입니다.

### 2. UI 구성
### 출석 방식 설정
- 출석 인식 방법 선택: QR코드 / 회원번호 입력 / 얼굴 인식 (복수 선택 가능)
- 기본 출석 방식 지정 (키오스크 첫 화면에 표시되는 방식)

### 화면 구성 설정
- 키오스크 표시 언어 선택
- 대기 화면 배경 이미지 또는 색상 설정
- 센터 로고 표시 여부
- 환영 메시지 텍스트 입력 (회원 이름 자동 삽입 태그 지원)
- 출석 완료 메시지 텍스트 입력

### 기능 표시 설정
- 키오스크에서 회원이 사용할 수 있는 기능 항목 ON/OFF
  - 이용권 잔여 조회
  - 수업 예약 조회
  - 락커 번호 확인
  - 공지사항 조회

### 미리보기
- 현재 설정값 기준 키오스크 화면 미리보기 표시

### 3. 액션 / 흐름
1. 화면 진입 시 현재 저장된 키오스크 설정값을 불러와 표시한다.
2. 항목 변경 시 우측 미리보기가 실시간으로 업데이트된다.
3. 저장 버튼 클릭 → 키오스크 기기에 설정이 즉시 반영된다.
4. 저장하지 않고 페이지 이탈 시 미저장 경고(DLG-080-001 / `D09-설정관리/설정관리.md:1202`)가 표시된다.

## SCR-082A 키오스크 IoT 설정

> 유형: 화면
> URL 경로: `/settings/iot`
> 연결 기능: IoT-01 / `D09-설정관리/설정관리.md:352`
> 연결 다이얼로그: DLG-080-001 / `D09-설정관리/설정관리.md:1202` 미저장경고

### 1. 목적
- 키오스크와 출입 장비의 기본 설정값을 관리합니다.
- 본 화면은 설정값 등록·수정·상태 확인만 담당하며, 키오스크 단말 운영 콘솔이나 원격 제어 화면이 아닙니다.
- 고정 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종입니다.
- SCR-082A와 SCR-083은 동일한 `/settings/iot` 설정 페이지의 탭입니다. SCR-082A는 키오스크 IoT 설정 탭, SCR-083은 IoT 출입 관리 탭입니다.

### 2. UI 구성
### 장비 기본값
| 항목 | 설명 |
|---|---|
| 기기 종류 | 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 중 1개 선택 |
| 기기명 | 운영자가 식별할 이름 |
| 설치 위치 | 예: 1층 입구, 데스크 우측, 남자 락커룸 |
| 사용 여부 | ON / OFF |
| 상태 확인 | OK / 불안정 / 오류 / 점검 중 배지와 마지막 수신 시각 |
| 헬스체크 주기 | 기본 5분. 본사(primary/superAdmin)가 전역 기준을 변경 가능 |

### 출입 규칙
- 출입 허용 시간
- 중복 출입 방지 시간
- 만료 회원 출입 허용 토글 (기본 OFF)
- 직원 출석 허용 토글 (기본 ON)

### 키오스크 연동 설정
- 키오스크에서 사용할 출석 방식과 SCR-082 / `D09-설정관리/설정관리.md:362` 화면 구성 설정을 함께 확인합니다.
- 키오스크 운영 화면 자체는 KIOSK 기획문서에서 정의합니다.

### 3. 액션 / 흐름
1. 사용자가 메뉴 또는 직접 URL로 화면에 진입합니다.
2. 등록된 기기의 종류, 위치, 상태 배지, 마지막 수신 시각을 확인합니다. 최신 헬스체크가 정상 응답이면 OK, 미응답·응답 지연·부분 응답이면 불안정, 인증 실패·하드웨어 오류이면 오류로 표시합니다.
3. 기기 설정 또는 출입 규칙을 수정합니다.
4. 저장 시 변경 요약을 표시하고 연결된 기기에 다음 동기화 시점부터 적용합니다.

### 4. 핵심 상태

## SCR-083 IoT 출입 관리

> 유형: 화면
> URL 경로: `/settings/iot`
> 연결 기능: IoT-02 / `D09-설정관리/설정관리.md:544`
> 연결 다이얼로그: DLG-080-001 / `D09-설정관리/설정관리.md:1202` 미저장경고

### 1. 목적
센터에 설치된 IoT 출입 관련 기기의 설정값과 출입 방식을 관리하는 화면입니다.
고정 기기 종류는 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기 4종이며, 밴드·QR코드·얼굴 인식·핀번호는 기기 종류가 아니라 출입 인증 수단입니다.
본 화면은 설정·상태 확인 화면이며, 키오스크 운영 콘솔이나 기기 원격 제어 화면이 아닙니다.
SCR-082A와 SCR-083은 동일한 `/settings/iot` 설정 페이지의 탭입니다. SCR-082A는 키오스크 IoT 설정 탭, SCR-083은 IoT 출입 관리 탭입니다.

### 2. UI 구성
### 연결 기기 현황
- 등록된 IoT 기기 목록 (기기 종류, 기기명, 설치 위치, 연결 상태, 마지막 통신 시각)
- 기기 종류: 출입 게이트 / 키오스크 / 락커 컨트롤러 / InBody 측정기
- 연결 상태 배지: OK / 불안정 / 오류
- 기기 헬스체크 주기: 기본 5분. 본사(primary/superAdmin)가 전역 기준을 변경할 수 있으며, 지점은 해당 기준을 임의 변경하지 않음
- 불안정 알림 기준: 헬스체크 미응답·응답 지연·부분 응답 감지 즉시 운영자 알림(NFR-19 / `_공통/공통참조_코드인덱스.md:33`) 발송
- 기기별 재연결 시도 버튼
- 새 기기 등록 버튼

### 출입 방식 설정
- 출입 수단별 활성화 ON/OFF
  - 스마트 밴드 (RFID)
  - QR코드 스캔
  - 얼굴 인식
  - 핀번호 입력
- 중복 출입 방지 시간 설정 (동일 회원 재입장 허용 간격)
- 출입 이력 보관 기간 설정

### 출입 알림 설정
- 입장 알림 발송 여부 (회원 앱 푸시)
- 퇴장 알림 발송 여부 (회원 앱 푸시)
- 비정상 출입 시도 알림 발송 여부 (관리자 알림)

### 3. 액션 / 흐름
1. 화면 진입 시 등록된 기기 목록과 현재 연결 상태를 자동으로 불러온다.
2. 최신 헬스체크가 정상 응답이면 OK 배지로 표시한다.
3. 기기가 불안정이면 오렌지 배지로 표시하고 즉시 운영자 알림을 발송하며, 재연결 버튼을 클릭하면 연결 시도 상태로 전환된다.
4. 기기가 인증 실패·하드웨어 오류 등 명시 오류를 반환하면 오류 배지와 운영자 확인 안내를 표시한다.
5. 출입 수단 ON/OFF 토글 변경 후 저장하면 연결된 모든 기기에 설정이 반영된다.
6. 새 기기 등록 버튼 클릭 시 기기 종류 4종 중 하나를 선택하고, 기기 코드 입력 → 설치 위치 지정 절차가 진행된다.

## SCR-084 구독 결제 관리

> 유형: 화면
> URL 경로: `/subscription`
> 연결 기능: SET-03 / `D09-설정관리/설정관리.md:644`
> 연결 다이얼로그: DLG-084-001 / `D09-설정관리/설정관리.md:1794` 플랜변경확인, DLG-084-002 / `D09-설정관리/설정관리.md:1872` 구독해지확인

### 1. 목적
센터가 현재 이용 중인 서비스 구독 플랜과 결제 수단을 확인하고 관리하는 화면입니다.
플랜 업그레이드·다운그레이드, 결제 수단 변경, 과거 청구 이력 조회를 처리합니다.

### 2. UI 구성
### 현재 구독 정보
- 현재 플랜명 및 주요 혜택 요약
- 다음 결제 예정일 및 금액
- 플랜 변경 버튼 (업그레이드 / 다운그레이드)
- 구독 해지 버튼

### 플랜 비교 섹션
- 이용 가능한 전체 플랜 목록 카드 형태 표시
- 각 플랜별 포함 기능 항목, 월 요금, 회원 수 한도
- 현재 플랜 강조 표시
- 타 플랜 선택 시 "이 플랜으로 변경" 버튼 활성

### 결제 수단 관리
- 현재 등록된 결제 수단 (카드사, 카드번호 마지막 4자리, 유효기간)
- 결제 수단 변경 버튼
- 결제 수단 삭제 버튼

### 청구 이력
- 과거 결제 내역 목록 (날짜, 플랜명, 금액, 상태)
- 영수증 다운로드 버튼 (PDF)
- 기간 필터 (최근 3개월 / 6개월 / 1년)

### 3. 액션 / 흐름
1. 화면 진입 시 현재 구독 정보와 청구 이력을 자동으로 불러온다.
2. 플랜 변경 버튼 클릭 → 플랜 변경 확인 다이얼로그(DLG-084-001 / `D09-설정관리/설정관리.md:1794`) 표시 (변경 후 요금 차액 안내 포함).
3. 구독 해지 버튼 클릭 → 구독 해지 최종 확인 다이얼로그(DLG-084-002 / `D09-설정관리/설정관리.md:1872`) 표시.
4. 결제 수단 변경 버튼 클릭 → 결제 수단 등록 페이지(PG사 연동)로 이동.
5. 영수증 다운로드 클릭 → 해당 월 PDF 영수증 자동 다운로드.

## SCR-086 출석 관리 설정

> 유형: 화면
> URL 경로: `/settings/attendance`
> 연결 기능: SET-EXT-05 / `D09-설정관리/설정관리.md:825`
> 연결 다이얼로그: DLG-080-001 / `D09-설정관리/설정관리.md:1202` 미저장경고

### 1. 목적
회원의 출석을 인정하는 기준과 직원 근태 자동 분류 기준을 설정하는 화면입니다.
수업/회원 출석 기준과 직원 근태 기준은 별도 정책값으로 관리합니다. 출석 인정 N분 전/후, 수업 지각 N분, 수업 지각 N회 알림 기준은 회원·수업 출석에만 적용하고, 직원 지각 기준은 `직원 근태 기준 설정`의 지점별 직원 지각 허용 시간을 따릅니다. 회원·수업 출석 기준값은 미설정을 허용하며, 값이 비어 있으면 해당 자동 분류·자동 알림·자동 노쇼 처리를 실행하지 않고 수동 처리 대상으로 남깁니다. 직원 지각 허용 시간의 기본값은 10분이며, 정규 출근 시각 + 10분까지는 정상, 10분 초과 출근부터 지각으로 분류합니다. 지점별 기준의 기본 수정 권한은 소속 지점 Owner(지점장)에게 있으며, 슈퍼관리자의 개별 지점값 수정은 비상 사유 입력 시에만 허용합니다.
QR 출석은 출석 수단 중 하나로 사용할지와 화면에 노출할지를 정하는 설정이며, QR 발급·토큰 만료·서버 검증 같은 실행 로직은 이 화면에서 설정하지 않습니다.

### 2. UI 구성
### 출석 인정 시간 설정
- 출석 인정 시작 시각 (수업 시작 기준 N분 전부터 인정)
- 출석 인정 마감 시각 (수업 시작 기준 N분 후까지 인정)
- 회원·수업 출석 기준값 미설정 허용. 미설정 항목은 자동 분류·자동 알림·자동 노쇼 처리에서 제외하고 수동 처리만 가능
- 주말/공휴일 출석 인정 여부

### 수업/회원 지각 기준 설정
- 수업 지각 판단 기준 시간 (수업 시작 후 N분까지는 정시 인정, N분 초과 입장부터 지각)
- 수업 지각 횟수 누적 시 알림 발송 기준 (집계 기간 내 N회 초과 시 자동 알림)
- 수업 지각 알림 수신자 (회원 / 관리자 / Owner(지점장) 중 지점 선택)
- 노쇼 판단 유예 시간 (출석 인정 마감 후 N분까지 미입장 시 노쇼. 기본 30분, 미설정 시 자동 노쇼 보류)

### 직원 근태 기준 설정
- 직원 지각 허용 시간 (기본값 10분. 정규 출근 시각 기준 10분까지는 정상, 설정값 초과 출근부터 지각)
- 직원 지각 누적 알림 기준 (월 또는 지점 설정 기간 내 N회 초과 시 알림)
- 직원 지각 알림 수신자 (Owner(지점장) / 매니저 / 직원 본인 중 지점 선택)
- 직원 근태 기준 적용 시점 (저장 이후 신규 출퇴근 기록부터 적용, 과거 기록 소급 변경 없음)

### QR 출석 설정
- QR 출석 사용 여부
- 키오스크 QR 표시 여부
- QR 출석은 출석 수단 옵션으로만 관리하며, 갱신 주기·유효 시간·스캔 검증 정책은 본 화면의 설정 항목에 포함하지 않음

### 자동 출석 규칙
- 자동 출석 처리 활성화 여부 (IoT 출입 기록 기반 자동 출석 인정)
- 수업 없이 자유 이용 입장 시 출석 인정 여부
- 하루 중 동일 회원 출석 중복 처리 방식 (첫 입장만 / 매 입장마다)

### 출석 기록 보관 설정
- 출석 이력 보관 기간 (개월 단위)

### 3. 액션 / 흐름
1. 화면 진입 시 현재 저장된 출석 설정값을 불러와 표시한다.
2. 각 항목 변경 후 저장 버튼 클릭 → 변경 내용 저장 완료 메시지 표시.
3. QR 출석 사용을 켜면 키오스크 QR 표시 토글이 활성화된다.
4. 자동 출석 처리를 활성화하면 IoT 출입 설정(SCR-083 / `D09-설정관리/설정관리.md:554`)과 연동 안내 메시지가 표시된다.
5. 직원 지각 허용 시간을 변경하면 D07 SCR-063 / `D07-직원관리/직원관리.md:333` 직원 근태 관리의 신규 출퇴근 자동 분류 기준이 변경된다.
6. 저장하지 않고 페이지 이탈 시 미저장 경고(DLG-080-001 / `D09-설정관리/설정관리.md:1202`)가 표시된다.

## SCR-087 커스텀 역할 생성

> 유형: 화면
> URL 경로: `/settings/custom-role`
> 연결 기능: SET-04 / `D09-설정관리/설정관리.md:933`
> 연결 다이얼로그: DLG-087-001 / `D09-설정관리/설정관리.md:2111` 역할삭제재배정

### 1. 목적
기본 제공 역할 8종: `superAdmin` / `primary` / `owner` / `manager` / `fc` / `trainer` / `staff` / `readonly`. 이 기본 역할 외에 센터 운영 특성에 맞는 맞춤형 역할을 생성·관리하는 화면입니다. `owner`는 화면에서 `Owner(지점장)`으로 표시합니다.
역할 이름과 설명을 정의하고, 필요한 권한 조합을 직접 구성합니다.

### 2. UI 구성
### 커스텀 역할 목록
- 현재 생성된 커스텀 역할 목록 (역할명, 설명, 배정된 직원 수, 생성일)
- 새 역할 만들기 버튼
- 역할별 수정·삭제 버튼

### 역할 생성·수정 폼
- 역할 이름 입력 (필수)
- 역할 설명 입력
- 기존 역할에서 권한 복사 선택 (선택적, 출발점 설정)
- 권한 매트릭스 (SCR-081과 동일 구조로 메뉴별 권한 선택)

### 역할 배정 현황
- 해당 역할이 배정된 직원 목록 미리보기

### 3. 액션 / 흐름
1. 화면 진입 시 생성된 커스텀 역할 목록을 표시한다.
2. 새 역할 만들기 버튼 클릭 → 역할 이름·설명 입력 및 권한 매트릭스 구성 화면으로 전환.
3. 기존 역할에서 복사 선택 시 선택한 역할의 권한이 매트릭스에 자동 로드된다.
4. 저장 → 역할 목록에 추가, 권한 설정(SCR-081 / `D09-설정관리/설정관리.md:245`)에서 선택 가능.
5. 역할 삭제 시 해당 역할이 배정된 직원이 있으면 역할 재배정 다이얼로그(DLG-087-001 / `D09-설정관리/설정관리.md:2111`) 표시.

## SCR-088 다국어 설정

> 유형: 화면
> URL 경로: `/settings/language`
> 연결 기능: SET-05 / `D09-설정관리/설정관리.md:1016`
> 연결 다이얼로그: DLG-080-001 / `D09-설정관리/설정관리.md:1202` 미저장경고

### 1. 목적
시스템 전체에서 사용하는 언어와 국가·지역 관련 표시 형식을 설정하는 화면입니다.
외국인 회원이 많은 센터나 해외 지점 운영 시 관리자 화면과 키오스크 화면의 표시 언어를 다르게 구성할 수 있습니다.

### 2. UI 구성
### 시스템 언어 설정
- 관리자 화면 언어 선택 (한국어 / 영어 / 일본어 / 중국어 간체)
- 키오스크 화면 언어 선택 (위와 동일 선택지, 복수 선택 가능)
- 키오스크 기본 표시 언어 지정

### 국가·지역 설정
- 날짜 표시 형식 선택 (YYYY-MM-DD / MM/DD/YYYY / DD/MM/YYYY)
- 시간 표시 형식 선택 (24시간제 / 12시간제)
- 화폐 단위 및 표시 형식 (₩ / $ / ¥ 등)
- 시간대(Timezone) 선택

### 자동 번역 설정
- 공지사항 자동 번역 활성화 여부 (선택 시 외부 번역 서비스 연동)

### 3. 액션 / 흐름
1. 화면 진입 시 현재 저장된 언어·지역 설정을 불러와 표시한다.
2. 관리자 화면 언어 변경 후 저장 → 저장 완료 즉시 화면 전체가 선택한 언어로 전환된다.
3. 키오스크 언어 변경 후 저장 → 연결된 키오스크 기기에 즉시 반영된다.
4. 저장하지 않고 이탈 시 미저장 경고(DLG-080-001 / `D09-설정관리/설정관리.md:1202`)가 표시된다.

## SCR-089 데이터 백업·복원

> 유형: 화면
> URL 경로: `/settings/backup`
> 연결 기능: SET-EXT-03 / `D09-설정관리/설정관리.md:1097`
> 연결 다이얼로그: DLG-089-001 / `D09-설정관리/설정관리.md:2184` 데이터복원확인, DLG-089-002 / `D09-설정관리/설정관리.md:2263` 백업설정

### 1. 목적
센터 운영 데이터의 백업 주기를 설정하고, 수동 백업을 실행하며, 이전 백업 시점으로 복원하는 화면입니다.
데이터 손실 사고에 대비한 안전망으로 운영되며, 복원 이력을 통해 과거 복원 내역을 추적할 수 있습니다.

### 2. UI 구성
### 자동 백업 설정
- 자동 백업 활성화 여부 토글
- 백업 주기 선택 (매일 / 매주 / 매월)
- 백업 실행 시각 설정
- 백업 데이터 보관 기간 설정 (30일 / 90일 / 180일 / 무제한)
- 자동 백업 설정 변경 버튼 → 백업 설정 다이얼로그(DLG-089-002 / `D09-설정관리/설정관리.md:2263`)
- v1 기본 정책: 자동 백업은 구현 대상입니다. 기본값은 자동 백업 ON, 주기 매일, 실행 시각 03:30, 보관 기간 90일이며 지점 운영 정책에 따라 주기·시각·보관 기간을 변경할 수 있습니다.

### 수동 백업
- 지금 백업 실행 버튼
- 마지막 백업 일시 표시
- 백업 데이터 다운로드 버튼 (최신 백업 파일)
- 다운로드는 슈퍼관리자 전용입니다. 백업 파일은 암호화 저장하고, 다운로드 URL은 발급 후 1시간 만료되며, 다운로드 요청·성공·실패는 감사 로그에 기록합니다.

### 백업 이력
- 백업 일시, 데이터 크기, 백업 방식(자동/수동), 상태 목록
- 각 이력 행에 복원 버튼 및 다운로드 버튼
- **기간 필터**: 오늘 / 이번 주 / 이번 달 / 사용자 지정 기간(시작일·종료일 직접 선택) 4종 프리셋

### 복원 이력
- 과거 복원 실행 이력 (복원 일시, 복원한 백업 시점, 실행자)
- 기간 필터: 백업 이력과 동일하게 오늘 / 이번 주 / 이번 달 / 사용자 지정 적용

### 3. 액션 / 흐름
1. 화면 진입 시 자동 백업 설정값과 백업 이력 목록을 불러와 표시한다.
2. "지금 백업 실행" 버튼 클릭 → 백업 진행 중 표시 → 완료 후 이력 목록에 추가.
3. 이력 목록의 "복원" 버튼 클릭 → 복원 최종 확인 다이얼로그(DLG-089-001 / `D09-설정관리/설정관리.md:2184`) 표시 → 확인 후 복원 진행.
4. 복원 완료 후 시스템이 재시작되고 선택한 시점의 데이터로 전환된다.
5. 자동 백업 설정 변경 버튼 클릭 → 백업 설정 다이얼로그(DLG-089-002 / `D09-설정관리/설정관리.md:2263`) 표시.

> 연결 다이얼로그: DLG-081-001 / `D09-설정관리/설정관리.md:1357` 권한초기화, DLG-081-002 / `D09-설정관리/설정관리.md:1425` 충돌경고, DLG-081-003 / `D09-설정관리/설정관리.md:1496` 역할생성, DLG-081-004 / `D09-설정관리/설정관리.md:1570` 역할삭제, DLG-081-005 / `D09-설정관리/설정관리.md:1642` 역할복사, DLG-081-006 / `D09-설정관리/설정관리.md:1715` 역할영향분석

### 1. 목적
권한 매트릭스를 변경하거나 역할 자체를 삭제·교체할 때, 영향 받는 범위를 사전에 확인해 운영 사고를 방지하는 확인 다이얼로그입니다. 매니저·Owner(지점장)이 의도하지 않은 권한 변경으로 직원 업무가 차단되는 상황을 막습니다.
본 다이얼로그의 영향 분석은 저장 전 확인 안내입니다. 별도 예측 산식, 영향 시뮬레이션, 분석 리포트 생성은 포함하지 않습니다.

### 2. UI 구성
- **요약 헤더**: 변경 또는 삭제 대상 역할 이름과 작업 유형(권한 변경 / 역할 삭제 / 역할 교체) 표시
- **영향 범위 카드 2종**:
  - 배정된 직원 수 (예: "이 역할에 배정된 직원 12명")
  - 그 직원이 담당하는 회원 수 합계 (예: "담당 회원 합계 384명")
- **영향 직원 목록 (펼침)**: 직원명 / 소속 지점 / 직무 / 현재 역할 / 변경 후 역할 또는 변경 후 권한 상태를 표시. 5명까지 기본 표시, 더보기로 확장.
- **변경 차이 요약**: 권한 변경의 경우 추가되는 권한, 제거되는 권한, 민감 기능 변경 여부를 before/after 비교로 표시.
- **확인 액션**: [취소] / [그대로 진행] 버튼. 진행 클릭 시 후속 단계(저장 또는 삭제 확인 DLG-081-004 / `D09-설정관리/설정관리.md:1570`)로 이동.

| 표시 항목 | 기준 |
|---|---|
| 배정 직원 수 | 현재 대상 역할이 배정된 활성 직원 수 |
| 담당 회원 합계 | 영향 직원들이 담당자로 연결된 회원 수 합계 |
| 권한 차이 | 접근/조회/생성/수정/삭제 권한의 추가·제거 항목 |
| 민감 기능 변경 | 회원 영구 삭제, 환불 처리, 급여 확정·지급, 슈퍼관리자 권한 부여/회수, 데이터 복원, 감사로그 내보내기 변경 여부 |
| 재배정 필요 여부 | 역할 삭제 시 배정 직원이 1명 이상이면 필수 |

### 3. 액션 / 흐름
1. SCR-081에서 권한 변경 후 [저장] 또는 [역할 삭제] 클릭 시 다이얼로그가 표시된다.
2. 영향 범위(직원·회원 수)와 영향 직원 목록을 확인한다.
3. 권한 변경의 경우 before/after 차이를 확인한다.
4. [그대로 진행] 클릭 → 후속 단계 진행 (저장 또는 DLG-081-004 / `D09-설정관리/설정관리.md:1570` 삭제 확인).
5. [취소] 클릭 → SCR-081로 돌아가 변경 사항을 재검토한다.

## DLG-084-001 플랜 변경 확인

> 유형: 다이얼로그
> URL 경로: `/subscription`
> 연결 기능: SET-03 / `D09-설정관리/설정관리.md:644`
> 연결 다이얼로그: DLG-084-001 / `D09-설정관리/설정관리.md:1794` 플랜변경확인, DLG-084-002 / `D09-설정관리/설정관리.md:1872` 구독해지확인

### 1. 목적
구독 플랜을 변경하기 전 변경 내용과 요금 차액을 상세히 안내하고 최종 동의를 받는 다이얼로그입니다.

### 2. UI 구성
- "플랜 변경 확인" 제목
- 현재 플랜 → 변경 플랜 비교 표시 (플랜명, 월 요금)
- 요금 차액 안내 (업그레이드 시 추가 요금, 다운그레이드 시 크레딧 또는 환불 정책 안내)
- 변경 적용 시점 안내 (즉시 적용 또는 다음 결제일부터 적용)
- 변경 후 포함/제외되는 기능 목록
- 취소 버튼
- 플랜 변경 확인 버튼

### 3. 액션 / 흐름
1. 구독 결제 관리 화면에서 플랜 변경 버튼 클릭 → 다이얼로그 표시
2. 변경 내용 및 요금 차액 확인
3. 취소 → 다이얼로그 닫힘, 현재 플랜 유지
4. 플랜 변경 확인 → 플랜 변경 처리 → 완료 메시지 표시 후 화면 갱신

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 표시 | 변경 내용 및 요금 정보 표시 |
| 처리 중 | 확인 버튼 클릭 후 처리 중 (버튼 비활성) |
| 변경 완료 | 다이얼로그 닫히고 구독 정보 화면 갱신 |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |

### 1. 목적
새 공지를 등록하거나 기존 공지를 수정하는 다이얼로그입니다.
제목, 내용, 게시 대상, 게시 기간을 설정하여 원하는 시점에 원하는 대상에게 공지를 전달합니다.

2. 수정 모드: 기존 공지 내용이 자동으로 채워진 상태
3. 내용 작성 후 저장 → 목록에 즉시 반영
4. 취소 → 다이얼로그 닫힘, 변경 사항 폐기

## DLG-089-002 백업 설정

> 유형: 다이얼로그
> URL 경로: `/settings/backup`
> 연결 기능: SET-EXT-03 / `D09-설정관리/설정관리.md:1097`
> 연결 다이얼로그: DLG-089-001 / `D09-설정관리/설정관리.md:2184` 데이터복원확인, DLG-089-002 / `D09-설정관리/설정관리.md:2263` 백업설정

### 1. 목적
자동 백업의 실행 주기, 시각, 데이터 보관 기간을 설정하는 다이얼로그입니다.

### 2. UI 구성
- "자동 백업 설정" 제목
- 자동 백업 활성화 여부 토글 (기본값 ON)
- 백업 주기 선택 (매일 / 매주 / 매월, 기본값 매일)
  - 매주 선택 시: 요일 선택 (월~일)
  - 매월 선택 시: 날짜 선택 (1~28일)
- 백업 실행 시각 선택 (시간 입력, 기본값 03:30)
- 백업 데이터 보관 기간 선택 (30일 / 90일 / 180일 / 무제한, 기본값 90일)
- 취소 버튼
- 저장 버튼

### 3. 액션 / 흐름
1. 데이터 백업·복원 화면에서 자동 백업 설정 변경 버튼 클릭 → 다이얼로그 표시
2. 현재 설정값이 자동으로 로드된 상태
3. 변경 후 저장 버튼 클릭 → 설정 저장 완료, 다이얼로그 닫힘
4. 취소 → 다이얼로그 닫힘, 변경 사항 폐기

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 표시 | 현재 백업 설정값 로드된 상태 |
| 비활성화 | 자동 백업 OFF 시 하위 설정 항목 비활성(회색) |
| 저장 완료 | 다이얼로그 닫히고 백업 화면에 변경 내용 반영 |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| superAdmin | 자동 백업 ON/OFF, 주기, 실행 시각, 보관 기간을 조회·수정·저장할 수 있습니다. |
| Owner(지점장) | 소속 지점 백업 설정과 이력을 조회할 수 있고 수동 백업을 실행할 수 있습니다. 자동 백업 설정 변경은 불가합니다. |
| manager / trainer / fc / staff / readonly | 접근 불가. 메뉴와 액션을 숨기거나 접근 제한 화면을 표시합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |
