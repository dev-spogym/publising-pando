# 공통 참조 코드 인덱스

## 1. 목적

이 문서는 여러 도메인에서 반복 참조되는 공통 코드의 의미와 기준 문서 위치를 한 곳에 모은 인덱스입니다.

개발자는 화면 문서에서 코드만 보고 다른 폴더를 역추적하지 않습니다. 화면 문서에 코드가 나오면 이 문서에서 코드의 정체를 확인하고, 실제 화면 UI와 동작은 해당 도메인 문서를 기준으로 구현합니다.

## 2. 작성 원칙

- 코드만 단독으로 쓰지 않고 기능명과 동작 설명을 함께 적습니다.
- `SCR-*`, `DLG-*`는 화면/다이얼로그 ID입니다.
- `AUTH-*`, `NFR-*`, `HQ-*`, `SET-*`, `MBR-*`, `PAY-*`, `PRD-*`, `A*`, `X*`는 기능/정책/자동화 참조입니다.
- 코드가 화면 ID인지 기능 ID인지 애매하면 이 문서의 구분을 우선합니다.
- 이 문서에 없는 새 공통 코드를 만들 때는 먼저 이 인덱스에 의미를 추가합니다.

## 3. 공통 코드 기준

### AUTH-01 관리자 인증 / 세션 / 보안 정책

- 성격: 인증·세션·계정 보안 정책.
- 기준 위치: D01-공통 운영정책, D01-공통 공통 문서.
- 포함 범위: 로그인 실패 5회 잠금, 30분 자동 해제, superAdmin / primary / Owner(지점장) 수동 해제, 비밀번호 만료, refresh 토큰, 신규 단말, IP 제한, 세션 종료.
- 구현 주의: `AUTH-01`은 화면 하나가 아니라 로그인과 보안 이벤트 전반의 정책 묶음입니다. 화면 문서에서는 "AUTH-01 기준으로 로그인 잠금 알림 생성"처럼 실제 동작을 같이 적습니다.

### NFR-05 만료/기한 알림 배치

- 성격: 날짜 기준 대상 산출과 배지·배너·알림 큐 동기화.
- 기준 위치: `_공통/비기능요구사항_NFR.md`.
- 포함 범위: 회원 이용권 만료, 결제기한 만료, 락커 만료.
- 제외 범위: 결제링크 만료. 결제링크는 PAY-06 기준의 7일 고정, D-3/D-1 고정 알림입니다.

### NFR-19 알림 센터 / 발송 큐

- 성격: 운영자·회원 대상 알림, 발송 큐, 미열람, 재시도, 실패 이력.
- 기준 위치: `_공통/비기능요구사항_NFR.md`, D01-공통 알림 센터.
- 포함 범위: 관리자 인앱 알림, 회원앱 Push, 카카오톡 fallback, 명시된 SMS.
- 구현 주의: `NFR-19 알림`만 적으면 부족합니다. 이벤트명, 수신자, 채널, 클릭 시 이동 위치를 함께 적습니다.

### HQ-09 정책 라이브러리

- 성격: 본사 표준 정책 세트 관리 화면/정책 기준.
- 기준 위치: D10-본사관리 HQ-09, D09 지점 자동화 적용.
- 포함 범위: 회원 이용권 만료, 결제기한 만료, 락커 만료 정책 step, 지점 적용 상태, 지점 추가 step 허용 여부. 결제기한 만료에는 미수/연체 알림 N일, 반복 주기, 발송 시각, 기본 채널 설정을 포함합니다.
- 구현 주의: 만료 정책은 `NFR-05`와 함께 읽습니다. HQ-09는 정책을 만들고 배포하는 쪽이고, NFR-05는 해당 정책 결과를 산출·동기화하는 쪽입니다.

### SCR-097 히스토리 로그

- 성격: 관리자 웹 변경·보안·정책 이벤트 이력 화면.
- 기준 위치: D10-본사관리 히스토리 로그, `_공통/01_ID매핑_v2_v3.md`.
- 포함 대상: 권한 변경, 역할 생성·삭제, 민감 권한 부여, 환불, 결제링크 무효화, 직원 퇴사 처리, 급여 확정/취소, 정책 저장, 백업/복원, 보안 이벤트.
- 구현 주의: 화면에서 "감사 로그 기록"이라고 쓰인 변경 액션은 SCR-097 조회 대상입니다. DB 테이블명은 개발사 설계 범위입니다.

### SCR-086 / SET-EXT-05 출석 관리 설정

- 성격: 출석 인정, 지각, 노쇼, 직원 근태 기준을 설정하는 화면/기능.
- 기준 위치: D09-설정관리 출석 관리 설정.
- 포함 범위: 출석 인정 N분 전/후, 수업 지각 N분, 수업 지각 N회 알림, 노쇼 유예 N분, 직원 지각 허용 시간.
- 구현 주의: 수업/회원 출석 기준과 직원 근태 지각 기준은 별도 정책값입니다. 직원 지각 기준은 D07 직원 근태 관리의 자동 분류 기준으로 사용합니다.

### A04 홀딩 자동 연장/해제 배치

- 성격: 회원 홀딩 종료와 만료일 연장을 자동 처리하는 배치.
- 기준 위치: `_공통/자동화_크론.md`, D02 회원관리.
- 포함 범위: 홀딩 만료일 도래 시 상태 복귀, 만료일 자동 연장.
- 구현 주의: A04는 회원 상태·이용권 기간 처리이며, 결제기한·락커 만료와 섞지 않습니다.

### A05 자동 노쇼 / 페널티 크론

- 성격: 수업 종료 후 미출석·미처리 예약을 노쇼로 분류하고 페널티를 부과하는 배치.
- 기준 위치: `_공통/자동화_크론.md`, D04 수업관리, D09 출석 관리 설정.
- 포함 범위: 출석 인정 마감 + 지점 노쇼 유예 N분 기준 자동 노쇼, 페널티 자동 부과.
- 구현 주의: 노쇼 유예 N분이 미설정이면 자동 노쇼를 보류하고 수동 처리 대상으로 남깁니다.

### X11 PII 마스킹 배치

- 성격: 개인정보 마스킹 및 민감정보 보호 정책.
- 기준 위치: `_공통/자동화_크론.md`, D02 회원관리.
- 포함 범위: 탈퇴/삭제 90일 경과 회원 이름·연락처·이메일 마스킹, 권한자 사유 입력 후 일시 해제.
- 구현 주의: 마스킹 해제는 조회 권한과 사유 입력 기준을 함께 확인합니다.

### MBR-EXT-04 세그먼트 자동 분류

- 성격: 회원 자동 세그먼트 산출 정책.
- 기준 위치: D02 회원관리, D02 운영정책.
- 포함 범위: 신규, 이탈위험, 만료임박, 충성, 활발, 관심필요, 만료후미등록 기본 자동 라벨과 충성 보조 라벨.
- 구현 주의: 첫 정상 결제 완료일을 신규 기준일로 사용합니다. 출석은 키오스크, 수업 출석, PT/GX 참여, 수동 출석을 포함하되 2시간 이내 중복 입장은 제외합니다.

### PRD-04 할인 설정

- 성격: 할인 정책 등록·관리 화면.
- 기준 위치: D05 상품관리, D03 매출관리, `_정책미확정_확인필요.md`.
- v1 포함 범위: 할인 정책 등록, 수정, 삭제, 조회.
- v1 제외 범위: 결제 화면 자동 적용, 할인 후보 조회, 중복 적용 우선순위, 직원 수동 변경 제한.
- 구현 주의: D03 결제 화면은 PRD-04를 개발 기준으로 자동 적용하지 않습니다. 운영자가 수동 확정한 할인 항목과 금액만 입력·표시합니다.

### PRD-08 시즌 가격 관리

- 성격: 기간 한정 시즌 특가 등록·수정·종료 관리.
- 기준 위치: D05 상품관리.
- 포함 범위: 시즌명, 기간, 대상 상품, 특가 금액/할인율, 진행 중 상품 강조, 종료 후 정상가 복구.
- 구현 주의: 같은 상품의 같은 기간에는 시즌 특가 1개만 활성화합니다. 할인/쿠폰/마일리지와의 결제 우선순위는 v1 제외이며 고객사 확인 후 후속 반영합니다.

### PAY-06 결제링크

- 성격: 비대면 전액 결제를 위한 결제링크 발송 흐름.
- 기준 위치: D03 매출관리.
- 현재 상태: 고객사 최종 확인 전 개발 대기 항목.
- 확정 후 포함 범위: 결제링크 발송, 재발송, 무효화, 만료, 결제완료, 회원상세 결제내역 `미결제` 표시, 무효화 사유/처리자 이력.
- 제외 범위: 계약금/잔액/할부 수납, 미수금 생성, 결제링크 열람됨 상태.
- 구현 주의: 결제링크는 7일 고정, D-3/D-1 고정 알림이며 NFR-05 정책 step을 사용하지 않습니다.

## 4. 개발 문서 표기 규칙

좋지 않은 표기:

```markdown
NFR-19 처리 결과 알림
```

권장 표기:

```markdown
수업 변경 저장 완료 시 예약 회원에게 회원앱 Push를 발송하고, 지점이 카카오톡 fallback을 켠 경우 Push 실패/미설치 회원에게 카카오톡을 발송한다. 알림 클릭 시 해당 수업 상세로 이동한다. 이 발송 이력은 NFR-19 알림 센터 기준으로 관리한다.
```

좋지 않은 표기:

```markdown
SCR-097 기록
```

권장 표기:

```markdown
권한 변경 저장 시 변경자, 변경 전/후 권한, 대상 직원 수, 변경 사유를 히스토리 로그(SCR-097)에서 조회할 수 있게 기록한다.
```
