## 4. 기기 연동

CRM에서 고정으로 인식하는 기기 종류는 출입 게이트, 키오스크, 락커 컨트롤러, InBody 측정기 4종입니다.

카드 리더, 얼굴인식 카메라, RFID/밴드, QR은 별도 기기 종류가 아니라 출입 게이트 또는 키오스크의 인증 수단/부속 모듈입니다.

기기 헬스체크는 본사 정책 기본 5분 간격입니다. 본사(primary/superAdmin)가 전역 기준을 변경할 수 있고, 지점은 해당 기준을 임의 변경하지 않습니다. 정상 응답은 OK, 미응답·응답 지연·부분 응답은 불안정, 인증 실패·하드웨어 오류는 오류로 표시합니다. 불안정 또는 오류 감지 시 운영자에게 즉시 알림을 보냅니다.

## 8. 문서 작성 규칙

외부연동 관련 내용을 추가할 때는 아래 항목을 반드시 적습니다.

| 필수 항목 | 설명 |
|---|---|
| 외부 대상 | 실제로 CRM 밖에 있는 채널, 기기, 앱, 시스템 이름 |
| CRM 책임 | CRM 화면에서 제공해야 하는 버튼, 상태, 표시, 입력, 이력 |
| 외부 책임 | 외부 시스템이 실제로 수행하는 처리 |
| 사용 조건 | 언제 해당 연동을 사용하는지 |
| 실패 처리 | 실패 시 화면 표시, 재시도, fallback, 운영자 알림 |
| 참조 | 관련 화면/정책 문서 경로와 라인 |

잘못된 예시는 아래와 같습니다.

```md
알림 센터 처리 결과를 회원에게 알림
결제 처리와 연동
기획 보강 완료
```

권장 표현은 아래와 같습니다.

```md
| 항목 | 정의 |
|---|---|
| 외부 대상 | 회원앱 Push |
| CRM 책임 | 수업 시간 변경 이벤트 발생 시 예약 회원에게 Push 발송 요청을 생성하고 발송 상태를 표시합니다. |
| 실패 처리 | Push 실패 또는 미설치 회원은 지점 KakaoTalk fallback ON일 때만 카카오톡으로 보조 발송합니다. |
| 참조 | `docs2/D04-수업관리/수업관리.md:1096`, `docs2/_공통/비기능요구사항_NFR.md:104` |
```
