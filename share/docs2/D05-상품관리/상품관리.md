### D05 다이얼로그 ID 원칙
- D05에서는 `P001`처럼 번호만 축약해 부르지 않습니다. 항상 DLG-P001 / `D05-상품관리/상품관리.md:934` 상품등록모달처럼 전체 ID를 사용합니다.
- 상품 등록과 상품 상세/수정은 실제 프론트 기준으로 같은 상품 패널 입력 UI를 공유합니다. 신규 등록은 `isNew=true` 상태, 기존 상품 조회/수정은 상품 데이터가 채워진 상태로 구분합니다.
- DLG-P001 / `D05-상품관리/상품관리.md:934` 상품등록모달은 별도 컴포넌트명이 아니라, 공유 상품 패널의 신규 등록 상태를 가리키는 기획 식별자입니다.
- 별도 상품 수정 전용 모달은 사용하지 않습니다. 기존 상품 수정은 SCR-P003 / `D05-상품관리/상품관리.md:320` 상품 상세/수정 패널에서 같은 패널 UI로 바로 처리합니다.
- 전 지점 배포부터 시즌가격 등록/수정까지 모든 다이얼로그는 P001~P023 번호대를 사용하되, 문서 참조 시에는 항상 DLG-P001 / `D05-상품관리/상품관리.md:934` 상품등록모달 같은 전체 ID로 표기합니다.

## 화면
## SCR-P001 상품 관리

> 유형: 화면
> URL 경로: `/products`

### 1. 목적
센터에서 판매하는 모든 상품(회원권, 수강권, 락커, 운동복 대여비, 일반 상품)을 한눈에 파악하고 관리하는 메인 화면입니다. 상품별 가격, 이용 기간, 요일·시간 제한, 판매 현황을 목록으로 조회하고, 우측 패널을 통해 상세 정보를 확인하거나 수정할 수 있습니다. 여기서 `운동복`은 실제 수량을 차감하는 재고 품목이 아니라 회원에게 청구하는 대여비/상품 비용입니다.

### 2. UI 구성
### 상단 통계 카드

| 카드 | 설명 |
|------|------|
| 전체 상품 수 | 센터에 등록된 상품 총 개수 |
| 활성 상품 수 | 현재 판매 중인 상품 수 |
| 비활성 상품 수 | 판매 중지된 상품 수 |
| 카테고리 수 | 등록된 상품 분류 개수 |

### 3단계 필터

| 단계 | 필터 유형 | 옵션 |
|------|-----------|------|
| 1단계 | 상품 대분류 탭 | 전체 / 회원권 / 수강권 / 락커 / 운동복 / 일반 |
| 2단계 | 세부 유형 필터 | 1단계 선택값에 따라 동적 표시. 수강권 선택 시 PT / GX / 골프 / 기타 수강권, GX 선택 시 요가 / 필라테스 / 스피닝 / 줌바 / GX 기타 |
| 3단계 | 상품그룹 필터 | 상품관리 "분류 관리" 탭에서 등록한 상품그룹. 선택한 대분류 안에서만 표시 |

- 우측 상단에 상품명 검색 입력창 제공

### 상품 마스터 분류 모델

상품 마스터 대분류는 회원권 / 수강권 / 락커 / 운동복 대여비 / 일반 5개입니다.

| 단계 | 필드 | 값 |
|------|------|------|
| 1단계 | 상품 대분류 `category` | `MEMBERSHIP`(회원권) / `LESSON_PASS`(수강권) / `LOCKER`(락커) / `WEAR`(운동복 대여비, 재고 수량 차감 없음) / `GENERAL`(일반) |
| 2단계 | 세부 유형 `lessonType` 또는 `subCategory` | 수강권은 PT / GX / 골프 / 기타 수강권으로 구분하고, GX 세부종목은 별도 고정 옵션으로 관리 |
| 3단계 | 상품그룹 `productGroupId` | 상품관리 "분류 관리" 탭의 그룹을 우선 표시 |

수강권에서 GX를 선택하면 GX 세부종목은 아래 5개로 고정합니다.

| 대분류 | 세부종목 |
|------|------|
| GX | 요가 / 필라테스 / 스피닝 / 줌바 / GX 기타 |

- `category=LESSON_PASS`이고 `lessonType=GX`인 상품은 GX 세부종목 선택이 필수입니다.
- `productGroupId`가 있는 상품은 연결된 `product_groups.name`을 목록 분류 필터와 상품명 배지에 우선 표시합니다.
- `productGroupId`가 없는 상품은 `category`의 한글 표시명(회원권/수강권/락커/운동복/일반)을 fallback으로 사용합니다.
- 이 세부종목 값은 SCR-S004 / `D03-매출관리/매출관리.md:353` 매출통계의 GX종목별 탭과 SCR-094 / `D10-본사관리/본사관리.md:405` KPI 대시보드의 GX 세부 종목 분포 집계 기준입니다.

### 3. 액션 / 흐름
1. **상품 조회**: 화면 진입 시 센터 전체 상품 목록 표시 → 필터/검색으로 원하는 상품 찾기
2. **상품 상세 보기**: 목록에서 상품 행 클릭 → 우측에 상세 패널이 슬라이드로 열림
3. **상품 등록**: 상단 "+ 상품 등록" 버튼 클릭 → 우측 패널이 등록 모드로 열림
4. **분류 관리**: "분류 관리" 탭 클릭 → 상품 분류 등록/수정/삭제
5. **전 지점 배포** (슈퍼관리자): "전 지점 배포" 버튼 → 배포 모달에서 상품·지점 선택 후 일괄 배포
6. **Excel 내보내기**: "Excel" 버튼 → 현재 필터된 상품 목록을 엑셀 파일로 다운로드

### 5. 권한
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|-----------|--------------|--------------|
| fc / trainer / staff / readonly | 상품 목록 조회, 통계 카드 | 상품 조회만 가능 |
| Owner(지점장) / manager | 소속 지점 상품 전체 화면 | 상품 등록, 수정, 삭제, Excel 다운로드 |
| superAdmin / primary | 전체 화면 + 전 지점 배포 버튼 | 상품 등록/수정/삭제 + 전체 지점 동시 배포 |

상품 등록·수정·삭제 권한 기준: 별도 개인 예외 권한을 두지 않고 역할 기준으로만 판단합니다. 상품 등록은 SCR-P001 / `D05-상품관리/상품관리.md:27` 상품 관리 상단의 `[+ 상품 등록]` 버튼에서 시작하며, 클릭 시 SCR-P002 / `D05-상품관리/상품관리.md:185` 상품 등록 / DLG-P001 / `D05-상품관리/상품관리.md:934` 상품등록모달의 신규 등록 상태 패널이 열립니다. 기존 상품 수정은 SCR-P001 / `D05-상품관리/상품관리.md:27` 상품 행 클릭 후 SCR-P003 / `D05-상품관리/상품관리.md:320` 상품 상세/수정 패널에서 수행합니다. 상품 생성·수정·삭제 버튼은 superAdmin / primary / Owner(지점장) / manager에게만 노출하고, fc / trainer / staff / readonly에는 노출하지 않습니다.

신규 등록과 기존 상품 수정은 같은 상품 패널 입력 UI를 공유합니다. 신규 등록은 빈 폼과 `상품등록` 제목, 기존 상품 수정은 기존 값이 채워진 폼과 `상품수정` 제목으로 구분합니다.

### 1. 목적
상품 판매 시 사용할 할인 정책을 등록·수정·삭제하고, 변경 이력을 확인하는 화면입니다. 정률(%) 또는 정액(원) 방식, 최소 계약 기간, 최대 할인 한도, 적용 상품, 적용 기간을 관리합니다.

> 유형: 화면

> 연결 기능: PRD-06 / `D05-상품관리/상품관리.md:604`

### 1. 목적
최대 3개 상품을 나란히 비교하여 회원 상담 시 최적의 상품을 안내할 수 있는 화면입니다. 가격, 이용 기간, 포함 서비스, 옵션 항목을 동일한 기준으로 나란히 표시하여 상품 간 차이를 직관적으로 파악할 수 있습니다.

## DLG-P002-전지점배포 전 지점 배포 확인

> 유형: 다이얼로그
> 연결 기능: PRD-01 / `D05-상품관리/상품관리.md:27`, PRD-EXT-01 / `D05-상품관리/상품관리.md:185`

### 1. 목적
슈퍼관리자가 현재 센터에 등록된 상품을 다른 지점 전체 또는 선택한 지점에 동시에 배포하는 모달입니다. 이미 같은 이름의 상품이 있는 지점은 자동으로 건너뛰어 중복 등록을 방지합니다.

### 2. UI 구성
### 배포할 상품 선택 영역

- 전체 상품 목록을 체크박스로 표시
- "전체 선택" 버튼 제공
- 선택된 상품 수 표시 (예: "3개 선택됨")

### 대상 지점 선택 영역

- "전체 지점" 체크박스 (기본 선택)
- 전체 지점 체크 해제 시: 개별 지점 체크박스 목록 표시
- 현재 로그인 지점은 목록에서 제외

하단 버튼: **취소** / **배포하기**

---

### 3. 액션 / 흐름
1. 배포할 상품 선택 (1개 이상 필수)
2. 대상 지점 선택 (1개 이상 필수)
3. "배포하기" 버튼 클릭
   - 상품 미선택 시: "배포할 상품을 선택하세요." 안내
   - 지점 미선택 시: "배포할 지점을 선택하세요." 안내
   - 정상 진행 시: 배포 최종 확인 단계로 이동
4. 최종 확인에서 승인 시: 지점별 배포 처리
   - 이미 같은 이름의 상품이 있는 지점은 건너뜀
   - 완료 후 "N건 추가, N건 건너뜀" 결과 메시지 표시

## DLG-P008-상품가져오기 상품 정보 가져오기

> 유형: 다이얼로그
> 연결 기능: PRD-04 / `D05-상품관리/상품관리.md:406`, PRD-EXT-01 / `D05-상품관리/상품관리.md:185`

### 1. 목적
기존에 등록된 상품의 정보를 복사하여 새 상품 등록의 기초 정보로 활용하는 모달입니다. 유사한 상품을 빠르게 등록할 때 유용하며, 가져온 정보는 새 상품으로 저장되어 원본 상품에 영향을 주지 않습니다.

### 2. UI 구성
- 상품명 검색 입력창
- 현재 편집 중인 상품을 제외한 전체 상품 목록
  - 상품명, 가격, 유형 표시
  - 행 클릭으로 선택

하단 버튼: **닫기**

---

### 3. 액션 / 흐름
1. 모달 열림 → 전체 상품 목록 표시 (현재 편집 중인 상품 제외)
2. 검색어 입력으로 원하는 상품 찾기
3. 상품 행 클릭 → 해당 상품 정보가 등록 폼에 자동 입력
4. 모달 닫힘 + "상품 정보를 불러왔습니다. 등록 시 새 상품으로 저장됩니다." 안내 메시지
5. 가져온 정보를 바탕으로 수정 후 새 상품으로 등록

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |

## DLG-P010-상품삭제최종확인 상품 삭제 최종 확인

> 유형: 다이얼로그
> 연결 기능: PRD-04 / `D05-상품관리/상품관리.md:406`, PRD-06 / `D05-상품관리/상품관리.md:604`, PRD-EXT-01 / `D05-상품관리/상품관리.md:185`

### 1. 목적
상품을 최종 삭제하기 전, 해당 상품에 연결된 이용권 또는 판매 이력을 안내하고 삭제 의사를 재확인하는 다이얼로그입니다. 연결된 데이터가 있을 경우 영향 범위를 명확히 알려줍니다.

### 2. UI 구성
- 삭제 대상 상품명 표시
- 연결된 이용권 수 / 판매 이력 안내 (있을 경우)
- 경고 문구: "삭제 후에는 복구할 수 없습니다."
- 하단 버튼: **취소** / **삭제**

---

### 3. 액션 / 흐름
1. 삭제 요청 → 대상 상품 정보 및 연결 데이터 조회
2. 이 다이얼로그에 요약 정보 표시
3. "취소" 클릭: 다이얼로그 닫힘, 작업 중단
4. "삭제" 클릭: 상품 삭제 처리 → 완료 메시지 → 화면 갱신

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |

| 정액 + 한도 입력 | 한도 필드 비활성 | 정률만 한도 |

| 적용 상품 0개 | "적용 상품을 선택해주세요" | 저장 차단 |

상품 카탈로그 또는 키오스크 화면에 표시될 상품 대표 이미지를 업로드하는 모달입니다. 이미지를 통해 고객에게 상품을 시각적으로 소개할 수 있습니다.

### 2. UI 구성
- 이미지 업로드 영역 (드래그 앤 드롭 또는 파일 선택)
- 업로드 가능 형식 안내: JPG, PNG, WebP
- 권장 크기 안내
- 미리보기 영역: 업로드한 이미지 확인
- 하단 버튼: **취소** / **적용**

---

### 3. 액션 / 흐름
1. 이미지 업로드 버튼 클릭 → 모달 열림
2. 파일 선택 또는 드래그 앤 드롭으로 이미지 업로드
3. 미리보기에서 이미지 확인
4. "적용" 클릭: 이미지 저장 → 상품 정보에 반영
5. "취소" 클릭: 모달 닫힘, 이미지 변경 없음

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |

## DLG-P014-가격이력조회 가격 이력 상세 조회

> 유형: 다이얼로그
> 연결 기능: PRD-04 / `D05-상품관리/상품관리.md:406`, PRD-EXT-02 / `D05-상품관리/상품관리.md:320`

### 1. 목적
선택한 기간 내의 상품 가격 변동 이력을 상세하게 조회하는 다이얼로그입니다. 특정 기간을 지정하여 그 기간 내 모든 가격 변경 내역을 확인할 수 있습니다.

### 2. UI 구성
- 조회 기간 선택 (시작일 ~ 종료일 날짜 피커)
- 가격 변동 이력 테이블

| 컬럼 | 내용 |
|------|------|
| 변경 일시 | 가격이 변경된 날짜와 시간 |
| 변경 전 가격 | 변경 이전 가격 |
| 변경 후 가격 | 변경 이후 가격 |
| 변경자 | 변경한 직원 이름 |

하단 버튼: **닫기**

---

### 3. 액션 / 흐름
1. 모달 열림 → 기본 기간(최근 3개월 등)으로 이력 표시
2. 기간 변경 시 해당 기간 이력 갱신
3. 이력 확인 후 "닫기" 클릭으로 모달 닫힘

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|

### 1. 목적
상품 카탈로그에 표시할 정보 항목과 레이아웃 옵션을 설정하는 모달입니다. 센터의 특성에 맞게 노출 항목을 조절할 수 있습니다.

| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

|---|---|

| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |
| readonly | 조회만 가능하며 저장·삭제·확정 같은 변경 액션은 숨기거나 비활성화합니다. |
| 권한 없음 | 다이얼로그 진입을 차단하고 부모 화면에서 권한 안내를 표시합니다. |
