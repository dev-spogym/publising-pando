# D05-상품관리 상품관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.

## 1. 핵심 원칙
- 상품 판매부터 계약, 결제, 이용권 생성, 환불까지를 하나의 운영 흐름으로 정리한다.
- 매출 화면과 회원 상세 화면이 서로 다른 상태를 말하지 않도록 기준을 통일한다.
- 상품 관리
- 할인 정책
- POS 판매
- 결제 처리
- 매출 현황
- 환불 관리
- 미수금 관리
- 선수익금 조회
- 회원권
- 수강권
- 락커
- 운동복
- 일반 상품
- 패키지는 상품 대분류가 아니라 상품 구성/판매 옵션으로 표시한다.
- GX 상품은 2단계 세부종목을 필수로 가진다.

## 2. 상태 / 운영 정의
- **Owner(지점장) 신규 오픈 / 시즌 라인업 정비**: 분류 관리에서 그룹 정비 → 회원권/수강권/락커/운동복/일반 상품 신규 등록 → 카탈로그(PRD-05 / `D05-상품관리/상품관리.md:503`)·키오스크 동시 노출 활성화
- **단종·시즌 종료 정리**: 비활성 토글로 신규 판매 차단(기존 보유 회원은 만료까지 유지), 카탈로그·키오스크에서 자동 노출 해제
- 활성 상품 수 | 판매 중 상품 |
- 비활성 상품 수 | 판매 중지 상품 |
- 상품명 | 유형 배지 + 패키지 배지 + 이름 |
- **재등록 할인**: 만료일 이전 재구매 시 정률 할인
- **할인 종료 / 비활성**: 캠페인 종료 시 비활성 토글 → 결제 화면 즉시 적용 제외
- 상태 | 활성 / 비활성 |
- 활성 토글 | ON/OFF |
- | PRD-04-01-02 / `D05-상품관리/운영정책.md:34` | 상태 배지 |
- 센터에서 판매 중인 상품을 고객이 보기 좋은 카탈로그 형태로 안내하는 화면이다. 회원 상담 시 직원이 태블릿이나 모니터로 보여주거나, 인쇄·공유하여 외부 홍보(전단지·웹 공유)에 활용한다. 상품 대분류별 색상 구분과 GX 5종 세부종목 배지로 시각적 구분이 명확해 회원이 직관적으로 상품을 비교할 수 있다. 카탈로그 노출 옵션·정렬·문구·이미지는 운영자가 카탈로그 설정(DLG-P017 / `D05-상품관리/상품관리.md:2143` 카탈로그설정)·편집(DLG-P018 / `D05-상품관리/상품관리.md:2230` 카탈로그편집)에서 제어하며, 미리보기(DLG-P016 / `D05-상품관리/상품관리.md:2063` 카탈로그미리보기)로 고객 노출 형태를 사전 확인한다.
- **시즌 캠페인 노출**: 시즌 가격(PRD-08 / `D05-상품관리/상품관리.md:821`) 진행 중 상품은 "할인" 배지 + 정상가 취소선 + 특가 표시
- GX 상품은 5종 세부종목 배지 표시 (요가·필라테스·스피닝·줌바·GX 기타)
- 시즌 특가 진행 중인 상품: "할인" 배지 + 정상가 취소선 + 특가 강조
- 패키지 구성 상품: "패키지" 배지
- 상품명 | 이름 + 유형 배지 + 패키지 배지 |
- **상품 0개 선택**: 빈 상태 "비교할 상품을 선택하세요" + 검색 강조
- **비활성 상품 선택**: 회색 처리 + "비활성" 배지 + 결제 점프 차단
- **결제 점프 + 비활성 상품**: 차단 + "이 상품은 신규 판매 불가" 안내
- **공유 URL 발급 + 비활성 상품 포함**: 외부 사용자에게 비활성 상품은 회색 처리

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
- 클라이언트 운영 정책값이 확정되지 않은 항목은 `docs2/_정책미확정_확인필요.md`에 별도 관리한다.
