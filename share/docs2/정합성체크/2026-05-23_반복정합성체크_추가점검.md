# docs2 반복 정합성 체크 추가 점검

- 기준일: 2026-05-23
- 범위: `docs2/**/*.md` 전체. 단, `정합성체크/**` 기존 보고서는 검사 기준에서 제외
- 목적: 개발팀이 기획서만 보고 화면 동작, 권한, 상태값, 정책 소유 위치, 자동화 대기 여부를 판단할 수 있는지 재점검
- 결론: 이번 추가 반복에서 발견한 self-fix 가능 drift는 모두 문서에 반영했다. 마지막 재검색 기준으로 남은 자가수정 대상 drift는 0건이다.

## 1. 이번 추가 점검에서 수정한 drift

|---|---|---|

## 2. 재검사 결과 drift 아님으로 확인한 항목

| 항목 | 판정 |
|---|---|
| `/` route 중복 | drift 아님. `superAdmin / primary`는 D01 SCR-101 전 지점 통합 대시보드, 그 외 권한은 D10 SCR-090 지점 대시보드로 분기한다. |
| `/locker/management` route 중복 | drift 아님. 사물함 배정 관리와 고정 물품 락커 관리는 같은 화면 안의 구분 영역이다. |
| `/rooms` route 중복 | drift 아님. SCR-059가 메인이고 SCR-053은 종속 구조다. |
| `/message/auto-alarm` route 중복 | drift 아님. SCR-072 설정과 SCR-072A 운영현황은 같은 페이지의 탭이다. |
| `/settings/iot` route 중복 | drift 아님. SCR-082A 키오스크 IoT 설정과 SCR-083 IoT 출입 관리는 같은 설정 페이지의 탭이다. |
| `/body-composition` route 중복 | drift 아님. D02는 회원 개인 추이/상담 활용 기준, D11은 전 지점 수신/검수/확정 운영 기준이다. |
| 상품 대분류 5개 vs GX 6종 | drift 아님. 상품 대분류는 회원권/수강권/락커/운동복/일반 5개이고, GX 6종은 수강권 하위 세부종목이다. |
| D04 강습 세션 4종 vs GX 6종 | drift 아님. 강습 세션 4종은 KPI 집계 축이고, GX 6종은 상품/매출 분류 축이다. |
| 회원 상태 8개 탭 vs 실제 상태 7개 | drift 아님. `전체`는 집계 탭이고 실제 회원 상태는 활성/만료/예정/임박/홀딩/미등록/탈퇴 7개다. |
| 자동 라벨 7종과 충성 중복 | drift 아님. 자동 7종에 충성이 포함되며, 충성은 보조 라벨로 다른 기본 라벨과 중복 표시 가능하다. |
| IoT 기기 종류 4종 | drift 아님. 출입 게이트/키오스크/락커 컨트롤러/InBody 측정기 4종으로 통일되어 있다. |

## 3. 고객사 확인 또는 후속 확정으로 남긴 항목

|---|---|

| 환불/취소 자동 계산 산식 | v1은 모든 환불 금액 수기 입력 + Owner(지점장) 승인. 자동 산식은 후속 확정 필요. |

## 4. 마지막 검색/검증 기준

마지막 재검색에서 아래 drift 후보는 발견되지 않았다.

- `센터장`, `센터 관리자`, 독립 표기 `지점장`
- `VIP`, `VVIP`
- `잔여 부족 마케팅`
- `회원권 유형`
- `재고 5개 이하`, `20 미만`
- `폐점 후 6개월`
- `PAY-06 결제링크`가 `PAY-03`으로 연결된 표현
- `회원 이용권 만료 알림 잡`
- `본사 표준 정책 변경 시도`
- `강사 액션`, `스태프 또는 강사가`
- `보류 또는 다음 결제 후 차감`

`git diff --check`도 통과했다. 출력에 남은 LF/CRLF warning은 Git의 line ending 안내이며 문서 정합성 오류나 trailing whitespace 오류는 아니다.
