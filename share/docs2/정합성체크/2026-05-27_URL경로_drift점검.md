# docs2 URL 경로 정합성 점검

## 기준

- 비교 대상은 `docs2` 내부 문서뿐입니다.
- `docs/admin` 원본 화면설계서/기능명세서는 비교하지 않았습니다.
- 점검 범위는 화면/기능/UX/정책 문서의 URL 경로 정의와 화면 이동 설명입니다.
- API, DB, table, endpoint 구현 방식은 판정 대상에서 제외했습니다.

## 요약

| 항목 | 결과 |
|---|---:|
| URL 경로가 명시된 섹션 | 150개 |
| 고유 URL 경로 | 106개 |
| 같은 화면 ID가 서로 다른 URL을 가진 경우 | 0건 |
| 실제 drift | 1건 |
| 표기 보완 필요 | 1건 |
| 의도된 공유 route | 5건 |

## 1. 실제 drift

| 구분 | 위치 | 현재 값 | 충돌 대상 | 판단 |
|---|---|---|---|---|
| `_공통/01_ID매핑` route 불일치 | SCR-055 / `_공통/01_ID매핑_v2_v3.md:180` | `/clothing` | SCR-055 / `D06-시설관리/시설관리.md:474`는 `/facility/inventory` | 실제 route drift. D06 정본이 `상품 재고 관리`로 변경되어 있으므로 `_공통/01_ID매핑_v2_v3.md`의 route를 `/facility/inventory`로 맞추는 것이 자연스럽다. |

## 2. 표기 보완 필요

| 구분 | 위치 | 현재 표현 | 문제 | 권장 정리 |
|---|---|---|---|---|
| 로그아웃 처리 흐름이 URL 경로처럼 추출됨 | SCR-109 / `D01-공통/공통.md:581` | `(로그아웃 처리 후 /login 이동)` | 실제 화면 route가 아니라 로그아웃 후 SCR-100 로그인으로 이동하는 처리 흐름이다. 개발자가 SCR-109를 `/login`의 별도 화면으로 오해할 수 있다. | `URL 경로`가 아니라 `공통 액션: 로그아웃 처리 후 SCR-100 / D01-공통/공통.md:20 로그인으로 이동`으로 표현하는 것이 안전하다. |
