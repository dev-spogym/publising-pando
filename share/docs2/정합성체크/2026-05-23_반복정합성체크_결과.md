## 2. 마지막 재검사에서 drift가 아닌 것으로 확인한 중복/수량 표현

| 항목 | 판정 |
|---|---|
| D04 강습 세션 4종 vs GX 6종 | drift 아님. 강습 세션 4종은 KPI 집계 축이고, GX 6종은 상품/매출 세부 종목 축이다. 문서에 섞지 않는다고 명시되어 있다. |
| D02 회원 상태 8개 탭 vs 실제 상태 7개 | drift 아님. `전체`는 조회 탭이고 실제 상태는 7개다. |
| D02 자동 라벨 7종 | drift 아님. `충성`은 7종에 포함되지만 보조 라벨로 중복 표시 가능하다. |
| 상품 분류 5개 vs 수강권 세부 유형 | drift 아님. 대분류 5개와 수강권 세부 유형/PT/GX/골프는 서로 다른 축이다. |
| IoT 기기 4종 | drift 아님. 출입 게이트/키오스크/락커 컨트롤러/InBody 측정기 4종으로 통일되어 있다. |

| `/locker/management` route 중복 | drift 아님. 같은 화면 안에서 D06 사물함 배정 탭과 D11 고정 물품 락커 탭으로 나뉜다. |
| `/body-composition` route 중복 | drift 아님. D02는 회원 개인 추이/상담 활용, D11은 전 지점 수신/검수/확정 운영 기준이다. |
| `/message/auto-alarm` route 중복 | drift 아님. SCR-072 설정과 SCR-072A 운영현황은 같은 페이지의 탭이다. |
| `/settings/iot` route 중복 | drift 아님. SCR-082A 키오스크 IoT 설정과 SCR-083 IoT 출입 관리는 같은 설정 페이지의 탭이다. |
| `/rooms` route 중복 | drift 아님. SCR-059가 메인이고 SCR-053은 종속 탭/섹션 구조다. |

## 4. 마지막 검색 기준

다음 drift 후보는 마지막 재검색에서 더 이상 발견되지 않았다.

- `회원권/PT`, `이용권·PT`, `category=GX`
- `테넌트 정책 활성 시`
- `결제링크 발송 경로는 별도로 유지`
- `FAC-EXT-04 청소 연계 ... 정책 활성 시`
- `횟수 차감 시점은 수업 유형과 정책에 따라`
- `역할·소속 지점은 운영 정책에 따라 수정 가능`
- `자동 취소 또는 유지`
- `보류 또는 다음 결제 후 차감`
- `상품 유형 4/6`, `상품 4/6종`처럼 대분류 수량이 충돌하는 표현
- `상태 8개`, `라벨 8개`, `IoT 장치 5종`

## 5. 현재 결론

현재 문서 기준에서 개발팀이 즉시 구현 기준으로 삼을 수 없는 항목은 `_정책미확정_확인필요.md`에 모았다. 그 외 이번 반복에서 발견된 self-fix 가능한 권한, 상태, 수량, 버튼 동작, route 분기, 정책 위치 drift는 문서에 직접 반영했다.

따라서 다음 반복의 시작 기준은 `자체 해결 가능한 신규 불일치 0건`이다. 미수/연체 운영 임계값은 미확정 항목이 아니라 HQ-09 본사 정책 세트 입력값으로 관리한다.
