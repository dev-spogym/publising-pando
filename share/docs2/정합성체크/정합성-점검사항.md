2. **스태프 권한이 같은 문서 안에서도 모순됩니다.**  
   `SCR-M001`에서는 스태프가 “회원 추가 가능”과 “조회 및 출석 처리만”으로 두 번 정의되어 있고, `SCR-M002`에서는 스태프가 “완전 접근”과 “접근 불가”에 동시에 들어갑니다.

3. **리드 `등록완료` 칸반 이동의 의미가 불명확합니다.**  
   문서상 `등록완료`는 첫 정상 결제 완료 시점에만 확정됩니다. 그런데 칸반에서 `등록완료` 컬럼으로 이동하는 액션도 제공됩니다. 이게 “상태 저장”인지 “회원 등록 화면으로 보내는 트리거”인지 분리하지 않으면 전환율 통계가 흔들릴 수 있습니다.

5. **결제링크 상태값과 `미결제` 용어가 섞여 있습니다.**  
   결제링크 저장 상태는 `발송됨 / 결제완료 / 만료 / 무효화 / 발송실패`인데, 회원상세 결제내역에는 `미결제/발송됨`으로 표시되고, 미수금 화면의 `미결제`는 계약 미수 의미입니다. API enum, 필터, 배지명을 분리해야 합니다.
