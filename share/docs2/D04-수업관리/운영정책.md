# D04-수업관리 수업관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.

## 1. 핵심 원칙
- 수업 생성부터 예약, 출석, 횟수 차감, 노쇼 처리까지 센터 운영의 실제 흐름을 정리한다.
- PT, GX, 골프 등 유형이 달라도 공통 운영 틀을 유지한다.
- 캘린더/수업 일정
- 수업 관리
- 예약/대기예약
- 출석 처리
- 서명 처리
- 횟수 차감
- 노쇼/페널티
- 수업 유형은 PT / GX / 골프 / 기타 4종으로 고정한다.
- 강습 세션 유형은 PT / GX / 골프 / 기타 4종으로 고정하며, 기본값은 수업 유형과 동일하게 설정한다. 이 값은 강습세션수 KPI의 월간·주차별 집계 축으로 사용한다.
- GX 세부종목은 요가 / 필라테스 / 스피닝 / 줌바 / GX 기타 5종으로 고정하며, 상품/매출 분류 축이고 수업 유형·강습 세션 유형과 섞지 않는다.
- 수업 등록/수정 화면에는 KPI 집계 대상 수업일 때 강습 세션 유형 선택값이 보이도록 한다.
- D04 수업관리 문서는 KPI 화면을 정의하지 않는다. 수업 도메인은 강습세션수, 출석률, 노쇼, 강사 NPS 등의 원천 데이터를 제공하고, 실제 KPI 표시 화면은 D10 SCR-094 / `D10-본사관리/본사관리.md:405` KPI 대시보드를 참조한다.
- 수업은 일정 단위, 예약은 좌석 단위, 출석은 실제 이용 단위다.
- v1 기준 모든 예약형 수업의 횟수 차감은 "출석 확정 후 1회 차감"으로 통일한다. 예약 시에는 잔여 횟수 보유 여부와 이용권 유효 상태를 검증하며, 잔여 0회 또는 이용권 만료 상태이면 예약·출석을 차단한다. 이 경우 별도 실패·횟수조정 상태를 만들지 않는다.
- 미승인 일괄 등록, 수업 템플릿 본사 동기화, 대기열 자동 알림/배정, 신규 회원 첫 노쇼 면제 같은 수업 정책 토글은 지점 화면에서 임의 설정하지 않고 D10 SCR-H1001 / `D10-본사관리/본사관리.md:894` 자동화 정책 라이브러리의 수업 운영 자동화 정책에서 본사(superAdmin/primary)가 관리한다.

## 2. 상태 / 운영 정의
- 일정표 / 수업 관리(SCR-C002 / `D04-수업관리/수업관리.md:110`) / 횟수 관리(SCR-C007 / `D04-수업관리/수업관리.md:508`) / 페널티 관리(SCR-C008 / `D04-수업관리/수업관리.md:582`, 건수 배지) / 유효 수업 목록(SCR-C011 / `D04-수업관리/수업관리.md:859`).
- 정원 상태 | 전체 / 여유 / 마감 토글 |
- | CLS-01-02-03 / `D04-수업관리/운영정책.md:27` | 정원 상태 토글 |
- | CLS-01-07-02 / `D04-수업관리/운영정책.md:28` | 건수 배지 |
- | CLS-01-08-02 / `D04-수업관리/운영정책.md:29` | 횟수·페널티 (배지) |
- 등록된 수업 전체를 목록으로 조회하고 상태별로 필터링하는 화면이다. 개별 수업의 출석 현황을 확인하고 수업 정보를 수정하거나 취소 처리할 수 있다. 캘린더 뷰(SCR-C001 / `D04-수업관리/수업관리.md:24`)의 보조 화면으로, 목록 기반의 정밀한 수업 관리가 필요할 때 사용한다. 강습 세션은 KPI 집계를 위해 PT / GX / 골프 / 기타 세션 유형을 함께 표시한다.
- **매니저 일·주간 정산**: 완료 상태 필터로 특정 강사 수업의 출석·노쇼 카운트 확인 → 급여(SCR-064 / `D07-직원관리/직원관리.md:451`) 정산 자료.
- 수업 캘린더와 동일한 탭 구조 (일정표 / 수업 관리 / 횟수 관리 / 페널티 / 유효 수업 목록). 현재 탭 "수업 관리" 활성.
- ### 상태 필터 버튼
- 상태 배지 | 예정·진행중·완료·취소 |
- | CLS-03-04-01 / `D04-수업관리/운영정책.md:35` | 활성 템플릿 드롭다운 |
- 본사 | 전 지점 시간표 통합 등록 | 통합 모드 + 지점 다중 선택 + 일괄 생성 | D10 SCR-H1001 / `D10-본사관리/본사관리.md:894` 수업 운영 자동화 정책이 ON인 지점에 동시 등록 |
- **시작일 > 종료일**: 인라인 에러 + 미리보기 비활성
- **템플릿 미선택**: 미리보기 비활성 + 강사·정원 자동 채움 차단
- **공휴일 데이터 미동기화**: 자동 제외 토글 비활성 + 안내 "공휴일 데이터를 가져올 수 없어요"
- 그룹 수업의 기본 정보(수업명, 유형, 정원, 기본 장소, 색상 등)를 템플릿으로 저장·관리하는 화면이다. 수업 템플릿을 미리 만들어 두면 시간표 일괄 등록(CLS-03 / `D04-수업관리/수업관리.md:192`)이나 개별 수업 등록(CLS-01 / `D04-수업관리/수업관리.md:24`)에서 반복 입력 없이 템플릿을 선택하는 것만으로 수업 정보가 자동 완성된다. 자주 사용되지 않는 템플릿은 비활성화하여 선택 목록에서 숨길 수 있다.
- **계절 수업 활성/비활성**: 여름 시즌 수영 템플릿은 활성, 겨울엔 비활성 → 신규 등록 목록에서 자동 숨김.
- **신규 트레이너 PT 템플릿 검토**: 트레이너는 활성 템플릿 조회만 가능, 등록 시 사용.

## 3. 권한·책임 분리
- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.

## 6. 정책 결정 근거
- 화면별 구현 판단은 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리` 순서로 확인한다.
- 여러 항목 값이 충돌하면 docs2 문서 안에서 하나의 기준값으로 보정해 반영한다.
