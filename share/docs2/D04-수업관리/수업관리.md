## SCR-C001 수업 캘린더

> 유형: 화면
> URL 경로: `/calendar`
> 연결 기능: CLS-01 / `D04-수업관리/수업관리.md:24`
> 연결 다이얼로그: DLG-C001 / `D04-수업관리/수업관리.md:1354` 수업등록수정_캘린더, DLG-C002 / `D04-수업관리/수업관리.md:1438` 일정상세, DLG-C004 / `D04-수업관리/수업관리.md:1607` 일괄변경

### 1. 목적
PT 및 그룹 수업 전체 일정을 캘린더 형태로 한눈에 파악하는 메인 수업 관리 화면입니다. 월·주·일·목록 뷰를 자유롭게 전환하며 수업 유형과 강사를 색상으로 구분해 일정을 직관적으로 확인하고, 직접 클릭하거나 드래그해 빠르게 수업을 등록·수정할 수 있습니다. 강습 세션은 KPI 집계를 위해 PT / GX / 골프 / 기타 중 하나의 세션 유형을 함께 관리합니다.

### 2. UI 구성
- **뷰 전환 탭**: 월·주·일·목록(list) 뷰 전환 버튼. 뷰에 따라 캘린더 표시 밀도가 달라짐
- **상단 액션 버튼**: 스케줄 일괄 변경(DLG-C004 / `D04-수업관리/수업관리.md:1607` 진입), 수업 등록(DLG-C001 / `D04-수업관리/수업관리.md:1354` 진입)
- **탭 내비게이션**: 일정표 / 수업 관리 / 횟수 관리 / 페널티 관리(건수 배지) / 유효 수업 목록 — 탭 클릭 시 각 전용 화면으로 이동
- **필터 바**:
  - 강사 필터: 드롭다운으로 특정 강사 일정만 표시
  - 수업명 필터: 드롭다운으로 특정 수업만 표시
  - 정원 상태 필터: 전체 / 여유 / 마감 버튼 토글
  - 분류 멀티필터: 상담·OT·체성분·방문·수업·PT·기타 복수 선택 가능
  - 강습 세션 유형 필터: PT / GX / 골프 / 기타 복수 선택 가능
- **색상 범례**: 강사별 색상, 수업 유형별 색상, 강습 세션 유형별 색상, 일정 분류별 색상을 화면 하단에 표시
- **캘린더 영역**: 수업 블록 클릭 시 일정 상세(DLG-C002 / `D04-수업관리/수업관리.md:1438`) 팝업, 빈 슬롯 클릭 시 수업 등록(DLG-C001 / `D04-수업관리/수업관리.md:1354`) 팝업
- **우측 패널**: 선택한 날짜의 수업 목록을 카드 형태로 표시. 수업명·강사·시간·예약 현황·정원 바 포함
- **미승인 일정 알림 배너**: 미승인 일정이 있을 때 상단에 경고 배너 표시

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 수업/캘린더 진입 시 이번 달 월 뷰로 기본 표시
2. 뷰 전환 버튼(월/주/일/목록)으로 캘린더 밀도 변경
3. 강사·수업명·분류 필터로 원하는 일정만 표시
4. 수업 블록 클릭 → 일정 상세(DLG-C002 / `D04-수업관리/수업관리.md:1438`)에서 정보 확인 및 수정 진입
5. 빈 시간 슬롯 클릭 또는 상단 "수업 등록" 버튼 → 수업 등록(DLG-C001 / `D04-수업관리/수업관리.md:1354`) 팝업
6. 드래그&드롭으로 수업 시간 이동, 블록 하단 드래그로 시간 연장
7. 우측 패널에서 해당 날짜 수업 목록 카드 형태로 빠르게 확인

## SCR-C002 수업 관리

> 유형: 화면
> URL 경로: `/lessons`
> 연결 기능: CLS-02 / `D04-수업관리/수업관리.md:110`
> 연결 다이얼로그: DLG-C003 / `D04-수업관리/수업관리.md:1523` 수업등록수정_관리, DLG-C005 / `D04-수업관리/수업관리.md:1684` 수업기록상세, DLG-C006 / `D04-수업관리/수업관리.md:1765` 서명

### 1. 목적
등록된 수업 전체를 목록으로 조회하고 상태별로 필터링하는 화면입니다. 개별 수업의 출석 현황을 확인하고 수업 정보를 수정하거나 취소 처리할 수 있습니다. 캘린더 뷰의 보조 화면으로, 목록 기반의 정밀한 수업 관리가 필요할 때 사용합니다. 강습 세션은 KPI 집계를 위해 PT / GX / 골프 / 기타 세션 유형을 함께 표시합니다.
드래그앤드롭 일정 이동, 시간 리사이즈, 캘린더 셀 충돌 검사는 수업 캘린더(SCR-C001 / `D04-수업관리/수업관리.md:24`)의 동작이며, 본 화면은 목록 기반 조회·정정·출석 처리 중심으로 사용합니다.

### 2. UI 구성
- **탭 내비게이션**: 수업 캘린더(SCR-C001 / `D04-수업관리/수업관리.md:24`)와 동일한 탭 구조. 현재 탭은 "수업 관리" 활성
- **필터 바**: 강사 드롭다운, 장소 드롭다운, 강습 세션 유형(PT/GX/골프/기타)으로 목록 좁히기
- **상태 필터 버튼**: 전체 / 예정 / 진행중 / 완료 / 취소 버튼 토글로 수업 상태별 분류
- **수업 목록 테이블**: 수업명, 유형(PT/GX/골프/기타), 강습 세션 유형(PT/GX/골프/기타), 강사, 장소, 날짜·시간, 예약 현황(현재/정원), 상태 배지
- **행 클릭**: 수업 등록/수정 다이얼로그(DLG-C003 / `D04-수업관리/수업관리.md:1523`) 열림
- **출석 현황 컬럼**: 수업별 출석자 수 / 노쇼 수 한눈에 확인
- **수업 등록 버튼**: 상단 우측, 새 수업 등록(DLG-C003 / `D04-수업관리/수업관리.md:1523`) 진입

### 3. 액션 / 흐름
1. 수업 캘린더 탭에서 "수업 관리" 탭 클릭하거나 `/lessons` 직접 진입
2. 강사·장소 필터 및 상태 버튼으로 원하는 수업 좁히기
3. 목록에서 수업 행 클릭 → 수업 등록/수정(DLG-C003 / `D04-수업관리/수업관리.md:1523`) 팝업으로 상세 정보 확인 및 수정
4. 완료된 수업 행에서 수업 기록 상세(DLG-C005 / `D04-수업관리/수업관리.md:1684`) 열어 출석·서명 기록 확인
5. 수업 취소 처리 시 취소 확인 안내 후 상태 변경

| 검색 결과 없음 | 조건에 맞는 수업 없음 안내 |

## SCR-C003 시간표 일괄 등록

> 유형: 화면
> URL 경로: `/class-schedule`
> 연결 기능: CLS-03 / `D04-수업관리/수업관리.md:192`
> 연결 다이얼로그: DLG-C008 / `D04-수업관리/수업관리.md:1925` 일괄생성확인

### 1. 목적
반복되는 수업 일정을 한 번에 대량으로 등록하는 화면입니다. 매주 같은 요일·시간에 열리는 그룹 수업이나 PT 일정을 개별 등록하는 번거로움 없이, 기간·요일·시간·강사·장소를 설정하면 지정된 기간 동안의 일정이 자동으로 생성됩니다.
수업 유형별 색상 체계와 색상 범례는 수업 캘린더(SCR-C001 / `D04-수업관리/수업관리.md:24`)와 그룹수업 템플릿(SCR-C004 / `D04-수업관리/수업관리.md:276`)의 표시 규칙이며, 본 화면은 반복 일정 생성에 한정합니다.

### 2. UI 구성
- **기간 설정**: 시작일 ~ 종료일 날짜 선택 (달력 피커)
- **요일 선택**: 월~일 중 복수 선택 가능한 토글 버튼
- **시간 설정**: 시작 시간 ~ 종료 시간 (30분 단위 드롭다운)
- **수업 선택**: 등록된 수업 템플릿 드롭다운 (수업명, 유형, 정원 자동 연동)
- **강사 배정**: 담당 강사 드롭다운 선택
- **장소 설정**: 룸·장소 드롭다운 선택
- **정원 입력**: 최대 예약 인원 수 입력 (템플릿 기본값 자동 채워짐)
- **예외 날짜 제외**: 공휴일·휴관일 등 특정 날짜를 제외 처리
- **미리보기 영역**: 생성될 일정 건수와 날짜 목록을 등록 전에 미리 확인
- **일괄 생성 버튼**: 클릭 시 최종 확인 다이얼로그(DLG-C008 / `D04-수업관리/수업관리.md:1925`) 표시

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 시간표 일괄 등록 진입
2. 적용 기간(시작일~종료일) 선택
3. 수업할 요일 복수 선택
4. 시작·종료 시간 설정
5. 수업 템플릿·강사·장소 선택
6. 제외할 날짜(공휴일 등) 추가
7. 미리보기에서 생성될 일정 건수·날짜 확인
8. "일괄 생성" 버튼 클릭 → 최종 확인 다이얼로그(DLG-C008 / `D04-수업관리/수업관리.md:1925`) → 확인 후 일정 생성

## SCR-C004 그룹 수업 템플릿

> 유형: 화면
> URL 경로: `/class-templates`
> 연결 기능: CLS-04 / `D04-수업관리/수업관리.md:276`
> 연결 다이얼로그: DLG-C009 / `D04-수업관리/수업관리.md:2008` 템플릿등록수정

### 1. 목적
그룹 수업의 기본 정보(수업명, 유형, 정원, 기본 장소, 색상 등)를 템플릿으로 저장·관리하는 화면입니다. 수업 템플릿을 미리 만들어 두면 시간표 일괄 등록이나 개별 수업 등록 시 반복 입력 없이 템플릿을 선택하는 것만으로 수업 정보가 자동 완성됩니다.

### 2. UI 구성
- **템플릿 목록**: 수업명, 유형(GX/기구필라테스 등), 정원, 기본 장소, 활성 여부를 카드 또는 테이블로 표시
- **활성/비활성 토글**: 더 이상 사용하지 않는 템플릿을 비활성화하여 선택 목록에서 숨김
- **템플릿 등록 버튼**: 상단 우측, 새 템플릿 등록(DLG-C009 / `D04-수업관리/수업관리.md:2008`) 진입
- **템플릿 수정**: 목록 행 클릭 또는 편집 아이콘 → DLG-C009 / `D04-수업관리/수업관리.md:2008` 수정 모드
- **템플릿 삭제**: 삭제 버튼 → 삭제 확인 후 처리. 이미 사용 중인 수업 일정이 있으면 삭제 불가 안내
- **색상 칩**: 각 템플릿에 지정된 수업 색상을 시각적으로 표시

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 그룹 수업 템플릿 진입
2. 기존 템플릿 목록 확인 및 활성 여부 필터링
3. "템플릿 등록" 버튼 클릭 → DLG-C009에서 수업명·유형·정원·장소·색상 입력 후 저장
4. 기존 템플릿 행 클릭 → DLG-C009 / `D04-수업관리/수업관리.md:2008` 수정 모드에서 정보 변경
5. 사용하지 않는 템플릿 비활성화 토글 → 수업 등록 시 선택 목록에서 숨겨짐
6. 삭제 시 사용 중인 일정 없는 경우에만 삭제 가능

## SCR-C005 그룹 수업 현황

> 유형: 화면
> URL 경로: `/class-stats`
> 연결 기능: CLS-05 / `D04-수업관리/수업관리.md:349`

### 1. 목적
그룹 수업별 예약 현황과 출석률, 잔여 자리를 한눈에 파악하는 현황판 화면입니다. 어떤 수업이 인기 있고, 어느 수업에 빈자리가 있는지 빠르게 파악해 운영 결정(수업 추가·취소·정원 조정)에 활용합니다.
운영 액션은 정원 조정과 수업 취소 진입으로 한정합니다. 정원 확대 후 대기자 알림/배정은 D10 SCR-H1001의 대기열 자동 알림/배정 정책이 ON인 경우의 후속 처리로 분리합니다.

### 2. UI 구성
- **기간 필터**: 조회 기간(주간/월간) 선택
- **수업 유형 필터**: 전체·PT·GX·골프·기타 유형별 필터
- **강사 필터**: 특정 강사 수업만 표시
- **현황 카드 또는 테이블**:
  - 수업명, 유형, 강사, 요일·시간, 정원, 예약 수, 잔여 자리
  - 출석률 (출석 / 예약 비율, %)
  - 노쇼 수
  - 잔여 자리 바 시각화 (여유: 초록, 마감 임박: 주황, 마감: 빨강)
- **정렬 기능**: 출석률 높은 순, 잔여 자리 적은 순, 예약 많은 순 정렬
- **수업 행 클릭**: 해당 수업의 예약 회원 목록 조회 (드로어 또는 서브 패널)

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 그룹 수업 현황 진입
2. 기간(주간/월간) 및 유형·강사 필터 설정
3. 현황 카드에서 수업별 예약·출석·잔여 자리 확인
4. 잔여 자리가 적은 수업 → 정원 확대 또는 동일 수업 추가 여부 판단
5. 출석률이 낮은 수업 → 수업 취소 또는 일정 조정 검토
6. 수업 행 클릭 → 해당 수업 예약 회원 명단 확인

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 현황 | 이번 주 전체 그룹 수업 현황 표시 |
| 마감 수업 강조 | 잔여 자리 0인 수업 빨간색 배지 표시 |
| 데이터 없음 | 해당 기간 그룹 수업 없음 안내 |
| 로딩 중 | 스켈레톤 카드 표시 |

| 트레이너 | 본인 담당 수업 현황 | 조회만 가능 |

## SCR-C006 강사 근무 현황

> 유형: 화면
> URL 경로: `/instructor-status`

> 연결 다이얼로그: DLG-C010 / `D04-수업관리/수업관리.md:2084` 강사상세

### 1. 목적
강사별 수업 배정 현황과 근무 시간을 요약해 파악하는 화면입니다. 특정 기간 동안 각 강사가 몇 건의 수업을 담당하고 총 몇 시간을 근무했는지 확인하고, 업무 부담이 편중된 강사가 없는지 점검하는 데 사용합니다.

### 2. UI 구성
- **기간 필터**: 주간 / 월간 / 사용자 지정 기간 선택
- **강사 목록 카드 또는 테이블**:
  - 강사명, 역할(트레이너/GX 강사 등)
  - 담당 수업 건수
  - 총 근무 시간 (수업 시간 합산)
  - 수업 유형별 분포 (PT/GX/골프/기타)
  - 강습 세션 유형별 분포 (PT/GX/골프/기타)
  - 출석 처리 완료율
  - O.T 배정 합계 / O.T 1 완료 / O.T 2 완료 / O.T 1·2 예정
- **강사 행 클릭**: 강사 상세 팝업(DLG-C010 / `D04-수업관리/수업관리.md:2084`)에서 담당 수업 목록 및 기본 정보 확인
- **강사별 색상 구분**: 캘린더와 동일한 강사 색상 코드 적용
- **요약 통계**: 평균 근무 시간, 최다 수업 강사, 수업 없는 강사 등 상단 요약 지표

6. 업무 편중 강사 파악 후 수업 재배정 검토 (캘린더 화면에서 수정)

### 1. 목적

### 2. UI 구성

## SCR-C009 일정 요청 처리

> 유형: 화면

### 1. 목적
회원이 앱에서 요청한 수업 변경·취소 요청을 관리자가 검토하고 수락 또는 거절하는 화면입니다. 회원이 직접 수업을 취소하거나 다른 일정으로 변경하고 싶을 때 요청을 보내면, Owner(지점장) / 매니저가 이 화면에서 일괄 처리합니다.

### 2. UI 구성
- **요청 목록 테이블**:
  - 요청 유형 (변경 / 취소 배지. 변경=파랑, 취소=주황)
  - 회원명, 연락처
  - 원래 수업 정보 (수업명, 일시, 강사)
  - 요청 내용 (변경 시: 희망 변경 일시, 취소 시: 취소 사유)
  - 요청 일시
  - 처리 상태 (대기 / 수락 / 거절 / 대안제시 / 응답대기 / 만료 배지)
- **요청 유형 필터**: 전체 / 변경 요청 / 취소 요청
- **처리 상태 필터**: 전체 / 대기 / 처리 완료
- **수락 버튼**: 요청 수락 → 수업 일정 자동 변경 또는 취소 처리
- **거절 버튼**: 거절 사유 입력 후 회원에게 안내
- **대안 일정 제시 버튼**: 취소 요청 시 대안 일정 제안(DLG-C016 / `D04-수업관리/수업관리.md:2558`) 팝업
- **미처리 건수 배지**: 화면 상단에 처리 상태가 `대기`, `응답대기`인 요청 수를 표시합니다. 24시간을 초과한 미처리 요청은 별도 `SLA 초과` 배지로 함께 표시합니다.
- **Today Tasks 연계**: SCR-098 / `D10-본사관리/본사관리.md:745` Today Tasks(오늘의 할 일)의 `일정 요청 미처리 N` 태스크에서 진입할 수 있습니다. 클릭 진입 시 `/schedule-requests?status=pending`으로 이동하고 기본 필터는 `대기/응답대기/24시간 초과`로 적용합니다.

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 일정 요청 처리 진입

## SCR-C010 운동 프로그램 관리

> 유형: 화면
> URL 경로: `/exercise-programs`
> 연결 기능: CLS-10 / `D04-수업관리/수업관리.md:771`
> 연결 다이얼로그: DLG-C011 / `D04-수업관리/수업관리.md:2170` 세션상세

### 1. 목적
트레이너가 회원에게 제공할 운동 프로그램을 수동으로 만들고 관리하는 화면입니다. 프로그램은 여러 운동 동작으로 구성되며, 트레이너가 직접 회원에게 배정하면 PT 세션에서 해당 프로그램을 기반으로 수업이 진행됩니다. v1에서는 프로그램 자동 추천·자동 생성 없이 수동 등록·수정·삭제·선택·배정만 제공합니다.

### 2. UI 구성
- **프로그램 목록**:
  - 프로그램명, 담당 트레이너, 배정 회원 수, 동작 수, 생성일, 마지막 수정일
  - 활성/비활성 상태 배지
- **프로그램 등록 버튼**: 새 운동 프로그램 생성 (드로어 또는 별도 등록 폼)
- **프로그램 상세 / 수정**: 행 클릭 시 동작 구성 목록 편집 가능
  - 동작명, 세트, 반복 횟수, 휴식 시간, 중량, 메모 항목 구성
  - 동작 순서 드래그&드롭 변경
- **회원 배정 기능**: 트레이너가 등록된 프로그램 중 하나를 직접 선택한 뒤 배정 회원 추가·제거
- **필터**: 내 프로그램만 보기, 배정 회원 있는 프로그램만 보기
- **프로그램 선택 / 배정**:
  - v1 범위: 트레이너가 프로그램을 수동 등록·수정·삭제하고, 등록된 프로그램 중 하나를 직접 선택해 회원에게 배정
  - v1 제외: 회원 목표·체성분·운동 이력을 분석해 프로그램을 자동 생성하거나 추천하는 로직
  - 자동 추천·자동 생성 버튼은 v1 화면에 노출하지 않으며, 데이터 기준과 추천 산식 확정 후 v2 또는 별도 협의 범위로 처리

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 운동 프로그램 관리 진입
2. 기존 프로그램 목록 확인, 필터로 내 프로그램만 표시
3. "프로그램 등록" 버튼 클릭 → 프로그램명 입력 후 동작 목록 구성
4. 각 동작에 세트·횟수·중량·메모 입력
5. 프로그램 저장 후 "회원 배정" 기능으로 대상 회원 선택
6. 수정이 필요한 경우 프로그램 행 클릭 → 편집 모드로 진입

### 회원 배정 / 프로그램 선택 흐름

1. 회원 상세 또는 프로그램 목록에서 "회원 배정"을 선택한다.
2. 트레이너가 배정할 프로그램을 직접 선택한다.
3. 회원 목표(다이어트/근력/재활 등)는 검색·필터 참고 정보로만 사용한다.
4. 저장하면 해당 회원의 다음 PT 세션에서 배정 프로그램이 노출된다.
5. 자동 추천·자동 생성 버튼은 v1 화면에 노출하지 않는다.

### 1. 목적
이용권이 유효하고 예약이 확정된 수업 중 실제로 진행 가능한 수업 목록을 보여주는 화면입니다. 만료되거나 취소된 수업을 제외한 "진행 예정 확정 수업"만 필터링해 출석 처리, 서명 진행, 당일 운영 확인에 활용합니다.

### 2. UI 구성
- **탭 내비게이션**: 수업 캘린더와 동일한 탭 구조. 현재 탭은 "유효 수업 목록" 활성
- **날짜 필터**: 오늘 / 이번 주 / 날짜 직접 선택
- **수업 목록 테이블**:
  - 수업명, 수업 유형(PT/GX/골프/기타), 강습 세션 유형(PT/GX/골프/기타), 강사명
  - 일시 (날짜, 시작~종료 시간)
  - 장소
  - 예약 회원명, 연락처
  - 이용권 잔여 횟수
  - 출석 상태 (미처리 / 출석 / 결석 / 노쇼)
- **출석 처리 버튼**: 각 행에서 바로 출석·결석·노쇼 처리 가능
- **서명 요청 버튼**: PT 수업 완료 후 회원 서명 받기(DLG-C006 / `D04-수업관리/수업관리.md:1765`) 진입
- **수업 기록 상세 버튼**: 완료 수업의 기록 확인(DLG-C005 / `D04-수업관리/수업관리.md:1684`) 진입

### 3. 액션 / 흐름
1. 수업 캘린더 탭 > "유효 수업 목록" 탭 클릭 또는 `/valid-lessons` 직접 진입
2. 날짜 필터로 오늘 또는 특정 날짜 수업 표시
3. 출석 처리가 필요한 수업 행에서 출석 상태 선택 (출석/결석/노쇼)
4. PT 수업 완료 후 "서명 요청" 버튼 → DLG-C006에서 회원 서명 받기
5. 노쇼 자동 페널티 적용 토글이 ON이면 페널티를 자동 부과하고, OFF이면 노쇼 기록만 저장
6. 완료 수업의 기록 확인 → DLG-C005 / `D04-수업관리/수업관리.md:1684` 세션 기록 조회

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 오늘 수업 | 오늘 날짜 기준 유효 수업 목록 기본 표시 |
| 출석 미처리 강조 | 수업 시작 시간 경과 후 미처리 항목 주황 표시 |
| 목록 없음 | 해당 날짜 유효 수업 없음 안내 |
| 전체 처리 완료 | 모든 수업 출석 처리 완료 시 완료 안내 |

### 5. 권한
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| Owner(지점장) / 매니저 | 전체 유효 수업 목록 | 출석 처리, 서명 요청 |
| 트레이너 | 본인 담당 유효 수업 목록 | 출석 처리, 서명 요청 |
| FC / 스태프 | 전체 유효 수업 목록 | 출석 처리 보조 가능. 서명 요청 Push 발송, 수업 정보 수정, 수업 취소는 불가 |

### 6. 필수 예외처리
| 케이스 | 표시 | 후속 조치 |
|---|---|---|
| 만료 또는 잔여 0 회원 출석 처리 시도 | 토스트 "유효한 잔여 횟수가 없어 출석 처리할 수 없습니다" | 출석 처리 차단 |

## SCR-C012 대기열 관리

> 유형: 화면
> URL 경로: `/class-waitlist`
> 연결 기능: CLS-12 / `D04-수업관리/수업관리.md:941`

### 1. 목적
만석이 된 그룹 수업에 대해 대기 신청한 회원 목록을 관리하는 화면입니다. 예약 취소 등으로 자리가 생기면 D10 SCR-H1001 / `D10-본사관리/본사관리.md:894` 대기열 자동 알림/배정 정책이 ON인 경우에만 대기 순번에 따라 자동 예약을 배정하고, 정책이 OFF이면 관리자가 수동으로 배정합니다. 인기 수업의 공정한 자리 배분을 위해 사용합니다.

### 2. UI 구성
- **수업 선택 드롭다운**: 대기열을 조회할 수업 선택
- **대기 신청자 목록**:
  - 대기 순번, 회원명, 연락처, 대기 신청 일시
  - 이용권 유효 여부 표시
  - 자동 배정 여부 (자동 배정 동의 / 알림만)
- **자동 배정 정책 상태**: D10 SCR-H1001 / `D10-본사관리/본사관리.md:894` 대기열 자동 알림/배정 정책 ON/OFF를 읽기 전용 배지로 표시합니다. 본 화면에서는 토글을 변경하지 않습니다.
- **수동 배정 버튼**: 특정 대기자를 관리자가 직접 예약 확정 처리
- **대기 취소 버튼**: 대기 신청 취소 처리
- **알림 발송 버튼**: 자리 생겼음을 대기자에게 수동으로 알림 발송

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 대기열 관리 진입
2. 대기열 확인할 수업 선택
3. 대기자 목록 및 순번 확인
4. 자리 발생 시 D10 SCR-H1001 / `D10-본사관리/본사관리.md:894` 대기열 자동 알림/배정 정책이 ON이면 자동 배정 동의 상태인 순번 1번에게 자동 예약 처리
5. D10 정책이 OFF이거나 대기자가 자동 배정에 동의하지 않았으면 관리자가 "수동 배정" 버튼으로 직접 처리
6. 배정 후 해당 회원에게 예약 확정 알림 자동 발송

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 대기자 없음 | 해당 수업 대기 신청자 없음 안내 |
| 대기자 있음 | 순번별 대기자 목록 표시 |
| 자동 배정 정책 ON | 자리 발생 시 D10 정책 기준으로 자동 처리 안내 |
| 자동 배정 정책 OFF | 자동 배정이 실행되지 않으며 수동 배정 버튼을 강조 |
| 배정 처리 중 | 배정 처리 중 로딩 표시 |

### 5. 권한
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| Owner(지점장) / 매니저 | 전체 수업 대기열 목록 | 수동 배정, 대기 취소, D10 자동 배정 정책 상태 조회 |
| 트레이너 | 담당 수업 대기열 조회 | 조회만 가능 |
| FC / 스태프 | 전체 수업 대기열 조회 | 수동 배정, 알림 발송 |

### 6. 필수 예외처리
| 케이스 | 표시 | 후속 조치 |
|---|---|---|
| 자동 배정 vs 미동의 1번 | 알림만, 자동 미수행 | 다음 순번 시도 |
| 자동 배정 vs 만료 회원 | 자동 스킵 + 안내 | 다음 순번 |
| 자동 배정 vs 페널티 회원 | 자동 스킵 + 안내 | 다음 순번 |
| 응답 만료 (24시간) | 자동 다음 순번 | 자동 |
| 시작 임박 자리 발생 | 정책 따라 차단/즉시 | - |
| 순번 변경 사유 미입력 | 차단 | - |
| 동시 순번 변경 충돌 | 낙관적 락 | 재로드 |
| 회원 탈퇴 | 자동 취소 + 재정렬 | 자동 |
| 권한 부족 (트레이너) | 액션 hidden | 조회만 |
| 이용권 만료 회원 수동 배정 시도 | 차단 + 안내 | 횟수 충전 안내 |
| 페널티 회원 수동 배정 시도 | 차단 + 안내 | 페널티 해제 후 가능 |
| 본사 통합 모드 권한 부족 | 지점 컬럼 hidden | - |
| 회원앱 동기화 실패 | retry 3회 | 운영자 수동 처리 |

### 7. 연동 / 자동화 참조
| 연계 대상 | 트리거 | 영향 |
|---|---|---|
| 응답 만료 잡 | 매분 | 24시간 초과 → 다음 순번 |
| NFR-19 / `_공통/공통참조_코드인덱스.md:33` 알림 | 배정·자리 발생 | 회원 알림 |
| 회원앱 동기화 | 신청·취소·배정 | 회원앱 즉시 반영 |
| 인기 수업 추가 개설 알림 잡 | 매일 04:00 | 대기자 {인원수}명 이상 → Owner(지점장)+ 알림 |
| 데이터 정리 잡 | 90일 경과 | 오래된 이력 정리 |

### 8. 핵심 디자인 요청사항
- 공통 색상, 간격, 폰트, 토스트 규칙은 `_공통/디자인_시스템.md`, `_공통/토스트_메시지.md`를 따른다.
- 본 섹션에 별도 명시된 UI 상태와 권한 차이가 있으면 도메인 공통보다 화면 정의를 우선한다.

## SCR-C013 수업 평가 피드백

> 유형: 화면
> URL 경로: `/class-feedback`
> 연결 기능: CLS-13 / `D04-수업관리/수업관리.md:1017`

### 1. 목적
수업 완료 후 회원이 남긴 평가·후기를 관리자가 조회하는 화면입니다. 강사별·수업별 평균 평점과 텍스트 후기를 확인해 서비스 품질을 개선하고, 우수 강사를 파악하거나 불만 사항에 대응하는 데 활용합니다.

### 2. UI 구성
- **기간 필터**: 조회 기간 선택 (주간/월간/사용자 지정)
- **강사 필터**: 특정 강사 피드백만 표시
- **수업 유형 필터**: PT / GX / 골프 / 기타
- **평점 요약 카드**: 전체 평균 평점, 강사별 평균 평점 랭킹
- **피드백 목록 테이블**:
  - 회원명, 수업명, 강사명, 수업 일시
  - 평점 (별점 1~5)
  - 후기 텍스트
  - 피드백 등록 일시
- **피드백 상세**: 행 클릭 시 후기 전문 표시
- **평점 분포 차트**: 1~5점 분포 막대 그래프 (선택)

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 수업 평가 피드백 진입
2. 기간 및 강사·유형 필터 설정
3. 평점 요약 카드에서 강사별 평균 평점 비교
4. 피드백 목록에서 낮은 평점 후기 우선 확인
5. 행 클릭으로 후기 전문 열람
6. 불만 사항 발견 시 해당 강사·수업 개선 검토

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 피드백 없음 | 해당 기간 피드백 없음 안내 |
| 피드백 있음 | 목록 및 평점 요약 카드 표시 |
| 낮은 평점 강조 | 3점 이하 후기 주황/빨강 강조 표시 |
| 로딩 중 | 스켈레톤 표시 |

### 5. 권한
| 계정 유형 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|
| Owner(지점장) / 매니저 | 전체 강사·수업 피드백, 평균 평점 | 조회만 가능 |
| 트레이너 | 본인 수업 피드백만 | 본인 피드백 조회만 가능 |
| FC / 스태프 | 접근 불가 | 이 기능 사용 불가 |

### 6. 필수 예외처리
| 케이스 | 표시 | 후속 조치 |
|---|---|---|
| 후기 없음 | "피드백이 없어요" | - |
| 익명 후기 회원명 마스킹 | *** 표시 | 신고/삭제 시에도 유지 |

## SCR-C014 수업 출석/완료 확인

> 유형: 화면
> URL 경로: `/attendance/lesson-completion`
> 연결 기능: CLS-14 / `D04-수업관리/수업관리.md:1096`

### 1. 목적
수업별 예약 회원의 출석, 수업 완료, 서명/확인 누락, 노쇼 상태를 확인하고 처리하는 화면입니다.

본 화면은 수업용 QR 인증 화면이 아닙니다. QR은 회원의 지점 입/출입 확인에 사용하는 수단이며, 수업 출석이나 수업 완료 인증에는 사용하지 않습니다. 입/출입 QR 발급·갱신·인증은 D11 통합운영 및 D09 설정관리 범위에서 관리하고, 본 화면에서는 분쟁 대응 참고값으로만 조회합니다.

### 2. UI 구성
- **수업 선택 영역**: 오늘 또는 선택 날짜의 진행 예정/진행중/종료 수업 목록
- **출석/완료 현황 패널**:
  - 예약 인원, 출석 처리 인원, 미처리 인원, 서명/확인 누락 인원, 노쇼 인원
  - 회원별 출석 상태, 처리 시각, 처리자
- **수동 처리 버튼**: 회원별 출석, 결석, 노쇼, 완료 상태 처리
- **서명 요청 Push 버튼**: 수업 종료 후 서명/확인이 누락된 회원에게 회원앱 Push 발송
- **입/출입 기록 참고 영역**: 회원의 지점 입/출입 기록이 있는 경우 분쟁 대응 참고값으로 표시

### 3. 액션 / 흐름
1. 수업 출석/완료 확인 화면 진입
2. 날짜·강사·수업 상태로 처리 대상 수업 필터링
3. 수업 선택 → 예약 회원별 출석/미처리/서명 누락/노쇼 상태 확인
4. 트레이너 또는 권한자가 회원별 출석 상태 처리
5. 수업 종료 후 서명/확인이 누락된 경우 트레이너가 회원에게 서명 요청 Push 발송
6. 회원이 회원앱에서 수업 정보를 확인하고 [확인] 클릭 → 수업 완료 처리
7. 출석 인정 마감 + 지점 노쇼 유예 N분 설정값이 있으면 수업 종료 후 미출석 회원을 자동 노쇼 처리하고, N분 미설정 시 수동 처리 대상으로 유지

### 4. 핵심 상태
| 상태 | 설명 |

## SCR-C015 수업 녹화 관리

> 유형: 화면
> URL 경로: `/class-recording`

> 연결 다이얼로그: DLG-C005 / `D04-수업관리/수업관리.md:1684` 수업기록상세

### 1. 목적
완료된 수업의 녹화 파일을 업로드하고 관리하는 화면입니다. 트레이너가 녹화한 수업 영상을 업로드하면, 담당 회원에게 공유해 복습 또는 피드백 자료로 활용할 수 있습니다. 영상 공개 범위를 설정해 해당 회원만 볼 수 있도록 접근을 제어합니다.

### 2. UI 구성
- **녹화 파일 목록**:
  - 수업명, 강사명, 수업 일시, 파일 이름, 파일 크기, 업로드 일시
  - 공유 상태 배지 (공유 중 / 비공개)
  - 공유 대상 회원 수
- **파일 업로드**: 드래그&드롭 또는 파일 선택으로 영상 파일 업로드
  - 업로드 시 연결할 수업 선택 (완료 수업 목록에서 선택)
- **공유 설정**: 업로드한 녹화 파일을 특정 회원에게만 공개 설정
  - 공개 기간 설정 (무기한 / 특정 날짜까지)
- **파일 삭제**: 녹화 파일 삭제 (삭제 전 확인 안내)
- **미리보기 / 다운로드**: 파일 재생 또는 다운로드 버튼

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 수업 녹화 관리 진입
2. "파일 업로드" 버튼 클릭 또는 드래그&드롭으로 영상 파일 선택
3. 연결할 완료 수업 선택 후 업로드 시작
4. 업로드 완료 후 공유 설정에서 대상 회원 선택 및 공개 기간 설정
5. 공유 상태를 "공유 중"으로 변경 → 해당 회원 앱에서 영상 열람 가능
6. 공개 기간 만료 시 자동으로 비공개 전환

전체 수업 예약을 예약 1건 단위로 조회·검색·필터링하는 예약 원장 화면입니다. 관리자는 회원 문의, 예약 취소 확인, 출석·노쇼 처리 현황을 한 화면에서 확인하고 필요한 후속 처리를 수행합니다.

### 2. UI 구성
- **요약 카드**: 전체 예약, 예약완료, 출석완료, 취소·노쇼, 대기 건수
- **필터 바**:
  - 기간: 수업일 기준 기본, 예약일 기준 전환 가능
  - 수업: 수업명 또는 수업 유형 선택
  - 강사: 담당 강사 선택
  - 상태: 전체 / 예약완료 / 출석완료 / 취소 / 노쇼 / 대기
  - 지점: 전체 지점(통합) 권한일 때만 노출
- **검색 입력**: 회원명 또는 연락처 검색. 연락처는 하이픈 제거 후 비교한다.
- **예약 목록 테이블**:
  - 예약일시
  - 수업일시
  - 수업명
  - 수업 유형
  - 강습 세션 유형
  - 강사
  - 회원명
  - 연락처
  - 지점
  - 상태
  - 취소 사유
  - 처리자 / 처리일시
- **행 액션**: 회원 상세 보기, 수업 상세 보기, 예약 취소, 출석 처리, 노쇼 처리
- **다운로드**: 현재 필터·검색 조건을 적용한 엑셀 다운로드

### 3. 액션 / 흐름
1. 사이드바 > 수업관리 > 예약 목록 진입
2. 기본값으로 오늘부터 7일간의 수업일 기준 예약을 표시
3. 기간, 수업, 강사, 상태 필터를 조합해 목록을 좁힘
4. 회원명 또는 연락처로 특정 회원 예약을 검색
5. 예약 행에서 회원 상세 또는 수업 상세로 이동
6. 권한이 있는 사용자는 예약 취소, 출석 처리, 노쇼 처리를 수행
7. 상태 변경 결과는 목록에 즉시 반영되고 히스토리 로그에 기록

## DLG-C001 수업 등록/수정 (캘린더)

> 유형: 다이얼로그
> 연결 기능: CLS-01 / `D04-수업관리/수업관리.md:24`
> 연결 다이얼로그: DLG-C001 / `D04-수업관리/수업관리.md:1354` 수업등록수정_캘린더, DLG-C002 / `D04-수업관리/수업관리.md:1438` 일정상세, DLG-C004 / `D04-수업관리/수업관리.md:1607` 일괄변경

### 1. 목적
캘린더 화면에서 새 수업 일정을 등록하거나 기존 일정을 수정하는 팝업입니다. 빈 시간 슬롯 클릭 또는 상단 "수업 등록" 버튼으로 진입하며, 수업 기본 정보와 일시·강사·장소를 한 화면에서 입력할 수 있습니다.

### 2. UI 구성
- **수업 선택**: 등록된 수업 템플릿 드롭다운 (수업명, 유형, 정원 자동 완성)
- **일정 분류**: 상담 / OT / 체성분 / 방문 / 수업 / PT / 기타 선택
- **강습 세션 유형**: PT / GX / 골프 / 기타 선택. 강습세션수 KPI 집계 대상 수업일 때 필수
- **대상 유형**: 회원 / 비회원 / 직원 선택 (비회원 선택 시 이름·연락처 직접 입력)
- **시작 일시**: 날짜 + 시간 선택 (30분 단위)
- **종료 일시**: 시작 시간 기준 수업 시간만큼 자동 계산, 직접 수정 가능
- **강사 배정**: 담당 강사 드롭다운
- **장소**: 룸·장소 드롭다운
- **정원**: 최대 예약 인원 수 입력
- **색상**: 캘린더에 표시될 수업 블록 색상 선택
- **메모**: 내부 메모 입력 (선택)
- **반복 일정**: 반복 여부 토글 — 반복 시 요일·종료일 추가 설정
- **저장 버튼** / **취소 버튼**

### 3. 액션 / 흐름
1. 빈 슬롯 클릭 → 팝업 열림, 클릭한 날짜·시간 자동 입력
2. 수업 템플릿 선택 → 수업명·정원·시간 자동 완성
3. 강사·장소·분류 선택
4. 반복 여부 설정 (반복 시 요일·종료일 추가 입력)
5. 저장 클릭 → 시간 충돌 감지 후 이상 없으면 캘린더에 즉시 반영
6. 충돌 감지 시 경고 표시 → 일시 수정 후 재저장 또는 강제 저장 선택

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |

캘린더에서 수업 블록을 클릭했을 때 해당 수업의 상세 정보를 빠르게 확인하는 팝업입니다. 수업 기본 정보와 예약 현황을 한눈에 보여주며, 수정·취소·출석 처리 등 후속 액션으로 바로 진입할 수 있습니다.

### 2. UI 구성
- **수업명 및 유형 배지**: 수업명, PT/GX/골프/기타 유형 색상 배지
- **강습 세션 유형 배지**: 강습세션수 KPI 대상 수업인 경우 PT / GX / 골프 / 기타 중 하나를 표시
- **일시 정보**: 날짜, 시작~종료 시간
- **강사명**
- **장소 / 룸**
- **예약 현황**: 현재 예약 인원 / 정원, 잔여 자리 수
- **예약 회원 목록**: 예약한 회원 이름 목록 (간략히 표시)
- **출석 상태 요약**: 출석 / 결석 / 노쇼 / 미처리 건수
- **분류**: 일정 분류 (상담·OT·수업·PT 등)
- **승인 상태**: 미승인 일정인 경우 승인/거절 버튼 표시
- **액션 버튼**:
  - 수정 → DLG-C001 / `D04-수업관리/수업관리.md:1354` 수정 모드 진입
  - 취소 → 수업 취소 확인 후 처리

### 3. 액션 / 흐름
1. 캘린더 수업 블록 클릭 → 팝업 열림
2. 수업 정보 및 예약 현황 확인
3. 수정이 필요한 경우 "수정" 버튼 → DLG-C001 / `D04-수업관리/수업관리.md:1354` 수정 모드
4. 미승인 일정인 경우 "승인" 또는 "거절" 버튼으로 바로 처리
5. 수업 취소 필요 시 "취소" 버튼 → 확인 후 상태 변경
6. 팝업 외부 클릭 또는 닫기 버튼으로 팝업 닫힘

수업 관리 화면(SCR-C002 / `D04-수업관리/수업관리.md:110`)에서 새 수업을 등록하거나 기존 수업 정보를 수정하는 팝업입니다. 캘린더 기반의 DLG-C001과 동일한 입력 항목을 제공하되, 목록 화면의 맥락에서 호출됩니다.

### 2. UI 구성
- **수업 템플릿 선택**: 드롭다운으로 기존 수업 선택 (수업명·유형·정원 자동 완성)
- **수업명**: 직접 입력 가능 (템플릿 미선택 시)
- **수업 유형**: PT / GX / 골프 / 기타 선택
- **강습 세션 유형**: PT / GX / 골프 / 기타 선택. 강습세션수 KPI 집계 대상 수업일 때 필수
- **일시**: 날짜 + 시작·종료 시간 선택
- **강사 배정**: 담당 강사 드롭다운
- **장소**: 룸·장소 드롭다운
- **정원**: 최대 예약 인원 입력
- **상태**: 예정 / 진행중 / 완료 / 취소 (수정 모드에서만 표시)
- **메모**: 내부 메모 입력 (선택)
- **저장 버튼** / **취소 버튼**

### 3. 액션 / 흐름
1. "수업 등록" 클릭 → 빈 폼으로 팝업 열림
2. 수업 템플릿 선택 또는 직접 입력으로 수업 정보 채우기
3. 강사·장소·일시 설정
4. 저장 클릭 → 시간 충돌 감지 후 수업 목록에 즉시 반영
5. 수정 모드: 기존 정보 자동 입력 → 변경 후 저장

### 1. 목적
반복 수업 일정을 수정할 때 변경 범위를 선택하는 팝업입니다. 반복 수업의 하나를 수정하면 "이 일정만 변경"할지, "이 일정 이후 모든 일정을 변경"할지, "전체 반복 일정을 변경"할지 선택하게 합니다. 의도하지 않은 대량 수정을 방지하는 안전장치 역할을 합니다.

### 2. UI 구성
- **안내 메시지**: "반복 수업을 수정합니다. 변경 범위를 선택하세요."
- **변경 범위 선택 라디오 버튼**:
  - 이 일정만 변경 (현재 클릭한 1건만)
  - 이 일정 이후 변경 (현재 날짜 포함 이후 모든 반복 일정)
  - 전체 반복 일정 변경 (처음부터 끝까지 모든 반복 일정)
- **확인 버튼**: 선택한 범위로 수업 등록/수정 팝업(DLG-C001 / `D04-수업관리/수업관리.md:1354` 또는 DLG-C003 / `D04-수업관리/수업관리.md:1523`) 진행
- **취소 버튼**: 팝업 닫기, 변경 없음

### 3. 액션 / 흐름
1. 반복 수업 블록에서 수정 클릭 → 일괄 변경 팝업 먼저 표시
2. 변경 범위 선택 (이 일정만 / 이후 / 전체)
3. 확인 클릭 → 선택 범위에 따라 수업 수정 팝업(DLG-C001 / `D04-수업관리/수업관리.md:1354` / C003) 열림
4. 수정 완료 후 선택 범위만큼 일정 일괄 업데이트
5. 취소 클릭 → 변경 없이 팝업 닫힘

### 1. 목적
완료된 수업의 출석 기록, 회원 서명, 운동 내용 등 수업 결과 전체를 상세 조회하는 팝업입니다. 수업이 실제로 이루어진 근거 자료로 활용되며, 분쟁 발생 시 출석·서명 이력을 확인하는 데 사용합니다.

### 2. UI 구성
- **수업 기본 정보**: 수업명, 일시, 강사명, 장소, 수업 유형, 강습 세션 유형
- **출석 현황 목록**:
  - 예약 회원 목록과 각 회원의 출석 상태 (출석 / 결석 / 노쇼)
  - 출석 처리 일시
- **서명 기록**: PT 수업의 경우 회원 서명 이미지 및 서명 일시 표시

- **강사 메모**: 수업 중 특이사항, 피드백 메모
- **횟수 차감 내역**: 해당 수업으로 차감된 이용권 횟수 정보
- **닫기 버튼**

### 3. 액션 / 흐름
1. 완료 수업 행에서 "기록 상세" 버튼 클릭 → 팝업 열림
2. 출석 현황, 서명, 운동 기록 확인
3. 서명 이미지 클릭 시 확대 보기
4. 닫기 버튼 또는 외부 클릭으로 팝업 닫힘

PT 수업 완료 후 회원이 수업 진행 사실을 확인하는 팝업입니다. 현장에서는 회원 서명을 입력하고, 현장 서명/확인이 누락된 경우 트레이너가 회원앱 Push로 확인 요청을 보낼 수 있습니다. 회원이 회원앱에서 수업 정보를 확인하고 [확인]을 누르면 수업 완료로 처리합니다.

### 2. UI 구성
- **안내 문구**: "수업이 완료되었습니다. 서명해 주세요."
- **수업 정보 요약**: 수업명, 일시, 담당 강사명 (서명 전 확인용)
- **서명 패드**: 터치 또는 마우스로 서명 입력하는 캔버스 영역
- **지우기 버튼**: 서명 초기화 후 재입력
- **확인 버튼**: 서명 저장 및 수업 기록 완료 처리
- **회원앱 Push 요청 버튼**: 서명/확인이 누락된 회원에게 확인 요청 Push 발송
- **건너뛰기 버튼**: 서명 없이 수업 완료 처리 (관리자 권한, 서명 미수집 사유 기록)

### 3. 액션 / 흐름
1. 트레이너가 "서명 요청" 버튼 클릭 → 서명 팝업 열림
2. 회원에게 기기를 건네주거나 화면을 보여줌
3. 회원이 서명 패드에 직접 서명 입력
4. "지우기"로 재서명 가능
5. "확인" 클릭 → 서명 이미지 저장, 수업 기록 완료 처리
6. 현장 서명/확인이 누락된 경우 트레이너가 "회원앱 Push 요청" 클릭
7. 회원앱에 수업명, 일시, 담당 트레이너, 확인 버튼 표시
8. 회원이 "확인" 클릭 → 수업 기록 완료 처리
9. 서명 불가 시 "건너뛰기" → 미서명 사유 입력 후 완료 처리

## DLG-C007 노쇼 정책

> 유형: 다이얼로그
> 연결 기능: CLS-08 / `D04-수업관리/수업관리.md:582`

### 1. 목적
예약 후 노쇼 발생 시 적용할 지점 정책을 설정하는 팝업입니다. 페널티를 자동으로 부과할 기준과 내용을 정의해 두면, 이후 노쇼 발생 시 수동 개입 없이 정책이 자동 적용됩니다. 늦은 취소는 페널티 대상이 아닙니다. D10 SCR-H1001 / `D10-본사관리/본사관리.md:894`에서 관리하는 수업 운영 자동화 4종 중 이 화면에 영향을 주는 항목은 `신규 회원 첫 노쇼 면제` 예외뿐이며, 일반 노쇼 페널티 기준은 지점 정책으로 관리합니다.

### 2. UI 구성
- **노쇼 정책 섹션**:
  - 노쇼 자동 페널티 적용 여부 토글
  - 페널티 내용 선택: 예약 제한 기본 7일 / 노쇼 처리된 예약 회차 사용 처리 / 병행 적용
  - 예약 제한 일수는 운영자가 직접 조정할 수 있으며 기본값은 7일입니다.
- **누적 노쇼 기준 추가 페널티** (선택):
  - 월 3회 이상 노쇼 시 추가 페널티 적용 여부
  - 추가 페널티 기본값은 예약 제한 14일이며 센터 정책으로 변경할 수 있습니다.
- **저장 버튼** / **취소 버튼**

### 3. 액션 / 흐름
1. 정책 설정 탭에서 편집 버튼 클릭 → 팝업 열림
2. 현재 설정된 정책 값이 자동 입력됨
3. 노쇼 페널티 내용과 누적 기준 수정
4. 저장 클릭 → 이후 발생하는 노쇼에 새 정책 적용
5. 취소 클릭 → 변경 없이 팝업 닫힘

### 4. 핵심 상태

|---|---|

  - 충돌 일정 건수 강조
  - 충돌 무시하고 생성 / 충돌 일정 제외하고 생성 선택 옵션
- **제외 날짜 목록**: 사용자가 설정한 예외 날짜 목록
- **확인 버튼**: 일정 생성 실행
- **취소 버튼**: 팝업 닫기, 설정 화면으로 돌아가기

### 3. 액션 / 흐름
1. "일괄 생성" 클릭 → 팝업 열림, 생성 예정 일정 요약 표시
2. 충돌 일정이 있으면 경고 표시 → 충돌 처리 방식 선택
3. 요약 내용 확인 후 "확인" 클릭 → 일정 대량 생성 시작
4. 생성 완료 후 성공 토스트 + 캘린더 화면으로 이동
5. "취소" 클릭 → 팝업 닫힘, 시간표 일괄 등록 화면으로 복귀

- **운동 기록 목록**:

  - 세트 수
  - 반복 횟수 (또는 시간)
  - 중량 (kg)
  - 완료 여부 체크박스
  - 동작별 메모
- **강사 노트**: 세션 전체에 대한 강사 코멘트, 특이사항, 다음 세션 제안

4. 저장 클릭 → 세션 이력에 기록 저장

특정 회원의 PT 횟수가 어떤 수업에서 언제 차감되었는지, 또는 관리자가 수동으로 조정한 이력을 시간순으로 조회하는 팝업입니다. 잔여 횟수가 예상과 다를 때 원인을 파악하거나, 회원의 이의 제기에 대응할 때 활용합니다.

### 2. UI 구성
- **회원 정보**: 회원명, 이용권명, 현재 잔여 횟수 (읽기 전용)
- **이력 목록 테이블**:
  - 날짜·시간
  - 이력 유형 (수업 차감 / 수동 추가 / 수동 차감 / 구매)
  - 관련 수업명 (수업 차감인 경우)
  - 변동 횟수 (+N 또는 -N)
  - 처리 후 잔여 횟수
  - 처리자 (강사명 또는 관리자명)
  - 사유 (수동 조정인 경우)
- **기간 필터**: 조회 기간 선택
- **닫기 버튼**

### 3. 액션 / 흐름
1. "이력" 버튼 클릭 → 팝업 열림, 전체 차감·조정 이력 최신순 표시
2. 기간 필터로 특정 기간 이력만 조회
3. 이력 확인 후 이상 항목 발견 시 횟수 조정(DLG-C012 / `D04-수업관리/수업관리.md:2248`)으로 정정
4. 닫기 버튼 또는 외부 클릭으로 팝업 닫힘

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
| 완료 | 저장 또는 처리 성공 후 부모 화면 데이터를 갱신하고 다이얼로그를 닫습니다. |
| 실패 | 실패 사유를 토스트 또는 인라인 메시지로 안내하고 입력값은 유지합니다. |

### 5. 권한
| 계정 | 동작 기준 |
|---|---|
| Owner(지점장) / manager | 부모 화면에서 허용된 경우 조회·입력·저장할 수 있습니다. |
| trainer / fc / staff | 담당 업무 범위에 포함된 데이터만 조회하며, 수정 권한은 부모 화면 정책을 따릅니다. |

## DLG-C014 페널티 등록

> 유형: 다이얼로그
> 연결 기능: CLS-08 / `D04-수업관리/수업관리.md:582`

### 1. 목적
자동 페널티 정책 외에, 관리자가 특정 예약의 노쇼 페널티를 수동으로 부과하는 팝업입니다. 자동 노쇼 처리 누락, 출석 기록 정정 등 운영 보정이 필요한 경우에만 사용합니다.

### 2. UI 구성
- **대상 회원 선택**: 회원 검색 또는 드롭다운으로 페널티 부과 대상 선택
- **페널티 유형 선택**: 노쇼 고정
- **관련 수업 선택**: 페널티 사유가 된 수업 선택 (선택)
- **페널티 내용**:
  - 예약 제한: 제한 기간(일수) 입력
  - 노쇼 회차 사용 처리: 해당 예약에 묶인 회차를 사용 처리
  - 두 가지 병행 적용 가능
- **적용 시작일**: 페널티 적용 시작일 (기본값: 오늘)
- **사유 입력**: 페널티 부과 사유 상세 기록 (필수)
- **확인 버튼** / **취소 버튼**

### 3. 액션 / 흐름
1. "페널티 등록" 클릭 → 팝업 열림
2. 대상 회원 검색 및 선택
3. 페널티 유형·내용·시작일 설정
4. 사유 입력 (필수)
5. "확인" 클릭 → 페널티 즉시 적용, 회원에게 알림 발송 (설정에 따라)
6. "취소" 클릭 → 팝업 닫힘, 변경 없음

## DLG-C015 자동 페널티 정책

> 유형: 다이얼로그
> 연결 기능: CLS-08 / `D04-수업관리/수업관리.md:582`
> 연결 다이얼로그: DLG-C014 / `D04-수업관리/수업관리.md:2401` 페널티등록, DLG-C015 / `D04-수업관리/수업관리.md:2482` 자동페널티정책, DLG-C007 / `D04-수업관리/수업관리.md:1849` 노쇼정책

### 1. 목적
예약 후 노쇼 발생 시 페널티를 자동으로 부과하는 지점 규칙을 설정하는 팝업입니다. 한번 설정해 두면 이후 해당 조건이 충족될 때마다 수동 개입 없이 정책이 자동 적용됩니다. 늦은 취소는 페널티 대상이 아닙니다. D10 SCR-H1001 / `D10-본사관리/본사관리.md:894`에서 관리하는 수업 운영 자동화 4종 중 이 화면에 영향을 주는 항목은 `신규 회원 첫 노쇼 면제` 예외뿐이며, 일반 자동 페널티 기준은 지점 정책으로 관리합니다.

### 2. UI 구성
- **자동 페널티 활성화 토글**: 전체 자동 페널티 기능 켜기/끄기
- **노쇼 자동 페널티 규칙**:
  - 적용 여부 토글
  - 페널티 내용: 예약 제한 기본 7일 / 노쇼 처리된 예약 회차 사용 처리 / 병행
  - 1회 노쇼 / 누적 3회 노쇼 / 누적 5회 노쇼 기준으로 각각 다른 페널티 적용 가능
- **자동 알림 설정**: 페널티 부과 시 회원에게 자동 알림 발송 여부 토글
- **저장 버튼** / **취소 버튼**

### 3. 액션 / 흐름
1. 정책 설정 탭에서 "자동 페널티 정책 설정" 클릭 → 팝업 열림
2. 현재 적용 중인 정책 값 자동 표시
3. 노쇼 규칙 설정
4. 자동 알림 발송 여부 설정
5. 저장 클릭 → 이후 발생하는 노쇼에 즉시 새 정책 적용
6. 취소 클릭 → 변경 없이 팝업 닫힘

### 4. 핵심 상태

## DLG-C016 대안 일정 제시

> 유형: 다이얼로그
> 연결 기능: CLS-09 / `D04-수업관리/수업관리.md:661`
> 연결 다이얼로그: DLG-C016 / `D04-수업관리/수업관리.md:2558` 대안일정제시

### 1. 목적
회원이 수업 취소를 요청했을 때, 단순히 취소 처리하는 대신 비슷한 조건의 다른 수업 일정을 대안으로 제안하는 팝업입니다. 회원 이탈을 줄이고 수업 자리 활용률을 높이기 위한 화면입니다.

### 2. UI 구성
- **원래 수업 정보**: 취소 요청된 수업명, 일시, 강사 (읽기 전용)
- **대안 일정 목록**:
  - 같은 수업 유형의 다른 날짜·시간 수업 목록 (잔여 자리 있는 수업만)
  - 수업명, 날짜·시간, 강사, 장소, 잔여 자리 수 표시
  - 각 대안 일정 옆에 "이 일정으로 제안" 버튼
- **직접 일정 입력**: 목록에 없는 경우 날짜·시간을 직접 입력해 제안
- **안내 메시지 입력**: 회원에게 전달할 추가 메시지 (선택)
- **제안 발송 버튼**: 선택한 대안 일정을 회원에게 알림으로 발송
- **취소 버튼**: 팝업 닫기 (대안 제시 없이 취소 요청 처리 계속)

### 3. 액션 / 흐름
1. 취소 요청 건에서 "대안 제시" 클릭 → 팝업 열림
2. 같은 유형의 잔여 자리 있는 수업 목록 자동 표시
3. 대안 일정 선택 또는 직접 입력
4. 회원에게 전달할 메시지 입력 (선택)
5. "제안 발송" 클릭 → 회원 앱에 대안 일정 제안 알림 발송
6. 회원이 제안 수락 시 기존 취소 처리 + 새 수업 예약 자동 완료
7. 회원이 제안 거절 시 원래 취소 요청 그대로 처리

### 4. 핵심 상태
| 상태 | 설명 |
|---|---|
| 기본 표시 | 부모 화면에서 선택한 대상의 최신 값을 불러와 표시합니다. |
| 입력/수정 중 | 필수값 누락, 형식 오류, 권한 제한을 인라인 메시지로 표시합니다. |
| 저장/처리 중 | 주요 버튼을 비활성화하고 중복 요청을 막습니다. |
