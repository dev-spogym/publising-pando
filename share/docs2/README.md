# FitGenie CRM 관리자 개발사용 납품 문서

이 폴더는 관리자 CRM 구현을 위해 기존 기획 내용을 개발자가 보기 쉬운 구조로 재정리한 납품 문서입니다. `docs2`만 보고 화면·기능·UX·운영정책 기준을 확인할 수 있도록 구성하며, 구현 판단을 위해 다른 기획 폴더를 확인하지 않습니다.

## 사용 방법
- 한 화면을 구현할 때는 해당 도메인의 `{도메인명}.md`와 `운영정책.md`를 우선 확인합니다.
- 공통 권한, 공통 참조 코드, 비기능 요구사항(NFR), KPI 지표, 공통 예외, 자동화, 토스트, 디자인 기준은 `_공통/` 문서를 확인합니다.
- API/DB 상세는 백엔드 설계 범위이므로 이 문서에서는 다루지 않습니다.
- 다이어그램은 개발사 요청 범위와 사용자 지시에 따라 납품 문서에서 제외했습니다.
- 정책값이 아직 확정되지 않은 항목은 `_정책미확정_확인필요.md`에서 확인합니다.

## 문서 기준
- `docs2`는 개발사용 납품 문서입니다.
- 화면별 구현 기준은 각 도메인 `{도메인명}.md`의 `UI 구성`, `액션 / 흐름`, `권한`, `필수 예외처리`를 기준으로 합니다.
- 운영 기준은 각 도메인 `운영정책.md`와 `_공통/` 문서를 기준으로 합니다.
- `AUTH-01`, `HQ-09`, `SCR-097`, `A04`, `A05`, `X11`처럼 여러 도메인에서 반복되는 공통 참조 코드는 `_공통/공통참조_코드인덱스.md`를 기준으로 합니다.
- `NFR-05`, `NFR-19`처럼 여러 도메인에 반복되는 비기능 요구사항 코드는 `_공통/비기능요구사항_NFR.md`를 기준으로 합니다.
- KPI 명칭, 산식, 역할별 보드, Today Tasks 관련 공통 기준은 `_공통/KPI_지표정의.md`를 기준으로 합니다.
- 문서 간 값이 충돌하면 docs2 안에서 값 하나로 보정해 반영해야 하며, 개발자는 다른 기획 폴더를 별도 확인 대상으로 보지 않습니다.
