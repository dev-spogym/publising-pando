# D03-매출관리 매출관리 — 운영정책

> 개발사용 운영정책입니다. 이 파일만으로 해당 도메인의 운영 기준, 정책값, 예외 기준을 확인할 수 있도록 재구성했습니다.

## 1. 핵심 원칙
- 상품 판매부터 계약, 결제, 이용권 생성, 환불까지를 하나의 운영 흐름으로 정리한다.
- 매출 화면과 회원 상세 화면이 서로 다른 상태를 말하지 않도록 기준을 통일한다.
- 상품 관리
- 할인 정책
- POS 판매
- 결제 처리
- 매출 현황
- 환불 관리
- 미수금 관리
- 선수익금 조회
- 회원권
- 수강권
- 락커
- 운동복
- 일반 상품
- 수강권은 PT / GX / 골프 / 기타 수강권을 세부 유형으로 가진다.
- GX 수강권은 2단계 세부종목을 필수로 가진다.
- 패키지는 상품 대분류가 아니라 상품 구성/판매 옵션으로 표시한다.

## 2. 상태 / 운영 정의
- POS 판매 화면(SCR-S002 / `D03-매출관리/매출관리.md:131`)에서 구성한 장바구니를 결제완료 상태로 확정하는 기능이다. 현장 판매에서는 CRM이 카드 PG 승인을 직접 수행하지 않는다. 인포데스크(FC)가 외부 POS 단말 또는 현금/계좌이체 수납으로 실제 결제를 먼저 완료하고, CRM에는 혼합결제 행, 포인트 사용액, 결제일시, 영수증 파일, VAN/POS 결과, 지점 귀속 필드를 등록한다. 등록이 성공하면 별도 매니저 승인 없이 결제완료와 회원권/수강권/락커/운동복/일반 상품 구매 완료가 즉시 반영된다.
- **카운터 매니저 일반 회원권 등록**: POS에서 1년 회원권 장바구니 구성 → 외부 POS 카드 결제 완료 → 영수증 첨부 → 회원권 즉시 활성
- 영수증 파일 첨부는 현장 판매 결제 등록의 필수 조건이다. 첨부가 없으면 결제 완료 등록 버튼은 활성화되지 않는다.
- v1 결제 화면은 PRD-04 / `D05-상품관리/상품관리.md:406` 할인 정책과 MBR-EXT-03 / `D02-회원관리/회원관리.md:753` 등급 혜택을 자동 적용하거나 후보로 조회하지 않는다. 할인 정책은 D05에서 등록·수정·삭제·조회까지만 가능하며, D03 결제 개발 기준으로 삼지 않는다. 할인 영역에는 운영자가 수동으로 확정한 할인 항목과 할인 금액만 입력·표시한다. 쿠폰/마일리지 선택 적용, 시즌특가·등급할인 중복 제한, 큰 할인 우선 적용 같은 결제 자동 적용정책은 고객사 확인 후 후속 기획으로 반영한다.
- `혼합결제 행 합계 + 포인트 사용액 = 장바구니 최종 합계`가 정확히 일치해야 결제 완료 등록이 가능하다. 불일치하면 차액을 붉은색으로 표시하고 버튼을 비활성화한다.

- **현장 환불 증빙 누락/결제링크 PG 실패**: 상태 "거절(빨강)" + 운영자 알림, 증빙 보완 또는 익영업일 재시도
- ## 화면 상태
- 회원이 실제 계약금·잔액·할부 등 계약상 납입 의무를 완료하지 못한 미수금을 추적하고 관리하는 화면이다. 결제링크 발송만 된 건은 계약금이나 계약상 미수금으로 보지 않으며, 회원상세 결제내역과 PAY-06 / `D03-매출관리/매출관리.md:233` 결제링크 상태로만 표시한다.
- **결제링크 만료 후 재발송**: 결제링크 상태가 `만료` 또는 `무효화`이면 회원상세 결제내역의 [결제링크 발송] 버튼으로 새 링크를 생성한다. `발송됨` 상태에서는 같은 링크 재발송 또는 결제링크 무효화만 가능하다.
- **미수금 연체 자동 독촉 (NFR-05 / `_공통/공통참조_코드인덱스.md:26`)**: HQ-09 / `D10-본사관리/본사관리.md:894` `결제기한 만료` 본사 정책 step + 지점 ON 상태 기준으로 자동 알림을 발송합니다. 미수/연체 알림의 N일 기준, 반복 주기, 발송 시각, 기본 채널은 본사(superAdmin/primary)가 HQ-09 / `D10-본사관리/본사관리.md:894` 정책 세트에서 설정해 지점에 배포합니다. 지점은 ON/OFF만 가능하고 지점 추가 step은 허용하지 않으며, 결제링크 만료 알림은 PAY-06 / `D03-매출관리/매출관리.md:233` 고객사 확인 후 D-3/D-1 고정값을 따릅니다.
- ### 상태 탭 (5개)
- 전체 / 미결제 / 일부결제 / 연체 / 완료. 각 탭에 건수 배지.
- 현장에서 확정한 상품·금액·운영자가 수동 확정한 할인 금액으로 회원이 직접 전액 결제할 수 있는 링크를 발송하는 기능이다. 결제 처리 화면(SCR-S003 / `D03-매출관리/매출관리.md:233`)에서 수납 방식을 `결제링크 발송`으로 선택하면 DLG-S016 / `D03-매출관리/매출관리.md:2638` 결제링크 발송 다이얼로그가 열리고, 운영자는 수신자·최종 결제금액·발송 채널·만료일을 확인한 후 링크를 보낸다. DLG-S016은 별도 화면이 아니라 발송 전 확인 팝업이다.
- **고객사 확인 대기**: 아래 결제링크 정책은 2026-05-23 판도 내부 기준안이며, 고객사 최종 확인 후 확정한다. 확정 전에는 PAY-06 / `D03-매출관리/매출관리.md:233` 자동 만료·자동 알림·활성 링크 제한·PG 결제 완료 webhook 구현 기준으로 사용하지 않고 개발 대기 항목으로 본다.
- **전액 결제 전용**: 결제링크는 계약금/잔액/할부 수납에 사용하지 않는다. 링크로 결제하면 항상 해당 링크 금액 전액 결제이며, 결제 완료 즉시 이용권을 활성화한다.
- **결제링크 상태값**: 발송됨 / 결제완료 / 만료 / 무효화 / 발송실패. `열람됨` 상태와 `미결제`는 결제링크 상태값으로 사용하지 않는다.
- **회원상세 표시상태**: 결제링크 발송 후 결제 완료 전 회원상세 결제내역 행에는 `미결제`만 표시한다. `발송됨`은 PAY-06 / `D03-매출관리/매출관리.md:233` 내부 링크 상태값이며, 화면에는 결제링크 재발송·무효화 버튼 노출 기준으로만 사용한다. 이 `미결제` 표시는 SCR-S008 / `D03-매출관리/매출관리.md:732` 미수금의 `미결제` 상태와 섞지 않는다.
- **활성 링크 1개 제한**: 동일 결제 건에는 `발송됨` 상태 링크를 1개만 허용한다. 새 링크가 필요하면 기존 링크를 먼저 무효화해야 한다.
- **결제링크 무효화**: 결제링크가 발송된 회원의 회원상세 결제내역 행에 [결제링크 무효화] 버튼을 노출한다. Owner(지점장)/manager/fc가 수행할 수 있으며 사유 입력은 필수, 무효화 처리자와 처리시각은 이력에 표시한다.
- **새 링크 생성**: 만료 또는 무효화 상태에서는 회원상세 결제내역 행의 [결제링크 발송] 버튼으로 DLG-S016을 다시 열어 새 링크를 생성한다.
- **만료/알림**: 링크 유효기간은 발송일시 + 7일 고정. 만료 D-3/D-1에 회원과 운영자에게 자동 알림을 발송하며 ON/OFF는 제공하지 않는다.
- **결제 완료 알림**: 결제 완료 즉시 회원과 운영자 모두에게 결제 완료 알림을 발송하고, 회원상세 결제내역 상태를 `결제완료`로 갱신한다.
- **환불 진입 제한**: 미결제 결제링크는 환불 목록과 환불 버튼에 노출하지 않는다. URL 직접 접근 등 방어 케이스에서만 "결제되지 않은 링크는 환불 대상이 아닙니다" 안내 후 차단한다.

- 본사 권한자는 통합 모드에서 전 지점 조회·정책 관리를 수행한다.
- 지점 권한자는 소속 지점 데이터의 일상 운영 처리를 수행한다.
- 변경·삭제·복구·정산·권한 변경은 화면별 최소 권한을 따른다.
- 공통 권한 기준은 `_공통/권한매트릭스.md`를 우선 참조하고, 화면별 차이는 도메인 문서에 따로 표기한다.
